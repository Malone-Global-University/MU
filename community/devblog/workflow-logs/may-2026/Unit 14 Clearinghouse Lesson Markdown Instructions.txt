Title
Unit 14 Clearinghouse Lesson Markdown Instructions

Date
2026-05-17

Area
MGU Wealth & Asset Operations Track / Unit 14: Clearinghouses and Central Counterparty Infrastructure

Problem
The Unit 14 lesson sequence existed as a topic outline, but Claude.ai needed a precise production prompt for writing the standard Markdown lessons and the unit capstone. Without a clear instruction sheet, lesson outputs could vary in structure, depth, naming, formatting, and instructional focus.

Intended Goal
Create a reusable Claude.ai instruction package that tells Claude how to write each Unit 14 lesson as a standalone Markdown course file, with consistent structure, filenames, depth, tone, review material, applied exercises, answer keys, and capstone synthesis.

Root Cause
The unit outline identified the lesson topics but did not yet define the full lesson production rules. Claude needed explicit guidance for standard lessons, capstone lessons, lesson-specific content emphasis, Markdown formatting, output behavior, and conceptual boundaries around clearinghouses and central counterparties.

Steps Taken

1. Converted the Unit 14 outline into a complete Claude.ai-ready instruction prompt.
2. Defined the unit purpose around post-trade stability, clearinghouse foundations, CCP novation, netting, margining, default management, and risk monitoring.
3. Created a standard lesson structure for Lessons 14.1 through 14.7, including overview, learning objectives, key terms, main content, operational example, risk perspective, controls, misunderstandings, summary, review questions, applied exercise, and answer key.
4. Created a separate capstone lesson structure for synthesizing the full clearinghouse and CCP operating model.
5. Added exact Markdown filenames for each lesson and the capstone.
6. Added lesson-specific guidance for all seven standard lessons so each topic has a distinct instructional focus.
7. Added depth, tone, formatting, and output rules to prevent Claude from producing blog-style content, unsupported legal claims, vague summaries, or multi-lesson batches when only one lesson is requested.

Key Change
The Unit 14 outline was transformed into a complete lesson-generation specification for Claude.ai, including standard lesson rules, capstone rules, filenames, structure, scope, and formatting requirements.

Outcome
Claude.ai now has clear instructions for producing consistent, substantial Markdown lessons for Unit 14. Each lesson can be generated one at a time with predictable structure, appropriate depth, applied examples, review questions, exercises, and answer keys. The capstone is also defined as an integrated synthesis rather than a simple repeat of the seven lessons.

Validation
The completed instruction package covers every listed lesson in Unit 14, includes the full capstone requirement, uses Markdown-only formatting, defines one-file-per-lesson output behavior, and gives specific conceptual boundaries for clearinghouses, central counterparties, novation, netting, margining, default management, and risk monitoring.

Rules Going Forward
Future MGU lesson-production prompts should include exact filenames, standard lesson structure, capstone structure when needed, lesson-specific content guidance, formatting rules, depth expectations, and output rules. Claude should be instructed to generate one Markdown lesson at a time unless a batch is explicitly requested. Course lessons should remain structured educational materials, not blog posts or short summaries.

Notes
This prompt can serve as a model for future units in the Wealth & Asset Operations Track. The same structure can be reused for other post-trade, custody, settlement, risk, compliance, or market infrastructure units by replacing the lesson-specific guidance and filenames.
