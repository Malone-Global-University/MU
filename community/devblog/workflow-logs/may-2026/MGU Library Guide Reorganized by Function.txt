WORKFLOW NOTE

Title
MGU Library Guide Reorganized by Function

Date
2026-05-17

Area
MGU Library / Library Guide / Navigation Structure

Problem
The draft Library Guide was initially organized too heavily around reference resource types such as infographics, charts, tables, research papers, whitepapers, blackpapers, legal materials, internal documentation, and textbooks. That structure described many useful sections, but it did not match the real top-level function of the MGU Library. It risked making the Library feel like a collection of disconnected content formats instead of a clear user-facing knowledge system.

Intended Goal
The Library Guide should explain the MGU Library by user function first. Users should be able to understand the six main Library areas, why each area exists, how to navigate to each one, and how to use each one. The correct top-level areas are Dictionary, Encyclopedia, Fiction, Guides, Malonicus Rex Educare, and Reference.

Root Cause
The earlier guide draft treated Reference subsections as if they were the whole Library structure. This happened because the recent work focused on Reference category index pages, including charts, legal-desk, mgu-docs, research, mgu-textbooks, whitepapers, blackpapers, infographics, and tables. That made the Reference area temporarily dominate the guide design, even though Reference is only one of the six major Library functions.

Steps Taken

1. Identified that the Library Guide should be based on function rather than only content type or folder depth.
2. Restated the six main Library areas as Dictionary, Encyclopedia, Fiction, Guides, Malonicus Rex Educare, and Reference.
3. Reframed the Library Guide around the user question: “What kind of resource do I need?” before “What subject do I need?”
4. Defined the functional purpose of each major area: Dictionary defines, Encyclopedia explains, Fiction imagines, Guides instruct, Malonicus Rex Educare philosophizes, and Reference preserves.
5. Explained navigation patterns for each area, including index pages, subject paths, breadcrumbs, category pages, and deeper resource pages.
6. Clarified that Reference remains important but should be treated as one major Library area with its own internal subcategories.
7. Listed Reference subsections separately, including Blackpapers, Charts, Infographics, Legal Desk, MGU Internal Documentation, MGU Textbooks, Research Papers, Tables, and Whitepapers.
8. Added recommended user paths for new visitors, students, teachers, researchers, and administrators.
9. Added a core rule for future organization: organize by function first, subject second.

Key Change
The Library Guide was shifted from a resource-type outline to a function-first navigation model based on the six true top-level Library areas.

Outcome
The MGU Library now has a clearer conceptual structure for a future guide page. Users can understand the Library as a system with distinct purposes instead of a large folder tree. The guide can now explain where to go for definitions, topic articles, fiction, instructions, educational philosophy, and formal reference materials without mixing those functions together.

Validation
The revised outline was checked against the uploaded Library structure and the user’s clarification that the complete Library should be understood through six main areas. The new model also matches the existing direction of the site: Dictionary for terms, Encyclopedia for topics, Fiction for literary works, Guides for instructions, Malonicus Rex Educare for educational philosophy, and Reference for formal documents and reusable materials.

Rules Going Forward
Organize the MGU Library by function first and subject second.
Do not let Reference subsections define the entire Library.
Keep Dictionary, Encyclopedia, Fiction, Guides, Malonicus Rex Educare, and Reference as the primary Library areas.
Use Reference for preserved, formal, reusable, visual, legal, research, textbook, and institutional materials.
Use Guides for user instruction, navigation help, workflows, and practical explanations.
Use Malonicus Rex Educare for the educational philosophy and institutional doctrine of MGU.
When writing future Library pages, explain what the area is for, how to navigate it, and how to use it.

Notes
The Library Guide should probably become a Markdown master draft first, then later be converted into an HTML page under the Guides area. A good final location would be something like `/library/guides/library-guide/index.html` or `/library/guides/how-to-use-the-library/index.html`. The guide can later include a quick decision table, examples of breadcrumb paths, and user-specific routes for students, teachers, researchers, and administrators.
