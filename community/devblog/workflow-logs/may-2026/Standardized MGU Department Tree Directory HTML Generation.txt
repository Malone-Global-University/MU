WORKFLOW NOTE

Title
Standardized MGU Department Tree Directory HTML Generation

Date
2026-05-17

Area
MGU department directory navigation pages

Problem
MGU department folders existed as local PowerShell directory trees, but they were not yet converted into clean, browsable HTML directory maps. Early versions needed to be aligned with the uploaded `all 5 volumes.html` style.

Intended Goal
Create standalone, downloadable HTML directory trees that match the uploaded expandable tabbed outline format and can be reused across departments.

Root Cause
The department structures were stored as filesystem hierarchies, not web-friendly navigation documents. Each department needed parsing, hierarchy reconstruction, and standardized presentation.

Steps Taken

1. Parsed PowerShell `Get-ChildItem -Recurse` exports for multiple MGU department folders.
2. Converted each folder/file hierarchy into structured tabbed HTML.
3. Matched the later outputs to the uploaded `all 5 volumes.html` style.
4. Added expandable sections, clickable file paths, item details, file sizes, dark-mode support, and ZIP fallbacks.
5. Generated directory trees for Computer Science, Art, Conspiracy, Culinary, Defense, History, Religion, Science, Professional Studies, Political Science, and Math.

Key Change
All future tree directories were standardized to the same expandable tabbed “all 5 volumes” style.

Outcome
MGU now has a repeatable directory-tree HTML format for large department structures, including very large trees such as Professional Studies, History, Political Science, and Math.

Validation
Each uploaded PowerShell tree was parsed into counts of folders/files, rendered into an HTML file, packaged into a ZIP fallback, and delivered for download.

Rules Going Forward
Use the matched expandable tabbed template for all future department tree directories. Preserve readable tab groupings, natural numeric sorting, clickable paths, file metadata, and ZIP fallback delivery.

Notes
For very large departments, split major folders into separate tabs so the page remains usable instead of becoming one long flat directory list.
