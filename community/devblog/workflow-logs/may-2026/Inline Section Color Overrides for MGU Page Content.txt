WORKFLOW NOTE

Title
Inline Section Color Overrides for MGU Page Content

Date
2026-05-17

Area
MGU HTML page styling / inline CSS / section headings and paragraph text

Problem
The page needed section-specific colors for MGU areas such as Main, Community, Department, Library, and News. The initial concern was that inline text coloring syntax was incorrect. After confirming the list text color syntax worked, the next issue was how to override the global `section h2` rule that used `var(--accent)` for both heading text and the bottom border.

Intended Goal
Allow individual sections to use their own identity colors directly in the HTML when needed, especially for heading text, heading borders, and optional paragraph text, without changing the global stylesheet.

Root Cause
The global CSS rule applied one shared accent color to all section headings:

```css
section h2 {
  color: var(--accent);
  border-bottom: 2px solid var(--accent);
}
```

This meant each section inherited the same accent color unless a more specific or inline style overrode it.

Steps Taken

1. Confirmed that inline text color syntax using `<span style="color: #hex;">...</span>` was valid and that the original issue came from a small typo rather than incorrect HTML structure.

2. Identified that heading colors and heading borders could be overridden inline by placing both `color` and `border-bottom` on the `<h2>` element:

```html
<h2 style="color: #2f8f57; border-bottom: 2px solid #2f8f57;">
  Community
</h2>
```

3. Clarified that if the global CSS already defines the border width and style, only the border color needs to be overridden:

```html
<h2 style="color: #2f8f57; border-bottom-color: #2f8f57;">
  Community
</h2>
```

4. Explained how to color paragraph text by placing the `style` attribute inside the same opening tag as the class:

```html
<p class="wide" style="color: #2f8f57;">
```

5. Explained two methods for multiple paragraphs: applying the same inline style to each paragraph, or applying `style="color: #2f8f57;"` to the parent `<section>` so child paragraphs inherit the color.

Key Change
Section-specific identity colors can now be applied inline by overriding `color` and `border-bottom-color` directly on each section heading.

Outcome
The page can now visually distinguish MGU areas using consistent identity colors: teal for Main, green for Community, blue for Department, red for Library, and purple for News. The heading text and border colors can be customized without editing the shared CSS file.

Validation
The original inline `<span>` color syntax was confirmed to work after the typo was corrected. The `h2` override pattern was validated against the existing CSS rule because inline styles have higher priority than the global `section h2` rule. The paragraph color pattern was also verified as valid HTML because `class` and `style` can both exist in the same opening tag.

Rules Going Forward
Use inline color overrides sparingly for one-off page-specific styling. For section headings, prefer:

```html
<h2 style="color: #2f8f57; border-bottom-color: #2f8f57;">
```

Use the full `border-bottom` declaration only when changing the border width, style, or color together. Keep long paragraph text in the normal theme text color unless there is a strong visual reason to color all body text.

Notes
The best visual pattern for readability is to color the section heading and border while leaving paragraph text controlled by the global theme. This preserves section identity without making long reading text harder to scan.
