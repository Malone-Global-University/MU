WORKFLOW NOTE

Title
PowerShell Tutorial Starter Series Created

Date
2026-05-17

Area
MGU PowerShell Tutorials / Department Tutorial Pages / Website Maintenance Instruction

Problem
MGU needed a beginner-friendly PowerShell tutorial sequence focused on practical file management and website maintenance. The tutorials needed to explain PowerShell through real MGU use cases such as scanning large folder trees, finding recent files, searching HTML contents, replacing stylesheet paths, creating reports, checking broken links, and using ImageMagick for image workflows. Early page generation also used `/component/main.css`, but the PowerShell tutorial pages should use the department stylesheet instead.

Intended Goal
Create a structured starter series of PowerShell HTML tutorial pages using the MGU page shell, with clear beginner explanations, practical commands, safety warnings, MGU-specific examples, and the correct stylesheet reference:

```html
<link rel="stylesheet" href="/component/dept.css" />
```

Root Cause
The PowerShell tutorial series was new and did not yet have a complete instructional sequence or page convention. Since the first pages were being generated from a general MGU shell, the stylesheet link initially followed the older/general pattern before being corrected to the department-page CSS convention.

Steps Taken

1. Defined the main use case for PowerShell as practical Windows automation, file management, system administration, and especially bulk MGU website maintenance.

2. Created the first introductory page, `what-powershell-is.html`, explaining PowerShell as both a command-line tool and scripting language.

3. Built the beginner tutorial sequence as separate HTML pages:
   `opening-powershell-safely.html`, `understanding-paths-and-folders.html`, `listing-files-with-get-childitem.html`, `finding-recent-files.html`, `searching-file-contents.html`, `copying-moving-and-renaming-files.html`, `using-recurse-for-large-folder-trees.html`, `making-backups-before-bulk-edits.html`, `replacing-text-across-html-files.html`, `generating-file-inventories.html`, `exporting-results-to-txt-or-csv.html`, `checking-broken-links-or-wrong-paths.html`, `using-powershell-with-imagemagick.html`, and `safe-rules-for-bulk-website-maintenance.html`.

4. Standardized the later PowerShell tutorial files to use `/component/dept.css` instead of `/component/main.css`.

5. Saved the CSS convention for future MGU PowerShell tutorial HTML generation.

6. Included practical PowerShell command examples for `Get-ChildItem`, `Get-Location`, `Set-Location`, `Select-String`, `Where-Object`, `Sort-Object`, `Select-Object`, `Measure-Object`, `Copy-Item`, `Move-Item`, `Rename-Item`, `New-Item`, `Test-Path`, `Out-File`, `Set-Content`, `Export-Csv`, and ImageMagick’s `magick` command.

7. Emphasized safe maintenance habits throughout the series: inspect first, back up second, test third, change fourth, verify fifth.

Key Change
The PowerShell tutorial series was established as a practical MGU website-maintenance curriculum, and future PowerShell tutorial pages were standardized to use `/component/dept.css`.

Outcome
MGU now has a complete starter run of PowerShell tutorial HTML pages covering beginner concepts through real maintenance workflows. The pages are aligned with the user’s practical needs: managing thousands of website files, correcting paths, searching content, creating reports, protecting work with backups, checking broken links, and processing image batches with ImageMagick.

Validation
The generated pages consistently used the MGU HTML shell with dynamic header, breadcrumb placeholder, main content area, bottom navigation placeholder, footer placeholder, and page date handling. After the stylesheet correction, subsequent pages used the correct department stylesheet link. The tutorial topics also followed the planned sequence and each page included concrete PowerShell examples plus MGU-specific use cases.

Rules Going Forward
All future generated MGU PowerShell tutorial HTML files should use:

```html
<link rel="stylesheet" href="/component/dept.css" />
```

PowerShell tutorial pages should remain practical, beginner-friendly, and tied to real website maintenance tasks. Bulk-edit lessons should always teach inspection, reports, backups, testing, and verification before changing files. Commands that can affect many files should include safety warnings and narrow-path examples.

Notes
The early pages that used `/component/main.css` may need to be manually updated to `/component/dept.css` for consistency. The completed tutorial set can later be connected with an index page, previous/next navigation, or a department tutorial landing page for the full PowerShell sequence.
