WORKFLOW NOTE

Title
Numerical Methods Master Instruction Sheet Creation

Date
2026-05-18

Area
MGU Mathematics Series textbook production pipeline; Claude.ai master instruction framework; Numerical Methods textbook planning

Problem
The Numerical Methods textbook outline existed, and a mature Algebra I master instruction sheet existed, but there was no dedicated master instruction framework for the Numerical Methods project. Without a unified instruction sheet, future chapter generation risked inconsistency in tone, structure, assessment patterns, numerical-method philosophy, formatting, computational emphasis, and textbook identity.

Intended Goal
Create a reusable master instruction sheet for Claude.ai that adapts the proven Algebra I instruction framework to a calculus-based Numerical Methods textbook while preserving MGU textbook standards, chapter structure consistency, computational philosophy, error-analysis emphasis, and long-form textbook formatting requirements.

Root Cause
The Numerical Methods project had a detailed conceptual outline and chapter architecture but lacked a standardized production instruction layer connecting the outline to the actual chapter-writing workflow. The Algebra I instruction system existed separately and needed to be generalized and transformed for computational mathematics.

Steps Taken

1. Reviewed the uploaded Numerical Methods outline and extracted the textbook identity, pedagogical philosophy, chapter sequence, computational themes, prerequisite assumptions, and assessment structure.

2. Reviewed the uploaded Algebra I `master_instructions.md` document to identify reusable structural patterns including:

   * textbook identity formatting,
   * instructional philosophy,
   * chapter requirements,
   * worked-example structure,
   * practice-problem categorization,
   * Markdown and LaTeX conventions,
   * MGU Library positioning,
   * HTML-awareness guidance,
   * chapter-purpose architecture.

3. Rebuilt the instruction framework specifically for Numerical Methods by replacing algebra-centered instructional goals with computational mathematics goals emphasizing:

   * approximation,
   * convergence,
   * floating-point arithmetic,
   * stability,
   * conditioning,
   * iterative algorithms,
   * error propagation,
   * scientific computing,
   * numerical modeling,
   * algorithm interpretation.

4. Added Numerical Methods–specific instructional structures including:

   * algorithm box formatting,
   * pseudocode guidance,
   * numerical interpretation workflows,
   * computational reliability analysis,
   * convergence/error expectations,
   * modeling-oriented problem-solving structure,
   * scientific-computing bridge notes.

5. Preserved the established MGU Library textbook identity and long-form textbook philosophy so the Numerical Methods book remains consistent with:

   * Calculus,
   * Linear Algebra,
   * Differential Equations,
   * Statistics,
   * Discrete Mathematics,
   * Combinatorics,
   * other MGU mathematics titles.

6. Integrated the complete 14-chapter Numerical Methods sequence into the instruction sheet, including:

   * part structure,
   * section lists,
   * conceptual chapter goals,
   * computational emphasis,
   * chapter back matter expectations,
   * textbook back matter architecture.

7. Added safeguards preventing future model drift by explicitly instructing Claude not to:

   * reduce the book to coding tutorials,
   * omit error analysis,
   * skip convergence or stability discussion,
   * overcompress explanations,
   * treat methods as disconnected formulas,
   * escalate beyond introductory numerical methods depth.

Key Change
A full production-grade master instruction framework was created for the MGU Numerical Methods textbook, transforming the outline into a stable long-form textbook generation system for Claude.ai.

Outcome
The Numerical Methods project now has a reusable instruction architecture aligned with the broader MGU Mathematics Series workflow. Future chapters can now be generated with:

* consistent textbook voice,
* stable section structure,
* unified assessment formatting,
* computational interpretation standards,
* error-analysis expectations,
* scientific-computing integration,
* clean Markdown output formatting.

Validation
Validation was performed by checking that the generated instruction sheet:

* preserved the original Numerical Methods chapter sequence,
* matched the organizational style of the Algebra I master instruction framework,
* retained MGU Library identity conventions,
* incorporated all major Numerical Methods conceptual themes,
* included standardized chapter/back matter expectations,
* properly integrated algorithm, approximation, and error-analysis workflows,
* remained suitable for Markdown-to-HTML conversion workflows.

Rules Going Forward

* Future MGU textbook projects should begin with a dedicated master instruction sheet before chapter production begins.
* Instruction sheets should inherit stable MGU structural conventions while adapting pedagogical philosophy to the discipline.
* Numerical/computational textbooks must explicitly address approximation, convergence, error, conditioning, and stability throughout the workflow.
* Long-form Markdown generation should remain the default textbook production format unless another format is explicitly requested.
* Chapter-generation prompts should always derive from the master instruction framework rather than ad hoc prompts.

Notes
The Numerical Methods framework now serves as a bridge-level computational mathematics instruction system positioned between Calculus/Linear Algebra and Scientific Computing/Numerical Analysis. The structure is flexible enough to later support:

* Python companion notebooks,
* algorithm reference appendices,
* HTML textbook conversion,
* computational labs,
* scientific-computing exercises,
* data-science integration,
* engineering simulation supplements.
