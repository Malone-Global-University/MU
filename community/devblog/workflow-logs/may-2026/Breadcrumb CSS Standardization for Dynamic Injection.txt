WORKFLOW NOTE

Title
Breadcrumb CSS Standardization for Dynamic Injection

Date
2026-05-17

Area
MGU website components: `/component/10-breadcrumbs.css`, breadcrumb injection markup, and dynamic breadcrumb JavaScript.

Problem
Breadcrumbs that previously appeared centered under the hero shifted to the far left after the site CSS was split into multiple files. The old static HTML page still displayed correctly, but the newer dynamic placeholder version using `<nav class="breadcrumb" aria-label="Breadcrumb" id="breadcrumb"></nav>` did not align correctly. Separator marks were also hardcoded white, making them visible in dark mode but weak or invisible in light mode. Some separator styling also risked being underlined because earlier approaches attached separators to links.

Intended Goal
Breadcrumbs should use one clean standard across the site. The modern system should rely on a single semantic breadcrumb placeholder, dynamically inject breadcrumb links from the URL path, center the breadcrumb row under the hero, use theme-aware separator colors, and avoid manual static breadcrumb typing.

Root Cause
Breadcrumb styles were duplicated across multiple CSS files after the CSS split. Older working styles remained in one file, while the newer breadcrumb file lacked some of the alignment behavior. The CSS also mixed support for multiple breadcrumb structures: old inline static breadcrumbs, generated list breadcrumbs, separator pseudo-elements, and newer dynamic placeholders. This made the system harder to maintain and caused inconsistent rendering between old and new pages.

Steps Taken

1. Compared the old working static breadcrumb page with the newer dynamic placeholder page and confirmed that the problem was not the semantic `<nav>` element itself but conflicting or incomplete CSS.
2. Identified that breadcrumb CSS had existed in multiple places, including base/navigation styles and `10-breadcrumbs.css`, creating a split source of truth.
3. Recommended making `/component/10-breadcrumbs.css` the only breadcrumb styling file and removing breadcrumb rules from other CSS files.
4. Reviewed several possible breadcrumb systems: fully static breadcrumbs, lightweight URL-generated breadcrumbs, and a heavier injected system with broader legacy support.
5. Rejected static breadcrumbs as unsuitable for MGU because the site needs scalable dynamic breadcrumb generation.
6. Confirmed the current preferred HTML placeholder as: `<nav class="breadcrumb" aria-label="Breadcrumb" id="breadcrumb"></nav>`.
7. Reviewed the existing JavaScript approach, which builds an ordered list inside the breadcrumb `<nav>` using URL path parts.
8. Narrowed the CSS recommendation to modern-only styling for the actual generated structure: `.breadcrumb > ol > li > a`.
9. Removed unnecessary legacy CSS concepts from the preferred direction, including inline breadcrumb support, `.sep`, `.breadcrumb-separator`, and old `a::after` separator logic.
10. Noted a JavaScript typo in `titleCaseSlug()` where `.replace(/[-_]/g,)` should be `.replace(/[-_]/g, " ")`.

Key Change
Breadcrumb styling should now be centralized in `/component/10-breadcrumbs.css` and written only for the modern dynamically injected `<nav id="breadcrumb" class="breadcrumb"><ol>...</ol></nav>` structure.

Outcome
The breadcrumb system now has a clearer path forward: one semantic placeholder, one dynamic JavaScript builder, and one authoritative CSS file. Centering is fixed, separator color can follow `var(--text)` for light/dark themes, and the separator is generated through list item CSS rather than being manually typed or attached to link text.

Validation
The centering issue was visually confirmed as solved using the heavier CSS version. The current refinement keeps the working alignment behavior while simplifying the stylesheet to match the actual JavaScript output. The CSS logic was checked against the generated structure from the provided JavaScript: a `<nav>` containing an `<ol>` with `<li>` breadcrumb items.

Rules Going Forward
All breadcrumb CSS should live only in `/component/10-breadcrumbs.css`.
All pages should use the semantic placeholder `<nav class="breadcrumb" aria-label="Breadcrumb" id="breadcrumb"></nav>`.
Breadcrumbs should be dynamically generated from the URL path rather than manually written.
Do not reintroduce static breadcrumb markup as the long-term standard.
Do not hardcode separator color as white; use theme variables such as `var(--text)` with opacity.
Do not add legacy support unless there is a specific migration need.
Avoid styling separators through linked anchor pseudo-elements if that causes underline or hover problems.

Notes
The current JavaScript is acceptable as the working breadcrumb generator. The main needed JavaScript correction is changing `.replace(/[-_]/g,)` to `.replace(/[-_]/g, " ")` so slugs such as `computer-science` render as `Computer Science`. The next clean-up step is to replace the heavier temporary breadcrumb CSS with a shorter modern-only version that styles only `.breadcrumb`, `.breadcrumb ol`, `.breadcrumb li`, `.breadcrumb li:not(:last-child)::after`, and `.breadcrumb a`.
