WORKFLOW NOTE

Title
MGU Library Guide Page Planning

Date
2026-05-17

Area
MGU Library / Guide Pages / Site Information Architecture

Problem
The MGU website needed a clearer guide-page structure for explaining how readers should use the Library. The Library already had multiple functional areas, such as Dictionary, Encyclopedia, Fiction, Guides, Malonicus Rex Educare, and Reference, but there was not yet a defined guide system to explain what each area is for or how users should navigate them.

Intended Goal
Create a practical guide-page framework for the MGU Library that helps students, readers, teachers, and researchers understand the purpose of each Library section and choose the correct resource type.

Root Cause
The Library was growing as a large content system, but its help and orientation pages had not yet been separated into a dedicated guide area. Without guide pages, users could see the Library categories but might not understand the difference between a dictionary entry, encyclopedia article, reference page, fiction page, or instructional guide.

Steps Taken

1. Identified `/library/guides/` as the proper location for a dedicated guide-page section.
2. Proposed core guide files, including `index.html`, `how-to-use-the-library.html`, `library-navigation-guide.html`, `dictionary-guide.html`, `encyclopedia-guide.html`, `fiction-guide.html`, `reference-guide.html`, `malonicus-rex-educare-guide.html`, `study-paths-guide.html`, `research-guide.html`, `citation-guide.html`, and `student-orientation.html`.
3. Defined the six main Library areas as functional categories: Dictionary, Encyclopedia, Fiction, Guides, Malonicus Rex Educare, and Reference.
4. Recommended that guide pages stay functional and instructional rather than becoming academic encyclopedia articles.
5. Established a reusable guide-page outline with a hero section, breadcrumb navigation, purpose section, explanation of Library areas, navigation guidance, and advice for choosing the correct resource type.
6. Suggested a first production batch of six pages: guide index, general Library guide, dictionary guide, encyclopedia guide, reference guide, and research guide.

Key Change
The MGU Library guide pages were framed as a dedicated instructional support system under `/library/guides/`, separate from encyclopedia articles, lessons, and reference materials.

Outcome
The Library now has a clearer plan for user-orientation pages that explain how the major Library sections work. This gives the site a scalable help structure that can support new users as the Library grows.

Validation
The proposed structure matches the existing MGU information architecture by keeping Guides as one of the Library’s functional areas. The page list also supports the current Library taxonomy without interfering with Encyclopedia, Dictionary, Fiction, Malonicus Rex Educare, or Reference content.

Rules Going Forward
Guide pages should explain how to use the site, not duplicate encyclopedia content.
Guide pages should answer what an area is for, how it is organized, and how a user should use it.
The `/library/guides/` folder should serve as the home for Library orientation, research help, navigation help, citation help, and study-path guidance.
The guide index should be built before specialized guide pages so the section has a clear entry point.
Guide pages should remain practical, readable, and student-facing.

Notes
The first guide-page batch should probably include only the most useful orientation pages before expanding into more specialized guides. A later phase can add pages for citation style, research workflows, teacher use, student study paths, and subject-specific Library navigation.
