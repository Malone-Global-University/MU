WORKFLOW NOTE

Title
MGU Physics Chapters 35–57 and Chapter 15 Semantic HTML Conversion

Date
2026-05-17

Area
MGU High School Physics textbook conversion pipeline; PDF-to-semantic-HTML workflow; Chapters 15 and 35–57.

Problem
The remaining MGU Physics chapters existed as stable PDF outputs from the textbook production pipeline, but they still needed to be converted into clean semantic HTML for web use. The earlier concern was that HTML conversions could drift if each chapter received its own styling decisions or embedded CSS. The user wanted the chapters converted as structure-only HTML so a shared stylesheet could control final presentation.

Intended Goal
Convert the uploaded physics PDF chapters into CSS-free semantic HTML files, package them as zipped downloads, and preserve the textbook structure without adding embedded styles. The correct output should be suitable for later MGU stylesheet integration and should use consistent semantic elements for chapters, sections, headings, paragraphs, lists, formulas, callouts, tables, key terms, review questions, problem sets, and checkpoints.

Root Cause
The PDFs were not the problem; they were consistent because they came from a stable shared-library production pipeline. The main issue was downstream conversion. If each PDF were treated as a separate design task, the HTML could acquire inconsistent CSS, class names, or layout assumptions. The better approach was to treat each PDF as a deterministic source artifact and convert it mechanically into a stable semantic structure.

Steps Taken

1. Confirmed that the batch conversion workflow should use CSS-free semantic HTML only, with styling deferred to shared external stylesheets.

2. Converted Chapters 35–37 into semantic HTML and bundled them as `mgu_physics_semantic_html_chapters_35_37.zip`.

3. Converted Chapters 38–42 into semantic HTML and bundled them as `mgu_physics_semantic_html_chapters_38_42.zip`.

4. Converted Chapters 43–47 into semantic HTML and bundled them as `mgu_physics_semantic_html_chapters_43_47.zip`.

5. Converted Chapters 48–52 into semantic HTML and bundled them as `mgu_physics_semantic_html_chapters_48_52.zip`.

6. Converted Chapters 53–57 into semantic HTML and bundled them as `mgu_physics_semantic_html_chapters_53_57.zip`.

7. Converted Chapter 15 separately into semantic HTML and bundled it as `mgu_physics_chapter_15_semantic_html.zip`.

8. Used a repeatable conversion script pattern that extracts PDF text, removes repeated headers/footers, detects chapter metadata, section headings, callouts, formulas, problem lists, key terms, and backmatter sections.

9. Preserved MathJax support in the HTML head while avoiding embedded CSS.

10. Used stable semantic structures such as `<article>`, `<header>`, `<section>`, `<h1>`, `<h2>`, `<p>`, `<ol>`, `<ul>`, `<dl>`, `<aside>`, and formula/table-caption class hooks.

Key Change
The conversion workflow was kept mechanical and semantic: no embedded CSS, no per-chapter design decisions, and no restructuring beyond preserving the textbook’s logical organization.

Outcome
Chapters 35–57 were converted in five zipped batches, and Chapter 15 was converted as an additional standalone zipped semantic HTML file. This effectively completed the later chapter conversion run and filled in an earlier chapter that needed corrected semantic HTML output.

The resulting files are ready for placement into the MGU web pipeline where external CSS can control appearance consistently across the textbook.

Validation
Each conversion produced a ZIP file and printed the generated HTML filenames after creation. The chapter titles and file names were verified during output generation, including:

`chapter-38-bernoulli-s-principle-semantic.html`
`chapter-43-series-and-parallel-circuits-semantic.html`
`chapter-48-atoms-and-spectra-semantic.html`
`chapter-53-physics-in-engineering-semantic.html`
`chapter-57-computational-and-data-based-physics-semantic.html`
`chapter-15-potential-energy-and-conservation-of-energy-semantic.html`

The uploaded PDF sources were cited in each delivery response, and the generated ZIP links were returned for download after each batch.

Rules Going Forward
Do not add embedded CSS to semantic HTML conversions.

Do not redesign each chapter during conversion.

Use the existing stable PDF/DOCX structure as the authority for chapter organization.

Preserve section numbering, backmatter, examples, callouts, tables, problem sets, key terms, and checkpoints.

Use stable class hooks only where useful for later stylesheet targeting.

Package multi-chapter conversions as ZIP files.

For best reliability, continue converting in batches of about five chapters unless a chapter is unusually large or needs individual handling.

Notes
The workflow confirms that the MGU Physics pipeline is now operating as a layered production system: Claude or another AI can produce structured chapter source, the shared library renders deterministic DOCX/PDF outputs, and downstream conversion can produce semantic HTML without spending additional AI effort on redesign.

The final HTML may still need a future polishing pass if the user wants fully reconstructed tables rather than preserved table text/caption structures, but the current files satisfy the immediate goal: CSS-free semantic HTML suitable for stylesheet-based web publication.
