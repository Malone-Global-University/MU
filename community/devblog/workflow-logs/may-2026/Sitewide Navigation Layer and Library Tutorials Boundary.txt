WORKFLOW NOTE

Title
Sitewide Navigation Layer and Library Tutorials Boundary

Date
2026-05-17

Area
MGU site architecture, navigation structure, library taxonomy, tutorials, reference organization

Problem
Malone Global University has grown to more than 11,000 pages, but users did not yet have a dedicated human-readable navigation layer explaining how the whole site is organized. The existing Library collection structure created uncertainty about whether navigation explanations belonged under Reference, Guides, Tutorials, or as a separate sitewide system. Tutorials also needed clearer placement because some tutorial pages are practical support pages, not full class lessons.

Intended Goal
Define a clear structure where users can understand the full MGU site layout without searching blindly through thousands of pages. The correct state is a dedicated navigation layer for sitewide orientation, while Tutorials remains a Library collection for practical task-based instruction not yet covered by formal classes.

Root Cause
The site outgrew a simple folder-and-menu model. At smaller scale, users could infer structure from menus, indexes, and breadcrumbs. At 11,000+ pages, the site needs an explanatory map that describes top-level domains, subdomains, departments, collections, and pathways. The confusion came from trying to force a sitewide orientation function into narrower content categories such as Reference or Tutorials.

Steps Taken

1. Reviewed whether navigation guides should be placed under Reference, Guides, Tutorials, or a separate top-level location.

2. Clarified that Reference should remain a lookup and verification area for tables, formulas, standards, specifications, source lists, and other quick-consultation materials.

3. Clarified that Tutorials should remain inside the Library for practical, task-based help not yet covered by formal classes or lessons.

4. Defined Tutorials as supplemental and potentially temporary: if a tutorial becomes part of a complete instructional sequence, it can later be moved into the appropriate class or department area.

5. Identified that the navigation layer must explain the entire MGU site, including Main, Community, Department, Library, and Malone Panther, so it should not be buried inside the Library.

6. Recommended a dedicated top-level Navigation area, such as `/navigation/`, to serve as the human-readable map and explanation layer for the full site.

7. Established a functional distinction between major content areas: Navigation explains where things are; Tutorials explain how to do tasks; Reference supports lookup; Dictionary defines terms; Encyclopedia explains subjects; Departments and classes provide formal instruction.

Key Change
MGU should use a dedicated top-level Navigation layer for whole-site orientation while keeping Tutorials inside the Library as supplemental practical instruction.

Outcome
The site architecture now has a clearer separation of responsibilities. Navigation becomes the user-facing map for the full 11,000+ page MGU system, while Library Tutorials remains a focused place for practical setup, workflow, and task-based pages that do not yet belong in formal classes. This reduces ambiguity and improves user experience by preventing navigation help, reference lookup, and tutorials from being mixed together.

Validation
The structure was checked against the actual purpose of each area. Since the navigation layer must explain multiple top-level domains, it does not belong inside Reference or Library Tutorials. Since Tutorials covers practical tasks such as software setup or focused procedures, it fits inside the Library as a supplemental support collection. The final model keeps each category aligned with a distinct user need.

Rules Going Forward
Navigation pages should explain site structure, major domains, subdomains, subject paths, user routes, and where to find information.

Tutorials should be task-based, practical, and supplemental to formal classes.

Reference should remain dedicated to quick lookup, verification, standards, tables, formulas, and structured support materials.

Formal course lessons should remain the primary home for complete instructional sequences.

If a tutorial grows into a full class lesson, move it into the appropriate department or course area.

Do not bury whole-site orientation pages inside a narrow collection when they explain the entire MGU system.

Notes
Recommended top-level navigation folder:

`/navigation/`

Recommended tutorials folder:

`/library/tutorials/`

Working distinction:

Navigation = where things are and how the site is organized.
Tutorials = how to complete practical tasks.
Reference = quick lookup and verification.
Dictionary = precise definitions.
Encyclopedia = subject explanation.
Department/classes = formal instructional pathways.
Library = organized knowledge collections.
