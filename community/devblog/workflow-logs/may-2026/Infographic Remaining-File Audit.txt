WORKFLOW NOTE

Title
Infographic Remaining-File Audit

Date
2026-05-17

Area
MGU How Things Work Infographic Collection

Problem
The infographic collection was growing faster than it could be tracked manually. A suspected “remaining files to make” list existed, but some of those files had already been created. The actual folder printout needed to be compared against the suspected list so duplicate work could be avoided.

Intended Goal
Identify which suspected infographic files were already present, which were near-duplicates by concept or naming, and which files still appeared to be missing from the actual collection.

Root Cause
The production pace exceeded the tracking system. Multiple batches had been generated across many categories, while folder names, file names, and category boundaries were not yet fully normalized. This made it hard to trust the “remaining” list without checking it against the actual directory inventory.

Steps Taken

1. Reviewed the suspected remaining infographic list by category, including appliance, agriculture, transportation, aerospace, energy, water, civil infrastructure, mechanical systems, defense/security, medical, computing, communications, materials science, manufacturing, environmental systems, public health, navigation, and sound/media topics.
2. Reviewed the actual PowerShell directory printout for `C:\MGU\image\infographic\how-things-work-series`, including existing folders and `.webp` filenames.
3. Compared suspected filenames against actual filenames using exact filename matching first.
4. Identified exact overlaps, including `how-a-transformer-works.webp`, `how-a-water-treatment-plant-works.webp`, and `how-a-smoke-detector-works.webp`.
5. Identified near-equivalent overlaps where the concept was already represented under a slightly different filename, such as `how-a-metal-detector-works.webp` versus `how-metal-detectors-work.webp`, and `how-a-storm-drain-system-works.webp` versus `how-a-storm-drainage-system-works.webp`.
6. Flagged possible concept overlap for router and modem files, since the suspected communications files had more specific names while the actual collection already contained general router and modem infographics.
7. Produced a cleaned missing-files list organized by category, removing confirmed duplicates and leaving the remaining candidates for future infographic production.
8. Noted cleanup issues in the existing folder structure, including the misspelled `material-sciience` folder and typo filenames in the bridges folder.

Key Change
The suspected remaining list was converted into a more reliable production checklist by removing confirmed overlaps and separating exact matches from near-equivalent and possible concept duplicates.

Outcome
The user now has a clearer view of which infographic files still need to be made and which should not be repeated. The comparison reduced duplicate production risk and exposed naming cleanup tasks that should be corrected before the collection expands further.

Validation
The audit was validated by comparing the uploaded suspected remaining list against the uploaded actual file printout. Confirmed overlaps were based on exact filename matches, while near-overlaps were based on highly similar concepts and naming patterns.

Rules Going Forward
Use the actual folder inventory as the source of truth before starting a new infographic batch.
Check for exact filename matches first, then check for concept-level duplicates with slightly different names.
Keep folder names standardized before expanding the collection further.
Avoid creating new category folders unless they are part of the cleaned taxonomy.
Fix spelling mistakes and duplicate-style filenames early, because they become harder to correct after index pages and links are generated.

Notes
