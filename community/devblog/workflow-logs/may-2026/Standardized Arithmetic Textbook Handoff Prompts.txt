Title
Standardized Arithmetic Textbook Handoff Prompts

Date
2026-05-17

Area
MGU Library / Textbooks / Mathematics / Arithmetic and Pre-Algebra Foundations

Problem
The existing Arithmetic 74 project was already functioning as an ebook-style online course, but it needed a clear path for becoming a longer MGU Library textbook source. The first proposed outline risked duplicating or replacing work that had already been completed. The project needed to preserve the stable 5-part, 14-unit, 73-lesson course structure while creating prompts Claude could use to draft fuller textbook chapters.

Intended Goal
The system should treat Arithmetic 74 as the existing course edition and use it as the spine for a longer textbook edition titled Arithmetic and Pre-Algebra Foundations. The textbook version should be organized into 14 chapters based on the 14 course units, with the existing lessons becoming chapter sections. Claude should receive stable, repeatable prompts that do not confuse or destabilize the chapter-writing workflow.

Root Cause
The initial planning treated Arithmetic and Pre-Algebra Foundations as a new textbook from scratch before recognizing that the user had already completed a compressed but accurate HTML ebook-course with lessons, reviews, tests, front matter, and back matter. The production plan needed to shift from invention to expansion.

Steps Taken

1. Reviewed the existing Arithmetic 74 course structure, including front matter, 5 parts, 14 units, 73 lessons, 14 unit reviews, 5 part tests, cumulative review, final exam, answer key, glossary, formula sheet, conversion chart, multiplication table, and symbols reference.
2. Reclassified the existing work as the “course edition” or ebook-course hybrid, rather than as a draft to be replaced.
3. Defined the new Library version as Arithmetic and Pre-Algebra Foundations, a fuller source textbook based on the same spine.
4. Converted the 14 course units into 14 textbook chapters while preserving the original instructional order.
5. Drafted chapter outline structures for all 14 chapters, including purpose paragraphs, numbered sections, and back matter categories.
6. Wrote a master Claude handoff prompt explaining the MGU textbook style, audience, structure, tone, formatting, and chapter requirements.
7. Reviewed Claude’s delivered Chapter 1 and confirmed that it was strong enough to use as the baseline style.
8. Decided not to overcorrect Claude while it was producing stable output; optional refinements will be handled later through human editing.
9. Generated clean chapter prompts for Chapter 2, Chapter 3, and Chapters 4–14 so Claude can continue drafting the full book one chapter at a time.

Key Change
The project shifted from creating a new arithmetic outline to expanding the existing Arithmetic 74 ebook-course into a 14-chapter MGU Library textbook edition.

Outcome
The Arithmetic and Pre-Algebra Foundations project now has a stable production model: Arithmetic 74 remains the guided online course, while the Library textbook becomes the fuller source version. Claude has a consistent prompt format for writing each chapter, and the chapter sequence now matches the existing course instead of competing with it.

Validation
The uploaded course files confirmed the ebook structure and scope. The table of contents showed 5 parts, 14 units, 73 lessons, 14 unit reviews, and 5 part tests. Claude’s Chapter 1 output was reviewed and found to match the intended MGU textbook structure: purpose, opening question, learning objectives, key terms, numbered sections, worked examples, warnings, applications, error analysis, checkpoint, and looking-ahead note.

Rules Going Forward
Preserve the existing Arithmetic 74 sequence. Do not invent a competing arithmetic textbook spine. Treat the 14 units as the 14 Library textbook chapters. Let Claude continue with the current stable chapter style before making refinements. Add improvements later through hand editing rather than changing the prompt style midstream. Keep the course edition and Library textbook edition distinct but cross-linked.

Notes
Future edits can add standardized MGU Library Connection boxes, visual model placeholders, tighter learning objectives, and consistent HTML conversion patterns. These should be applied after Claude finishes the full draft so the writing pipeline remains stable.
