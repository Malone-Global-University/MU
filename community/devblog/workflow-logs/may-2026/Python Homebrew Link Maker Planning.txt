WORKFLOW NOTE

Title
Python Homebrew Link Maker Planning

Date
2026-05-17

Area
Computer Science / Applied Computing / Programming and Development / Python / Homebrew

Problem
MGU needed a small reusable Python script that could scan folders and generate HTML link lists for files. The first version grouped files by file type, but that layout was not ideal for infographic and resource folders where the subfolder itself represents the meaningful category. There was also no defined public location yet for sharing the script as part of the MGU website.

Intended Goal
Create a practical `link-maker.py` utility that can be run from PowerShell, ask what folder to scan, optionally include subfolders, and generate clean HTML link sections grouped by subfolder. Each section should use the folder name as the section title and contain a list of links to files in that folder. The script should also have a logical home inside the MGU Computer Science Python area as a shareable homebrew utility.

Root Cause
The original file-type grouping model worked for mixed file collections but flattened folder meaning when scanning organized collections such as `how-things-work-series`. Since MGU folders often already encode subject categories, grouping everything under “WebP Images” erased the useful taxonomy.

Steps Taken

1. Drafted a first `link-maker.py` script that runs from PowerShell, prompts for a folder path, asks whether to include subfolders, asks for an output HTML filename, scans files, groups them by type, and writes a full HTML page.

2. Revised the script logic so output sections are grouped by containing subfolder instead of file extension. Each folder becomes its own `<section>`, with a clean section id and an `<h2>` derived from the folder title.

3. Updated the expected output pattern from one large “WebP Images” section into category sections such as `Aerospace Images` and `Agriculture Images`, each with its own `<ul>` of links.

4. Corrected the example HTML closing structure by replacing the incorrect `</h2>` ending with the proper `</ul>` and `</section>` sequence.

5. Discussed placing the script in a public MGU code-sharing area rather than leaving it as a private local helper.

6. Selected a sensible site location for the script:
   `C:\MGU\department\computer-science\applied-computing\programming-and-development\python\homebrew\`

7. Proposed a simple folder structure for the homebrew section, including an `index.html`, a `link-maker` subfolder, `link-maker.py`, a `README.md`, and sample output.

Key Change
The link maker was redefined from a file-type index generator into a folder-category index generator, which better matches MGU’s subject-based directory structure.

Outcome
MGU now has a clear plan for a reusable Python utility that can generate categorized HTML link lists from local folders. The script is suitable for image collections, static-site resource indexes, file inventories, and future MGU code examples. A permanent home was also identified for sharing it publicly under the Computer Science Python Homebrew section.

Validation
The revised output was checked against the desired transformation: files from `aerospace-and-space-systems` and `agriculture` no longer appear in one combined “WebP Images” section, but instead appear under separate folder-based sections. The HTML structure was also reviewed for correct section, heading, list, and closing-tag order.

Rules Going Forward
Use folder-based grouping when the folder name represents the subject category. Use file-type grouping only when the scan target contains mixed file types in the same folder and the extension is the main organizing principle. Keep generated links relative to the scanned base folder so the output can be moved into static-site pages without hardcoded Windows paths. Ignore generated output files and common system files such as `Thumbs.db`, `desktop.ini`, and `.DS_Store`.

Notes
The `homebrew` path is a better fit than a generic `experimental-code` path because the script is a practical Python utility. Future tools that could live in the same section include `breadcrumb-maker.py`, `folder-list-maker.py`, `html-date-checker.py`, `missing-file-checker.py`, `image-index-maker.py`, and `sitemap-helper.py`.
