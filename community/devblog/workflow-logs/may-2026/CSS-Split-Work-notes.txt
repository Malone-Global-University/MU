Work Log — CSS Modular Split

The original single CSS file was reviewed and divided by responsibility instead of by page. Existing comment sections were used as the main guide so the split would remain predictable and easy to verify.

A new modular folder structure was created under:

/component/css/

The original CSS was separated into these files:

main.css
00-theme.css
01-base.css
02-header.css
03-navigation.css
04-hero.css
05-content.css
06-components.css
07-tables.css
08-utilities.css
09-responsive.css

The main.css file was made into the control file. It does not contain styling rules. It only imports the other CSS modules in the correct order.

Theme variables, light mode, and dark mode were moved into 00-theme.css.

Global reset rules, body styling, links, paragraphs, headings, lists, and the skip link were moved into 01-base.css.

Header rules, logo styling, school name styling, and the theme toggle button were moved into 02-header.css.

Main navigation, hamburger toggle styling, local navigation, breadcrumbs, pagination, and bottom page navigation were moved into 03-navigation.css.

Hero section rules were moved into 04-hero.css.

Main section layout, section headings, step grids, equation styling, and note styling were moved into 05-content.css.

Reusable interface components were moved into 06-components.css, including summary/details blocks, buttons, footer, back button, accordion, and accordion panels.

Table styling was moved into 07-tables.css.

Small helper classes, including color classes and spacing utilities, were moved into 08-utilities.css.

All media queries were grouped into 09-responsive.css so responsive behavior can be adjusted from one place.

After the files were created, they were bundled into a ZIP file named:

mgu_modular_css_bundle.zip
