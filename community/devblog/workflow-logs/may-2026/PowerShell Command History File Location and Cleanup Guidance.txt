WORKFLOW NOTE

Title
PowerShell Command History File Location and Cleanup Guidance

Date
2026-05-17

Area
PowerShell / Windows command history / PSReadLine user profile data

Problem
The location of PowerShell’s saved command history was unclear. The user needed to know where PowerShell stores previous commands and whether the history file could be safely deleted.

Intended Goal
Identify the exact PowerShell history file path, explain how to open and inspect it, and clarify whether deleting or clearing the file is safe.

Root Cause
PowerShell has more than one kind of history. `Get-History` only shows the current session’s temporary history, while persistent command history is saved by the PSReadLine module in a text file under the user’s AppData profile. This distinction can make the real saved-history file hard to locate.

Steps Taken

1. Identified the usual PSReadLine history file location:

```powershell
$env:APPDATA\Microsoft\Windows\PowerShell\PSReadLine\ConsoleHost_history.txt
```

2. Provided a command to reveal the exact history path on the local machine:

```powershell
(Get-PSReadLineOption).HistorySavePath
```

3. Provided commands to open, view, search, delete, clear, or back up the file:

```powershell
notepad "$env:APPDATA\Microsoft\Windows\PowerShell\PSReadLine\ConsoleHost_history.txt"
```

```powershell
Get-Content (Get-PSReadLineOption).HistorySavePath
```

```powershell
Select-String -Path (Get-PSReadLineOption).HistorySavePath -Pattern "Get-ChildItem"
```

```powershell
Remove-Item (Get-PSReadLineOption).HistorySavePath
```

```powershell
Clear-Content (Get-PSReadLineOption).HistorySavePath
```

```powershell
Copy-Item (Get-PSReadLineOption).HistorySavePath "$env:USERPROFILE\Desktop\ConsoleHost_history_backup.txt"
```

Key Change
The session clarified that PowerShell’s persistent command history is stored in the PSReadLine `ConsoleHost_history.txt` file and that it can be safely deleted or cleared without damaging PowerShell.

Outcome
The user now has the exact commands needed to locate, inspect, search, delete, clear, or back up the PowerShell command history file.

Validation
The guidance was validated conceptually by separating temporary session history from persistent PSReadLine history and by using PowerShell’s own setting command, `(Get-PSReadLineOption).HistorySavePath`, to identify the active file path instead of relying only on a hard-coded default.

Rules Going Forward
Use `(Get-PSReadLineOption).HistorySavePath` when locating the active PowerShell history file, because the exact path may vary by host, profile, or configuration. Treat `ConsoleHost_history.txt` as disposable user history data, not a required system file. Back it up first when command history might be useful later.

Notes
Deleting the file removes saved command recall across sessions but does not remove PowerShell itself, installed modules, scripts, profiles, or system configuration. The file may contain sensitive commands, paths, tokens, or accidentally typed secrets, so clearing it can be useful for privacy.
