WORKFLOW NOTE

Title
Main CSS Modular Split

Date
8:23 PM 2026-05-06

Area
Global CSS system / `/component/css/`

Problem
The main CSS file contained theme variables, base rules, header styles, navigation, hero styling, content layout, buttons, tables, utilities, and responsive rules in one large file. This made the stylesheet harder to maintain, scan, edit, and expand safely.

Intended Goal
Separate the CSS into modular files by responsibility while keeping one main stylesheet entry point for HTML pages.

Root Cause
The stylesheet had grown organically as new features were added. Different CSS responsibilities were grouped by comment blocks, but they were still stored in one file instead of being separated into maintainable modules.

Steps Taken

1. Reviewed the existing CSS and identified major responsibility groups.
2. Created a modular CSS structure under `/component/css/`.
3. Created `main.css` as the import controller file.
4. Moved theme variables and light/dark mode rules into `00-theme.css`.
5. Moved reset, body, typography, links, lists, and skip-link rules into `01-base.css`.
6. Moved header, logo, school name, and theme toggle rules into `02-header.css`.
7. Moved main navigation, hamburger menu, local navigation, breadcrumbs, pagination, and bottom navigation into `03-navigation.css`.
8. Moved hero section styling into `04-hero.css`.
9. Moved main section layout, headings, steps grid, equations, and notes into `05-content.css`.
10. Moved reusable UI pieces such as buttons, summary/details, footer, back button, accordion, and panels into `06-components.css`.
11. Moved table styling into `07-tables.css`.
12. Moved helper classes such as text colors and spacing utilities into `08-utilities.css`.
13. Moved all media queries into `09-responsive.css`.
14. Packaged the finished modular CSS files into `mgu_modular_css_bundle.zip`.

Key Change
The original single stylesheet was converted into a modular CSS system controlled by one `main.css` import file.

Outcome
The CSS is now easier to maintain because each file has a clear purpose. Future edits can be made to the correct module without searching through one large stylesheet.

Validation
The split preserved the original CSS rules and ordering through the import structure. The bundled files were generated and delivered as a downloadable ZIP.

Rules Going Forward
Keep baseline rules separate from feature rules.
Place new styles in the file that matches their responsibility.
Use `main.css` only for imports.
Keep responsive overrides in the responsive file unless a component requires a very specific local override.
Avoid putting page-specific experiments into the global baseline file.

Notes
The next useful improvement would be separating `footer` and `buttons` into their own files if those areas grow larger.
