WORKFLOW NOTE

Title
How Things Work Infographic Batch Expansion and ZIP Delivery

Date
2026-05-17

Area
MGU image library / how-things-work-series infographic folders

Problem
The image batch workflow had been producing inconsistent delivery behavior. Some batches were generated correctly as separate files, while earlier attempts sometimes produced collages, raw `/mnt/data/` paths, failed upload states, or ZIP files that were not immediately given as clickable download links. The folder structure also needed to follow the cleaned scalable taxonomy instead of mixing unrelated systems into broad miscellaneous folders.

Intended Goal
Each requested batch should produce separate `.webp` infographic files, place them under the correct `how-things-work-series/` category folder, package them into a ZIP bundle, and immediately deliver a clickable sandbox download link. No collages should be used for these series batches.

Root Cause
The image-generation workflow interpreted some “batch” requests as one combined visual instead of multiple separate assets. Packaging was also sometimes separated from link delivery, which caused delays and confusion. The category taxonomy was still being normalized, so some folder targets needed to be corrected into cleaner long-term names.

Steps Taken

1. Confirmed that all future How Things Work image batches should be separate files, not collages.

2. Key Change
The workflow was corrected so each requested infographic batch is treated as separate `.webp` files inside the proper category folder, then delivered as a ZIP download link.

Outcome
Multiple underbuilt and new How Things Work folders were expanded with consistent educational infographic assets. The site image library now has stronger coverage across mechanical systems, home appliances, agriculture, aerospace, energy, water infrastructure, civil construction, safety/security, medical technology, computing, materials science, manufacturing, environmental systems, public health, measurement, and media technology.

Validation
Each ZIP was created from the generated source images and delivered with a clickable sandbox link. Packaging steps converted the generated PNG assets into `.webp` files, placed them in the appropriate folder path, and zipped the batch for download. File counts were checked during packaging for several batches, and final links were delivered after ZIP creation.

Rules Going Forward
Do not make collages for How Things Work batches. Each listed topic should become its own separate `.webp` file. Use the cleaned folder taxonomy under `how-things-work-series/`. Deliver the ZIP download link immediately after packaging. Keep filenames slugged exactly as requested unless a correction is necessary for folder consistency. Use 10-file batches when possible, and keep every batch aligned with the larger MGU infographic series style.

Notes
Some batches initially produced only raw `/mnt/data/` paths before the final clickable link was requested. The preferred future response pattern is to generate the batch, package it, and return the clickable download link in the same completion step whenever possible.
