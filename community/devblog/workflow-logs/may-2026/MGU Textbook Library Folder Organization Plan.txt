WORKFLOW NOTE

Title
MGU Textbook Library Folder Organization Plan

Date
2026-05-17

Area
MGU textbook library / `C:\MGU\desk\books`

Problem
The textbook library contained many useful projects, but the folder structure was inconsistent. Textbooks were grouped under broad areas such as business, computer science, history, language skills, math, and science, but naming conventions varied across folders. Some folders included temporary status labels such as `-good`, some contained typos, and some production folders were mixed directly into subject folders instead of being separated by planning, source, schema, build, and final output stages. The uploaded tree showed examples such as `coumper science foundations`, `artificiall intelligence`, `combinotorics-good`, `environmnetal science`, `markdown-by josephmlaone253`, and miscellaneous `New folder` entries. 

Intended Goal
Create a predictable textbook library structure where top-level folders act as library shelves, each textbook has a consistent internal production structure, and temporary or typo-based folder names are cleaned up. The correct state is a library that separates subject shelves, textbook titles, planning files, Markdown source, schema files, build outputs, final exports, and archive material.

Root Cause
The library grew organically as different textbook projects were created at different times. Some folders reflected production status, temporary sorting decisions, or tool-specific output instead of permanent library organization. Because there was no single repeated structure for every textbook, project files, schemas, prompts, Markdown, generated outputs, and temporary folders became unevenly arranged.

Steps Taken

1. Reviewed the current `C:\MGU\desk\books` folder tree and identified the main existing categories: business, computer science, government/civics, history, language skills, math, science, misc, and shared schema folders.
2. Proposed a clean top-level shelf structure: `00-admin`, `01-humanities`, `02-history-and-social-science`, `03-business-and-economics`, `04-mathematics`, `05-science`, `06-computer-science-and-technology`, `07-professional-and-life-skills`, `98-archive`, and `99-shared-schema`.
3. Proposed a standard internal textbook structure using `00-planning`, `01-source`, `02-schema`, `03-build`, and `04-final`.
4. Mapped current textbook projects into the new shelves, including humanities, social science, business, mathematics, science, and computer science/technology.
5. Identified typo and cleanup fixes, including `coumper science foundations` → `Computer Science Foundations`, `artificiall intelligence` → `Artificial Intelligence Foundations`, `combinotorics-good` → `Combinatorics`, and `environmnetal science` → `Earth and Environmental Science`.
6. Recommended removing temporary labels such as `-good` from permanent folder names and tracking status inside each book folder instead.
7. Recommended preserving the specialized physics pipeline because it already contains a more advanced multi-format production structure with text, JavaScript, DOCX, PDF, HTML, and Markdown folders.
8. Recommended moving `Data Science Foundations`, `Machine Learning Mathematics`, and `Artificial Intelligence Foundations` from Mathematics into Computer Science and Technology because they function more like applied computing and AI curriculum than pure mathematics.
9. Provided a safe PowerShell starter script to create the new top-level shelf folders without moving existing files yet.

Key Change
The textbook library was redefined as a shelf-based publishing system where each textbook follows the same internal production structure.

Outcome
The library now has a clear proposed organization model. Subject areas can be migrated one at a time without disrupting existing work. Textbook projects will become easier to locate, maintain, build, convert, archive, and publish. Math keeps its ordered curriculum sequence, physics keeps its specialized production pipeline, and computer science gains the applied computing titles that were previously grouped under mathematics.

Validation
The proposed structure was validated against the actual folder tree provided from `C:\MGU\desk\books`. The plan accounted for existing subject folders, existing textbook titles, current Markdown/schema folders, chapter prompt bundles, physics build folders, and cleanup issues shown in the tree. The migration plan was intentionally staged so new shelf folders can be created first before any files are moved.

Rules Going Forward
Top-level folders should describe library shelves, not production states.
Textbook folders should use clean title names, not temporary status labels.
Every textbook should use the same internal structure: planning, source, schema, build, and final.
Typos should be corrected before the library grows larger.
Temporary folders such as `New folder` should be inspected, archived, or deleted.
Math curriculum folders may remain numbered when sequence matters.
Applied computing, data science, machine learning, and AI titles should live under Computer Science and Technology, not pure Mathematics.
Physics may keep a specialized pipeline when the production process requires it.

Notes
The safest next step is to create the new shelf folders first, then build a migration map before moving files. Math and physics should be migrated last because they have the most internal complexity.
