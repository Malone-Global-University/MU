WORKFLOW NOTE

Title
Department Template Structural Cleanup

Date
2026-05-08

Area
MGU HTML page template system
Department page base template
Shared semantic layout structure

Problem
The department HTML template contained several structural inconsistencies that could cause layout instability, semantic issues, styling conflicts, and future maintenance problems. The page accidentally opened the <main> element twice, reused the .hero class for unrelated sections, contained an incorrect heading closing tag (</h1> instead of </h2>), and lacked a clearly standardized section hierarchy. These issues increased the likelihood of rendering inconsistencies and CSS conflicts across department pages.

Intended Goal
Create a clean, reusable, semantically correct HTML template that can serve as a stable foundation for department pages across the MGU ecosystem. The template should support dynamic header/footer injection, breadcrumb navigation, internal page anchors, and scalable section organization without structural ambiguity.

Root Cause
The template evolved incrementally while adding new sections and injection logic, resulting in duplicated structural wrappers and inconsistent semantic patterns. Reusing presentation classes such as .hero outside their intended purpose also blurred the distinction between layout structure and visual styling.

Steps Taken

1. Removed the accidental duplicate <main> opening tag to restore valid page structure.

2. Corrected the malformed heading element in the “On This Page” section by replacing:


   <h2>On This Page</h1>


   with:


   <h2>On This Page</h2>


3. Replaced the secondary .hero usage with a dedicated semantic section:


   <section class="page-index">


   to prevent hero-specific CSS from affecting unrelated content blocks.

4. Added consistent section IDs for anchor navigation:

   * #overview
   * #departments
   * #resources

5. Connected “On This Page” ordered-list links to matching section IDs for reliable internal navigation.

6. Improved semantic grouping and indentation throughout the document for readability and long-term maintainability.

7. Preserved lightweight fetch-based injection logic for:

   * Header
   * Footer
   * Bottom navigation

8. Maintained dynamic uploaded/updated date injection using:

javascript
   document.body.dataset.uploaded
   document.body.dataset.updated


Key Change
The single most important correction was separating structural section roles from visual hero styling by limiting .hero usage to the primary page banner only.

Outcome
The department template is now structurally valid, semantically cleaner, easier to scale, and more predictable for future CSS development. Internal page navigation works correctly, section hierarchy is clearer, and the template is safer for reuse across hundreds or thousands of pages.

Validation

* Confirmed only one <main> element exists.
* Verified all headings properly open and close.
* Verified internal anchor links scroll correctly.
* Confirmed injected header/footer components still load successfully.
* Confirmed uploaded/updated dates populate correctly.
* Performed visual review to ensure the hero section styling no longer affects secondary content sections.

Rules Going Forward

* Use only one .hero section per page.
* Use exactly one <main> wrapper per document.
* Every navigable section must have a unique id.
* Avoid reusing visual utility classes for unrelated semantic purposes.
* Separate structural semantics from presentation styling whenever possible.
* Keep injected component logic lightweight and modular.

Notes
Future improvements may include:

* Automated breadcrumb generation from file paths
* Shared section utility classes for indexes and references
* Reusable HTML partials for “On This Page” blocks
* Standardized section naming conventions across all MGU page systems
* Automatic validation scripts for structural HTML consistency across the site
