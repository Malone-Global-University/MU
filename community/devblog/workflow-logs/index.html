Workflow Logs

Repeatable records of build sessions, fixes, decisions, and maintenance work.

Purpose

The Workflow Logs page preserves detailed records of work sessions. It is for documenting what happened, what was attempted, what worked, what failed, and what rule should guide future work.

This section is especially useful for large maintenance tasks where the same process may need to be repeated later.

What to Record

Workflow logs should be concrete and reproducible. They should describe the problem, intended goal, root cause, steps taken, key change, outcome, validation method, and rules going forward.

Useful entries may cover batch page creation, CSS-link rewrites, breadcrumb migrations, textbook conversion routines, image bundle production, file renaming, index generation, and deployment checks.

Suggested Entry Types

Session summaries, troubleshooting logs, migration records, production checklists, command histories, page-generation procedures, and post-fix notes all belong here.

When commands or scripts are involved, include enough detail to repeat the process safely without relying on memory.

Connection to the Devblog

Workflow Logs is the operational memory of MGU development. It helps prevent repeated mistakes and makes large site-building tasks easier to resume, audit, and improve.
