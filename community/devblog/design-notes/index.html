Design Notes

Layout, interface, navigation, and visual design decisions for MGU.

Purpose

The Design Notes page records decisions about the visual and structural design of Malone Global University. This includes page layout, color systems, hero sections, navigation patterns, typography, spacing, cards, lists, and accessibility improvements.

Use this section when the main concern is how a page looks, reads, feels, or guides the visitor through the site.

What to Record

Design notes should explain the reasoning behind a design decision. They should identify what was confusing, inconsistent, unattractive, inaccessible, or hard to maintain, then describe the new design rule.

Useful entries may include hero design standards, department color conventions, breadcrumb placement, index-page structure, course-link presentation, footer behavior, mobile layout changes, and dark-mode corrections.

Suggested Entry Types

Layout revisions, visual hierarchy changes, accessibility adjustments, theme corrections, color-palette notes, responsive-design decisions, and page-template improvements all belong here.

Entries should be written so the same design rule can be applied again on future pages without rethinking the problem from scratch.

Connection to the Devblog

Design Notes helps keep MGU visually consistent as the site grows. It turns one-time fixes into reusable design standards that can guide future departments, library pages, community pages, and course materials.
