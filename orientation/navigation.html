Navigation Guide

A practical guide to moving through Malone Global University, understanding the major site areas, and finding the right learning, reference, library, community, and support resources.

How to Use This Navigation Guide

Malone Global University is a large educational website with thousands of pages across departments, library collections, tutorials, reference materials, fiction projects, community pages, and special educational systems. This guide explains the main structure of the site so students, readers, teachers, and visitors can understand where to begin and how to move from one area to another.

The site is organized around function. Some areas are designed for formal study, some are designed for reference, some are designed for exploration, and some are designed to explain how MGU itself works. The goal of this page is to make the site feel less like a maze and more like a campus map.

If you are new to Malone Global University, begin with the overview below. If you already know what you are looking for, use the main sections to jump directly into the area that best matches your purpose.

How the Site Is Organized

The MGU website is built as a layered structure. The top-level areas help visitors choose the kind of resource they need. Inside each top-level area, pages are grouped by topic, function, subject, or collection. This makes it possible for the site to grow without forcing every page into one giant menu.

A visitor should usually begin by asking a simple question: “What am I trying to do?” If the goal is to study a subject, the department area is usually the best place to start. If the goal is to look something up, the library is usually the best place to start. If the goal is to learn how to perform a practical task, tutorials may be the best place to start. If the goal is to understand the website itself, orientation is the correct area.

This functional structure allows Malone Global University to support several kinds of learning at the same time: formal study, independent reading, research, quick reference, guided tutorials, and exploration.

Library

The library area is the organized knowledge system of Malone Global University. It is designed for lookup, reading, research, reference, exploration, and long-term knowledge organization.

The MGU Library is not limited to one kind of material. It can contain dictionaries, encyclopedias, guides, reference works, fiction, educational collections, and special projects. The library is best understood as the place where knowledge is stored, classified, explained, and made searchable.

Use the library when you want to look up a concept, read an article, explore a discipline, find a reference page, browse a collection, or study a topic outside the structure of a formal course.

Major Library Areas

  • Dictionary

    A structured reference system for definitions, terms, phrases, and specialized vocabulary used across the MGU website.

  • Encyclopedia

    A broad article-based knowledge system organized by domains, disciplines, subdisciplines, and individual topics.

  • Fiction

    A creative library area for stories, literary projects, fictional worlds, characters, and narrative collections.

  • Guides

    Practical and explanatory guide pages that help readers understand systems, processes, resources, and learning pathways.

  • Malonicus Rex Educare

    A special educational collection connected to the MGU learning ecosystem, creative identity, and educational storytelling structure.

  • Reference

    A reference area for tables, charts, quick facts, structured data, study aids, and lookup resources.

  • Tutorials

    The tutorials area is for practical instruction. It can include software setup guides, technical walkthroughs, computer-use help, applied skill pages, and other resources that teach a specific task.

    Tutorials are placed at a major navigation level because they serve a different purpose from textbooks, encyclopedia articles, and dictionary entries. A tutorial is often about doing something: installing a tool, using a program, setting up a workflow, solving a practical problem, or completing a technical process.

    Some tutorial topics may eventually move into formal courses if they become part of a stable class pathway. Other tutorials may remain independent because they are useful as quick practical references.

News

This area is separate from the formal academic and library systems because it serves a different role. It helps give the site personality, continuity, and creative identity while supporting the larger educational mission.

Recommended Starting Points

Different visitors should begin in different places depending on what they want to accomplish.

  • New visitors

    Start with the main page and this orientation section. These pages provide the broadest overview of the site before you move into departments, library collections, or tutorials.

  • Students

    Start in the department area if you want structured lessons, textbook pathways, academic subjects, or course-style learning.

  • Researchers and readers

    Start in the library if you want articles, definitions, references, guides, or organized knowledge collections.

  • People learning a practical task

    Start in tutorials if you need step-by-step help with software, technical workflows, setup instructions, or applied skills.

  • Returning users

    Use breadcrumbs, section indexes, and bottom navigation to move between related pages without needing to return to the homepage each time.

Navigation Tips

Breadcrumbs show where a page sits inside the site. They are especially useful on deep pages because they let you move back up to a parent section, collection, department, or index.

Page indexes are useful when you want to browse a group of related pages. A department index may show courses or subjects. A library index may show collections or topics. A tutorial index may show practical guides grouped by task or tool.

Bottom navigation (bottom-navs) helps connect nearby pages and gives visitors another way to continue after finishing a page. On a large website, repeated navigation tools help prevent dead ends. Bottom-navs contain a back to top link which return s the user to the top of the page where they can click breadcrumb lonks to return to previous pages.

Using a keyboard and holding the alt key they pressing back or forward arrow with go to the previous page or next pages visited. Using a mouse with side click buttons is faster and achieves the same effect.

Bookmarking frequently visited pages, such as the home page, saves time. Users should also bookmark pages that are especially deep to save time and bandwidth navigating to them. There are many different ways to bookmark a page that are covered in computing basics here → Bookmarking basics.

Manual entry can always be used to visit a webpage by typing in the full url in the address bar of a browser. For advanced users this site is designed so that urls end with forward slashes instead of old-school ".html" so that they can type an ending partial url quickly. This is helpful in some contexts for people who are faster using keyboard inputs than mouse clicks.
