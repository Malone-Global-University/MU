I see the problem üëÄ ‚Äî your hamburger click works, but the mobile menu never shows because of a CSS conflict:

Right now you have two different mechanisms hiding/showing the menu:

.mobile-menu {
  display: none;   /* <--- This blocks it */
  flex-direction: column;
  ...
}
.mobile-menu.open {
  display: flex;
  max-height: 500px;
}


But since display: none; is declared in the base rule, toggling open sometimes doesn‚Äôt override properly (depending on cascade).

‚úÖ Fix: remove display: none; from .mobile-menu and let max-height handle the hide/show animation.

Here‚Äôs the corrected CSS section:

/* ===== MOBILE MENU ===== */
.mobile-menu {
  flex-direction: column;
  background: var(--card);
  position: absolute;
  top: 100%;
  left: 0;
  width: 100%;
  max-height: 0;
  overflow: hidden;
  transition: max-height 0.4s ease;
}
.mobile-menu.open {
  max-height: 500px; /* enough to fit all items */
}


That way:

Default = collapsed (max-height: 0).

When you click hamburger, JS toggles .open ‚Üí expands smoothly.

üí° Extra improvement: add a little z-index to the mobile menu so it doesn‚Äôt get hidden under content:

.mobile-menu {
  z-index: 999;
}
