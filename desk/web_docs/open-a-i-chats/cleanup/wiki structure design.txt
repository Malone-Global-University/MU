    }
  });

Right ‚Äî you can use class attributes for styling or identifying elements, but for jump links specifically, browsers need an id, not just a class.

Why id is needed for jump links
When you link to #something, the browser searches the page for an element with id="something".

It completely ignores class="something" for scrolling/jumping.

Common pattern you might be thinking of
Some devs do this:

<h2 class="jump-link">Web Hosting</h2>
‚Ä¶and then they run JavaScript at load time to turn the class into an ID:

document.querySelectorAll('.jump-link').forEach(el => {
  el.id = el.textContent.trim().toLowerCase().replace(/\s+/g, '-');
});
This way:

You mark any heading you want as a jump link using a class.

JS generates the matching id for you automatically.

When to use which
Direct id in HTML ‚Üí Best if you know the exact target name when writing content.

Class + JS-generated id ‚Üí Best if you want to quickly tag multiple elements without worrying about typing consistent IDs manually.
