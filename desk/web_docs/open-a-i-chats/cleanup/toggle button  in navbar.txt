ChatGPT said:
Yes — you can definitely put a toggle button in a navbar list like that, but there are two small issues in your snippet:

The last <li> is missing its closing </li> tag.

For consistent styling, you’ll probably want to make the <button> visually match your <a> links.

Here’s a cleaned-up example:

<header>
  <nav class="navbar">
    <div class="logo-container">
      <a href="/">
        <img src="/images/MU_logo.png" alt="Malone University Logo" class="logo" />
      </a>
      <span class="school-name">Malone University</span>
    </div>
    <ul class="nav-links">
      <li><a href="/">Home</a></li>
      <li><a href="/static/gayle_king">About</a></li>
      <li><a href="/static/jm">Founder</a></li>
      <li><a href="/static/mission">Mission</a></li>
      <li><a href="/static/hardened">FINAL OFFER</a></li>
      <li><a href="/departments/catalog">Catalog</a></li>
      <li>
        <button id="theme-toggle" class="nav-button">Toggle Dark/Light</button>
      </li>
    </ul>
  </nav>
</header>
Styling tip (to make the button blend with the other nav items):

.nav-links li .nav-button {
  background: none;
  border: none;
  color: inherit;
  font: inherit;
  cursor: pointer;
  padding: 0;
}

.nav-links li .nav-button:hover {
  text-decoration: underline;
}
This way, it still behaves like a button for accessibility but visually matches your links.
