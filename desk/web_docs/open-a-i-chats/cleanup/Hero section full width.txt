ChatGPT said:
Your .hero is set to width: 100%, but the reason it’s not stretching across the entire viewport visually is because your section selector later in the CSS adds this rule:

section {
  padding: 4rem 2rem;
  max-width: 900px;
  margin: 0 auto;
}
If your hero is inside a <section> tag, that max-width: 900px and margin: 0 auto are centering and constraining it.

So you’ve got two ways to fix it:

Option 1 — Remove the constraint for .hero

.hero {
  background: linear-gradient(135deg, #0f2027, #203a43, #2c5364);
  padding: 5rem 2rem;
  text-align: center;
  width: 100%;
  max-width: none; /* override the section max-width */
  margin: 0;       /* override the section margin */
}
Option 2 — Don’t wrap hero in <section>
Instead, use:

<div class="hero">
  <h1>Welcome</h1>
  <p>Your intro text...</p>
</div>
That way it won’t inherit the max-width: 900px rule.

I’m guessing your hero is currently inside a <section> so it’s being “boxed in” by that shared section style. If you want it truly edge-to-edge, you’ll need to break it free from those constraints.
