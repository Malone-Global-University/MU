Right now your toggle is working functionally ‚Äî the html element gets dark-mode or light-mode ‚Äî but your CSS only changes a few things between themes.

You never actually tell lists, tables, and section backgrounds to inherit the theme‚Äôs text and background color, so in light mode they stay dark or mismatched.

You can fix that by adding inheritable color rules and ensuring all block elements share the same background. Here‚Äôs a minimal addition to your CSS that will make dark/light mode look right everywhere:

/* Dark mode full coverage */
html.dark-mode,
html.dark-mode body,
html.dark-mode section,
html.dark-mode header,
html.dark-mode footer {
  background-color: #121212; /* off-black */
  color: #f0f0f0; /* light text */
}

html.dark-mode ol,
html.dark-mode li,
html.dark-mode table,
html.dark-mode th,

  color: inherit;

}

Why this fixes it:
Background inheritance: We explicitly set background-color on html, body, section, header, and footer, so no element keeps an old dark/light background.

List & table color: We use color: inherit so <li> and <th>/<td> match the mode‚Äôs text color.

No hardcoded colors: Except for theme backgrounds/text, everything else inherits.

If you drop this block at the end of your current CSS, your dark/light mode will look consistent ‚Äî headers, lists, and tables will all match.
