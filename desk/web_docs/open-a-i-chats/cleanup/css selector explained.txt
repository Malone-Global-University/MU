ChatGPT said:
Got it — you basically want to know “what selects what” in that CSS so you can understand which rules apply to which parts of the page.

Let’s break it into three layers:

1. Theme Variables
These are CSS custom properties (--bg, --text, etc.) set on :root and overridden in .dark mode.

:root { … }
Applies to the root element of the document (<html>), so all elements inherit these variables unless overridden.
These define your light mode defaults.

.dark, .dark-mode { … }
Applies whenever the <html> or any ancestor element has class dark or dark-mode.
These override the light mode variables to switch colors for dark mode.

How it’s used:
For example,

body { background: var(--bg); color: var(--text); }
will change depending on which set of variables is active.

2. Element & Class Selectors
These match elements in your HTML directly:

Selector	Matches	Purpose
body	<body>	Sets base font, background, padding
.container	Any element with class="container"	Limits width, centers content
header	<header> tag	Flex layout for top section
.brand	<a class="brand">	Logo + name styling
.logo	<div class="logo"> or <img class="logo">	Circular background gradient with shadow
h1	All <h1>	Title font size
p.lead	<p class="lead">	Subheading with muted color
section	<section>	Section padding/margin
.theme-toggle	Button with class="theme-toggle"	The dark/light toggle button
.hero	<div class="hero"> or <header class="hero">	Two-column grid layout for intro content
.card	Content card with padding, shadow	
.btn	Button styles	
.features	Container for .feature cards	
.feature	Individual feature card styling	
footer	<footer>	Small muted text
3. Navigation & Responsive
These target the navbar (from your /partials/navbar.html) and change depending on screen size.

.nav-toggle
The hamburger menu button. Hidden by default, shown on small screens.

.nav-links
The container for <li> menu links.
By default:

Desktop: flex row layout

Mobile: hidden unless .open class is added

Media queries:

@media (max-width: 880px) → Shrinks hero/features to 1 column, reduces padding

@media (max-width: 768px) → Hides nav links, shows hamburger menu, .open makes menu visible vertically

@media (min-width: 768px) → Forces nav links to be visible horizontally and hides hamburger

Key Connections
The theme colors are not hardcoded into elements — they’re pulled from --variables, so .dark or .light just swaps those variable values.

Layout is controlled by flexbox (header, nav) and grid (hero, features).

.open is a state class used in JavaScript to show/hide the mobile menu.

.dark on <html> is toggled by JS to switch theme.

what is controlling the text color of <p> tags?
ChatGPT said:
In your CSS, <p> tags get their text color from two places:

Global body rule

body {
  color: var(--text);
}
This sets the default color for all text, including <p>, to the CSS variable --text (light mode: #0b1320, dark mode: #e6eef6).

Overrides for special cases

.intro p { color: var(--muted); } → Applies only inside elements with class="intro".

p.lead { color: var(--muted); } → Applies to <p class="lead">.

section p { color: #e0e0e0; } → Applies to <p> inside a <section>. This rule overrides the body color in those sections because it’s more specific.

So for a normal <p>:

Outside a <section> → inherits var(--text) from body.

Inside a <section> → #e0e0e0 (light gray) from section p.

If it has .lead → var(--muted) instead.
