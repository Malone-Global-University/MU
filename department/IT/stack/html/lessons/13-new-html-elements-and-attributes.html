Overview

HTML5 introduced a set of semantic elements and utility elements that improve structure, accessibility, and expressiveness without extra ARIA. In this lesson we focus on four high-value items: <figure> / <figcaption>, <details> / <summary>, and <time>. You will learn when to use each, accessibility considerations, useful attributes, and practical examples you can drop into lessons or projects.

<figure> and <figcaption> â€” media with context

The <figure> element groups self-contained content (images, diagrams, code listings, tables) and the optional <figcaption> provides a caption or explanation. Use <figure> when the media has a caption or when the media can be considered a unit (it could be moved or syndicated).

Why use them?

  • They create a semantic association between media and its caption (better for screen readers and search).
  • They help document structure â€” a figure is a self-contained unit that can be referenced from text.

Example â€” image with caption

<figure>
  <img src="/images/campus-panorama.jpg" alt="Panoramic view of Malone campus at sunrise" width="1200" height="480">
  <figcaption>Panoramic view of Malone campus. Photo by the Communications team.</figcaption>
</figure>

Example â€” chart with long description

<figure>
  <img src="/images/revenue-chart.png" alt="Quarterly revenue chart; see long description below" width="800" height="400">
  <figcaption>Quarterly revenue, Q1â€“Q4. <a href="#chart-longdesc">Long description</a>.</figcaption>
</figure>

<p id="chart-longdesc">Long description: In Q1 revenue was ... (provide accessible long description or linked text).</p>

Accessibility notes

  • If an image is purely decorative, keep it out of the figure or use alt="" so assistive tech ignores it.
  • For complex images (diagrams, charts) provide a short alt and a longer human-readable description nearby or linked from the figcaption.
  • When inlining SVG in a figure, include accessible attributes (for example, role="img" + aria-labelledby or <title> within the SVG).

<details> and <summary> â€” progressive disclosure

The <details> element creates a native disclosure widget; the first child <summary> represents the summary/label that users click to expand or collapse the hidden content. Because this is native HTML, it works without JavaScript and is keyboard accessible by default.

Basic usage

<details>
  <summary>Show full assignment brief</summary>
  <p>Here is the full brief and additional resources for the assignment...</p>
</details>

Attributes and state

  • open â€” boolean attribute; when present the details start expanded (e.g., <details open>).
  • Use CSS to style the <summary> and the expanded state: details[open] { ... }.

Accessibility and behavior

  • The <summary> is focusable and receives keyboard events (Enter/Space toggles it).
  • Prefer native <details> to JavaScript-only accordions when possible â€” native semantics are simpler and more robust.
  • If you must build a custom widget for complex interactions, replicate keyboard behavior and ARIA roles (e.g., aria-expanded, proper focus management).

Example â€” FAQ item

<details>
  <summary>What is the refund policy?</summary>
  <p>Students may request a refund within 30 days of purchase. The policy details are...</p>
</details>

Progressive enhancement pattern

If you want animated open/close transitions, enhance the native control with a small script that toggles a CSS class, but keep the native <details> functionality as the baseline so it works without JS.

<time> â€” machine-readable dates & times

The <time> element expresses dates and times in a machine-readable way using the datetime attribute. This helps scripts, search engines, and assistive tech interpret temporal data consistently.

Examples

<!-- Date only -->
<time datetime="2025-08-20">August 20, 2025</time>

<!-- Date + time with timezone offset -->
<time datetime="2025-08-20T14:30:00-04:00">Aug 20, 2:30 PM (EDT)</time>

<!-- Duration or time range hint (use text for human-readable form) -->
<time datetime="PT90M">90 minutes</time>

Why use <time>?

  • Search engines can extract event dates or publish dates more reliably.
  • Scripting and progressive enhancement (e.g., converting to user's local timezone) become straightforward.
  • Screen readers can announce dates more precisely when markup is explicit.

Practical pattern â€” published date & updated date

<p class="meta">Published: <time datetime="2024-11-01">Nov 1, 2024</time>
 – Updated: <time datetime="2025-08-20">Aug 20, 2025</time></p>

Other useful HTML5 attributes & small notes

While this lesson emphasizes the four named elements, remember that HTML5 also introduced practical global/useful attributes that pair well with them:

  • hidden â€” hides content visually and from assistive tech by default (use carefully).
  • download on anchors â€” hints to the browser that the link should download a resource.
  • placeholder (inputs), required, and other form attributes (covered in Lesson 15) â€” useful when creating figure captions that reference form inputs or embeds.

Accessibility checklist for these elements

  • Figure/Caption: Provide a meaningful alt for images and a concise <figcaption>. For complex images, link to a long description.
  • Details/Summary: Ensure summary is descriptive and the reveal content is focusable if contains interactive controls.
  • Time: Use the datetime attribute with ISO 8601 format (YYYY-MM-DD or YYYY-MM-DDThh:mm:ssZ).
  • Prefer native semantics over ARIA for these patterns; only add ARIA when extra semantics are required.

Practice

  1. Create 13-html5-elements.html using this lesson as the main content. Include:
    • One <figure> with an image and <figcaption> that links to a longer description elsewhere on the page.
    • At least two <details> items: one for an FAQ and one that contains a small form control (e.g., a subscribe checkbox). Ensure the form control is focusable when details are expanded.
    • Use <time datetime="..."> for a published and updated date and demonstrate a script-free, human-readable form.
  2. Test the page with keyboard-only navigation and a screen reader (or an automated accessibility tool). Confirm that:
    • The summary elements are focusable and toggle with Enter/Space.
    • Captions are announced in context and long descriptions are reachable.
    • Date values are machine-readable via the datetime attribute.

Key Takeaways

  • <figure> and <figcaption> group media and captions semantically â€” use them for images, charts, and diagrams with a caption.
  • <details> and <summary> provide native, accessible disclosure widgetsâ€”prefer them over custom JS accordions when appropriate.
  • <time> enables machine-readable dates/times via the datetime attribute and aids search, scripts, and assistive tech.
  • Use native HTML semantics first; enhance with CSS/JS only for presentation or improved UX while preserving underlying accessibility.
