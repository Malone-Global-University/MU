Using PowerShell with ImageMagick

PowerShell can control ImageMagick to convert, resize, rename, and organize images in bulk. This is useful for website icons, WebP files, infographics, thumbnails, and folder-wide image cleanup.

Overview

ImageMagick is an image-processing program that can convert, resize, crop, compress, and inspect image files. PowerShell is useful with ImageMagick because it can find many images in a folder tree and send them through ImageMagick commands one after another.

This combination is especially useful for website work. A large website may contain PNG files, JPG files, WebP files, icons, diagrams, logos, and generated images. Instead of opening every image manually, PowerShell can automate repeated image tasks.

For Malone Global University, PowerShell and ImageMagick can help convert PNG images to WebP, create 256 by 256 icon files, standardize infographic formats, resize oversized images, and build reports of image folders.

What ImageMagick Does

ImageMagick works from the command line. It can read an input image, apply instructions, and write a new output image. For example, it can take a PNG file and create a WebP file from it. 

magick input.png output.webp

In this example, magick is the ImageMagick command. input.png is the source file. output.webp is the new file ImageMagick creates.

Check Whether ImageMagick Is Installed

Before using ImageMagick from PowerShell, check whether the magick command is available. 

magick -version

If ImageMagick is installed correctly and available in the system path, this command shows version information. If PowerShell says the command is not recognized, ImageMagick may not be installed or may not be available through the PATH environment variable.

Convert One Image

To convert one PNG file to WebP, use: 

magick "C:\MGU\image\example.png" "C:\MGU\image\example.webp"

Quotation marks are important because paths may contain spaces. Even when paths do not contain spaces, quotation marks are a safe habit.

Convert an Image in the Current Folder

If PowerShell is already inside the folder that contains the image, you can use shorter paths. 

Set-Location -Path "C:\MGU\image"

magick "example.png" "example.webp"

The first command moves PowerShell into the image folder. The second command converts example.png into example.webp.

Resize One Image

ImageMagick can resize an image while converting it. This example creates a 256 by 256 PNG copy. 

magick "input.png" -resize 256x256 "output.png"

This resizes the image so it fits within a 256 by 256 pixel box while preserving the aspect ratio. If the original image is not square, the result may not be exactly 256 by 256 in both dimensions.

Create an Exact Square Image

To create an exact square image, use resize, crop, and extent carefully. This example creates a 256 by 256 image with centered content. 

magick "input.png" -resize 256x256^ -gravity center -extent 256x256 "output.png"

The caret-like symbol after 256x256 tells ImageMagick to fill the target size. The -gravity center option keeps the crop centered. The -extent option sets the final canvas size.

Create a 256 by 256 ICO File

ICO files are often used as Windows icons or website favicons. To create a 256 by 256 ICO file from a PNG, use: 

magick "input.png" -resize 256x256 "output.ico"

For a square icon with centered cropping, use: 

magick "input.png" -resize 256x256^ -gravity center -extent 256x256 "output.ico"

This is useful for creating folder icons, site icons, or project icons from a larger source image.

Convert All PNG Files in One Folder to WebP

PowerShell can find every PNG file in a folder and convert each one to WebP. 

Get-ChildItem -Path "C:\MGU\image" -Filter "*.png" -File |
  ForEach-Object {
    $Output = Join-Path $_.DirectoryName ($_.BaseName + ".webp")
    magick $_.FullName $Output
  }

This command converts PNG files directly inside C:\MGU\image. It does not search subfolders unless -Recurse is added.

Convert PNG Files in Subfolders

To convert PNG files across a folder tree, add -Recurse. 

Get-ChildItem -Path "C:\MGU\image" -Filter "*.png" -File -Recurse |
  ForEach-Object {
    $Output = Join-Path $_.DirectoryName ($_.BaseName + ".webp")
    magick $_.FullName $Output
  }

This searches C:\MGU\image and all subfolders for PNG files, then creates WebP files in the same folders as the originals.

Avoid Overwriting Existing Files

Before creating new image files, check whether the output file already exists. This prevents accidental overwriting. 

Get-ChildItem -Path "C:\MGU\image" -Filter "*.png" -File -Recurse |
  ForEach-Object {
    $Output = Join-Path $_.DirectoryName ($_.BaseName + ".webp")

    if (!(Test-Path $Output)) {
      magick $_.FullName $Output
    }
  }

This converts only PNG files that do not already have a matching WebP output file.

Set WebP Quality

WebP output can use a quality setting. Higher quality usually means a larger file. Lower quality usually means a smaller file. 

magick "input.png" -quality 85 "output.webp"

A quality value around 80 to 90 is often a reasonable starting range for website images, but the best value depends on the image and the visual standard required.

Resize and Convert to WebP

ImageMagick can resize and convert in the same command. 

magick "input.png" -resize 1536x1024 "output.webp"

This creates a WebP file that fits within 1536 by 1024 pixels while preserving the original aspect ratio.

Batch Resize and Convert PNG to WebP

This command converts all PNG files in a folder tree to WebP and resizes them to fit within 1536 by 1024 pixels. 

Get-ChildItem -Path "C:\MGU\image\infographic" -Filter "*.png" -File -Recurse |
  ForEach-Object {
    $Output = Join-Path $_.DirectoryName ($_.BaseName + ".webp")

    if (!(Test-Path $Output)) {
      magick $_.FullName -resize 1536x1024 -quality 85 $Output
    }
  }

This is useful for standardizing infographic images for website use.

Send Converted Images to a Different Folder

Sometimes you do not want converted files placed beside the originals. You can send output files to a separate folder. 

$InputFolder = "C:\MGU\image\source"
$OutputFolder = "C:\MGU\image\webp-output"

if (!(Test-Path $OutputFolder)) {
  New-Item -Path $OutputFolder -ItemType Directory
}

Get-ChildItem -Path $InputFolder -Filter "*.png" -File |
  ForEach-Object {
    $Output = Join-Path $OutputFolder ($_.BaseName + ".webp")
    magick $_.FullName -quality 85 $Output
  }

This keeps original PNG files in one folder and converted WebP files in another folder.

Preserve Folder Structure in Output

For large projects, you may want the output folder to preserve the same subfolder structure as the input folder. 

$InputRoot = "C:\MGU\image\source"
$OutputRoot = "C:\MGU\image\webp-output"

Get-ChildItem -Path $InputRoot -Filter "*.png" -File -Recurse |
  ForEach-Object {
    $RelativePath = $_.FullName.Substring($InputRoot.Length).TrimStart("\")
    $RelativeFolder = Split-Path $RelativePath -Parent
    $OutputFolder = Join-Path $OutputRoot $RelativeFolder
    $OutputFile = Join-Path $OutputFolder ($_.BaseName + ".webp")

    if (!(Test-Path $OutputFolder)) {
      New-Item -Path $OutputFolder -ItemType Directory -Force | Out-Null
    }

    magick $_.FullName -quality 85 $OutputFile
  }

This is useful when converting a large image library while keeping its category folders organized.

Inspect Image Information

ImageMagick can inspect an image and show information such as format, size, colorspace, and file details. 

magick identify "C:\MGU\image\example.webp"

To inspect many images, combine Get-ChildItem with ImageMagick. 

Get-ChildItem -Path "C:\MGU\image" -Filter "*.webp" -File |
  ForEach-Object {
    magick identify $_.FullName
  }

Create an Image Inventory

PowerShell can create a basic image inventory without using ImageMagick. This report lists image files, paths, dates, and sizes. 

Get-ChildItem -Path "C:\MGU\image" -Include "*.png","*.jpg","*.jpeg","*.webp","*.ico" -File -Recurse |
  Select-Object FullName, DirectoryName, Name, Extension, LastWriteTime, Length |
  Export-Csv "C:\MGU\reports\image-inventory.csv" -NoTypeInformation

This is useful before conversion work because it shows what image files currently exist.

Find the Largest Images

Large image files can slow down websites. PowerShell can find the largest images so they can be reviewed or optimized. 

Get-ChildItem -Path "C:\MGU\image" -Include "*.png","*.jpg","*.jpeg","*.webp" -File -Recurse |
  Sort-Object Length -Descending |
  Select-Object -First 50 FullName, Extension, Length, LastWriteTime |
  Export-Csv "C:\MGU\reports\largest-images.csv" -NoTypeInformation

This creates a report of the largest images under the image folder.

Create ICO Files from PNG Files

This example creates 256 by 256 ICO files from PNG files in one folder. 

Get-ChildItem -Path "C:\MGU\icons-source" -Filter "*.png" -File |
  ForEach-Object {
    $Output = Join-Path $_.DirectoryName ($_.BaseName + ".ico")
    magick $_.FullName -resize 256x256^ -gravity center -extent 256x256 $Output
  }

This is useful for folder icons, local project icons, or favicon experiments.

Convert WebP to PNG

Sometimes a WebP image needs to be converted back to PNG for editing or compatibility. 

magick "input.webp" "output.png"

To convert many WebP files in one folder: 

Get-ChildItem -Path "C:\MGU\image" -Filter "*.webp" -File |
  ForEach-Object {
    $Output = Join-Path $_.DirectoryName ($_.BaseName + ".png")
    magick $_.FullName $Output
  }

Make a Backup Before Batch Conversion

Before running a large conversion or resizing task, make a backup of the image folder. This is especially important if output files may overwrite existing files. 

$Stamp = Get-Date -Format "yyyy-MM-dd_HH-mm-ss"
$BackupPath = "C:\MGU-backups\image_backup_$Stamp"

Copy-Item -Path "C:\MGU\image" -Destination $BackupPath -Recurse

This creates a timestamped backup of the image folder.

Preview the File List First

ImageMagick itself does not use PowerShell's -WhatIf in the same way that PowerShell file commands do. Instead, preview the files with Get-ChildItem before running the ImageMagick command. 

Get-ChildItem -Path "C:\MGU\image\infographic" -Filter "*.png" -File -Recurse |
  Select-Object FullName

After reviewing the file list, run the conversion command. This keeps the workflow controlled.

Test on a Small Folder First

Before converting hundreds of images, copy a few images to a test folder and run the command there. 

New-Item -Path "C:\MGU\image-test" -ItemType Directory

Copy-Item -Path "C:\MGU\image\sample.png" -Destination "C:\MGU\image-test\sample.png"

magick "C:\MGU\image-test\sample.png" -resize 1536x1024 -quality 85 "C:\MGU\image-test\sample.webp"

Open the output image and inspect quality before applying the command to a large folder tree.

Safe Habits

Always inspect the file list before batch conversion. Work on a test folder first. Avoid overwriting files unless that is clearly intended. Use output folders when possible. Make a backup before large image operations.

After conversion, open a few output images and inspect them visually. Also compare file sizes, dimensions, and locations. A command can complete successfully but still produce images that are too large, too small, too compressed, or stored in the wrong folder.

Summary

PowerShell and ImageMagick work well together. PowerShell finds, filters, loops through, and organizes files. ImageMagick converts, resizes, crops, and writes image outputs.

This combination is useful for converting PNG to WebP, creating ICO files, resizing infographics, inspecting image details, finding large images, and maintaining website image folders.

For large website projects, the safe workflow is to inventory first, back up important folders, test on a small batch, convert carefully, and verify the output.
