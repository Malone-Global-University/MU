Finding Recent Files

PowerShell can quickly locate files that were created or changed recently. This is useful when you misplaced a file, want to review today’s work, or need to inspect recent website changes.

Overview

Finding recent files is one of the most practical beginner uses of PowerShell. When a project contains many folders, it can be difficult to remember where a file was saved. PowerShell can search a folder tree and sort files by date so the newest items appear first.

This is especially useful for large website projects. If a file was created today, edited this week, or accidentally saved in the wrong folder, PowerShell can help locate it without manually opening many folders.

Modified Date and Created Date

Windows tracks several dates for files. Two of the most useful are the modified date and the created date.

The modified date shows when the file was last changed. The created date shows when the file was first created at its current location. For most recent-work searches, the modified date is the better place to start because it shows the files you edited most recently.

Show the Newest Files in a Folder

To show the newest files in a folder, use Get-ChildItem and sort the results by LastWriteTime. 

Get-ChildItem -Path "C:\MGU" -File |
  Sort-Object LastWriteTime -Descending

This lists files directly inside C:\MGU, with the most recently modified files at the top.

Search Subfolders Too

To search inside the folder and all folders beneath it, add -Recurse. 

Get-ChildItem -Path "C:\MGU" -File -Recurse |
  Sort-Object LastWriteTime -Descending

This can return many results in a large project. It is useful when you do not remember which subfolder contains the file.

Show Only the Top Results

Large folder trees can produce too much output. To show only the newest 20 files, pipe the sorted results into Select-Object. 

Get-ChildItem -Path "C:\MGU" -File -Recurse |
  Sort-Object LastWriteTime -Descending |
  Select-Object -First 20 FullName, LastWriteTime

This is one of the most useful commands for finding a misplaced file.

Find Files Modified Today

To find files modified today, compare each file’s LastWriteTime to today’s date. 

Get-ChildItem -Path "C:\MGU" -File -Recurse |
  Where-Object { $_.LastWriteTime -ge (Get-Date).Date } |
  Sort-Object LastWriteTime -Descending |
  Select-Object FullName, LastWriteTime

This searches C:\MGU and its subfolders for files modified since the beginning of today.

Find Files Modified in the Last 24 Hours

Today and the last 24 hours are not exactly the same. Today starts at midnight. The last 24 hours counts backward from the current time. 

Get-ChildItem -Path "C:\MGU" -File -Recurse |
  Where-Object { $_.LastWriteTime -ge (Get-Date).AddDays(-1) } |
  Sort-Object LastWriteTime -Descending |
  Select-Object FullName, LastWriteTime

This is useful when the file may have been edited yesterday but still within the last day.

Find Files Modified in the Last Week

To find files changed in the last seven days, use AddDays(-7). 

Get-ChildItem -Path "C:\MGU" -File -Recurse |
  Where-Object { $_.LastWriteTime -ge (Get-Date).AddDays(-7) } |
  Sort-Object LastWriteTime -Descending |
  Select-Object FullName, LastWriteTime

This is useful for reviewing the most recent work in a large project folder.

Find Files Created Today

Sometimes you want to find files that were newly created, not merely edited. In that case, use CreationTime instead of LastWriteTime. 

Get-ChildItem -Path "C:\MGU" -File -Recurse |
  Where-Object { $_.CreationTime -ge (Get-Date).Date } |
  Sort-Object CreationTime -Descending |
  Select-Object FullName, CreationTime

This searches for files created today. If a file was copied from another location, its created date may not always mean the original document creation date. It usually means the time it was created in that folder location.

Find Recent Files by Type

To search only for a certain file type, add -Filter. For example, this command finds recently modified text files. 

Get-ChildItem -Path "C:\MGU" -Filter "*.txt" -File -Recurse |
  Sort-Object LastWriteTime -Descending |
  Select-Object -First 20 FullName, LastWriteTime

This is helpful when you know the file type but not the folder where it was saved.

The same pattern works for HTML, CSS, JavaScript, images, documents, and other file types. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Sort-Object LastWriteTime -Descending |
  Select-Object -First 20 FullName, LastWriteTime

Show Name, Folder, and Date

Sometimes the full path is long. You can display the file name, folder path, and modified date in separate columns. 

Get-ChildItem -Path "C:\MGU" -File -Recurse |
  Sort-Object LastWriteTime -Descending |
  Select-Object -First 20 Name, DirectoryName, LastWriteTime

This makes the results easier to read when scanning recent files.

Save the Results to a Text File

If you want to keep a record of recent files, save the results to a text file. 

Get-ChildItem -Path "C:\MGU" -File -Recurse |
  Sort-Object LastWriteTime -Descending |
  Select-Object -First 100 FullName, LastWriteTime |
  Out-File "C:\MGU\recent-files.txt"

This creates a file named recent-files.txt inside C:\MGU. Open the text file to review the newest files.

Save the Results to a CSV File

A CSV file can be opened in spreadsheet software. This is useful when you want to sort, filter, or keep a larger inventory. 

Get-ChildItem -Path "C:\MGU" -File -Recurse |
  Sort-Object LastWriteTime -Descending |
  Select-Object -First 100 FullName, LastWriteTime, Length |
  Export-Csv "C:\MGU\recent-files.csv" -NoTypeInformation

This creates a spreadsheet-friendly report of the 100 most recently modified files.

Safe Habits

Finding recent files is a safe task because the commands in this tutorial only read file information. They do not change the files. This makes them good commands for beginners.

Be careful when adding commands that move, rename, delete, or overwrite files. Use recent-file searches first to inspect what exists before taking action.

Useful MGU Example

To find the 30 most recently changed HTML files inside the MGU folder, use: 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Sort-Object LastWriteTime -Descending |
  Select-Object -First 30 FullName, LastWriteTime

This is helpful after editing pages, generating new indexes, fixing stylesheet links, or checking whether files were updated correctly.

Summary

PowerShell can find recent files by sorting files according to LastWriteTime or CreationTime. LastWriteTime shows when a file was last modified. CreationTime shows when a file was created in its current location.

Use Get-ChildItem to collect files, Where-Object to filter by date, Sort-Object to put the newest files first, and Select-Object to choose the information you want to display.

For large projects, recent-file searches are one of the fastest ways to locate misplaced files and review recent work.
