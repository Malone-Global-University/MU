Understanding Paths and Folders

Paths tell PowerShell where files and folders are located. Learning how paths work is one of the first steps toward using PowerShell safely and confidently.

Overview

A path is the written address of a file or folder on a computer. In File Explorer, you move through folders by clicking. In PowerShell, you move through folders by typing paths. Once you understand paths, you can tell PowerShell exactly where to look, where to search, where to save results, and which files should be affected by a command.

Paths are important because PowerShell commands usually act on a location. That location might be the folder you are currently in, or it might be a specific folder or file named in the command.

What a Folder Is

A folder is a container used to organize files and other folders. A website folder might contain HTML files, CSS files, JavaScript files, images, documents, and subfolders. A large project such as Malone Global University depends on clear folder structure so that pages, images, and components can be found consistently.

In PowerShell, folders are often called directories. The words folder and directory usually mean the same thing in beginner PowerShell work.

What a Path Is

A path is the route to a file or folder. On Windows, a path often begins with a drive letter, followed by folder names separated by backslashes. 

C:\MGU\department\science\physics

This path points to a physics folder inside science, which is inside department, which is inside the MGU folder on the C drive.

Drive Letters

Windows uses drive letters to identify storage locations. The main drive is usually C:. Other drives might appear as D:, E:, or another letter. A full Windows path often begins with one of these drive letters. 

C:\MGU

This means the MGU folder is located on the C drive.

Root Folders

A root folder is the top folder for a project or storage location. For a website project, the root folder is the main folder that contains the website structure. If a website is stored in C:\MGU, then C:\MGU is the project root folder.

On a live website, a forward slash often represents the web root. For example, /component/main.css means the browser should look for main.css inside the component folder at the website root. On Windows, the local file path might be written differently, such as C:\MGU\component\main.css.

Absolute Paths

An absolute path gives the full location of a file or folder. It usually starts from the drive letter and includes every folder needed to reach the target. 

C:\MGU\component\main.css

This path points directly to the main.css file inside the component folder. Absolute paths are useful because they are specific and do not depend on the current PowerShell location.

Relative Paths

A relative path starts from the current location instead of starting from the drive letter. If PowerShell is already inside C:\MGU, then component\main.css can point to the same file as C:\MGU\component\main.css. 

component\main.css

Relative paths are shorter, but they depend on where PowerShell is currently working. This is why checking the current location is important.

Current Location

PowerShell always has a current location. This is the folder PowerShell is currently using. Commands without a full path usually run from this location. 

Get-Location

This command shows the current location.

Changing Location

To move PowerShell into another folder, use Set-Location. This is similar to opening a folder in File Explorer. 

Set-Location -Path "C:\MGU"

After running this command, PowerShell is working inside the C:\MGU folder. 

Get-Location

Running Get-Location afterward confirms that the move worked.

Spaces in Paths

If a path contains spaces, place the path inside quotation marks. This tells PowerShell to treat the whole path as one location. 

Set-Location -Path "C:\MGU\PowerShell Tutorials"

Quotation marks are a safe habit even when a path does not contain spaces.

Listing a Folder

To see what is inside the current folder, use Get-ChildItem. 

Get-ChildItem

To list a specific folder without moving into it, give the command a path. 

Get-ChildItem -Path "C:\MGU"

Website Paths and Windows Paths

Website paths and Windows paths often look different. A website path usually uses forward slashes. A Windows path usually uses backslashes. 

/component/main.css
C:\MGU\component\main.css

Both examples can point to the same file in different contexts. The first path is written for the browser. The second path is written for the local Windows file system.

Safe Habits

Before running a command that changes files, check your current location. A command that works correctly in one folder may cause problems if it is run in the wrong folder. 

Get-Location

Use full paths when precision matters. Use quotation marks around paths. Test commands on a small folder before using them on a large project.

Summary

A folder is a container for files. A path is the written address of a file or folder. Windows paths often begin with a drive letter, such as C:, and use backslashes between folders.

Absolute paths give the full location. Relative paths begin from the current location. PowerShell commands depend on location, so beginners should learn to use Get-Location, Set-Location, and Get-ChildItem early.

Understanding paths and folders makes PowerShell safer because it helps you know exactly where your commands are working.
