Replacing Text Across HTML Files

PowerShell can replace repeated text across many HTML files. This is useful for website maintenance, but it must be done carefully because one command can change hundreds or thousands of pages.

Overview

Replacing text across HTML files is one of the most useful PowerShell skills for maintaining a large website. It can update stylesheet links, script references, navigation snippets, image paths, metadata, repeated wording, or old folder paths.

This kind of task is also risky. A replacement command does exactly what it is told to do. If the old text is too broad, PowerShell may change files that should not be changed. If the new text contains a typo, the typo may be copied into many pages at once.

The safest method is to search first, save a list of affected files, make a backup, test on a small folder, run the replacement, then search again to verify the result.

The Safe Replacement Workflow

A safe text replacement workflow follows a clear order: 

1. Search for the old text.
2. Save a list of matching files.
3. Make a backup.
4. Test the replacement in a small folder.
5. Run the replacement.
6. Search again to verify the old text is gone.
7. Open sample files to check the result.

Do not begin with the replacement command. Begin with inspection. PowerShell is safest when it is used first to show what will be affected.

Example Problem

Suppose many HTML files contain this old stylesheet link: 

<link rel="stylesheet" href="/component/main.css" />

The correct stylesheet link should be: 

<link rel="stylesheet" href="/component/dept.css" />

PowerShell can search for the old line and replace it with the new line across many HTML files.

Search Before Replacing

Before replacing anything, search for the old text. This command searches all HTML files under C:\MGU for the old stylesheet path. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-String -Pattern '<link rel="stylesheet" href="/component/main.css" />' -SimpleMatch

This command only reads files. It does not change anything. Review the results before continuing.

Save Matching File Paths Only

For a cleaner review, save only the unique file paths that contain the old text. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-String -Pattern '<link rel="stylesheet" href="/component/main.css" />' -SimpleMatch |
  Select-Object -ExpandProperty Path |
  Sort-Object -Unique |
  Out-File "C:\MGU\files-using-old-stylesheet.txt"

This creates a report showing which files are expected to change.

Make a Backup First

Before running a replacement command, make a backup. For a site-wide replacement, a full backup is safest. 

$Stamp = Get-Date -Format "yyyy-MM-dd_HH-mm-ss"
$BackupPath = "C:\MGU-backups\before_text_replace_$Stamp"

Copy-Item -Path "C:\MGU" -Destination $BackupPath -Recurse

This creates a timestamped backup of the MGU folder. If the replacement causes problems, the original files are preserved.

Test on a Small Folder

Before changing the full website, copy a small folder into a test folder. 

Copy-Item -Path "C:\MGU\library\tutorials\powershell" -Destination "C:\MGU\test-powershell" -Recurse

Run the replacement on the test folder first. Open several changed files and confirm that the result is correct before editing the main folder tree.

Basic Replacement Pattern

A common replacement pattern reads the file content, replaces the old text with new text, and writes the result back to the same file. 

$OldText = '<link rel="stylesheet" href="/component/main.css" />'
$NewText = '<link rel="stylesheet" href="/component/dept.css" />'

Get-ChildItem -Path "C:\MGU\test-powershell" -Filter "*.html" -File -Recurse |
  ForEach-Object {
    $Content = Get-Content -Path $_.FullName -Raw
    $NewContent = $Content.Replace($OldText, $NewText)
    Set-Content -Path $_.FullName -Value $NewContent -Encoding UTF8
  }

This example uses a test folder. Do not point this command at the full website until you have tested it and made a backup.

Replace Only in Matching Files

It is safer to replace text only in files that actually contain the old text. This avoids rewriting files unnecessarily. 

$OldText = '<link rel="stylesheet" href="/component/main.css" />'
$NewText = '<link rel="stylesheet" href="/component/dept.css" />'

$Files = Get-ChildItem -Path "C:\MGU\test-powershell" -Filter "*.html" -File -Recurse |
  Select-String -Pattern $OldText -SimpleMatch |
  Select-Object -ExpandProperty Path |
  Sort-Object -Unique

foreach ($File in $Files) {
  $Content = Get-Content -Path $File -Raw
  $NewContent = $Content.Replace($OldText, $NewText)
  Set-Content -Path $File -Value $NewContent -Encoding UTF8
}

This version first builds a list of matching files. Then it changes only those files.

Replacing a Path Instead of a Whole Line

Sometimes it is better to replace only the path instead of the entire link tag. For example, several files may use slightly different spacing in the same stylesheet line. 

$OldText = "/component/main.css"
$NewText = "/component/dept.css"

$Files = Get-ChildItem -Path "C:\MGU\test-powershell" -Filter "*.html" -File -Recurse |
  Select-String -Pattern $OldText -SimpleMatch |
  Select-Object -ExpandProperty Path |
  Sort-Object -Unique

foreach ($File in $Files) {
  $Content = Get-Content -Path $File -Raw
  $NewContent = $Content.Replace($OldText, $NewText)
  Set-Content -Path $File -Value $NewContent -Encoding UTF8
}

This approach catches more variations because it focuses on the part of the line that actually needs to change.

Replacing Multiple Old Forms

A project may contain several incorrect versions of the same link. For example, old pages may use different spacing, a different folder, or a missing closing slash. 

$Replacements = @{
  '<link rel="stylesheet" href="/component/css/main.css" />' = '<link rel="stylesheet" href="/component/dept.css" />'
  '<link rel="stylesheet" href="/component/css/main.css"/>'  = '<link rel="stylesheet" href="/component/dept.css" />'
  '<link rel="stylesheet" href="/component/css/main.css">'   = '<link rel="stylesheet" href="/component/dept.css" />'
  '<link rel="stylesheet" href="/component/main.css" />'     = '<link rel="stylesheet" href="/component/dept.css" />'
  '<link rel="stylesheet" href="/component/main.css"/>'      = '<link rel="stylesheet" href="/component/dept.css" />'
  '<link rel="stylesheet" href="/component/main.css">'       = '<link rel="stylesheet" href="/component/dept.css" />'
}

Get-ChildItem -Path "C:\MGU\test-powershell" -Filter "*.html" -File -Recurse |
  ForEach-Object {
    $File = $_.FullName
    $Content = Get-Content -Path $File -Raw
    $NewContent = $Content

    foreach ($OldText in $Replacements.Keys) {
      $NewContent = $NewContent.Replace($OldText, $Replacements[$OldText])
    }

    if ($NewContent -ne $Content) {
      Set-Content -Path $File -Value $NewContent -Encoding UTF8
    }
  }

This script replaces several old stylesheet forms with one standardized line. It only writes the file if the content actually changes.

SimpleMatch and Replace

Select-String can treat a search pattern as regular expression text unless -SimpleMatch is used. Beginners should use -SimpleMatch when searching for ordinary HTML code, paths, and links. 

Select-String -Pattern "/component/main.css" -SimpleMatch

In the replacement examples, the .Replace() method treats the old text as literal text. This is usually easier for beginners than regular expression replacement.

Encoding Warning

When PowerShell writes a file back to disk, it uses an encoding. For modern HTML projects, UTF-8 is usually appropriate. The examples in this tutorial use: 

Set-Content -Path $File -Value $NewContent -Encoding UTF8

If a project has special encoding requirements, test carefully before running a site-wide edit.

Verify That the Old Text Is Gone

After running the replacement, search again for the old text. 

Get-ChildItem -Path "C:\MGU\test-powershell" -Filter "*.html" -File -Recurse |
  Select-String -Pattern "/component/main.css" -SimpleMatch

If no results appear, the old path is no longer present in the tested folder.

Verify That the New Text Exists

Also search for the new text to confirm that it appears where expected. 

Get-ChildItem -Path "C:\MGU\test-powershell" -Filter "*.html" -File -Recurse |
  Select-String -Pattern "/component/dept.css" -SimpleMatch |
  Select-Object -ExpandProperty Path |
  Sort-Object -Unique

This shows the files that now contain the updated stylesheet path.

Open Sample Files

Command-line verification is important, but you should also open a few changed HTML files and inspect the result directly. Make sure the link is written correctly and appears in the expected part of the head section.

A replacement can technically succeed while still producing the wrong layout or wrong line placement. Visual inspection helps catch mistakes that a search count may not reveal.

Full Site Example After Testing

After testing in a small folder and making a full backup, the same replacement pattern can be pointed at the full C:\MGU folder. 

$OldText = "/component/main.css"
$NewText = "/component/dept.css"

$Files = Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-String -Pattern $OldText -SimpleMatch |
  Select-Object -ExpandProperty Path |
  Sort-Object -Unique

foreach ($File in $Files) {
  $Content = Get-Content -Path $File -Raw
  $NewContent = $Content.Replace($OldText, $NewText)

  if ($NewContent -ne $Content) {
    Set-Content -Path $File -Value $NewContent -Encoding UTF8
  }
}

This replaces the old path with the new path only in HTML files that contain the old path.

Avoid Replacing Text That Is Too Broad

Do not replace a short or common word unless you are completely sure it is safe. A broad replacement can change text in places you did not intend. 

$OldText = "main"
$NewText = "dept"

This would be dangerous because the word main might appear in many unrelated places. A safer target is the full path: 

$OldText = "/component/main.css"
$NewText = "/component/dept.css"

Restore from Backup if Needed

If the replacement causes problems, restore the affected file or folder from the backup. For one file, copy the backup file over the changed file. 

Copy-Item -Path "C:\MGU-backups\before_text_replace_2026-05-15_14-30-00\library\tutorials\powershell\example.html" -Destination "C:\MGU\library\tutorials\powershell\example.html"

Replace the backup folder name and file path with the actual locations on your computer.

Useful MGU Example

For MGU department tutorial pages, the desired stylesheet line is: 

<link rel="stylesheet" href="/component/dept.css" />

To replace the old main.css path with the department stylesheet path across HTML files after backing up, use: 

$OldText = "/component/main.css"
$NewText = "/component/dept.css"

$Files = Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-String -Pattern $OldText -SimpleMatch |
  Select-Object -ExpandProperty Path |
  Sort-Object -Unique

foreach ($File in $Files) {
  $Content = Get-Content -Path $File -Raw
  $NewContent = $Content.Replace($OldText, $NewText)

  if ($NewContent -ne $Content) {
    Set-Content -Path $File -Value $NewContent -Encoding UTF8
  }
}

Afterward, search for both old and new paths to verify the result.

Summary

PowerShell can replace text across many HTML files by reading each file, changing matching text, and writing the updated content back to disk. This is useful for large website maintenance.

Safe replacement work begins with searching, saving a list of affected files, making a backup, and testing in a small folder. Only after those steps should a replacement be run across a large folder tree.

Use precise old text, avoid broad replacements, verify the old text is gone, confirm the new text exists, and open sample files to inspect the final result.
