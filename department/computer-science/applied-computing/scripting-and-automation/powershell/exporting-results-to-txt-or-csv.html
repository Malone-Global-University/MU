Exporting Results to .txt or .csv

PowerShell can save command results into text files and CSV files. This makes it useful for creating reports, inventories, checklists, audits, and maintenance records.

Overview

Many PowerShell commands show results directly in the PowerShell window. That is useful for quick checks, but it is not always enough. If you are reviewing many files, preparing a website cleanup, checking recent changes, or creating an inventory, it is better to save the results into a file.

Two of the most common export formats are .txt and .csv. A .txt file is a plain text report that is easy to open and read. A .csv file is a spreadsheet-friendly report that can be opened in Excel, Google Sheets, LibreOffice Calc, or another spreadsheet program.

For Malone Global University, exporting results is useful for tracking HTML files, images, recent edits, broken paths, old stylesheet links, file inventories, and large folder-tree audits.

Why Export Results?

Exporting results turns temporary command output into a saved record. Instead of letting the information disappear when the PowerShell window closes, you keep a report that can be reviewed, compared, archived, or shared.

Exported reports are especially useful before and after bulk edits. Before editing, a report can show which files match a search. After editing, a second report can help verify that the expected changes happened.

.txt Compared with .csv

A .txt file is best for simple reading. It works well for lists, command output, notes, and quick reports. If you only need to read a list of file paths or save search results, a .txt file is often enough.

A .csv file is best for structured data. It works well when the results have columns, such as file name, full path, folder name, extension, modified date, and file size. CSV files are easier to sort, filter, and analyze in a spreadsheet.

Export to a .txt File with Out-File

Use Out-File to send PowerShell results into a plain text file. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-Object FullName |
  Out-File "C:\MGU\html-files.txt"

This creates a text file named html-files.txt inside C:\MGU. The file contains the selected output from the command.

Export Simple Lists with Set-Content

Set-Content can also write text into a file. It is often useful when you are exporting a clean list of values. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-Object -ExpandProperty FullName |
  Set-Content "C:\MGU\html-file-paths.txt"

This exports only the full file paths, one path per line. The -ExpandProperty option removes the table formatting and returns the value itself.

Append to an Existing Text File

Out-File normally creates or replaces the output file. To add results to the end of an existing file, use -Append. 

Get-Date | Out-File "C:\MGU\maintenance-log.txt" -Append

This adds the current date and time to the maintenance log instead of replacing the whole file. 

"HTML inventory generated." | Out-File "C:\MGU\maintenance-log.txt" -Append

This adds a short message to the same log file.

Export to a .csv File with Export-Csv

Use Export-Csv when the results have columns and you want to open the report in a spreadsheet. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-Object FullName, DirectoryName, Name, Extension, LastWriteTime, Length |
  Export-Csv "C:\MGU\html-inventory.csv" -NoTypeInformation

This creates a CSV file with columns for full path, folder path, file name, extension, modified date, and size.

Why Use -NoTypeInformation?

The -NoTypeInformation option prevents PowerShell from adding an extra type-information line at the top of the CSV file. For normal spreadsheet reports, this option keeps the CSV cleaner. 

Export-Csv "C:\MGU\report.csv" -NoTypeInformation

For beginner reports, it is usually a good habit to include -NoTypeInformation when using Export-Csv.

Do Not Use Out-File for Spreadsheet Reports

Out-File saves what PowerShell displays on the screen. That can be useful for a readable text report, but it is not the best choice for structured spreadsheet data.

Use Export-Csv when you want real columns. Use Out-File when you want a plain text record. 

Get-ChildItem -Path "C:\MGU" -File -Recurse |
  Select-Object FullName, LastWriteTime, Length |
  Export-Csv "C:\MGU\file-report.csv" -NoTypeInformation

Export Search Results

Search results from Select-String can be saved to a text file. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-String -Pattern "/component/main.css" -SimpleMatch |
  Out-File "C:\MGU\main-css-search-results.txt"

This creates a text report showing matching lines, file paths, and line numbers.

Export Matching Paths Only

Sometimes you only need the list of files that contain a match. Select the Path property and remove duplicates. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-String -Pattern "/component/main.css" -SimpleMatch |
  Select-Object -ExpandProperty Path |
  Sort-Object -Unique |
  Set-Content "C:\MGU\files-using-main-css.txt"

This creates a cleaner report with one file path per line.

Export Search Results to CSV

Search results can also be converted into a CSV report with useful columns. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-String -Pattern "/component/main.css" -SimpleMatch |
  Select-Object Path, LineNumber, Line |
  Export-Csv "C:\MGU\main-css-search-results.csv" -NoTypeInformation

This creates a spreadsheet with the file path, line number, and matching line.

Export Recent Files

To save a report of recently modified files, sort by LastWriteTime and export the newest results. 

Get-ChildItem -Path "C:\MGU" -File -Recurse |
  Sort-Object LastWriteTime -Descending |
  Select-Object -First 100 FullName, DirectoryName, Name, Extension, LastWriteTime, Length |
  Export-Csv "C:\MGU\recent-files.csv" -NoTypeInformation

This report is useful after a large work session because it shows which files were changed most recently.

Export Counts by File Type

To see how many files exist for each extension, group the results and export them. 

Get-ChildItem -Path "C:\MGU" -File -Recurse |
  Group-Object Extension |
  Sort-Object Count -Descending |
  Select-Object Count, Name |
  Export-Csv "C:\MGU\file-counts-by-extension.csv" -NoTypeInformation

This creates a summary report showing how many HTML, CSS, JavaScript, image, text, document, and other file types exist in the project.

Encoding

Encoding controls how text is written into the file. For modern website and text work, UTF-8 is usually a good choice. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-Object -ExpandProperty FullName |
  Out-File "C:\MGU\html-files.txt" -Encoding UTF8

In many modern PowerShell versions, UTF-8 is already the default, but writing the encoding explicitly can make the command clearer.

Overwrite Warning

Export commands can overwrite existing report files. If you use the same output path again, the older report may be replaced. 

Export-Csv "C:\MGU\html-inventory.csv" -NoTypeInformation

If you want to preserve older reports, use timestamped file names.

Create Timestamped Report Names

A timestamped report includes the date and time in the file name. This prevents accidental overwriting and helps you track when the report was created. 

$Stamp = Get-Date -Format "yyyy-MM-dd_HH-mm-ss"
$ReportPath = "C:\MGU\html-inventory_$Stamp.csv"

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-Object FullName, DirectoryName, Name, LastWriteTime, Length |
  Export-Csv $ReportPath -NoTypeInformation

This creates a report with a name such as html-inventory_2026-05-15_14-30-00.csv.

Create a Reports Folder

It is helpful to keep reports in a dedicated folder instead of scattering them across the project root. 

if (!(Test-Path "C:\MGU\reports")) {
  New-Item -Path "C:\MGU\reports" -ItemType Directory
}

After creating the reports folder, export files there. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-Object FullName, LastWriteTime, Length |
  Export-Csv "C:\MGU\reports\html-inventory.csv" -NoTypeInformation

Open an Exported Report

After exporting a report, you can open it from File Explorer. You can also ask PowerShell to open it with the default program. 

Invoke-Item "C:\MGU\reports\html-inventory.csv"

For a CSV file, this usually opens the file in the default spreadsheet application. For a text file, it usually opens in the default text editor.

Useful MGU Examples

Export a text list of all HTML pages: 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-Object -ExpandProperty FullName |
  Set-Content "C:\MGU\reports\all-html-pages.txt"

Export a CSV inventory of all HTML pages: 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-Object FullName, DirectoryName, Name, LastWriteTime, Length |
  Export-Csv "C:\MGU\reports\all-html-pages.csv" -NoTypeInformation

Export a CSV of files still using the old stylesheet path: 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-String -Pattern "/component/main.css" -SimpleMatch |
  Select-Object Path, LineNumber, Line |
  Export-Csv "C:\MGU\reports\old-stylesheet-results.csv" -NoTypeInformation

Export a text list of unique files still using the old stylesheet path: 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-String -Pattern "/component/main.css" -SimpleMatch |
  Select-Object -ExpandProperty Path |
  Sort-Object -Unique |
  Set-Content "C:\MGU\reports\old-stylesheet-files.txt"

Safe Habits

Exporting results is usually safe because it reads information and writes a report file. It does not change the source files being inspected.

Be careful not to overwrite a report you still need. Use timestamped report names for important audits. Keep reports outside areas that may be included in later scans if you do not want reports to appear in future search results.

Summary

PowerShell can export results to .txt files with Out-File or Set-Content. It can export structured spreadsheet reports to .csv files with Export-Csv.

Use .txt files for simple lists and readable reports. Use .csv files for structured inventories with columns that can be sorted and filtered.

Exported reports are useful for website maintenance because they preserve search results, file inventories, recent-file lists, and before-and-after audit records.
