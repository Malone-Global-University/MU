Searching File Contents

PowerShell can search inside files for words, code, links, stylesheet references, script references, and other text patterns. This is one of the safest ways to inspect a large website before making changes.

Overview

Searching file contents means looking inside files, not just looking at file names. This is useful when you need to find a word, a phrase, a broken link, an old CSS reference, a JavaScript script tag, a title, a heading, or a repeated code pattern.

In PowerShell, the main command for searching inside text files is Select-String. It works like a search tool for file contents. It can search one file, many files, or an entire folder tree.

For large website projects, content search is essential. It lets you answer questions such as: Which pages still use an old stylesheet? Which files mention a certain folder path? Which HTML pages contain a broken link? Which pages still have an outdated word or title?

The Basic Command

The basic content-search command uses Select-String with a path and a pattern. 

Select-String -Path "C:\MGU\index.html" -Pattern "Malone Global University"

This searches inside index.html for the text Malone Global University. If the text is found, PowerShell displays the matching file, line number, and matching line.

Search All HTML Files in a Folder

To search all HTML files directly inside a folder, use a wildcard. 

Select-String -Path "C:\MGU\*.html" -Pattern "stylesheet"

This searches every HTML file directly inside C:\MGU for the word stylesheet. It does not search subfolders unless you use Get-ChildItem with -Recurse.

Search Inside Subfolders

To search an entire folder tree, first collect the files with Get-ChildItem, then send them to Select-String. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-String -Pattern "stylesheet"

This searches every HTML file inside C:\MGU and all of its subfolders for the word stylesheet.

Search for an Exact CSS Link

A common website maintenance task is finding pages that still use an old stylesheet link. To search for a specific line of HTML, place the search pattern in quotes. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-String -Pattern '<link rel="stylesheet" href="/component/main.css" />'

This searches all HTML files for the exact stylesheet reference. The results show which files still contain that line.

Search for More Than One Pattern

Select-String can search for more than one pattern at the same time. Separate the patterns with commas. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-String -Pattern "/component/main.css", "/component/css/main.css"

This is useful when a project has several old versions of a link or code pattern.

Show File Paths Only

Sometimes you only want to know which files contain the searched text. Use Select-Object to show the Path property, then remove duplicates with Sort-Object -Unique. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-String -Pattern "/component/main.css" |
  Select-Object -ExpandProperty Path |
  Sort-Object -Unique

This creates a cleaner list of matching files without showing every matching line.

Count Matching Files

To count how many unique files contain a pattern, combine the search with Measure-Object. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-String -Pattern "/component/main.css" |
  Select-Object -ExpandProperty Path |
  Sort-Object -Unique |
  Measure-Object

This is useful before a bulk edit because it tells you how many files may need attention.

Case-Sensitive Searching

By default, Select-String is usually case-insensitive. That means a search for powershell can also match PowerShell. To make the search case-sensitive, add -CaseSensitive. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-String -Pattern "PowerShell" -CaseSensitive

Case-sensitive searching is useful when capitalization matters.

Search as Simple Text

Select-String patterns can behave like regular expressions. Regular expressions are powerful, but they can surprise beginners because some characters have special meaning. To search for plain text, use -SimpleMatch. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-String -Pattern "/component/dept.css" -SimpleMatch

This tells PowerShell to treat the pattern as ordinary text instead of a regular expression.

Search for Website Links

To find pages that link to a certain folder, search for that path. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-String -Pattern 'href="/department/science/' -SimpleMatch

This can help locate pages connected to a department, library section, image folder, stylesheet, script, or navigation file.

Show Lines Around a Match

Sometimes the matching line alone is not enough. You may want to see nearby lines to understand the surrounding code. Use -Context. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-String -Pattern "bottom-nav" -Context 2,2

This shows two lines before and two lines after each match.

Save Search Results to a Text File

Search results can be saved for review. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-String -Pattern "/component/main.css" |
  Out-File "C:\MGU\main-css-search-results.txt"

This creates a text report containing the matches.

Save Matching File Paths Only

If you only want a clean list of files that contain the pattern, save the unique paths. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-String -Pattern "/component/main.css" |
  Select-Object -ExpandProperty Path |
  Sort-Object -Unique |
  Out-File "C:\MGU\files-using-main-css.txt"

This is useful before replacing old code across many files.

Safe Habits

Searching file contents is a safe inspection step because it does not change files. It only reads them. This makes Select-String one of the best commands to use before bulk edits.

Before replacing text, moving files, deleting files, or changing links, search first. Confirm the files and lines that match. Save a report if the change is important. Then make a backup before editing.

Useful MGU Example

To find every HTML file that still references the old main.css stylesheet, use: 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-String -Pattern '/component/main.css' -SimpleMatch |
  Select-Object -ExpandProperty Path |
  Sort-Object -Unique

This produces a clean list of pages that still need stylesheet cleanup.

Summary

Select-String searches inside files for text. It can search one file, many files, or an entire folder tree when combined with Get-ChildItem and -Recurse.

Use content searches to find old links, repeated code, page titles, CSS references, JavaScript references, and text patterns. For beginners, searching is one of the safest and most useful PowerShell skills because it helps inspect a project before changing it.
