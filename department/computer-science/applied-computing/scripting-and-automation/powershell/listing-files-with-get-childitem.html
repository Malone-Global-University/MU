Listing Files with Get-ChildItem

Get-ChildItem is one of the most important PowerShell commands. It lets you see the files and folders inside a location before you search, edit, move, copy, or organize anything.

Overview

Get-ChildItem lists the items inside a folder. In PowerShell, child items are the files and folders contained inside another folder. This command is useful because it lets you inspect a location before making changes.

For website maintenance, Get-ChildItem can show HTML files, CSS files, JavaScript files, images, documents, and subfolders. It can also be combined with options to search only for certain file types, list items recursively, or send results into another command.

The Basic Command

To list the files and folders in the current location, type: 

Get-ChildItem

This shows the contents of the folder where PowerShell is currently working. Before using it, you can check your current location with: 

Get-Location

List a Specific Folder

You do not have to move into a folder before listing it. You can give Get-ChildItem a path. 

Get-ChildItem -Path "C:\MGU"

This lists the files and folders inside C:\MGU. Using a specific path is safer when you want PowerShell to inspect one clear location.

List Files Only

A folder can contain both files and subfolders. To show only files, use the -File option. 

Get-ChildItem -Path "C:\MGU" -File

This is useful when you want to count, inspect, or export only files without including folders.

List Folders Only

To show only folders, use the -Directory option. 

Get-ChildItem -Path "C:\MGU" -Directory

This is useful when studying the structure of a project or building a directory map.

List a File Type

To list one kind of file, use a wildcard pattern. A wildcard is a symbol that stands for other characters. The asterisk means “anything.” 

Get-ChildItem -Path "C:\MGU" -Filter "*.html"

This lists HTML files directly inside C:\MGU. It does not automatically search every subfolder unless recursion is added.

Common File Filters

Filters help narrow the results. These examples show common website file types. 

Get-ChildItem -Path "C:\MGU" -Filter "*.css"
Get-ChildItem -Path "C:\MGU" -Filter "*.js"
Get-ChildItem -Path "C:\MGU" -Filter "*.webp"
Get-ChildItem -Path "C:\MGU" -Filter "*.txt"

List Files in Subfolders

To list files inside a folder and its subfolders, use -Recurse. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -Recurse

This command searches through C:\MGU and all folders beneath it for HTML files. Use -Recurse carefully on large projects because it can return thousands of results.

Show Full File Paths

By default, PowerShell shows a formatted view. To show only the full path of each file, use Select-Object with the FullName property. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -Recurse | Select-Object FullName

This is helpful when you need a clear list of exactly where matching files are located.

Count Matching Files

To count how many files match a search, pipe the results into Measure-Object. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -Recurse | Measure-Object

This is useful for checking the size of a website section or confirming how many files a command might affect.

Sort the Results

Results can be sorted by name, date, size, or other properties. To sort by file name, use Sort-Object. 

Get-ChildItem -Path "C:\MGU" -File | Sort-Object Name

To see the newest files first, sort by LastWriteTime in descending order. 

Get-ChildItem -Path "C:\MGU" -File | Sort-Object LastWriteTime -Descending

Why Get-ChildItem Is Safe

Get-ChildItem is safe because it only reads information. It does not delete, overwrite, rename, or move files. This makes it one of the best commands for beginners.

Many safe PowerShell workflows begin with Get-ChildItem. First list the files. Then check the results. Then decide whether another command should be used.

Example for an MGU Website Folder

To list every HTML file inside the MGU website folder, use: 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -Recurse

To show only the full paths: 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -Recurse | Select-Object FullName

To count the HTML files: 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -Recurse | Measure-Object

Summary

Get-ChildItem lists files and folders. It can list the current folder, a specific folder, only files, only folders, one file type, or an entire folder tree with -Recurse.

This command is one of the safest and most useful PowerShell commands because it helps you inspect a project before changing anything.
