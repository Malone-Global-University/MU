Generating File Inventories

A file inventory is a list of files in a folder or project. PowerShell can generate inventories for websites, image folders, document collections, backups, and large folder trees.

Overview

A file inventory is a structured list of files. It can show file names, full paths, folder locations, file sizes, extensions, creation dates, and modified dates. A good inventory helps you understand what exists inside a project before you edit, move, rename, archive, or reorganize files.

File inventories are especially useful for large website projects. When a site contains thousands of HTML pages, images, CSS files, scripts, documents, and generated folders, it becomes difficult to track everything by memory. PowerShell can scan the folder tree and create a report.

For Malone Global University, file inventories can help track HTML pages, WebP images, department pages, library articles, tutorials, textbook chapters, component files, and recent updates.

Basic File Inventory

The simplest file inventory lists the files inside a folder. 

Get-ChildItem -Path "C:\MGU" -File

This lists files directly inside C:\MGU. It does not include files inside subfolders unless -Recurse is added.

Recursive File Inventory

To list files inside a folder and all of its subfolders, use -Recurse. 

Get-ChildItem -Path "C:\MGU" -File -Recurse

This command creates a full file listing for the C:\MGU folder tree. On a large website, it may return thousands of files.

Choose Useful File Details

A raw file listing may show more information than you need. Use Select-Object to choose the columns that matter. 

Get-ChildItem -Path "C:\MGU" -File -Recurse |
  Select-Object FullName, DirectoryName, Name, Extension, LastWriteTime, Length

This shows the full path, folder path, file name, extension, modified date, and file size in bytes.

Inventory HTML Files

To inventory only HTML files, use -Filter. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-Object FullName, DirectoryName, Name, LastWriteTime, Length

This is useful for counting pages, reviewing recently modified pages, checking generated indexes, and planning bulk edits.

Inventory Image Files

Image inventories are useful when managing icons, infographics, logos, diagrams, and converted image formats. 

Get-ChildItem -Path "C:\MGU\image" -Filter "*.webp" -File -Recurse |
  Select-Object FullName, DirectoryName, Name, LastWriteTime, Length

This creates an inventory of WebP images inside the image folder tree.

You can use the same pattern for PNG, JPG, SVG, ICO, or other image types. 

Get-ChildItem -Path "C:\MGU\image" -Filter "*.png" -File -Recurse |
  Select-Object FullName, DirectoryName, Name, LastWriteTime, Length

Inventory Folders

A folder inventory lists directories instead of files. This is useful when studying the structure of a website or preparing navigation pages. 

Get-ChildItem -Path "C:\MGU" -Directory -Recurse |
  Select-Object FullName, Parent, Name, LastWriteTime

This lists folders under C:\MGU and shows where they are located.

Count Files

To count matching files, pipe the inventory into Measure-Object. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Measure-Object

This counts HTML files in the MGU folder tree.

Count Files by Type

To see how many files exist for each extension, group files by extension. 

Get-ChildItem -Path "C:\MGU" -File -Recurse |
  Group-Object Extension |
  Sort-Object Count -Descending |
  Select-Object Count, Name

This helps you understand the file makeup of a project, such as how many HTML, CSS, JavaScript, WebP, PNG, TXT, CSV, or JSON files exist.

Save an Inventory as a Text File

Use Out-File to save an inventory as a plain text report. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-Object FullName, LastWriteTime, Length |
  Out-File "C:\MGU\html-inventory.txt"

This creates a readable text report inside C:\MGU.

Save an Inventory as a CSV File

CSV is often better than plain text because it can be opened in spreadsheet software. Use Export-Csv to create a spreadsheet-friendly inventory. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-Object FullName, DirectoryName, Name, Extension, LastWriteTime, Length |
  Export-Csv "C:\MGU\html-inventory.csv" -NoTypeInformation

This creates a CSV inventory that can be sorted and filtered by path, name, date, extension, or size.

Inventory Recent Files

To create an inventory of recently changed files, sort by LastWriteTime and select the newest results. 

Get-ChildItem -Path "C:\MGU" -File -Recurse |
  Sort-Object LastWriteTime -Descending |
  Select-Object -First 100 FullName, DirectoryName, Name, Extension, LastWriteTime, Length |
  Export-Csv "C:\MGU\recent-file-inventory.csv" -NoTypeInformation

This report is useful after a large editing session because it shows which files were changed most recently.

Inventory Large Files

To find large files, sort by Length in descending order. 

Get-ChildItem -Path "C:\MGU" -File -Recurse |
  Sort-Object Length -Descending |
  Select-Object -First 50 FullName, Length, LastWriteTime |
  Export-Csv "C:\MGU\largest-files.csv" -NoTypeInformation

This can help locate oversized images, large documents, or unexpected files that may need review.

Find Empty Folders

Empty folders can appear during cleanup, testing, or file moves. This command finds folders that do not contain child items. 

Get-ChildItem -Path "C:\MGU" -Directory -Recurse |
  Where-Object { -not (Get-ChildItem -Path $_.FullName -Force) } |
  Select-Object FullName |
  Export-Csv "C:\MGU\empty-folders.csv" -NoTypeInformation

Review empty folders before deleting anything. Some empty folders may be placeholders.

Inventory One Website Section

You do not always need to inventory the whole website. To inspect one section, target a narrower folder. 

Get-ChildItem -Path "C:\MGU\department\science" -Filter "*.html" -File -Recurse |
  Select-Object FullName, DirectoryName, Name, LastWriteTime, Length |
  Export-Csv "C:\MGU\science-html-inventory.csv" -NoTypeInformation

This is faster and cleaner than scanning the entire site when you only need one area.

Add Relative Paths

Full Windows paths are useful, but relative paths are often easier to compare with website paths. This example creates a relative path column. 

$Root = "C:\MGU"

Get-ChildItem -Path $Root -Filter "*.html" -File -Recurse |
  Select-Object FullName,
                @{Name="RelativePath"; Expression={$_.FullName.Substring($Root.Length).TrimStart("\")}},
                DirectoryName,
                Name,
                LastWriteTime,
                Length |
  Export-Csv "C:\MGU\html-inventory-with-relative-paths.csv" -NoTypeInformation

Relative paths are helpful when comparing local files to website routes.

Create URL-Like Paths

Website paths use forward slashes instead of Windows backslashes. This example adds a URL-like path column. 

$Root = "C:\MGU"

Get-ChildItem -Path $Root -Filter "*.html" -File -Recurse |
  Select-Object FullName,
                @{Name="WebPath"; Expression={"/" + ($_.FullName.Substring($Root.Length).TrimStart("\") -replace "\\", "/")}},
                Name,
                LastWriteTime,
                Length |
  Export-Csv "C:\MGU\html-webpath-inventory.csv" -NoTypeInformation

This is useful for creating link lists, checking routes, or preparing website navigation maps.

Before-and-After Inventories

File inventories are useful before and after bulk edits. Before editing, create a report of the current state. After editing, create another report and compare them. 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-Object FullName, LastWriteTime, Length |
  Export-Csv "C:\MGU\html-inventory-before.csv" -NoTypeInformation
Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-Object FullName, LastWriteTime, Length |
  Export-Csv "C:\MGU\html-inventory-after.csv" -NoTypeInformation

These reports help show what changed during maintenance.

Safe Habits

Generating inventories is a safe PowerShell activity because it only reads file information. It does not change, move, delete, or overwrite files.

Inventories should be created before major cleanup or bulk edits. They give you a record of what existed before changes were made. They also help you plan, audit, and verify work.

Useful MGU Example

To create a complete HTML inventory for the MGU website, use: 

Get-ChildItem -Path "C:\MGU" -Filter "*.html" -File -Recurse |
  Select-Object FullName, DirectoryName, Name, Extension, LastWriteTime, Length |
  Export-Csv "C:\MGU\mgu-html-inventory.csv" -NoTypeInformation

To create a WebP image inventory, use: 

Get-ChildItem -Path "C:\MGU\image" -Filter "*.webp" -File -Recurse |
  Select-Object FullName, DirectoryName, Name, Extension, LastWriteTime, Length |
  Export-Csv "C:\MGU\mgu-webp-inventory.csv" -NoTypeInformation

These reports can help track page growth, image production, folder cleanup, and maintenance planning.

Summary

A file inventory is a report showing what files exist in a folder or project. PowerShell can generate inventories for HTML pages, images, documents, scripts, folders, recent files, large files, and full website trees.

Use Get-ChildItem to collect files, Select-Object to choose useful details, Measure-Object to count results, Out-File for text reports, and Export-Csv for spreadsheet-friendly reports.

For large website maintenance, inventories help you see the project clearly before making changes.
