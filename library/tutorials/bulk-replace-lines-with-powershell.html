Bulk Replace Lines With Powershell

Short Tutorial: Bulk Replace One Line Across Many HTML Files with PowerShell.

Tutorial Overview

This tutorial shows how to walk through a folder tree and replace one exact line in every HTML file. This is useful when updating shared stylesheet links, script links, headers, footers, or other repeated code across a large website.

Example

Example goal:


<link rel="stylesheet" href="/css/main.css">

Change it to:


<link rel="stylesheet" href="/css/assets.css">

1. Open PowerShell

Open PowerShell on your computer. For an example site stored at: 


powershell
C:\example

you can use that folder as the root folder for the replacement.

2. Run a Dry Run First

A dry run checks which files would be changed without actually editing anything. 


powershell
$root = "C:\example"

$old = '<link rel="stylesheet" href="/css/main.css">'

Get-ChildItem -Path $root -Recurse -File -Include *.html | ForEach-Object {
    $path = $_.FullName
    $content = Get-Content -Path $path -Raw

    if ($content -like "*$old*") {
        Write-Host "Would update: $path"
    }
}

This prints every HTML file that contains the old stylesheet line.

3. Make a Backup

Before changing thousands of files, create a backup. 


powershell
Copy-Item "C:\example" "C:\example_backup_before_css_swap" -Recurse


This gives you a complete copy of the site before the replacement.

4. Run the Replacement

After checking the dry run, run the real replacement script. 


powershell
$root = "C:\example"

$old = '<link rel="stylesheet" href="/css/main.css">'
$new = '<link rel="stylesheet" href="/css/assets.css">'

Get-ChildItem -Path $root -Recurse -File -Include *.html | ForEach-Object {
    $path = $_.FullName
    $content = Get-Content -Path $path -Raw

    if ($content -like "*$old*") {
        $updated = $content.Replace($old, $new)
        Set-Content -Path $path -Value $updated -Encoding UTF8
        Write-Host "Updated: $path"
    }
}

PowerShell will walk through every `.html` file under `C:\example`, find the exact old line, and replace it with the new line.

5. Optional: Limit the Change to One Area

To update only the assetsrary section, change the root folder: 


powershell
$root = "C:\example\assets"


To update only the department section:

powershell
$root = "C:\example\department"

6. Safer Version with Per-File Backups

This version creates a `.bak` backup beside each file before editing it. 


powershell
$root = "C:\example"

$old = '<link rel="stylesheet" href="/css/main.css">'
$new = '<link rel="stylesheet" href="/css/assets.css">'

Get-ChildItem -Path $root -Recurse -File -Include *.html | ForEach-Object {
    $path = $_.FullName
    $content = Get-Content -Path $path -Raw

    if ($content -like "*$old*") {
        Copy-Item $path "$path.bak"
        $updated = $content.Replace($old, $new)
        Set-Content -Path $path -Value $updated -Encoding UTF8
        Write-Host "Updated with backup: $path"
    }
}

Final Check

After running the script, open a few updated HTML files and confirm that the stylesheet line now says: 


<link rel="stylesheet" href="/css/assets.css">

This method is best for exact, repeated lines. For more complicated patterns with inconsistent spacing or multiple stylesheet formats, use a regex-based version instead.

To restore the .bak files, you copy each backup file back over its original file name.

For example, this backup: 


C:\example\library\index.html.bak

should restore to: 


C:\example\library\index.html

Run this from PowerShell: 


$root = "C:\example"

Get-ChildItem -Path $root -Recurse -File -Filter "*.bak" | ForEach-Object {
    $bakPath = $_.FullName
    $originalPath = $bakPath -replace '\.bak$',''

    Copy-Item -Path $bakPath -Destination $originalPath -Force

    Write-Host "Restored: $originalPath"
}

That restores every .bak file under C:\example by overwriting the matching original file.

Before restoring, you can do a dry run: 


$root = "C:\example"

Get-ChildItem -Path $root -Recurse -File -Filter "*.bak" | ForEach-Object {
    $bakPath = $_.FullName
    $originalPath = $bakPath -replace '\.bak$',''

    Write-Host "Would restore: $bakPath -> $originalPath"
}

After restoring, you can delete the .bak files only when you are sure everything is correct: 


$root = "C:\example"

Get-ChildItem -Path $root -Recurse -File -Filter "*.bak" | Remove-Item

I would run the dry run first, then the restore, then check a few pages before deleting the .bak files.
