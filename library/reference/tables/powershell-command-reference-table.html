PowerShell Command Reference Table

PowerShell commands, also called cmdlets, use a verb-noun naming pattern that makes them easier to read, search, and remember. This table lists common commands for navigating folders, managing files, viewing system information, working with processes, and controlling command output.

Command Purpose Example Practical Use
Get-Location Shows the current working folder. Get-Location Confirms where PowerShell is currently operating.
Set-Location Changes the current working folder. Set-Location C:\MGU Moves into a project folder before running commands.
Get-ChildItem Lists files and folders in a location. Get-ChildItem Displays folder contents, similar to dir or ls.
New-Item Creates a new file or folder. New-Item index.html -ItemType File Creates new project files directly from the terminal.
Copy-Item Copies files or folders. Copy-Item source.html backup.html Makes a duplicate before editing or replacing a file.
Move-Item Moves or renames files and folders. Move-Item old.html new.html Renames a file or moves it into another folder.
Remove-Item Deletes files or folders. Remove-Item temp.txt Removes unwanted files from a project directory.
Get-Content Reads the contents of a text file. Get-Content notes.txt Displays a file directly in the terminal.
Set-Content Writes or replaces content in a file. Set-Content output.txt "Hello" Creates or overwrites a text file with new content.
Add-Content Adds text to the end of an existing file. Add-Content log.txt "Build complete" Appends notes, logs, or status messages without replacing the file.
Select-String Searches text for matching words or patterns. Select-String -Path *.html -Pattern "hero" Finds text inside files, useful for locating CSS classes or HTML sections.
Where-Object Filters command results by condition. Get-ChildItem | Where-Object {$_.Extension -eq ".html"} Shows only files that match a rule.
Sort-Object Sorts command output. Get-ChildItem | Sort-Object Name Arranges files, commands, or results alphabetically or numerically.
Select-Object Chooses specific properties from command output. Get-ChildItem | Select-Object Name, Length Creates cleaner output by showing only useful columns.
Group-Object Groups results by a shared property. Get-Command | Group-Object Source Summarizes commands, files, or objects by category.
Get-Command Lists available PowerShell commands. Get-Command Discovers installed cmdlets, functions, aliases, and scripts.
Get-Help Displays help information for a command. Get-Help Get-ChildItem Shows syntax, parameters, examples, and command explanations.
Get-Process Lists running processes. Get-Process Checks what programs and background tasks are active.
Stop-Process Stops a running process. Stop-Process -Name notepad Closes a frozen or unwanted process from the terminal.
Clear-Host Clears the PowerShell screen. Clear-Host Removes clutter from the terminal window.

Common PowerShell Pipeline Patterns

PowerShell becomes especially powerful when commands are connected with the pipeline symbol |. The pipeline passes objects from one command into the next, allowing results to be filtered, sorted, grouped, selected, exported, or reformatted.

Pattern Example What It Does
List and sort Get-ChildItem | Sort-Object Name Lists files and folders in alphabetical order.
Filter by extension Get-ChildItem | Where-Object {$_.Extension -eq ".html"} Shows only HTML files in the current folder.
Select columns Get-ChildItem | Select-Object Name, Length, LastWriteTime Displays only the file name, size, and modified date.
Group results Get-Command | Group-Object Source Groups available commands by their module or source.
Export results Get-ChildItem | Out-File files.txt Saves command output into a text file.
Search inside files Select-String -Path *.html -Pattern "breadcrumb" Finds matching text inside multiple HTML files.
