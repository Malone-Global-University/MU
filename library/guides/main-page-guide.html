Main Page Guide

Welcome to Main Page Guide!

The main page of this website located at:

https://maloneuniversity.org/index.html

There are five main ways it can be reached:

  1. Entering https://maloneuniversity.org into a web browser.
  2. Clicking the Home link on the top navigation bar of MGU webpages.
  3. Clicking the the MGU Logo image link in the top left corner of MGU webpages.
  4. Clicking on the Site Home link in bottom navigation menus.
  5. Clicking on its link on external sites such as social media apps or Google search pages.

Purpose of Main Page

The Main Page serves as the main landing page for the entire site. Its job is to communicate basic nature, navigation, and function of the site. Typically it is the first page people see when visiting the site so it must clear in its branding and philosophy to establish its identity early on.

Features of the Main Page

The main page features include introductory text, explanation of the school's general philosophy, and guilding principles.

The main layout and design of the main page is stored using a template. This template serves as a model for all other pages on the website. Pages may vary from time to time but they all tend to follow the same construction. Pay attention to commented sections:

The Head

The head contains important metadata such as html title, favicon link, and CSS stylesheet link.The title can be read by hovering mouse pointer about tabs of webpages. The idea is that if you have many active tabs hovering the mouse to read the tab's title allows you to find a page quickly without viewing each page.

The favicion is a small icon image that when linked properly dispays a small picture in the tab. This picture is intended to be the logo of the website so that people can identify what pages are open just ny looking at the icon; looking at the icon is faster than hovering mouse to read title; both favicons and page title do the same job of telling you what page you are on but title sare more specific.

A favicon will tell you what website but titles can tell you what exact page of what website. When viewing a page directly all metadata items that are enclosing with <head></head> tags are in invisible to visistors of the page.

The Body

The Body contains everything that is visible on the page. This includes the header, hero, breadcrumbs, bottom-nav, and footer. The body is the primary part of a HTML document (webpage) and acts as a container by wrapping all visible HTML elements inside <body></body> tags.

The Header

This is the uppermost visible element on the page. MGU uses a combination header + navigation menu. A header acts as the top formatting block for the page. It contains information to communicate what website you are on. Think of it as the top section of a webpage that usually contains the site logo, navigation menu, and search bar. It helps users orient themselves and navigate the site.

The Hero

A hero is a wide banner that displays the title of a page and sometimes additional subtitle text. The different subdomians of MGU use color coded heroes to identify which subdomain a user is currently navigating.

  • Main and Orientation pages uses dark teal hex color #2c5364.
  • Community uses green hex color #2f8f57.
  • Department uses blue hex color #3d6db5.
  • Library uses red hex color #9b2c2c.
  • News uses purple hex color #7b4ca8.

The Breadcrumbs

Breadcrumbs are links that take you back to the home page by following either the hierarchy of folder indices, or by tracking the actual path taken.

Common breadcrumb navigation types are:

  1. Location-based breadcrumbs

    Shows where the current page sits in the site hierarchy. Example:

    Home > Library > Encyclopedia > Science > Physics

  2. Path-based breadcrumbs

    Shows the actual route the user took to reach the page. Example:

    Home > Search Results > Article

  3. Attribute-based breadcrumbs

    Common in stores or catalogs. Shows filters/categories applied. Example:

    Home > Shoes > Men > Black > Size 10

  4. Hierarchical breadcrumbs

    Almost the same as location-based, but the name emphasizes parent-child structure. Example:

    Home > Department > Course > Lesson

  5. History-based breadcrumbs

    A “back trail” based on browsing history. Less common today. Example:

    Previous Page > Previous Page > Current Page

  6. Dynamic breadcrumbs

    Generated automatically with JavaScript or server code from the URL/path.

    Example URL:

    /library/encyclopedia/science/physics/

    Breadcrumb:

    Home > Library > Encyclopedia > Science > Physics

  7. Static breadcrumbs

    Hard-coded directly into each HTML page.

  8. Responsive/mobile breadcrumbs

    A compact version for small screens, often showing only:

    ‹ Back to Physics or Home > … > Current Page

MGU originally used static breadcrumbs but has since transitioned to dynamic, location-based, hierarchical breadcrumbs, generated from the URL path.

More specifically:

It is dynamic because JavaScript builds it automatically from /component/breadcrumbs.js using window.location.pathname.

It is location-based because it reflects where the page lives in the site structure.

It is hierarchical because each URL folder becomes a parent-child breadcrumb level.

Example:

/library/encyclopedia/science/physics/index.html

becomes:

Home > Library > Encyclopedia > Science > Physics

The Main

The <main></main> tags wrap body sections. This site uses H2 headings to start new sections using the <section> tag. CSS is used to create side margins for anything inside <main> tags. The "Main" section of pages begins right after breadcrumbs and ends righ before the footer. Breadcrumbs on this site are controlled by separate CSS to center it so it does not rely on from "Main" CSS for formatting. The Hero and Footer must exist outside <main> tags or they would become boxed in the margins set by CSS. This would stop them form bing 100% full width of screens and not aesthetically pleasing.

The Footer

The footer is the bottom banner of all pages that does things like list social media contacts and page upload and last updated links. Pages that are updated more often are fresher and tend to be better written and structured. Early pages that have not been updated in a while are ewither very basic or in due for an overhaul. Periodically pages are reviewed for maintenance. If they need kater upcoding the dates are unmodified until changes occur. If the pages do not need updating then the dates are given current time to say they are current up to that moment in time. Basic permenent pages hardly need updating so may carry older dates but still be as current as possible. Deep pages, especially those for prototype lessons carry older dates because updoing is still in progress.
