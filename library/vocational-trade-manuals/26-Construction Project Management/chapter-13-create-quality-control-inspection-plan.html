Job Overview

This project qualifies you to build and issue a project-specific quality-control plan — the controlling document that identifies what will be checked, against what standard, by whom, and at what point in construction, before work is concealed, handed off, inspected by an authority, or accepted by the owner. The finished work product is one issued quality-control plan supported by a quality register, document-based acceptance criteria, a controlled checklist set covering preparatory, first-work, in-process, and preconcealment checks, a deficiency log, and verification records showing corrective action was confirmed rather than merely reported.

You will collect the approved drawings, specifications, code notes, manufacturer instructions, samples, mockups, submittals, permits, and contract quality requirements; build a quality register aligned with the work breakdown structure; identify high-risk and concealed work; define measurable acceptance criteria without inventing tolerances; build preparatory, first-work, in-process, preconcealment, trade-handoff, preinspection, and final checklists; assign inspection and approval responsibility; define deficiency categories and thresholds; test the plan on one representative high-risk assembly; and issue the completed plan to the project team.

This project depends on the completed scope statement from Project 01, the work breakdown structure from Project 03, and the trade-coordination matrix from Project 08. It in turn supplies the controlling checklists used in Project 08's trade handoffs, the hold points referenced in Project 16's inspection-scheduling system, and the verified punch-closure standard used in Project 14.

Where This Skill Fits

Contractor quality control does not replace design review, code inspection, special inspection, testing, manufacturer inspection, or owner acceptance; it is the contractor's own planned verification that work is ready for each of those reviews. The quality plan built here supplies the preconcealment hold points that Project 08 references when a predecessor trade requests area release, the preinspection readiness checks that Project 16 uses before requesting an authority inspection, and the acceptance-criteria language that Project 14 relies on to distinguish a legitimate punch item from an owner preference or a warranty issue.

The discipline of citing a governing document for every acceptance criterion, rather than describing quality in general terms, is the same discipline the capstone project in Project 20 audits across every trade package before substantial completion.

Scope and Intended Use

This project covers:

  • collecting and reviewing the approved drawings, specifications, code notes, manufacturer instructions, samples, mockups, submittals, permits, and contract quality requirements
  • building a quality register aligned with the approved work breakdown structure and trade packages
  • identifying high-risk, concealed, water-sensitive, structural, safety-critical, finish-critical, and owner-visible work, and identifying which items require special inspection, authority inspection, testing agency, manufacturer representative, designer review, or owner approval
  • defining document-based acceptance criteria using measurable dimensions, tolerances, attachment, material, sequence, finish, test, and documentation requirements
  • building preparatory-phase, first-work, in-process, preconcealment, trade-handoff, preinspection, and final acceptance checklists
  • assigning inspection, witness, approval, and record-owner responsibility, and defining deficiency categories, thresholds, and the nonconformance process
  • testing the plan on one representative high-risk assembly, issuing the plan, and performing scheduled inspections with verified corrective action

This project does not authorize the learner to:

  • perform code inspection, special inspection, testing-agency verification, or design review; those remain with the authority having jurisdiction, the identified testing agency, and the licensed design professional
  • invent a tolerance, load value, cure time, or acceptance standard that is not present in the governing drawings, specifications, code, or manufacturer instructions
  • accept an unauthorized deviation from approved documents merely because correction is inconvenient or would affect the schedule
  • conceal work before required internal checks, photographs, tests, and authority inspections are complete

Any conflict between the drawings, specifications, manufacturer instructions, and contract quality requirements must be identified and resolved in writing, through the RFI process established in Project 10, before the affected acceptance criterion is finalized in the quality plan.

Trade Note

Every acceptance criterion in this plan must cite the document that establishes it — a drawing note, a specification section, a code reference, or a manufacturer instruction. A criterion with no cited source is not a quality standard; it is a guess, and a guess cannot be defended when a trade disputes a deficiency.

Shop Tip

Write acceptance criteria as a pass/fail statement with a cited source, not as a description. "Fastener spacing does not exceed 6 inches on center at panel edges per Sheet S-4, Note 3" can be checked and failed. "Properly fastened panel edges" cannot.

Common Field Fundamentals

Every quality-control plan in this book, from this chapter's residential quality register to the Chapter 20 capstone's full-project quality audit, follows the same sequence: identify, define acceptance, checkpoint, assign, hold, deficiency, verify, close. Learning that sequence here means every later concealment, handoff, and inspection decision in the project starts from a checklist already tested against real work.

Acceptance Criteria and Hold Points

Every controlled work package must have a written acceptance criterion citing its governing document, and every package identified as concealed, water-sensitive, structural, or safety-critical must have a defined preconcealment hold point. Do not allow a preconcealment hold point to be treated as optional because the trade is confident the work is correct; the hold point exists precisely so that confidence is verified before the work becomes inaccessible.

Documentation Cue — Cite the Governing Document for Every Criterion

Record the specific drawing sheet, specification section, code reference, or manufacturer instruction that establishes each acceptance criterion. An acceptance criterion with no cited source cannot be defended when a trade disputes a deficiency, and it is not a substitute for the governing document itself.

Critical Quality Cue — Do Not Cover Work Before Required Checks Are Complete

Do not allow work identified with a preconcealment hold point to be covered until required internal checks, photographs, tests, and authority inspections are complete. Covering held work before verification is complete removes the ability to confirm the work meets the governing documents and may require exposing the work again at the responsible party's cost.

Field Coordination Cue — Confirm Attendance for First-Work and Special Inspections

Confirm which parties must attend the first-work review or a required special inspection before scheduling it. A first-work review conducted without the required attendee does not establish the approved standard for the remainder of the package.

Manufacturer Instruction Cue — Verify Installer Qualification and Environmental Conditions

Where a manufacturer instruction requires a qualified or certified installer, a minimum or maximum temperature, humidity, or substrate condition, confirm that condition before work begins and record it on the preparatory-phase checklist. Do not accept installed work when the required condition was not met and not documented as an approved exception.

E. Preparation — Collect Documents and Build the Quality Register

  1. Collect approved drawings, specifications, code notes, manufacturer instructions, samples, mockups, submittals, permits, and contract quality requirements. Assemble every available document before building the register.
  2. Create a quality register aligned with the WBS and trade packages. List every work package from the approved WBS that requires a quality check.
  3. Identify high-risk, concealed, water-sensitive, structural, safety-critical, finish-critical, and owner-visible work. Mark each such package in the register.
  4. Identify work requiring special inspection, authority inspection, testing agency, manufacturer representative, designer review, or owner approval. Record the specific authority for each identified package.
  5. Distinguish contractor QC from code inspection, design review, testing, and owner acceptance. State this distinction directly in the quality plan introduction.

F. Define Acceptance Criteria

  1. Define acceptance criteria for each controlled work package. Write one acceptance criterion, or a set of criteria, for every package identified in Step 2.
  2. Use measurable dimensions, tolerances, attachment, material, sequence, finish, test, and documentation requirements. State each criterion as a pass/fail standard.
  3. Do not invent tolerances not present in the governing documents. Cite the drawing, specification, code, or manufacturer instruction that establishes each value; where none exists, route the gap through the RFI process before finalizing the criterion.
  4. Identify required submittals, samples, mockups, shop drawings, and approved products. Link each controlled package to its approved submittal record.

G. Build the Checklist Set

  1. Create a preparatory-phase checklist to confirm documents, material, substrate, equipment, labor, and conditions before work. Include every prerequisite identified for the package.
  2. Create a first-work checklist for the initial installation or representative area. State the specific acceptance criteria the first-work review will confirm.
  3. Identify who must attend and approve first-work review. Name the required attendees for each high-risk package.
  4. Create in-process inspection points for layout, concealed components, fastening, alignment, support, flashing, sealing, testing, and finish. Assign each in-process point to the applicable package.
  5. Create preconcealment hold points for structure, utilities, air and water control, fire blocking, waterproofing, and other covered work. State the exact condition that must be verified before covering.
  6. Create trade-handoff quality checks. Align these with the handoff conditions established in Project 08.
  7. Create preinspection readiness checks before authority inspections. Align these with the inspection-scheduling system established in Project 16.
  8. Create final-area and system acceptance checks. State the condition that defines the package as complete and ready for owner acceptance.

H. Assign Responsibility and Define Deficiency Management

  1. Assign inspection responsibility, witness responsibility, approval authority, and record owner. Name one party for each role on every checklist.
  2. Link quality checks to schedule activities and lookahead tasks. Confirm each checklist appears on the relevant schedule activity.
  3. Link inspections to current drawings, details, specifications, and approved submittals. Confirm the referenced documents are the current approved revision.
  4. Define photograph requirements by location, direction, scale, identifier, and concealment stage. State the required photograph set for each preconcealment hold point.
  5. Define testing records, calibration, witness, result, and acceptance requirements. State the required test record for each applicable package.
  6. Define deficiency categories such as incomplete, nonconforming, damaged, unapproved, untested, or design question. State the distinguishing definition of each category.
  7. Create a deficiency log with unique number, location, requirement, responsible party, due date, status, and verification. Confirm the log fields match this structure exactly.
  8. Define stop-work, repair, replacement, engineering, mockup, and reinspection thresholds. State the condition that triggers each threshold.
  9. Do not allow field staff to accept unauthorized deviations merely because replacement is inconvenient. State this rule directly in the plan.
  10. Define the nonconformance and deviation-approval process. State who may approve a deviation and what documentation the approval requires.
  11. Define protection of accepted work from weather, traffic, subsequent trades, and cleaning. State the protection requirement for each finish-critical package.

I. Test, Issue, and Operate the Plan

  1. Conduct a preconstruction quality meeting with affected trades. Review the quality register, acceptance criteria, and checklist set with the trades responsible for high-risk packages.
  2. Review installer qualifications, manufacturer requirements, storage, environmental conditions, and warranty conditions. Confirm these are documented on the applicable preparatory checklists.
  3. Test the plan on one representative high-risk assembly. Perform the preparatory, first-work, and in-process checks on the selected assembly.
  4. Revise checklists that are ambiguous or impossible to verify. Correct any checklist item the field test could not be completed against as written.
  5. Issue the quality plan and controlled checklist set. Distribute the plan with its revision number and issuance date recorded.
  6. Perform scheduled inspections and issue deficiencies promptly. Enter each identified deficiency in the log at the time it is found.
  7. Verify corrective action rather than accepting photographs or statements without adequate evidence. Reinspect the corrected condition before closing the deficiency.
  8. Update the plan for changes, RFIs, substitutions, and lessons learned. Revise affected acceptance criteria and checklists when a controlling document changes.
  9. Archive completed checklists, photographs, tests, approvals, deficiencies, and acceptance records. File records in the job system established in Project 02.

Common Quality-Plan Inspection

Inspect every quality-control plan using the same checks regardless of project size or complexity.

  • Register check: confirm every work package in the approved WBS appears in the quality register.
  • Criteria check: confirm every acceptance criterion cites a specific governing document and no tolerance has been invented.
  • Checklist check: confirm preparatory, first-work, in-process, and preconcealment checklists exist for every high-risk or concealed package.
  • Responsibility check: confirm every checklist item names one inspector, witness, approval authority, and record owner.
  • Deficiency check: confirm the deficiency log uses the complete required field set and tracks each item to verified closure.
  • Field-test check: confirm the checklists were tested on one representative high-risk assembly before issuance and revised where necessary.

Failure Points and Troubleshooting

Work Is Concealed Before Required Checks Are Complete

Probable cause: the preconcealment hold point was not identified in Step 15 for that package, or the trade covered the work before the required internal checks, photographs, or tests were completed and verified.
Confirmation: check the quality register and checklist set for the package and confirm whether a preconcealment hold point exists and whether it was recorded as satisfied before the cover date.
Correction: where safely possible, expose the work for verification; where exposure is not possible, document the condition through available evidence, escalate the gap, and record the deviation and its resolution.
Reinspection: confirm the hold point is added or corrected in the checklist set and communicated to the responsible trade before the next occurrence of that package.

A Deficiency Is Disputed Because the Standard Is Unclear

Probable cause: the acceptance criterion for the item was written without citing a specific governing document, or no acceptance criterion exists for that package.
Confirmation: check the quality register for a cited source for the disputed criterion; confirm whether the drawing, specification, code, or manufacturer instruction actually establishes the standard being enforced.
Correction: resolve the ambiguity through the RFI process where the governing documents are genuinely silent or in conflict, record the resolved criterion with its citation, and update the checklist.
Reinspection: repeat the criteria check for the full quality register, not only the disputed item.

A Checklist Item Cannot Be Verified in the Field

Probable cause: the checklist item was written from the specification language directly without adapting it into an observable, measurable field check, or the representative-assembly test in Step 31 was skipped.
Confirmation: attempt to complete the checklist item against the actual installed condition and confirm whether a pass/fail determination can be made.
Correction: rewrite the item as a measurable, observable check, retest it against a representative condition, and reissue the corrected checklist.
Reinspection: repeat the field-test check on the corrected checklist before the next scheduled use.

Field Staff Accept an Unauthorized Deviation

Probable cause: the nonconformance and deviation-approval process from Step 27 was not communicated to field staff, or a deviation was accepted informally to avoid schedule impact.
Confirmation: check whether a written deviation approval exists for the condition and whether it was issued by the authority identified in Step 27.
Correction: stop further work relying on the unauthorized condition where safely possible, document the actual condition, and route it through the nonconformance process for a documented decision.
Reinspection: confirm the deviation-approval process has been reissued to the field team and acknowledged.

A Deficiency Is Closed Without Verification

Probable cause: the deficiency log entry was closed based on a trade's statement or an unverified photograph rather than a reinspection against the stated acceptance criterion.
Confirmation: check the deficiency log entry for a recorded verification date and verifier, and compare the closure evidence against the original cited criterion.
Correction: reopen the item, perform the required reinspection, and record the verification date and verifier before closing it again.
Reinspection: audit a sample of recently closed deficiency-log items for the same gap.

A Controlled Package Is Missing From the Quality Register

Probable cause: the quality register in Step 2 was not fully reconciled against the approved WBS, or a work package added by a later change was not added to the register.
Confirmation: compare the current WBS against the current quality register and identify any package without a corresponding register entry.
Correction: add the missing package to the register, define its acceptance criteria and checklist set, and issue the updated quality plan.
Reinspection: repeat the full register-to-WBS reconciliation check.

Field Terms

Quality Register
The controlled list of work packages requiring quality checks, aligned to the WBS.
Preparatory-Phase Check
The check performed before work begins, confirming documents, material, substrate, equipment, labor, and conditions.
First-Work Check
The check performed on the initial installation or a representative area before the rest of the package proceeds.
In-Process Check
Checks performed during installation at defined points such as layout, fastening, alignment, and sealing.
Preconcealment Hold Point
A defined point after which work may not be covered until required checks, photographs, and tests are complete.
Deficiency
An identified instance of incomplete, nonconforming, damaged, unapproved, or untested work.
Nonconformance
Work that does not meet the governing acceptance criteria and has not received an approved deviation.
Deviation Approval
Documented, authorized approval of a departure from the governing acceptance criteria.
Contractor QC
The contractor's planned verification of work before acceptance; it does not replace design review, code inspection, special inspection, testing, manufacturer inspection, or owner acceptance.
Acceptance Criterion
A measurable, document-cited pass/fail standard for a controlled work package.
