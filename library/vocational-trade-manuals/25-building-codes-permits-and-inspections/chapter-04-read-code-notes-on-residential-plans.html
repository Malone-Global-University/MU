Job Overview

This project qualifies you to locate, read, and organize the code notes printed on a residential plan set into one sheet-referenced code-note summary that a field crew can actually use. The finished work product is one complete note register and code-note summary tied to a specific plan revision, covering project data, fire and life safety, structural, MEP, and energy notes, with every note linked to its controlling sheet, detail, or specification.

You will confirm the plan date and approval status, build a note register keyed to sheet and subject, identify occupancy, construction-type, and fire-separation notes, identify alarm, egress, and emergency-escape notes, identify structural design-criteria and framing notes, identify plumbing, electrical, HVAC, and energy notes, compare notes against the actual drawings for conflicts, convert unresolved items into formal clarification questions, and issue a controlled summary for field planning while the approved documents remain controlling.

This project depends directly on the authority register and adopted-code sources built in Project 01. The code-note summary produced here becomes a working reference for every later inspection-readiness project in this book, including the foundation, framing, envelope, and MEP rough-in inspections and the Chapter 20 capstone.

Skills You Will Demonstrate

  • CP-019 — Locate code notes on a residential plan set and organize them into a sheet-referenced register.
  • CP-020 — Identify occupancy information stated on the plans.
  • CP-021 — Interpret construction-type notes as printed on the approved documents.
  • CP-022 — Identify fire-safety notes, including separation, alarm, and egress requirements shown on the plans.
  • CP-023 — Identify energy-code notes shown on the plans and supporting energy documents.
  • CP-024 — Create a code-note summary linking every note to its controlling sheet, detail, or specification.

Where This Skill Fits

A plan set carries the project's actual design decisions in its notes, not only in its drawn geometry. A learner who skips the notes and reads only the drawing risks missing a fire-separation requirement, an energy assembly value, or a structural design criterion that never appears as a dimension line. This project teaches the discipline of reading every note, tracing it to its supporting detail or specification, and recording it where the field crew and later inspection-readiness projects can find it.

The site and zoning checklist in Project 05, the foundation-inspection preparation in Project 06, and every later rough-in and envelope project in this book reuse the code-note summary built here as their starting reference for project-specific requirements. The Chapter 20 capstone repeats this exact review with the final issued plan set.

Scope and Intended Use

This project covers:

  • confirming the plan date, revision, approval status, and jurisdiction of the plan set under review
  • locating cover-sheet project data, general notes, code notes, life-safety notes, structural notes, MEP notes, and energy notes
  • building a note register keyed to sheet, subject, affected trade, project stage, and verification method
  • comparing notes against the drawings and against each other for conflicts or missing implementation detail
  • converting unclear or conflicting notes into formal clarification questions and recording approved responses
  • producing one code-note summary organized by project stage and trade, tied to the exact plan revision reviewed

This project does not authorize the learner to:

  • redo, verify, or approve the structural, energy, or life-safety design shown in the notes
  • resolve a conflict between notes and drawings by selecting the least costly interpretation
  • treat the code-note summary as a replacement for the approved plan set, which remains the controlling document
  • assume a listed code edition or design criterion is correct without comparing it against the Chapter 1 authority register

Any note that cannot be reconciled with the drawings, specifications, or authority register must be recorded as an open clarification question rather than resolved by assumption.

Trade Note

A code-note summary is only accurate for the plan revision it was built from. A revised sheet issued after this project closes out invalidates any note register entry tied to the superseded sheet. Confirm the reviewed revision date every time the summary is reused.

Setup Standard

Do not begin the note register until the plan date, revision, and approval status are confirmed. A note pulled from a superseded sheet or an unapproved preliminary drawing produces a summary that misdirects the field crew rather than assisting it.

Shop Tip

Read the cover sheet and general notes completely before touching a single trade-specific sheet. Project data, code editions, and design criteria stated once on the cover sheet govern every later note and are easy to miss if you start with the framing plan.

Common Field Fundamentals

Every plan set carries requirements in three places: the drawn geometry, the printed notes, and the referenced specifications or details. A note that says a wall is rated, an assembly meets an energy value, or a stair meets a code dimension is only as good as the detail or specification it points to. This project teaches the habit of tracing every note to its controlling reference before treating it as understood, the same habit used when reading approved documents throughout the rest of this book.

Reading Notes Against Their Controlling Reference

Every note that references a detail, section, schedule, or specification must be traced to that reference before the note is considered read. A note stating a requirement without a traceable controlling reference, or a note using generic language requiring local confirmation, must be recorded as an item requiring clarification rather than resolved by assumption.

Documentation Cue — Record Sheet and Reference for Every Note

Do not enter a note in the register without recording its sheet number, its subject, and the specific detail, section, or specification it points to. A note recorded without a traceable reference cannot be verified later and must be flagged for clarification.

Critical Safety Cue — Do Not Resolve a Conflict by Assumption

When a code note conflicts with the drawings, with another note, or with the Chapter 1 authority register, stop and record the conflict as a formal clarification question. Do not select the least costly interpretation, and do not instruct a trade to proceed on an assumed resolution.

Field Safety Cue — Superseded Sheets

Before entering any note in the register, confirm against the revision log that the sheet is the current controlling revision. Remove or clearly mark any note pulled from a superseded sheet, and do not distribute a code-note summary built from a mixture of current and superseded sheets.

Professional Boundary Cue — Interpretation Authority

The design professional, plans examiner, or building official interprets ambiguous or conflicting plan notes, not the learner or this summary. Record unresolved notes as open clarification questions and route them through the responsible party rather than interpreting them independently.

E. Preparation — Confirm the Plan Set and Establish the Register

  1. Obtain the current approved or permit-submittal plan set, specifications, code analysis, energy documents, and revision log. Record each document's title and date on the Plan Set and Review Record above.
  2. Confirm the plan date, revision, approval status, and jurisdiction. Compare the stated revision against the revision log before proceeding.
  3. Locate cover-sheet project data, code editions, local amendments, design criteria, occupancy or use, construction type where stated, building area, height, and story information. Enter each item in the Project Data register section.
  4. Do not assume a listed code edition is adopted or correct without comparing it with the authority register. Cross-check every cited edition and amendment against the Chapter 1 authority register and record any discrepancy.
  5. Locate general notes, code notes, life-safety plans, energy notes, structural notes, MEP notes, and standard details. Confirm the location of each note category before beginning detailed entry.
  6. Create a note register with note number, sheet, exact subject, affected trade, stage, and verification method. Use a single running numbering system across the entire plan set.

F. Identify Occupancy, Construction, and Life-Safety Notes

  1. Identify use and occupancy information and any accessory or mixed-use conditions. Enter this in the Project Data section with its sheet reference.
  2. Identify construction-type, fire-resistance, separation-distance, opening-protection, garage-separation, and shaft or chase notes where applicable. Trace each to its supporting detail.
  3. Identify smoke alarm, carbon-monoxide alarm, address, emergency escape, egress door, corridor, and exit notes. Enter these in the Life Safety register section.
  4. Identify stair, landing, handrail, guard, glazing, and safety-surface notes. Record the sheet and detail reference for each.

G. Identify Structural, MEP, and Energy Notes

  1. Identify structural design criteria for soil, frost, wind, snow, seismic, loads, and special inspection without redoing the engineering. Record the stated criteria and its source sheet in the Structural section.
  2. Identify foundation, anchor, hold-down, bracing, connector, nailing, and framing notes tied to inspections. Cross-reference these with the Project 06 and Project 07 inspection-readiness scope.
  3. Identify plumbing, electrical, gas, HVAC, ventilation, combustion, and equipment-clearance notes. Enter these in the MEP register section with their trade assignment.
  4. Identify energy requirements for insulation, fenestration, air sealing, ducts, equipment, controls, ventilation, and testing. Enter these in the Energy register section, tracing each to the supporting energy document.
  5. Identify accessibility or visitability notes when applicable to the project. Record these with their applicable sheet reference.

H. Identify Site, Product-Approval, and Reference Notes

  1. Identify zoning, setback, height, lot coverage, parking, flood, wildfire, or site notes shown on the plans. Enter these in the Zoning and Site register section.
  2. Identify product-approval, listed-system, manufacturer-instruction, testing, commissioning, and deferred-submittal notes. Enter these in the Product Approvals register section.
  3. Follow detail, section, schedule, and specification references from each critical note. Confirm the traced reference actually contains the implementation information the note promises.

I. Compare, Reconcile, and Clarify

  1. Compare notes with the actual drawings and identify contradictions or missing implementation details. Record each finding in the Conflicts and Clarifications register section.
  2. Compare architectural, structural, plumbing, electrical, mechanical, and energy notes for conflicts. Cross-check notes that address the same location or system across different sheets.
  3. Identify notes that use generic language requiring local or project-specific confirmation. Flag these rather than substituting an assumed value.
  4. Distinguish mandatory requirements from design recommendations and contractor coordination notes. Label each register entry accordingly.
  5. Identify field items that must remain visible for inspection. Cross-reference these with the applicable rough-in and framing inspection-readiness projects.
  6. Identify documents that must be available onsite for each inspection stage. Record this requirement in the corresponding register section.
  7. Convert unclear or conflicting notes into formal clarification questions. Use the clarification log format established in Project 01's Field Safety Cue.
  8. Record approved responses and update the note register. Do not close a clarification item until the response is recorded with date and source.

J. Assemble, Review, and Issue the Summary

  1. Create a code-note summary organized by project stage and trade. Group register entries so each trade can locate its applicable notes without reading the full register.
  2. Link each summary item to the controlling sheet, detail, specification, or approved response. Confirm every item includes a traceable reference.
  3. Review the summary with the project manager, superintendent, and responsible trades. Record the names and date of this review.
  4. Issue the controlled summary for field planning while stating that approved documents remain controlling. Include this statement on the summary's cover page.
  5. Archive the summary with the exact plan revision reviewed. Record the revision date and number on the archived copy.

Common Summary Inspection

Inspect the finished note register and code-note summary using the same checks regardless of which section is being reviewed.

  • Revision check: confirm the reviewed plan date and revision number match the current controlling set.
  • Traceability check: confirm every register entry links to a controlling sheet, detail, or specification.
  • Cross-check: confirm every cited code edition matches the Chapter 1 authority register or is flagged as a discrepancy.
  • Conflict check: confirm every identified conflict was converted into a clarification question rather than resolved by assumption.
  • Clarification check: confirm at least one clarification question was answered and the response recorded.
  • Usability check: confirm the code-note summary is organized by stage and trade and can be located quickly by a responsible trade representative.

Complete the Code-Note Verification Worksheet in the Inspection Checklist section below before beginning Project 05.

Failure Points and Troubleshooting

A Register Entry Cannot Be Traced to Its Controlling Reference

Probable cause: the note was copied into the register without following its detail, section, or specification callout back to the actual referenced content.
Confirmation: locate the referenced detail, section, or specification and confirm it actually contains the implementation information the note promises.
Correction: update the register entry with the confirmed reference, or flag the note as a clarification item if no supporting reference exists.
Reinspection: repeat the traceability check on the corrected entry.

A Cited Code Edition Does Not Match the Chapter 1 Authority Register

Probable cause: the plan set's cover sheet or general notes reference a code edition that has since been superseded, or the plans were drafted using a different jurisdiction's default edition.
Confirmation: compare the plan's stated edition and amendment date directly against the Chapter 1 authority register entry for the same code.
Correction: record the discrepancy as a formal clarification question to the design professional or plans examiner rather than assuming either source controls.
Reinspection: repeat the code-edition cross-check after the clarification response is recorded.

A Note Was Resolved by Assumption Instead of Clarification

Probable cause: a conflicting or generic note was interpreted in the field or office to keep the review moving, rather than being logged and routed to the design professional or plans examiner.
Confirmation: review the Conflicts and Clarifications section for entries missing a recorded question and response.
Correction: remove the assumed interpretation from the summary, submit the formal clarification question, and update the register once the response is recorded.
Reinspection: repeat the conflict check on the corrected register.

The Summary Was Issued from a Superseded Plan Revision

Probable cause: a revised sheet was issued after the register was built, and the summary was not rechecked against the current revision log before distribution.
Confirmation: compare the revision date recorded in the summary against the current revision log for the plan set.
Correction: rebuild or revise the affected register entries using the current sheet, and reissue the summary with the corrected revision date.
Reinspection: repeat the revision check on the reissued summary.

The Code-Note Summary Cannot Be Used Quickly by a Trade

Probable cause: the summary retained the register's raw sheet-by-sheet order instead of being organized by project stage and trade.
Confirmation: ask a responsible trade representative to locate their applicable notes and time the search.
Correction: reorganize the summary by stage and trade grouping as required in Section J.
Reinspection: repeat the usability check with the same trade representative.

Field Terms

Code Note
A printed statement on a plan sheet that states a code, design, or administrative requirement applicable to the shown work.
General Note
A note applying broadly across the sheet or project rather than to one specific detail or location.
Controlling Sheet
The specific plan sheet, detail, or specification section a note refers to for its full implementation requirement.
Note Register
The working document recording note number, sheet, subject, trade, stage, and verification method for every note reviewed.
Plan Revision
A specific dated and numbered version of a plan sheet; only the current, approved revision controls field work.
Clarification Question
A formal question sent to the design professional or plans examiner when a note is unclear, conflicting, or missing implementation detail.
Code-Note Summary
The stage-and-trade organized field-planning document produced from the note register, subordinate to the approved plan set.
Design Criteria
Structural values such as soil, frost, wind, snow, seismic, and load assumptions stated on the plans and confirmed rather than re-derived.
Deferred Submittal
A design element submitted for approval after the initial permit, tracked through its own note and clarification entry.
Verification Method
The recorded means by which a register entry's controlling reference was confirmed, such as detail cross-check or clarification response.
