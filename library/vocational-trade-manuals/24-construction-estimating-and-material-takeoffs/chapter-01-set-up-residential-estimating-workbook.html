Job Overview

This project qualifies you to build the estimating workbook that every later chapter in this book depends on. The finished work product is one validated residential estimating workbook containing a project-information sheet, a document register, a revision log, a cost-code structure, quantity and pricing columns, labor formulas, indirect-cost sections, error checks, a reconciliation sheet, cell-status controls, and summary sheets by trade, cost code, room, phase, material, labor, equipment, subcontract, allowance, and total.

You will define the estimate purpose and project data, build the document register and revision log, select and align a cost-code structure, build quantity and pricing columns with visible formulas, build labor and indirect-cost sections, build error checks and a reconciliation sheet, enter and verify one test item through every calculation, intentionally break the workbook to confirm the checks catch common errors, correct the workbook, and save a controlled master template with a one-page use guide.

This is the first project in the book. Every chapter takeoff from Chapter 3 through Chapter 16 is entered into a copy of this workbook. The pricing controls built in Chapter 18, the proposal built in Chapter 19, and the full capstone estimate in Chapter 20 all depend on the cost codes, formulas, and version-control system established here.

Skills You Will Demonstrate

  • ET-001 — Organize estimate categories by defined purpose, project data, and document controls.
  • ET-002 — Create cost-code sections aligned with accounting, project management, procurement, and field reporting.
  • ET-003 — Set up quantity columns with visible, traceable sources and units.
  • ET-004 — Set up unit-cost columns with dated pricing, waste, and purchase-quantity conversion.
  • ET-005 — Set up labor columns with production rate, crew, hours, and labor cost formulas.
  • ET-006 — Prepare an estimating workbook for use, including error checks, reconciliation, protection, and a controlled master template.

Where This Skill Fits

Nothing measured, priced, or bid in this book has a home until this workbook exists. Every quantity takeoff chapter (Chapters 3 through 16) enters its component schedules into the cost-code sections built here. Chapter 17 builds its labor hours from the labor columns built here. Chapter 18 builds its selling price from the pricing, indirect-cost, and markup sections built here. Chapter 19 issues a proposal that must reconcile line-for-line with this workbook. Chapter 20 opens a fresh copy of this exact template to run the capstone estimate.

The error-checking and version-control discipline in this chapter — locked formulas, visible sources, a revision log, and a reconciliation sheet — is the same discipline used to prevent silent quantity and price errors from reaching a client proposal in Chapter 19.

Scope and Intended Use

This project covers:

  • defining estimate purpose, project data, document controls, and revision tracking
  • selecting and aligning a cost-code structure across sitework, concrete, framing, roofing, envelope, insulation, drywall, finishes, MEP, exterior work, labor, equipment, subcontractors, and indirect costs
  • building quantity, unit-cost, waste, purchase-quantity, labor, and total columns with visible formulas
  • building indirect-cost sections for general conditions, overhead, escalation, contingency, profit, and tax
  • building error checks, a reconciliation sheet, and cell-status controls
  • entering and verifying one test item through every calculation
  • saving a controlled master template with a workbook-use guide

This project does not authorize the learner to:

  • enter real project quantities, prices, or labor rates into the workbook; this chapter uses one test item only
  • issue the workbook as a client-facing bid proposal; that document is prepared separately in Chapter 19
  • substitute a personal cost-code structure for an established company cost-code structure where one exists
  • apply overhead, profit, escalation, or contingency percentages that are not supplied by company policy; this chapter labels those fields and leaves values blank pending policy input

Any workbook section that fails its own error check in this chapter must be corrected before the master template is saved.

Trade Note

Every quantity, price, and rate entered anywhere in this book must trace to a drawing, specification, field condition, approved assumption, or quotation. A workbook cell with no visible source is the single most common cause of an estimate that cannot be defended during a later review, which is why the source column built in this chapter is never optional.

Setup Standard

Shop Tip

Build the source column before you build the price column. Deciding where a quantity or price came from after the number is already in the cell is how untraceable estimates end up in a bid.

Input Cells, Formula Cells, and Cell Protection

Every column in this workbook is either an input column or a formula column, never both. Mark the difference so it does not depend on color alone, protect verified formula cells from direct entry, and confirm approved input cells remain available after protection is applied. A workbook where an estimator can accidentally overtype a formula is a workbook that will eventually produce an untraceable number.

Traceability Cue — Record the Source With Every Entry

Enter a source for every quantity and price the moment it is entered, not afterward. Do not leave a source field blank with the intention of filling it in later. A quantity or price without a recorded source cannot be defended in a later review and must be treated as unverified until a source is added.

Critical Control Cue — Protect Formulas Before Entering Real Data

Lock every verified formula cell and every reference table before any real project quantity is entered. An unprotected formula that is accidentally overwritten will silently corrupt every calculation that depends on it, including summaries, labor cost, and the final total bid, without producing any visible error.

Documentation Cue — Log Every Revision

Record workbook version, date, person, reason for change, and approval status in the revision log for every change to the master template, including formula corrections made during this chapter. Do not overwrite a prior version without a logged reason.

Software Cue — Confirm Protection and Filter Behavior

Confirm how the spreadsheet application's protection feature interacts with filters, sorting, and frozen headings before relying on it. Some protection settings will block a filter or sort operation on a protected sheet; identify this behavior before building summary sheets that depend on filtering.

E. Preparation — Purpose, Project Data, and Document Controls

  1. Define the estimate purpose, project type, estimate class, intended user, pricing date, currency, market, and required output. Record these on the project-information sheet before building any other section.
  2. Create the project-information sheet. Include project, client, address, estimator, reviewer, estimate number, document date, revision, issue date, and validity.
  3. Create the document register. Add columns for plans, specifications, addenda, RFIs, clarifications, field reports, quotations, and assumptions, each with title, date, revision, and status.
  4. Create the revision log. Add columns for workbook version, date, person, reason for change, and approval status, and enter the first row for this initial build.
  5. Select the cost-code structure. Use the company cost-code list if one exists; otherwise use the default structure in Section C. Confirm alignment with accounting, project management, procurement, and field reporting before continuing.
  6. Confirm the selected structure with a second person where a company structure exists. Record the confirmation on the project-information sheet.
  7. Review the full list of required sections in Section B before building any single section. Confirm none will be omitted or merged.

F. Cost-Code, Column, and Formula Build Procedure

  1. Create the fourteen cost-code sections listed in Section C. Give each its own labeled area of the workbook.
  2. Create the columns listed in Section D for each cost-code section. Include cost code, description, location, source, quantity, unit, waste, order quantity, unit cost, material cost, labor rate, crew, hours, equipment, subcontract, tax, and total.
  3. Create quotation fields. Add supplier, quote number, date, expiration, freight, lead time, minimum order, and qualifications for each priced item.
  4. Create fields for assumptions, exclusions, allowances, alternates, owner-supplied items, and unresolved questions. Link each field to the affected cost-code row.
  5. Define approved units and visible conversion factors. Cover length, area, volume, package coverage, stock length, and weight, and place the conversion table on its own reference sheet.
  6. Build the order-quantity formula. Net quantity multiplied by one plus the waste percentage, converted into the approved purchase unit.
  7. Build the material-cost formula. Order quantity multiplied by unit cost.
  8. Build the labor-hours and labor-cost formulas. Hours from the production-rate basis; cost from hours multiplied by the burdened labor rate.
  9. Build the direct-cost formula. Material cost plus labor cost plus equipment plus subcontract, for each row and each cost-code section total.
  10. Build the general conditions, overhead, escalation, contingency, profit, and tax sections. Label the calculation basis for every percentage and distinguish markup from margin; leave rate values blank pending company policy or Chapter 18 input.
  11. Build the total-bid formula. Direct cost plus general conditions, overhead, escalation, contingency, profit, and tax.
  12. Label the calculation basis for every percentage field. State plainly whether each is applied to direct cost, a subtotal, or another defined base.
  13. Create summary sheets. Build summaries by trade, cost code, room, phase, material, labor, equipment, subcontract, allowance, and total, each linked to the cost-code sections.
  14. Create the assumptions, exclusions, clarifications, and risk register. Link every entry to the affected estimate row or section.

G. Error Checks, Protection, and Layout Controls

  1. Build checks for blank units, zero prices, duplicate codes, missing sources, negative values, broken links, and overwritten formulas. Place these checks on a dedicated error-check sheet with a visible pass or fail indicator for each check.
  2. Build the reconciliation sheet. Include a place to compare major quantities calculated by an independent method and to record the result of that comparison.
  3. Distinguish input, formula, linked, reviewed, and locked cells without relying on color alone. Use a labeled status field or cell note in addition to any color coding.
  4. Protect formulas and reference tables. Lock every verified formula cell and reference table while leaving approved input cells available for entry.
  5. Create filters, frozen headings, print areas, page headers, and export settings. Confirm filters function correctly on protected sheets per the Software Cue above.

H. Test Item and Verification (Perform Once, Using One Test Item Only)

  1. Enter one test item through quantity, waste, material, labor, tax, markup, and total. Use a clearly labeled placeholder cost code such as "TEST-001" so the item cannot be mistaken for real project data.
  2. Verify the test item appears correctly in every summary and review sheet. Confirm the trade, cost code, room, phase, material, labor, equipment, subcontract, allowance, and total summaries each show the test item's values without discrepancy.
  3. Intentionally enter each error type from the Section G checklist and confirm the workbook flags it. Test a blank unit, a zero price, a duplicate cost code, a missing source, a negative value, and a broken link one at a time, confirming the error-check sheet correctly identifies each.
  4. Correct formulas, rounding, units, broken links, display, and print issues found during testing. Log each correction in the revision log with the reason for the change.
  5. Remove the test item and all intentional test errors before saving the master template. Confirm the error-check sheet shows a clean result with the test data removed.

I. Closeout — Master Template and Use Guide

  1. Save a controlled master template. Use a clearly labeled master filename and document the project-copy procedure that will be used to start each later chapter's estimate.
  2. Prepare a one-page workbook-use guide. Cover inputs, formulas, versioning, review, and issue controls in language a new estimator could follow without additional instruction.

Common Workbook Inspection

Inspect the finished workbook using the same checks regardless of which cost-code section or summary sheet is being reviewed.

  • Structure check: confirm all fourteen cost-code sections exist and match the approved company or default structure.
  • Formula check: confirm order quantity, material cost, labor cost, direct cost, and total-bid formulas calculate correctly for the test item and are protected from direct entry.
  • Error-check function: confirm the error-check sheet correctly flags every intentional error entered in Section H, Step 3.
  • Summary check: confirm the test item appears correctly in all nine required summary sheets.
  • Documentation check: confirm the project-information sheet, document register, and revision log are complete and the initial version is logged.
  • Cleanliness check: confirm all test data and intentional test errors are removed before the master template is saved.

Complete the Workbook Verification Worksheet in the Inspection Checklist section below before beginning Project 02.

Failure Points and Troubleshooting

Summary Sheet Total Does Not Match a Cost-Code Section Total

Probable cause: the summary sheet references a stale range, a manually re-entered value instead of a live link, or a row inserted into the cost-code section after the summary link was built.
Confirmation: trace the summary cell's formula back to the source cost-code section and compare the referenced range to the section's current row range.
Correction: rebuild the summary link as a live reference to the full current row range and remove any manually re-entered value.
Reinspection: repeat the test item through the corrected summary and confirm the totals match.

The Error-Check Sheet Does Not Flag a Known Bad Entry

Probable cause: the check formula references the wrong column or range, or the check was built before a column was added or reordered.
Confirmation: re-run the Section H, Step 3 intentional-error test for the specific error type that failed to flag and inspect the check formula's referenced range.
Correction: rebuild the check formula against the current column layout and re-run all six intentional-error tests.
Reinspection: repeat the full Section H, Step 3 test sequence after correction.

A Formula Cell Was Overwritten With a Typed Value

Probable cause: the cell was not locked before data entry began, or protection was removed for an edit and not reapplied.
Confirmation: compare the cell's current content to the Section D formula basis for that column; a typed number where a formula is expected confirms the error.
Correction: restore the correct formula from the Section D formula basis, then reapply cell protection.
Reinspection: repeat the test item through the corrected cell and confirm cell protection is active.

A Quantity or Price Has No Recorded Source

Probable cause: the source field was left blank at entry with the intention of completing it later.
Confirmation: the error-check sheet's missing-source flag identifies the affected row.
Correction: add the specific document, field condition, approved assumption, or quotation supporting the value; if no source can be identified, mark the value as unverified and do not use it in a proposal.
Reinspection: repeat the missing-source check on the corrected row.

Filters or Sorting Fail on a Protected Sheet

Probable cause: the spreadsheet application's protection settings block filter or sort operations by default.
Confirmation: attempt to filter or sort the protected sheet and note the specific restriction reported by the application.
Correction: adjust the protection settings to explicitly allow filtering and sorting while keeping formula cells locked, per the application's own instructions.
Reinspection: repeat the filter and sort test on the corrected protection settings.

Two Cost-Code Sections Use Overlapping or Duplicate Codes

Probable cause: the cost-code structure was not fully confirmed against the company or default list before sections were built.
Confirmation: the error-check sheet's duplicate-code flag identifies the conflicting rows; compare both against Section C.
Correction: reassign the incorrect code to the correct value from Section C and update every formula or summary link referencing the corrected row.
Reinspection: repeat the duplicate-code check and confirm all downstream summaries reflect the corrected code.

The Test Item Was Left in the Workbook After Closeout

Probable cause: Section H, Step 5 was skipped or performed before all test errors were corrected.
Confirmation: search the workbook for the "TEST-001" placeholder code and review the error-check sheet for a non-clean result.
Correction: remove the test item and all intentional test errors, then re-verify a clean error-check result.
Reinspection: repeat the full Common Workbook Inspection section before saving the master template again.

Field Terms

Cost Code
The identifier organizing quantities and prices by trade, category, or function.
Net Quantity
The measured quantity before waste is added.
Order Quantity
The net quantity plus waste, converted into actual purchase units.
Direct Cost
Material, labor, equipment, and subcontract cost before general conditions, overhead, escalation, contingency, and profit.
Indirect Cost
General conditions, overhead, escalation, contingency, and profit applied after direct costs are established.
Markup vs. Margin
Two distinct calculation bases for adding profit to cost; the approved basis must be labeled and applied consistently.
Input Cell
A cell where the estimator enters a value directly; never contains a formula.
Formula Cell
A cell that calculates a result from input cells and is protected once verified.
Reconciliation
Comparing a major quantity by an independent method or reviewer to confirm accuracy.
Revision Log
The document recording workbook version, date, person, reason for change, and approval status.
