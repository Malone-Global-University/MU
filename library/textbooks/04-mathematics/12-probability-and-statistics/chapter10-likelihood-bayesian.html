Chapter 10 — Front Matter

Purpose and Overview

Every inference method rests on a question: given data, what can we learn about the unknown process that produced it? Chapters 8 and 9 developed two classical answers. Confidence intervals quantify uncertainty in estimation. Hypothesis tests evaluate evidence against a null claim. Both approaches use sample statistics whose sampling distributions are known — from the central limit theorem, from earlier distribution theory, from the careful analysis of Part II.

This chapter introduces a different level of the question. Rather than asking how a fixed procedure behaves across many samples, we ask: for this specific dataset, how does the evidence bear on different possible parameter values? That question leads to two powerful frameworks: maximum likelihood estimation and Bayesian inference. Both use the same mathematical object — the likelihood function — as their foundation, though they ask different questions of it.

By the end of this chapter, you should be able to construct likelihood functions for discrete and continuous models, find maximum likelihood estimates using calculus, understand priors and posteriors in the Bayesian framework, and connect these ideas to practice in statistics, machine learning, and data science.

Opening Question

A clinical trial tests a new drug. Of 20 patients enrolled, 14 respond positively. What is the most plausible value of the true response rate p? And if prior evidence from similar drugs suggests response rates cluster around 50%, how should that prior knowledge change your estimate? By the end of this chapter, you will have two rigorous answers — one from maximum likelihood, one from Bayesian inference — and the tools to choose between them in context.

Section 10.1

Probability Versus Likelihood

When you work with a probability distribution, two different things are in play: the parameter that defines the distribution and the data that the distribution generates. In most of Chapters 3 through 9, parameters were fixed or assumed unknown, while data was the random quantity. Probability answers: if the parameter is θ, what is the probability of observing data x?

Likelihood reverses the question. Data has already been observed — it is fixed. The parameter is unknown. Likelihood asks: given that we observed x, how does the probability of x change as we vary θ?

Formal Definitions

Probability (before observing data): For fixed θ, the probability of observing x is P(X = x ; θ) in the discrete case, or the density f(x ; θ) in the continuous case.

Likelihood (after observing data x): The same mathematical expression — now evaluated as a function of θ with x held fixed:

L(θ) = P(X = x ; θ) [discrete]    L(θ) = f(x ; θ) [continuous]

Critical Distinction

Likelihood is not a probability distribution over θ. The values L(θ) do not need to integrate or sum to 1 over the parameter space. Saying "the likelihood that θ = 0.5 is 0.3" is not a probability statement about θ — it is a comparative measure of how consistent θ = 0.5 is with the data.

A Concrete Contrast: Binomial Model

Suppose X ~ Binomial(n = 10, p) for unknown p. We observe x = 7 successes.

QuestionFormulaWhat VariesWhat Is Fixed
Probability — if p = 0.5, how likely is x = 7?P(X = 7 ; p = 0.5) = C(10,7)(0.5)⁷(0.5)³ ≈ 0.117Data xParameter p
Likelihood — given x = 7, how does p affect the probability?L(p) = C(10,7) p⁷(1−p)³ for p ∈ [0,1]Parameter pData x = 7
Example 10.1.1
Evaluating the likelihood at selected parameter values
  • SetupX ~ Binomial(10, p), observed x = 7. The likelihood is L(p) = C(10,7) p⁷(1−p)³ = 120 · p⁷(1−p)³.
  • Evaluate L(0.3) = 120 · (0.3)⁷ · (0.7)³ ≈ 120 · 0.000219 · 0.343 ≈ 0.009 L(0.5) = 120 · (0.5)⁷ · (0.5)³ = 120 · (0.5)¹⁰ ≈ 0.117 L(0.7) = 120 · (0.7)⁷ · (0.3)³ ≈ 120 · 0.0824 · 0.027 ≈ 0.267 L(0.9) = 120 · (0.9)⁷ · (0.1)³ ≈ 120 · 0.478 · 0.001 ≈ 0.057
  • Interpretp = 0.7 produces the highest likelihood. When 7 of 10 trials succeed, p = 0.7 makes that outcome more probable than any other value shown. The data "prefers" p = 0.7.
The likelihood peaks near p = 0.7. We will confirm this with calculus in Section 10.6.
Conceptual Note

The distinction between probability and likelihood is one of the most important conceptual shifts in statistics. Probability flows from parameter to data — it predicts what we might see. Likelihood flows from data back to parameters — it evaluates what is consistent with what we saw. Both use the same formula but ask fundamentally different questions.

Section 10.2

What a Likelihood Function Measures

When a model has a single parameter θ, the likelihood L(θ) is a curve — a function from the parameter space into the non-negative real numbers. High values indicate parameter values that make the observed data relatively probable. Low values indicate parameter values under which the data would be rare.

Likelihood Ratios

One of the most useful properties of likelihood is that it supports direct comparison through the likelihood ratio:

L(θ₁) / L(θ₂)

If this ratio equals 8, then θ₁ makes the observed data 8 times more probable than θ₂ does. Likelihood ratios do not depend on the overall scale of the likelihood — only on relative values. This is a natural tool for comparing competing parameter values or hypotheses.

Example 10.2.1
Computing and comparing likelihood values — coin flipping
  • ProblemA coin is flipped 5 times and lands heads 4 times. Let p be the probability of heads. Compare L(0.5) and L(0.8).
  • SetupX ~ Binomial(5, p), observed x = 4. The likelihood is L(p) = C(5,4) p⁴(1−p)¹ = 5p⁴(1−p).
  • Evaluate L(0.5) = 5(0.5)⁴(0.5) = 5/32 ≈ 0.156 L(0.8) = 5(0.8)⁴(0.2) = 5 · 0.4096 · 0.2 ≈ 0.410
  • RatioL(0.8) / L(0.5) = 0.410 / 0.156 ≈ 2.63
  • InterpretUnder p = 0.8, the observed data (4 heads in 5 flips) is about 2.63 times more probable than under p = 0.5. The data favors p = 0.8 by this factor.
Likelihood ratio L(0.8)/L(0.5) ≈ 2.63. The data is approximately 2.6 times more consistent with p = 0.8 than with p = 0.5.
Key Insight

The likelihood function is the data's voice about the parameter. It does not tell you the parameter with certainty — it tells you, relatively speaking, which parameter values the data finds more or less plausible. The shape of the likelihood — sharp peak versus broad plateau — also reflects how much information the data contains about θ.

Section 10.3

Likelihood for Discrete Models

In most problems, we observe not one data point but a sample of n independent observations. Because the observations are independent, the probability of the entire dataset is the product of individual probabilities. Viewed as a function of θ, this product is the likelihood.

Likelihood for Independent Discrete Observations

Let x₁, x₂, …, xₙ be observed values from independent random variables each with PMF P(Xᵢ = xᵢ ; θ). The likelihood is:

L(θ) = P(X₁ = x₁ ; θ) · P(X₂ = x₂ ; θ) · ⋯ · P(Xₙ = xₙ ; θ)

Example 10.3.1
Bernoulli likelihood — building from a specific sequence
  • SetupEight independent Bernoulli trials with unknown p. Observed sequence: 1, 0, 1, 1, 0, 1, 0, 1 — five successes, three failures.
  • ConstructEach observation contributes p^xᵢ(1−p)^(1−xᵢ). The full likelihood is their product:
    L(p) = p⁵(1−p)³
  • Evaluate L(0.5) = (0.5)⁵(0.5)³ = (0.5)⁸ ≈ 0.00391 L(0.625) ≈ (0.625)⁵(0.375)³ ≈ 0.00502 L(0.8) = (0.8)⁵(0.2)³ ≈ 0.00262
  • NoteThe likelihood peaks near p = 5/8 = 0.625, which equals the sample proportion of successes.
L(p) = p⁵(1−p)³. Likelihood peaks near p = 0.625. Section 10.6 confirms this with calculus.
Example 10.3.2
Poisson likelihood — three observations
  • SetupX₁, X₂, X₃ independent Poisson(λ). Observed: x₁ = 2, x₂ = 0, x₃ = 4.
  • ConstructPMF: P(X = k ; λ) = e^(−λ) λᵏ / k!
    L(λ) = [e⁻λλ²/2!] · [e⁻λλ⁰/0!] · [e⁻λλ⁴/4!] = e⁻³λ · λ⁶ / (2 · 1 · 24) = e⁻³λ · λ⁶ / 48
  • NoteThe denominator 48 is constant in λ. The likelihood's shape is determined by e^(−3λ)λ⁶. Total count = 2+0+4 = 6 over n = 3 observations, so x̄ = 2. The MLE for λ will be x̄ = 2.
L(λ) ∝ e^(−3λ)λ⁶. The likelihood depends on the data only through the total count — the sufficient statistic.
Pattern: Sufficient Statistics

For discrete exponential family distributions (Bernoulli, Binomial, Poisson, Geometric), the likelihood always depends on the data through a sufficient statistic — a summary that captures all the likelihood information. For Bernoulli models, it is the count of successes and failures. For Poisson, it is the total count. Identifying the sufficient statistic is the first step toward finding the MLE efficiently.

Section 10.4

Likelihood for Continuous Models

For continuous random variables, individual probabilities are zero — P(X = x) = 0 for any specific x. The likelihood uses the probability density function evaluated at the observed values. For independent observations, the likelihood is the product of densities:

L(θ) = f(x₁ ; θ) · f(x₂ ; θ) · ⋯ · f(xₙ ; θ)
Example 10.4.1
Exponential likelihood
  • SetupX₁, …, Xₙ independent Exponential(λ) with PDF f(x ; λ) = λe^(−λx) for x ≥ 0. Observed: x₁, …, xₙ.
  • Construct L(λ) = ∏ᵢ λe^(−λxᵢ) = λⁿ · exp(−λ∑xᵢ) = λⁿe^(−λS) where S = x₁ + x₂ + ⋯ + xₙ.
  • CheckFor n = 4 observations 0.5, 1.2, 0.8, 2.1: S = 4.6, x̄ = 1.15.
    L(0.87) ≈ (0.87)⁴ e^(−0.87·4.6) ≈ 0.01048 L(0.5) = (0.5)⁴ e^(−0.5·4.6) ≈ 0.00627 L(1.5) = (1.5)⁴ e^(−1.5·4.6) ≈ 0.00510
  • ObserveThe likelihood peaks near λ = 1/x̄ ≈ 0.870. Section 10.6 proves this with calculus.
L(λ) = λⁿe^(−λS). Likelihood depends on data only through sample sum S (sufficient statistic). MLE: λ̂ = 1/x̄.
Example 10.4.2
Normal likelihood with unknown mean (σ² known)
  • SetupX₁, …, Xₙ iid Normal(μ, σ²), σ² known, μ unknown.
  • Construct L(μ) = (1/√(2πσ²))ⁿ · exp(−(1/2σ²)∑(xᵢ − μ)²)
  • Key insightThe prefactor is constant in μ. Maximizing L(μ) is equivalent to minimizing ∑(xᵢ − μ)². This sum is minimized at μ = x̄.
  • ConclusionThe MLE for μ in the Normal model with known σ² is x̄. This is a deep connection: least squares is maximum likelihood under normally distributed errors.
MLE: μ̂ = x̄. Minimizing sum of squared deviations is equivalent to maximizing the Normal likelihood.
Normal MLE — Both Parameters Unknown

When both μ and σ² are unknown in the normal model, the MLEs are μ̂ = x̄ and σ̂² = (1/n)∑(xᵢ − x̄)². Note the MLE divides by n, not n − 1. The MLE for σ² is biased downward — a point examined in Section 10.8. The standard unbiased estimator S² = (1/(n−1))∑(xᵢ − x̄)² corrects this.

Section 10.5

Log-Likelihood

The likelihood function is a product of probabilities or densities. For large samples this product becomes extremely small, causing numerical precision problems. More importantly, products are harder to differentiate than sums. The natural logarithm transforms products into sums and does not change where the maximum occurs — because the logarithm is strictly increasing. If L(θ) is maximized at θ*, then so is ln L(θ).

Log-Likelihood Definition

The log-likelihood function is:

ℓ(θ) = ln L(θ) = ∑ᵢ ln f(xᵢ ; θ)

We denote the log-likelihood by ℓ (lowercase ell). The maximum of L(θ) and ℓ(θ) occur at the same θ. In virtually all applied statistics, machine learning, and software, the log-likelihood is the quantity being maximized.

Example 10.5.1
Bernoulli log-likelihood and its derivative
  • SetupFrom Section 10.3: L(p) = p⁵(1−p)³ for 5 successes in 8 trials.
  • Log-likelihoodℓ(p) = 5 ln p + 3 ln(1−p)
  • Derivativedℓ/dp = 5/p − 3/(1−p)
  • Set to zero5(1−p) = 3p → 5 − 5p = 3p → p̂ = 5/8 = 0.625
The MLE for p is 5/8 = 0.625, the sample proportion. The log-likelihood made the calculus straightforward.
Example 10.5.2
Poisson log-likelihood
  • SetupL(λ) ∝ λˢe^(−nλ) where S = ∑xᵢ is the total count and n is sample size.
  • Log-likelihoodℓ(λ) = S ln λ − nλ (plus constants)
  • Derivativedℓ/dλ = S/λ − n = 0 → λ̂ = S/n = x̄
MLE for Poisson rate λ is the sample mean x̄.
Example 10.5.3
Normal log-likelihood, both parameters unknown
  • Log-likelihood ℓ(μ, σ²) = −(n/2)ln(2π) − (n/2)ln(σ²) − (1/2σ²)∑(xᵢ − μ)²
  • Partial wrt μ∂ℓ/∂μ = (1/σ²)∑(xᵢ − μ) = 0 → μ̂ = x̄
  • Partial wrt σ² (let v = σ²)∂ℓ/∂v = −n/(2v) + ∑(xᵢ−μ)²/(2v²) = 0 → σ̂² = (1/n)∑(xᵢ − x̄)²
MLE: μ̂ = x̄ and σ̂² = (1/n)∑(xᵢ − x̄)². Note: MLE for variance divides by n, not n−1.

Section 10.6

Maximum Likelihood Estimation

The maximum likelihood estimate (MLE) of parameter θ, given observed data, is the value θ̂ that maximizes the likelihood — equivalently, the log-likelihood:

θ̂ = argmax L(θ) = argmax ℓ(θ)
Finding MLEs by Calculus — The Score Equation

For differentiable log-likelihoods, the MLE satisfies the score equation:

dℓ/dθ = 0

For multiple parameters, solve the system ∂ℓ/∂θⱼ = 0 simultaneously. Always verify the second derivative is negative (confirming a maximum, not a minimum or saddle point).

Example 10.6.1
Binomial MLE — deriving p̂ = x/n
  • SetupX ~ Binomial(n, p), n known, p unknown. Observed x successes.
  • Log-likelihoodℓ(p) = x ln p + (n−x) ln(1−p) + constant
  • Score equationdℓ/dp = x/p − (n−x)/(1−p) = 0 → x(1−p) = (n−x)p → x − xp = np − xp → x = np → p̂ = x/n
  • Verify maxd²ℓ/dp² = −x/p² − (n−x)/(1−p)² < 0 for 0 < x < n. ✓ Confirmed maximum.
MLE: p̂ = x/n. The MLE for a Binomial probability is the sample proportion. Intuitively: the value that makes the data most probable is the proportion actually observed.
Example 10.6.2
Exponential MLE
  • Log-likelihoodℓ(λ) = n ln λ − λ∑xᵢ
  • Score equationdℓ/dλ = n/λ − ∑xᵢ = 0 → λ̂ = n/∑xᵢ = 1/x̄
  • Verify maxd²ℓ/dλ² = −n/λ² < 0. ✓
MLE: λ̂ = 1/x̄. The MLE for the Exponential rate is the reciprocal of the sample mean.
Example 10.6.3
Poisson MLE
  • Log-likelihoodℓ(λ) = −nλ + (∑xᵢ) ln λ − ∑ ln(xᵢ!)
  • Score equationdℓ/dλ = −n + ∑xᵢ/λ = 0 → λ̂ = ∑xᵢ/n = x̄
  • Verify maxd²ℓ/dλ² = −∑xᵢ/λ² < 0 for xᵢ > 0. ✓
MLE: λ̂ = x̄. The MLE for the Poisson rate is the sample mean.

Special Case: Uniform Distribution MLE

The Uniform(0, θ) case shows that the score equation approach can fail. The likelihood L(θ) = (1/θⁿ) · 1[θ ≥ max(xᵢ)] is decreasing in θ (wherever it is positive), so it is maximized by making θ as small as possible — subject to θ ≥ max(x₁, …, xₙ). Therefore θ̂ = max(x₁, …, xₙ). This estimate is biased downward: E[x₍ₙ₎] = n/(n+1) · θ < θ. An unbiased correction is (n+1)/n · x₍ₙ₎.

Example 10.7.1
Normal MLE in Practice — Breaking strength data
  • DataBreaking strengths (MPa) of 5 ceramic rods: 312, 298, 325, 307, 319.
  • MLE for μμ̂ = x̄ = (312 + 298 + 325 + 307 + 319)/5 = 1561/5 = 312.2 MPa
  • MLE for σ²Deviations from 312.2: −0.2, −14.2, 12.8, −5.2, 6.8. Squared: 0.04, 201.64, 163.84, 27.04, 46.24.
    σ̂² = 438.8/5 = 87.76 σ̂ ≈ 9.37 MPa
  • NoteThe unbiased sample variance divides by n−1 = 4: s² = 438.8/4 = 109.7. The MLE (dividing by n = 5) is biased downward.
MLE: Normal(312.2, 87.76). The MLE for σ² underestimates; the unbiased S² = 109.7 corrects this.
Example 10.7.2
Poisson MLE in Practice — Bird arrivals
  • DataBird arrivals at a feeder in 10 one-minute intervals: 3, 1, 4, 2, 3, 0, 2, 5, 1, 3.
  • MLEλ̂ = x̄ = (3+1+4+2+3+0+2+5+1+3)/10 = 24/10 = 2.4 birds/minute
Fitted model: Poisson(2.4). This is the rate that makes the observed arrival counts most probable.

Section 10.8

Properties of Maximum Likelihood Estimators: An Introduction

MLE is not just a computational recipe — it produces estimators with strong theoretical properties. This section introduces the most important properties; full proofs are left to mathematical statistics courses.

PropertyStatementSignificance
Consistencyθ̂ converges in probability to the true θ* as n → ∞Larger samples produce more accurate estimates
Asymptotic normalityStandardized MLE is approximately N(0,1) for large nJustifies large-sample confidence intervals and tests
Asymptotic efficiencyMLE achieves the Cramér–Rao lower bound asymptoticallyMost precise consistent estimator in large samples
InvarianceIf θ̂ is MLE of θ, then g(θ̂) is MLE of g(θ)No need to re-derive likelihood for transformed parameters
Possible biasMLE may be biased in finite samplesUnbiased correction may be needed (e.g., divide by n−1 for variance)
The Cramér–Rao Lower Bound (Introduction)

For any unbiased estimator T of θ:

Var(T) ≥ 1/I(θ)

where I(θ) is the Fisher information:

I(θ) = E[(d/dθ ln f(X;θ))²] = −E[d²/dθ² ln f(X;θ)]

Fisher information measures how sensitive the log-likelihood is to the parameter. The MLE achieves this bound asymptotically — in large samples, it is the most precise consistent estimator.

Invariance Property in Practice

If λ̂ = 1/x̄ is the MLE for the Exponential rate λ, then the MLE for the mean μ = 1/λ is μ̂ = x̄. The MLE for P(X > t) = e^(−λt) is e^(−λ̂t) = e^(−t/x̄). No re-derivation is needed — plug in the MLE wherever the parameter appears.

Section 10.9

Bayesian Reasoning Review

Bayesian inference takes a different view of uncertainty from frequentist inference. Rather than treating parameters as fixed unknown constants, the Bayesian framework expresses uncertainty about parameters directly as a probability distribution. Before seeing the data, the analyst specifies a prior distribution P(θ) encoding beliefs or prior knowledge. After observing data x, the prior is updated through Bayes' theorem to produce a posterior distribution P(θ | x).

Bayes' Theorem for Statistical Inference

In the inferential context, Bayes' theorem becomes:

P(θ | x) = [P(x | θ) · P(θ)] / P(x)

Or compactly: Posterior ∝ Likelihood × Prior

The symbol ∝ means "proportional to." The marginal likelihood P(x) is a normalizing constant that ensures the posterior integrates to 1. Since it does not depend on θ, the essential computation is the product of likelihood and prior.

TermSymbolMeaning
PriorP(θ) or f(θ)Probability distribution over the parameter before seeing data
LikelihoodP(x | θ) = L(θ)Probability of observed data given parameter θ
Marginal likelihoodP(x)Normalizing constant — probability of data averaged over all θ
PosteriorP(θ | x)Updated distribution over θ after observing data
Example 10.9.1
Discrete Bayesian update — coin type
  • SetupA bag contains two coin types: Type A (fair, p = 0.5) and Type B (biased, p = 0.8). Prior: P(Type A) = 0.60, P(Type B) = 0.40. Observe two heads.
  • Likelihoods P(HH | Type A) = (0.5)² = 0.25 P(HH | Type B) = (0.8)² = 0.64
  • Joint probabilities P(HH and Type A) = 0.60 × 0.25 = 0.150 P(HH and Type B) = 0.40 × 0.64 = 0.256 P(HH) = 0.150 + 0.256 = 0.406
  • Posteriors P(Type A | HH) = 0.150/0.406 ≈ 0.370 P(Type B | HH) = 0.256/0.406 ≈ 0.630
  • InterpretPrior said 60% Type A, 40% Type B. After two heads, the posterior says 37% Type A, 63% Type B. Evidence of two heads shifted belief toward the biased coin.
Posterior: 37% Type A, 63% Type B. The likelihood of the data under Type B was higher, so observing HH shifted probability toward Type B.

Section 10.10

Priors

A prior distribution f(θ) encodes uncertainty about the parameter θ before the data is observed. Priors must be valid probability distributions — they integrate (or sum) to 1 over the parameter space. The choice of prior is one of the most discussed topics in Bayesian statistics.

Prior TypeDescriptionWhen to Use
InformativeReflects genuine prior knowledge; concentrated near plausible valuesWhen previous studies, expert knowledge, or historical data are available
Weakly informativeEncodes mild constraints (e.g., non-negativity, plausible range) without being restrictiveGeneral default; guards against implausible extremes while letting data dominate
Non-informative (flat)Uniform over parameter space; all values equal probabilityWhen little prior knowledge exists; requires care — flatness depends on parameterization
ImproperIntegrates to infinity; not a valid distribution but may yield valid posteriorsAdvanced settings; used with caution
Flat Priors Are Not Assumption-Free

A uniform prior on p ∈ [0, 1] looks assumption-free, but flatness depends on parameterization. A flat prior on p does not correspond to a flat prior on the log-odds logit(p) = log(p/(1−p)). The notion of "knowing nothing" is more subtle than it appears. Weakly informative priors that encode basic physical constraints are generally preferred to flat priors.

Example 10.10.1
Choosing a prior for a proportion — Beta distribution
  • ContextA political scientist believes voter support rate p is likely around 0.5 with standard deviation about 0.1. She wants to encode this as a Beta prior.
  • Beta momentsFor Beta(α, β): E[p] = α/(α+β), Var(p) = αβ/[(α+β)²(α+β+1)].
  • Solve for parametersSet α = β (for symmetry at 0.5). Then Var(p) = 1/(4(2α+1)) ≈ 0.01, giving 4(2α+1) = 100, so α = 12.
    Prior: p ~ Beta(12, 12)
  • InterpretBeta(12, 12) has mean 0.5 and standard deviation ≈ 0.1. It acts like having seen 11 prior successes and 11 prior failures — a moderate prior with real information.
Beta(12, 12) encodes prior belief that p ≈ 0.5 ± 0.1. Prior information corresponds to an effective prior sample size of about 22 observations.

Section 10.11

Posteriors

Given a prior f(θ) and likelihood L(θ) = f(x | θ), the posterior is proportional to their product. For conjugate priors (Section 10.12), the posterior falls in a recognizable family and the normalizing constant is automatic. From the posterior distribution, Bayesian inference extracts point estimates and intervals.

Posterior SummaryDefinitionInterpretation
Posterior meanExpected value E[θ | x] under the posteriorPoint estimate that minimizes expected squared error
MAP estimateMode of the posterior densityMost probable parameter value; equals MLE when prior is flat
Credible interval[a, b] such that P(a ≤ θ ≤ b | x) ≥ 0.95Direct probability statement: 95% posterior probability that θ ∈ [a, b]
Example 10.11.1
Beta-Binomial model — clinical trial
  • SetupResponse rate p ~ Beta(α, β) (prior). Observe n patients, x respond.
  • PosteriorLikelihood: L(p) ∝ p^x(1−p)^(n−x). Prior: f(p) ∝ p^(α−1)(1−p)^(β−1).
    f(p | x) ∝ p^(x+α−1)(1−p)^(n−x+β−1) This is the kernel of Beta(α + x, β + n − x).
  • Posterior familyp | x ~ Beta(α + x, β + n − x)
  • NumericalPrior Beta(2, 2), data n = 20, x = 14:
    Posterior: Beta(2+14, 2+6) = Beta(16, 8) Posterior mean = 16/(16+8) = 16/24 ≈ 0.667 Prior mean = 0.5 MLE = 14/20 = 0.70
  • InterpretThe data (14/20 = 0.70) and the prior (mean 0.50) blend to give a posterior mean of 0.667. The posterior pulls toward the data but not all the way — the prior's information acts as a modest counterweight.
Posterior: Beta(16, 8). Mean = 0.667. The posterior blends prior belief (mean 0.5) with data evidence (sample proportion 0.70).
Example 10.11.2
Credible interval versus confidence interval
  • Bayesian credible intervalFrom Beta(16, 8), the 95% credible interval is approximately [0.46, 0.84]. Interpretation: Given data and prior, there is a 95% posterior probability that p lies between 0.46 and 0.84.
  • Frequentist confidence intervalUsing Wald formula: p̂ ± 1.96√(p̂(1−p̂)/n) = 0.7 ± 1.96√(0.21/20) ≈ [0.499, 0.901].
  • CompareThe credible interval is narrower because the prior added effective observations. Both intervals contain 0.7 and are centered near 0.7, but their interpretations differ fundamentally.
Credible interval [0.46, 0.84] carries a direct probability interpretation. Confidence interval [0.50, 0.90] is a procedure — it captures the true p 95% of the time over many replications, but says nothing about this specific interval.

Section 10.12

Conjugate Priors: An Introduction

A prior is conjugate to a likelihood if the posterior falls in the same distributional family as the prior. Conjugacy produces closed-form posteriors and simple update rules. It is not required for Bayesian inference — modern computational methods handle non-conjugate settings — but it provides clean intuition and efficient computation for many standard models.

Example 10.12.1
Gamma-Poisson conjugate update
  • SetupData scientist models server crashes per hour as Poisson(λ). Prior: λ ~ Gamma(2, 1) — mean 2 crashes/hour. Observes 5 hours: 1, 3, 0, 2, 2. Total S = 8, n = 5.
  • Update ruleGamma-Poisson: Gamma(α, β) prior + Poisson data → Gamma(α + S, β + n) posterior.
    Posterior: λ | x ~ Gamma(2 + 8, 1 + 5) = Gamma(10, 6)
  • Summaries Posterior mean = 10/6 ≈ 1.67 crashes/hour Prior mean = 2.0 MLE = 8/5 = 1.6
  • InterpretThe posterior (1.67) lies between the prior (2.0) and the MLE (1.6), pulled toward the data but not all the way.
Posterior: Gamma(10, 6). Mean ≈ 1.67. The posterior blends prior knowledge with observed data.
Example 10.12.2
Normal-Normal conjugate update
  • SetupResting heart rate in rare-condition patients. Prior: μ ~ N(72, 25) (mean 72 bpm, τ² = 25). Known measurement σ = 8 bpm. Observe n = 9 patients with x̄ = 68 bpm.
  • Posterior precision 1/τ₁² = 1/τ² + n/σ² = 1/25 + 9/64 = 0.04 + 0.1406 = 0.1806 → τ₁² ≈ 5.54, τ₁ ≈ 2.35 bpm
  • Posterior mean μ₁ = τ₁² · (μ₀/τ² + nx̄/σ²) = 5.54 · (72/25 + 9·68/64) = 5.54 · (2.88 + 9.5625) ≈ 68.93 bpm
  • InterpretThe data (x̄ = 68) pulled the estimate below the prior mean (72). The posterior standard deviation (2.35 bpm) is smaller than both prior (5 bpm) and standard error of data (8/3 ≈ 2.67 bpm), reflecting gains from combining both information sources.
Posterior: μ | x ~ N(68.93, 5.54). The posterior mean is a precision-weighted average of prior and data means.

Likelihood and Bayesian Methods in Data Science

The frameworks developed in this chapter are not confined to traditional statistics. They underlie the training of modern machine learning models, the comparison of competing models, and probabilistic forecasting across science, engineering, and industry.

ApplicationConnection to Likelihood/Bayesian Methods
Linear regressionLeast squares = MLE under Normal error assumption. Minimizing residual sum of squares is maximizing the Normal log-likelihood.
Logistic regressionParameter estimation = maximizing Bernoulli log-likelihood. Cross-entropy loss = negative log-likelihood.
Neural network trainingTraining with cross-entropy loss = maximizing log-likelihood of a categorical distribution. Gradient descent on the log-likelihood is MLE in high dimensions.
Bayesian optimizationUses Gaussian process prior over objective function; updates posterior as new evaluations arrive. Widely used for hyperparameter tuning.

Key Insight — MLE and Machine Learning

Maximum likelihood estimation is the conceptual foundation of most machine learning training procedures. When a neural network minimizes cross-entropy loss, it is performing MLE on a categorical model. When a regression minimizes squared error, it is performing MLE on a Gaussian model. The mathematical framework of this chapter — likelihood as a function of parameters, differentiation, maximization — is the underlying principle behind a vast range of modern methods.

Section 10.14

Common Likelihood and Bayesian Mistakes

Mistake 1 — Confusing likelihood with probability
"The likelihood L(θ) integrates to 1 because probability distributions must integrate to 1."

Correction: Likelihood is not a probability distribution over θ. L(θ) values do not sum or integrate to 1. Likelihood is used comparatively — L(θ₁)/L(θ₂) > 1 means θ₁ is more consistent with the data than θ₂. It is not a probability statement about θ.

Mistake 2 — Treating the MLE as the "true" parameter
"The MLE is p̂ = 0.72, so the true probability is 0.72."

Correction: The MLE is an estimate — the most probable parameter given observed data. For a different sample, the MLE would differ. The MLE is a random variable with its own sampling distribution. Always accompany it with a standard error or confidence interval.

Mistake 3 — Assuming MLE is always unbiased
"MLE is the best estimator, so it must be unbiased."

Correction: MLE is consistent and asymptotically efficient, but not always unbiased in finite samples. The normal variance MLE σ̂² = (1/n)∑(xᵢ−x̄)² is biased downward; the unbiased estimator S² divides by n−1. The uniform MLE x₍ₙ₎ underestimates θ systematically.

Mistake 4 — Confusing credible intervals and confidence intervals
"The 95% confidence interval [0.4, 0.8] means there is a 95% probability that θ lies between 0.4 and 0.8."

Correction: A 95% confidence interval is a random procedure that captures the true θ in 95% of repeated samples. It says nothing about the probability that θ lies in the specific interval already computed. A 95% credible interval does carry that direct interpretation: P(θ ∈ [a, b] | x) = 0.95.

Mistake 5 — Thinking priors make Bayesian inference invalid
"Bayesian inference is subjective and therefore less reliable than frequentist methods."

Correction: All statistical inference involves choices — model family, estimator, significance level. Bayesian inference makes the prior explicit and open to scrutiny. When prior information is genuine, incorporating it improves inference. Sensitivity analysis (varying the prior) checks how robust conclusions are.

Mistake 6 — Thinking a flat prior means no assumptions
"A uniform prior on p ∈ [0,1] is completely objective because it treats all values equally."

Correction: Flatness depends on parameterization. A uniform prior on p ∈ [0,1] is not flat when reparameterized as logit(p). No prior is assumption-free. Weakly informative priors that encode basic physical constraints are generally preferred.

Mistake 7 — Forgetting to solve the full system in multi-parameter MLE
For Normal(μ, σ²) with both unknown, solving only ∂ℓ/∂μ = 0 and ignoring σ².

Correction: In multi-parameter problems, both partial derivatives must be set to zero and solved simultaneously. Optimizing over only one parameter while holding others fixed does not give the joint MLE — it gives a conditional optimum, which may be different.

Mistake 8 — Confusing MAP estimate and posterior mean
"The posterior mode and posterior mean are always the same."

Correction: The MAP estimate (posterior mode) and the posterior mean coincide only when the posterior is symmetric. For skewed posteriors (Beta, Gamma), they differ. The posterior mean minimizes expected squared error loss; the posterior mode is the most probable single value. Choose the appropriate summary for context.

Mistake 9 — Interpreting log-likelihood as probability
"ℓ(θ) = −10.4 means there is only a 10.4% chance the parameter equals θ."

Correction: Log-likelihoods are always ≤ 0 (since probabilities ≤ 1 and log(1) = 0). They cannot be interpreted as probabilities. Log-likelihoods are used for comparison and optimization — a less negative log-likelihood means a better fit. Use likelihood ratios or AIC/BIC for formal comparison.

Chapter 10 — Back Matter

Chapter Summary

  • Sec 10.1Likelihood and probability use the same formula but ask different questions. Probability flows from parameter to data (what might we see?). Likelihood flows from data back to parameters (what is consistent with what we saw?). Likelihood is not a probability distribution over θ.
  • Sec 10.2The likelihood function assigns a value to every parameter specification. Likelihood ratios L(θ₁)/L(θ₂) compare parameter values directly. A sharp, narrow likelihood indicates high information; a flat likelihood indicates low information about θ.
  • Sec 10.3For independent discrete observations, the likelihood is the product of individual PMF values. For exponential family models, the likelihood depends on the data only through a sufficient statistic — enabling efficient estimation.
  • Sec 10.4For continuous models, the likelihood is the product of density values at observed points. For the Normal model with known σ², maximizing the likelihood is equivalent to minimizing squared deviations — connecting MLE to least squares.
  • Sec 10.5The log-likelihood ℓ(θ) = ln L(θ) converts products to sums, avoids numerical underflow, and simplifies calculus. The maximum of L(θ) and ℓ(θ) occur at the same θ. Log-likelihoods are always ≤ 0 and used comparatively, not as probabilities.
  • Sec 10.6The MLE is the value θ̂ maximizing the log-likelihood. For smooth models, it satisfies the score equation dℓ/dθ = 0, confirmed by negative second derivative. Key MLEs: p̂ = x/n (Binomial), λ̂ = x̄ (Poisson), λ̂ = 1/x̄ (Exponential), μ̂ = x̄ (Normal).
  • Sec 10.7Common MLEs follow a pattern: for exponential family models, the MLE is a simple function of the sample mean. The Uniform upper bound MLE is the sample maximum — found by logical argument, not calculus — and is biased downward.
  • Sec 10.8MLE properties: consistency (converges to truth), asymptotic normality (enables large-sample inference), asymptotic efficiency (achieves Cramér–Rao bound), invariance (g(θ̂) is MLE of g(θ)). MLE may be biased in finite samples — notably for normal variance, where dividing by n rather than n−1 yields a biased estimate.
  • Sec 10.9Bayesian inference expresses parameter uncertainty as a probability distribution. Bayes' theorem: Posterior ∝ Likelihood × Prior. The prior encodes pre-data belief; the likelihood encodes what data reveals; the posterior combines them. As sample size grows, the likelihood dominates.
  • Sec 10.10Priors can be informative, weakly informative, or non-informative. Flat priors are not assumption-free — flatness depends on parameterization. Weakly informative priors that encode basic physical constraints while letting data drive inference are generally preferred in practice.
  • Sec 10.11The posterior f(θ | x) ∝ L(θ) · f(θ) is the complete updated distribution over θ. Point summaries include the posterior mean (minimizes squared error loss) and MAP estimate (posterior mode). A 95% credible interval carries a direct probability statement: P(θ ∈ [a, b] | x) = 0.95.
  • Sec 10.12Conjugate priors produce posteriors in the same family as the prior, with simple update rules. Key pairs: Beta-Binomial (add successes/failures to parameters), Gamma-Poisson (add count and n), Normal-Normal (precisions add, means are precision-weighted averages). Conjugacy is convenient but not required for Bayesian inference.
  • Sec 10.13MLE underlies linear regression (least squares = Normal MLE), logistic regression (cross-entropy loss = Bernoulli negative log-likelihood), and neural network training. Bayesian methods power optimization, A/B testing, and probabilistic forecasting. AIC and BIC compare models by balancing fit and complexity via the log-likelihood.
  • Sec 10.14Nine recurring mistakes: confusing likelihood with probability; treating MLE as the truth; assuming MLE is unbiased; confusing credible and confidence intervals; treating priors as making inference invalid; assuming flat priors are objective; solving only part of a multi-parameter system; confusing MAP and posterior mean; interpreting log-likelihood as probability.

Key Terms Review

likelihood function
L(θ) = f(x | θ): the joint probability or density of observed data, viewed as a function of the parameter with data fixed. Not a probability distribution over θ.
log-likelihood
ℓ(θ) = ln L(θ). Converts products to sums, avoids numerical underflow, and simplifies optimization. Maximum occurs at the same θ as L(θ).
maximum likelihood estimate
θ̂ = argmax ℓ(θ). The parameter value making observed data most probable under the model.
score equation
The condition dℓ/dθ = 0 whose solution gives the MLE. In multi-parameter problems, a system of partial derivative equations.
sufficient statistic
A function of the data that captures all likelihood information about θ. For Bernoulli models: count of successes. For Poisson: total count. Enables efficient estimation.
consistency
Property that the estimator converges in probability to the true parameter as n → ∞. A minimum requirement for any sensible estimator.
asymptotic normality
Property that the standardized MLE approaches N(0,1) for large n. Justifies large-sample confidence intervals and hypothesis tests based on the MLE.
Cramér–Rao bound
Lower bound on the variance of any unbiased estimator: Var(T) ≥ 1/I(θ). The MLE achieves this bound asymptotically.
Fisher information
I(θ) = E[(d/dθ ln f(X;θ))²]. Measures how much information a single observation carries about θ. Related to the curvature of the log-likelihood.
invariance property
If θ̂ is the MLE of θ, then g(θ̂) is the MLE of g(θ) for any function g. No re-derivation needed for transformed parameters.
prior distribution
f(θ): probability distribution over the parameter encoding uncertainty before observing data. Can be informative, weakly informative, or non-informative.
posterior distribution
f(θ | x) ∝ L(θ) · f(θ): updated distribution over θ after observing data. Combines prior belief with likelihood evidence.
marginal likelihood
The normalizing constant ∫ f(x | θ) f(θ) dθ ensuring the posterior integrates to 1. Does not depend on θ; treated as constant in posterior computation.
conjugate prior
A prior whose family is preserved under updating: if prior ∈ 𝒫 and likelihood is L(θ), then posterior ∈ 𝒫. Produces closed-form posteriors with simple update rules.
credible interval
An interval [a, b] such that P(a ≤ θ ≤ b | x) = 0.95 (or chosen level). Carries a direct probability statement about the parameter — unlike frequentist confidence intervals.
MAP estimate
Maximum A Posteriori estimate: the mode of the posterior distribution. Equals the MLE when the prior is flat. Differs from the posterior mean when the posterior is asymmetric.
Beta-Binomial model
Conjugate Bayesian model for a proportion: p ~ Beta(α, β) prior, Binomial likelihood. Posterior: Beta(α + x, β + n − x).
Gamma-Poisson model
Conjugate Bayesian model for a rate: λ ~ Gamma(α, β) prior, Poisson likelihood. Posterior: Gamma(α + S, β + n).
likelihood ratio
L(θ₁)/L(θ₂): the ratio comparing how well two parameter values explain the observed data. The foundation of likelihood ratio tests.
AIC
Akaike Information Criterion: 2k − 2ℓ(θ̂). Lower is better. Balances model fit against complexity by penalizing for the number of parameters k.
BIC
Bayesian Information Criterion: k ln n − 2ℓ(θ̂). Applies a stronger complexity penalty than AIC, especially for larger n.

Bridge to Chapter 11
Chapter 11: Linear Regression and Correlation

Chapter 10 developed two foundational frameworks: maximum likelihood estimation and Bayesian reasoning. Both use the likelihood function to connect data to parameters. Both produce principled estimates with interpretable uncertainty.

Chapter 11 applies these frameworks — and the estimation theory built throughout Part III — to regression analysis. Regression models the relationship between two or more variables as a function with unknown parameters. The most important case is linear regression, where the relationship is modeled as a line and the parameters (intercept and slope) must be estimated from data.

The connection runs deep. As Section 10.4 showed, maximizing the Normal log-likelihood with unknown mean is equivalent to minimizing the sum of squared deviations from that mean. In linear regression, the unknown "mean" depends on a predictor variable x, and the parameters govern how the mean changes with x. Minimizing the sum of squared residuals — the least squares criterion — is therefore maximum likelihood estimation under the assumption that errors are normally distributed. Every idea from this chapter carries forward.
