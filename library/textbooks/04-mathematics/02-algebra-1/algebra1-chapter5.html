Chapter 5 — Front Matter

Purpose and Overview

This chapter introduces one of the most important ideas in all of mathematics: the function. A function is a rule that assigns each input to exactly one output. That idea sounds simple, but it opens the door to nearly every topic that follows in algebra, geometry, statistics, science, finance, and computing.

Before this chapter, you worked with expressions that describe quantities and equations that describe conditions. Now you will begin studying relationships — situations where one quantity depends on another. Functions are the mathematical tool for describing, analyzing, and predicting those relationships.

By the end of this chapter, you will be able to read and write function notation, evaluate functions from equations, tables, and graphs, identify domains and ranges, and describe functions in words and symbols. This chapter prepares you for the deep work with linear functions in Chapters 6, 7, and 8, and for every function family you will encounter throughout Algebra I, Algebra II, and beyond.

Opening Question

A vending machine accepts a dollar bill and dispenses one item when you press a button. If you press button A3, you always get the same product. If you press button B7, you always get a different but consistent product. The machine never gives you two different items for the same button press.

In what ways is this vending machine behaving like a mathematical function? In what ways would it fail to be a function if it sometimes gave you two different items for the same button press? Keep this image in mind as you work through the chapter. A function is, at its core, a reliable rule: one input, one output, every time.

Why This Chapter Matters

Before algebra, most of the mathematics you studied focused on computing specific numbers: the sum of two values, the area of a shape, the percent of a total. In algebra, the focus shifts toward relationships between quantities. How does the total cost change as the number of items increases? How does the height of a thrown ball change over time? How does a person's pay depend on the number of hours worked?

The concept of a function gives mathematicians and scientists a precise language for describing these relationships. Nearly every equation you write in Algebra I can be thought of as a function. Nearly every graph you draw represents a function. And nearly every real-world model in finance, science, and technology is built from functions.

Section 5.1

What a Function Is

A Starting Point: Relations

Before defining a function precisely, it helps to think about a broader idea. A relation is any pairing of input values with output values. A relation simply associates some set of inputs with some set of outputs. There are no restrictions on how the pairing works.

Here is an example of a relation expressed as a set of ordered pairs:

{(1, 3), (2, 5), (3, 3), (4, 7)}

In this relation, the input 1 is paired with the output 3, the input 2 is paired with the output 5, and so on. Notice that the inputs 1 and 3 are both paired with the same output, 3. That is perfectly acceptable in a relation — different inputs can share the same output.

Now consider a different relation:

{(1, 3), (1, 5), (2, 6), (3, 4)}

Here, the input 1 is paired with two different outputs: both 3 and 5. This kind of ambiguity is the key distinction that separates a function from a general relation.

Definition — Function

A function is a relation in which every input is paired with exactly one output.

The input set is called the domain. The set of actual outputs is called the range.

Key Distinction — Inputs vs. Outputs

Repeated outputs are allowed. If input 3 always maps to output 7, and input 5 also maps to 7, the relation can still be a function. But if input 3 maps to both 7 and 9, that input is ambiguous and the relation is not a function. Focus only on repeated inputs when testing for functions.

Example 5.1.1
Identifying functions from ordered pairs
  • ProblemDetermine whether each relation is a function.

    (a) {(2, 4), (3, 9), (4, 16), (5, 25)}
    (b) {(1, 6), (2, 7), (1, 8), (3, 9)}
    (c) {(0, 0), (1, 1), (2, 4), (3, 9), (4, 16)}
  • ThinkFor each relation, list the input values. Check whether any input appears more than once with different outputs.
  • Solve (a) Inputs: 2, 3, 4, 5 — all different. → Function. (b) Input 1 appears twice: paired with 6 and with 8. → Not a function. (c) Inputs: 0, 1, 2, 3, 4 — all different. → Function.
  • CheckIn part (b), re-read (1, 6) and (1, 8). Both have the same input. If asked "What is the output for input 1?", there would be no single correct answer.
Answer: (a) Function.   (b) Not a function.   (c) Function.

Mapping Diagrams

Another way to represent a relation is a mapping diagram. A mapping diagram shows two sets — one for inputs and one for outputs — with arrows showing which inputs connect to which outputs. In a function, every input has exactly one outgoing arrow. No input can have two arrows leaving it.

Diagram Instruction — Mapping Diagrams

Draw two vertical ovals side by side. Label the left oval "Input" and the right oval "Output." Inside the left oval, list the values 1, 2, 3. Inside the right oval, list the values 5, 7, 9. Draw one arrow from 1 to 5, one arrow from 2 to 7, and one arrow from 3 to 9. This mapping represents a function because each input has exactly one outgoing arrow. Now draw a second mapping where the input 2 has arrows going to both 7 and 9 simultaneously — this represents a non-function because one input has two outgoing arrows.

Inputs and Outputs

Every function involves two roles: the input and the output. The input is the value you start with — what you put into the function. The output is the result — what the function produces from that input.

Section 5.3

Independent and Dependent Variables

VariableRoleDescription
Independent variableInputChosen freely; does not depend on the other variable
Dependent variableOutputDetermined by the input; its value depends on the independent variable
Tip — Which Variable Depends on Which?

Ask yourself: "Which quantity is being controlled or chosen, and which quantity is being produced?" The one being controlled is independent. The one being determined is dependent. In "total cost depends on hours worked," hours is independent and cost is dependent.

In most algebra problems, the independent variable is called x and the dependent variable is called y. When you write y = 2x + 3, the variable x is the independent variable (input), and y is the dependent variable (output). In real-world problems, letters are often chosen to reflect the quantities involved:

t for time, h for height: h = −16t² + 64t n for number of items, C for cost: C = 1.5n + 0.75 d for distance, s for speed, t for time: d = st

Section 5.4

Function Notation

When you write y = 2x + 3, the notation works, but it does not tell you the function's name, or how to clearly indicate evaluation at a specific input. Function notation was created to solve these problems.

Function Notation — Standard Form

f(x) = [rule]

Read: "f of x equals [rule]." Here, f is the function's name. The x inside the parentheses is the input variable. The rule describes what happens to the input.

Example: f(x) = 2x + 3 — the function named f multiplies the input by 2 and adds 3.

Critical Clarification — f(x) Is Not Multiplication

The notation f(x) does not mean f multiplied by x. The parentheses here are a notation convention meaning "the function f applied to the input x." This is one of the most common misreadings in Algebra I. f(3) does not mean f × 3; it means "evaluate the function at the input 3."

Naming Functions in Context

Functions are most commonly named f, g, or h, but any letter can be used. In applied problems, the function is often named after the quantity it produces:

Function NameInput VariableWhat It Represents
C(n)n = number of itemsCost function
h(t)t = time in secondsHeight function
A(s)s = side lengthArea function
P(h)h = hours workedPay function

Section 5.5

Evaluating Functions

Evaluating a function means substituting a specific value for the input variable and computing the output. The process always follows the same steps: identify the input value → substitute using parentheses → simplify using the order of operations → state the output.

Example 5.5.1
Evaluating a function at a single value
  • ProblemLet f(x) = 3x − 5. Find f(2), f(−1), and f(0).
  • ThinkReplace x with the given value in each case. Simplify completely.
  • Solve f(2) = 3(2) − 5 = 6 − 5 = 1 f(−1) = 3(−1) − 5 = −3 − 5 = −8 f(0) = 3(0) − 5 = 0 − 5 = −5
  • CheckFor f(2): 3(2) − 5 = 6 − 5 = 1. Correct. ✓
Answer: f(2) = 1,   f(−1) = −8,   f(0) = −5.
Example 5.5.2
Evaluating a quadratic function with negative inputs
  • ProblemLet g(x) = x² − 4x + 1. Find g(3) and g(−2).
  • ThinkSubstitute carefully. When squaring a negative number, use parentheses around the entire substituted value first.
  • Solve g(3) = (3)² − 4(3) + 1 = 9 − 12 + 1 = −2 g(−2) = (−2)² − 4(−2) + 1 = 4 + 8 + 1 = 13
  • CheckFor g(3): 9 − 12 + 1 = −2. Correct. ✓
Answer: g(3) = −2,   g(−2) = 13.
Example 5.5.3
Evaluating with a variable expression as input
  • ProblemLet f(x) = 2x + 5. Find f(a) and f(x + 1).
  • ThinkThe input does not have to be a number. Replace every instance of x with the new input expression.
  • Solve f(a) = 2a + 5 f(x+1) = 2(x + 1) + 5 = 2x + 2 + 5 = 2x + 7
  • CheckTest with x = 0: f(1) = 2(1) + 5 = 7. From the simplified form: 2(0) + 7 = 7. These agree. ✓
Answer: f(a) = 2a + 5,   f(x + 1) = 2x + 7.
Student Warning — Parentheses for Negative Inputs

Always put parentheses around a negative substituted value. Writing f(−3) = 2 · −3 + 5 is sloppy and leads to sign errors. Write f(−3) = 2(−3) + 5 instead. This habit prevents the most common evaluation mistake in Algebra I.

Section 5.6

Tables of Values

A table of values is an organized way to record several input-output pairs for a function. Tables make the function's behavior visible across multiple inputs at once, allow you to look for patterns, and serve as a bridge between the rule and the graph.

Example 5.6.1
Building a table of values
  • ProblemBuild a table of values for f(x) = 2x + 1 using x = −2, −1, 0, 1, 2, 3.
  • ThinkSubstitute each input into the rule and compute the output.
  • Solve
    xf(x) = 2x + 1f(x)
    −22(−2) + 1−3
    −12(−1) + 1−1
    02(0) + 11
    12(1) + 13
    22(2) + 15
    32(3) + 17
  • CheckFor x = 2: 2(2) + 1 = 5. The table shows 5. ✓
Answer: Table as shown above. Notice the output increases by 2 each time x increases by 1.
Reading Patterns from Tables

The output column above reads: −3, −1, 1, 3, 5, 7. Each time the input increases by 1, the output increases by 2. This constant increase reflects the structure of f(x) = 2x + 1 — the coefficient 2 controls how fast the output grows. You will study this idea in depth in Chapters 6 and 7 as slope and rate of change.

Example 5.6.2
Identifying a function from a table
  • ProblemDoes the following table represent a function?

    Input12334
    Output58111417
  • ThinkCheck whether any input value appears more than once with different outputs.
  • SolveInput value 3 appears twice, paired with outputs 11 and 14. Same input, two different outputs.
  • CheckIf asked "What is the output for input 3?", there is no single answer. That ambiguity confirms it is not a function.
Answer: Not a function — input 3 maps to two different outputs.

Section 5.7

Graphs of Functions

The graph of a function is a visual picture of all its input-output pairs. Each ordered pair (x, f(x)) becomes a point in the coordinate plane. When all those points are plotted, they form the shape of the function — a line, a curve, a scatter of dots, or some other pattern.

The Vertical Line Test

The vertical line test formalizes the definition on a graph: if any vertical line crosses a graph in more than one point, then the graph does not represent a function. If every vertical line crosses the graph in at most one point, then the graph represents a function.

The test works because a vertical line represents a single input value. If the line hits the graph in two places, that input has two outputs — which violates the definition.

Diagram Instruction — Vertical Line Test

Draw two graphs side by side. On the left, draw a straight diagonal line across the coordinate plane. Then draw several vertical dashed lines, each crossing the diagonal line at exactly one point. Label this "Passes the Vertical Line Test — Is a Function." On the right, draw a circle. Then draw one vertical dashed line cutting through the circle at two points (one on the top arc and one on the bottom arc). Label this "Fails the Vertical Line Test — Not a Function."

Example 5.7.1
Applying the vertical line test
  • ProblemDetermine whether each graph represents a function.

    (a) A straight line crossing the plane diagonally from lower-left to upper-right.
    (b) A circle centered at the origin.
    (c) A U-shaped parabola opening upward, lowest point at the origin.
  • ThinkFor each graph, imagine drawing vertical lines at different x-values. Would any vertical line hit the graph more than once?
  • Solve (a) Diagonal line: every vertical line crosses it exactly once. → Function. (b) Circle: a vertical line through the interior crosses it twice. → Not a function. (c) Parabola opening upward: every vertical line crosses it at most once. → Function.
Answer: (a) Function.   (b) Not a function.   (c) Function.
Diagram Instruction — Graphing from a Table

Using the table from Example 5.6.1, plot the ordered pairs (−2, −3), (−1, −1), (0, 1), (1, 3), (2, 5), (3, 7) in the coordinate plane. Draw a straight line through all six points. Label the line f(x) = 2x + 1. Notice that the points are collinear — they all lie on the same straight line — which reflects the constant rate of change built into the rule.

Section 5.8

Domain and Range

Two Key Descriptors of Every Function

Domain — the set of all allowable input values; all x-values for which the function is defined. Think of the domain as everything you are allowed to put into the function.

Range — the set of all output values the function actually produces when applied to its domain. Think of the range as everything that comes out.

Domain and Range from Tables and Graphs

When a function is given as a finite table of ordered pairs, the domain is the list of all input values and the range is the list of all output values. When a function is given as a graph, the domain is the set of all x-values the graph covers (reading left to right), and the range is the set of all y-values the graph covers (reading bottom to top).

Domain Restrictions

Some functions have natural domain restrictions — values that are not allowed because they would make the output undefined or impossible. The most common situations in Algebra I are:

SituationRestrictionExample
Division by zeroExclude values that make the denominator zerof(x) = 1/x; x ≠ 0
Square root of a negativeExclude inputs that produce a negative radicandf(x) = √x; x ≥ 0
Real-world contextExclude values that are physically impossibleHours worked ≥ 0; items ordered ≥ 0
Example 5.8.1
Identifying domain and range
  • ProblemState the domain and range of each function.

    (a) {(0, 1), (1, 4), (2, 9), (3, 16)}
    (b) f(x) = x + 5, with no context restrictions.
    (c) A graph: horizontal line segment from (−3, 4) to (5, 4), including both endpoints.
  • ThinkFor ordered pairs, list inputs and outputs. For an equation with no restrictions, domain and range are all real numbers. For a graph, read horizontal span for domain and vertical span for range.
  • Solve (a) Domain: {0, 1, 2, 3} Range: {1, 4, 9, 16} (b) Domain: all real numbers (no inputs make the rule undefined) Range: all real numbers (x + 5 takes every real value) (c) Domain: −3 ≤ x ≤ 5 (horizontal span from −3 to 5) Range: {4} (graph stays at y = 4 throughout)
Answer: (a) D: {0,1,2,3}, R: {1,4,9,16}.   (b) D: all reals, R: all reals.   (c) D: [−3, 5], R: {4}.
Memory Aid — Domain Before Range

D comes before R in the alphabet. Inputs come before outputs in the function process. Domain is inputs; Range is outputs. The domain and range are not always the same — pay attention to what the output actually does, not just what the inputs are.

Section 5.9

Discrete and Continuous Relationships

TypeDomainReal-World ExamplesGraph Appearance
DiscreteIsolated, separate values (often whole numbers)Number of tickets sold, items ordered, students in a classSeparate dots — never connected
ContinuousUnbroken interval of valuesTime, temperature, distance, weightConnected curve or line
Tip — How to Decide

Ask: can the input take any value within a range, including fractions and decimals? If yes, it is likely continuous. If the input can only take specific isolated values (typically whole numbers), it is likely discrete. Selling 2.7 tickets makes no sense — discrete. A balloon at height 4.37 feet at time 1.85 seconds makes perfect sense — continuous.

Example 5.9.1
Identifying discrete and continuous functions
  • ProblemDecide whether each situation is discrete or continuous.

    (a) Cost of buying whole boxes of crackers at $3 each, as a function of the number of boxes.
    (b) Height of a rising balloon as a function of time in seconds.
    (c) {(1, 3), (2, 6), (3, 9), (4, 12)}.
  • ThinkCan the input take any value in a range (continuous), or only specific isolated values (discrete)?
  • Solve (a) Only whole numbers of boxes: {0, 1, 2, 3, ...} → Discrete. (b) Time flows continuously; height exists at every fraction of a second → Continuous. (c) Only four specific input values given → Discrete.
Answer: (a) Discrete.   (b) Continuous.   (c) Discrete.
Student Warning — Do Not Connect Dots on a Discrete Graph

If a function is discrete, plot points only. Do not connect them with a line or curve. Connecting the points suggests the function is defined between them — which it is not. Graphing the number of books checked out per day calls for separate points, not a connected curve.

Section 5.10

Function Rules from Patterns

A table of values shows the behavior of a function, but not always the rule. For linear functions, the output changes by a constant amount each time the input increases by 1. When you find that constant change, you can identify the rule.

Example 5.10.1
Finding a linear rule from a table
  • ProblemFind the function rule for this table.

    x01234
    f(x)47101316
  • ThinkLook at how the output changes as x increases by 1. Then identify the starting value when x = 0.
  • Solve Change from x=0 to x=1: 7 − 4 = 3 Change from x=1 to x=2: 10 − 7 = 3 Change is constant: +3 each time → coefficient of x is 3. When x = 0, f(x) = 4 → constant term is 4. Rule: f(x) = 3x + 4
  • CheckFor x = 3: f(3) = 3(3) + 4 = 9 + 4 = 13. Table shows 13. ✓
Answer: f(x) = 3x + 4.
Connecting to Real Situations

A gym charges a one-time sign-up fee of $20 and then $15 per month. The total cost C after m months is C(m) = 15m + 20. The rule comes directly from the verbal description: the recurring monthly charge is the coefficient, and the one-time fee is the constant. This pattern — constant rate plus starting value — defines linear functions, which are the focus of Chapters 6 and 7.

Comparing Representations

A single function can be expressed in four main ways. Understanding how to move between representations is a core algebra skill that you will use throughout the entire course.

RepresentationStrengthsLimitations

Example 5.11.1
Moving between all four representations
  • ProblemA plumber charges a flat fee of $50 for a service visit, plus $30 per hour of work. Provide all four representations.
  • VerbalThe total cost equals 30 times the number of hours worked, plus the flat fee of 50 dollars.
  • Equation C(h) = 30h + 50
  • Table
    h (hours)01234
    C(h) (dollars)5080110140170
  • GraphThe graph begins at (0, 50) and rises steadily to the right. For every 1-unit increase in hours, the cost increases by $30. The graph is a straight line with positive slope, starting at the point (0, 50) on the vertical axis.

Same Function, Four Languages

The verbal description, equation, table, and graph in the example above all describe the same function. They are four different languages for one mathematical object. One of the great skills of algebra is the ability to read any of these representations and translate freely between them. This skill grows with every chapter you study.

Section 5.12

Functions in Real Situations

Functions allow you to take a real situation, strip it down to its mathematical structure, and use that structure to compute, predict, and explain. Here are four examples from contexts you will encounter throughout the MGU Mathematics Series.

Temperature Conversion

The function that converts Celsius temperatures to Fahrenheit is:

F(C) = (9/5)C + 32

The input is a Celsius temperature C, and the output is the equivalent Fahrenheit temperature. Evaluating: F(0) = 32 (freezing point), F(100) = 212 (boiling point).

Hourly Wages

If a worker earns $14.50 per hour with no overtime:

P(h) = 14.5h (h = hours worked, h ≥ 0)

The domain is restricted to non-negative values. The range is also non-negative.

Area as a Function of Side Length

The area of a square with side length s:

A(s) = s² (s > 0)

This is a quadratic function — a function family you will study in depth in Chapter 12.

Simple Interest Savings

A savings account earning simple interest of 4% per year on a $1,000 deposit:

B(t) = 1000(1 + 0.04t) (t = years)

The input is time, the output is the account balance. This is a linear function of t. Compound interest, studied in Algebra II, produces an exponential function.

Section 5.13

Common Function Mistakes

Mistake 1 — Misreading f(x) as Multiplication
f(4) means f × 4, so if f(4) = 14 then f = 3.5

The parentheses in f(x) mean "applied to," not "multiplied by." f(4) means "evaluate the function at input 4." It does not mean f times 4.

Mistake 2 — Confusing Domain and Range
The range of a function is the set of all input values.

Domain is always inputs (independent variable). Range is always outputs (dependent variable). D comes before R; inputs come before outputs.

Mistake 3 — Repeated Outputs Disqualify a Function
The table {(1,3), (2,7), (3,3), (4,11)} is not a function because output 3 appears twice.

Repeated outputs are allowed. What disqualifies a function is the same input giving two different outputs. In the table above, each input is unique, so it is a valid function.

Mistake 4 — Omitting Parentheses on Negative Inputs
g(−3) = −3² − 5(−3) = −9 + 15 = 6

Without parentheses, −3² evaluates as −(3²) = −9. The correct approach: g(−3) = (−3)² − 5(−3) = 9 + 15 = 24. Always substitute negative values inside parentheses.

Mistake 5 — Connecting Dots on a Discrete Graph
A student graphs the number of siblings per student by connecting the plotted points with a smooth curve.

A discrete function's graph should show individual points only. Connecting them claims the function is defined between the dots — which it is not. You cannot have 2.4 siblings.

Mistake 6 — Not Recognizing a Constant Function
f(x) = 5 is not a real function because x does not appear in the rule.

A constant function is a valid function. Every input maps to exactly one output — always 5 in this case. The graph is a horizontal line. Every input has exactly one (constant) output.

Chapter 5 — Back Matter

Chapter Summary

  • Sec 5.1A function is a relation in which every input is paired with exactly one output. A relation that sends one input to two different outputs is not a function. Mapping diagrams and ordered pairs both reveal whether a relation is a function.
  • Sec 5.2Every function has inputs (values put in) and outputs (values produced). The function machine model emphasizes that the same input always returns the same output — that consistency defines a function.
  • Sec 5.3The independent variable represents the input; the dependent variable represents the output. In most algebra contexts, x is the independent variable and y is the dependent variable, but any descriptive letters can be used.
  • Sec 5.4Function notation f(x) names a function and denotes the input. It is not multiplication. The notation f(3) means "evaluate f at input 3." Functions can be named f, g, h, or contextual names like C(n) or h(t).
  • Sec 5.5To evaluate a function, substitute the input value for the variable and simplify. Always use parentheses around negative inputs to avoid sign errors with exponents. Inputs can be numbers, variables, or expressions.
  • Sec 5.6A table of values records specific input-output pairs. Reading the output column reveals patterns such as a constant rate of change. A table represents a function if no input appears more than once with different outputs.
  • Sec 5.7The graph of a function plots all input-output pairs as points in the coordinate plane. The vertical line test checks whether a graph represents a function: if any vertical line hits the graph more than once, it is not a function.
  • Sec 5.8The domain is the set of all allowable inputs. The range is the set of all actual outputs. Common domain restrictions include division by zero, square roots of negatives, and real-world context limitations.
  • Sec 5.9A discrete function has isolated input values; its graph is a scatter of separate points. A continuous function has an unbroken domain; its graph is a connected curve or line. Never connect the dots on a discrete graph.
  • Sec 5.10Linear function rules can be found from tables by identifying the constant change in output as input increases by 1. The starting value at x = 0 gives the constant term. Rules can also be built from verbal descriptions.
  • Sec 5.11A single function can be expressed as a verbal description, equation, table, or graph. All four representations describe the same mathematical object. Translating between them is a core algebra skill used in every subsequent chapter.
  • Sec 5.12Functions model temperature conversion, wages, area, savings, and countless real situations. Algebra is not just a symbolic system — it is a tool for computing, predicting, and explaining the world.
  • Sec 5.13Six recurring errors: misreading f(x) as multiplication; reversing domain and range; concluding repeated outputs disqualify a function; omitting parentheses on negative inputs; connecting discrete graph dots; not recognizing constant functions.
  • Sec 5.14Function notation, tables, domain, range, and the four representations all carry forward into Chapters 6, 7, and 8 — and into every function family in Algebra I and Algebra II. The central questions are always: input, output, rule, graph.

Key Terms Review

function
A relation in which each input is assigned exactly one output. Same input always produces same output.
input
The value put into a function; the independent variable; the x-value in most algebra contexts.
output
The value produced by a function; the dependent variable; the result of applying the rule to an input.
relation
Any pairing of input values with output values, with no restriction on how inputs and outputs are matched.
mapping
A diagram showing two sets (inputs and outputs) with arrows connecting each input to its output(s).
independent variable
The variable representing the input; chosen freely; does not depend on the other variable in the relationship.
dependent variable
The variable representing the output; its value is determined by the independent variable.
function notation
The form f(x) used to name a function and denote its input. Not multiplication — means "function f applied to input x."
evaluate
Substitute a specific input value into a function rule and simplify to find the output.
table of values
An organized list of input-output pairs for a function; used to identify patterns and build graphs.
vertical line test
A visual test: if any vertical line crosses a graph more than once, the graph does not represent a function.
domain
The set of all allowable input values for a function. Think: everything you can put in.
range
The set of all actual output values produced by a function. Think: everything that comes out.
discrete function
A function with a domain of isolated, separate values. Graphed as individual points — never connected.
continuous function
A function defined over an unbroken interval. Graphed as a connected curve or line.
function rule
The algebraic expression describing how inputs are mapped to outputs. For example: f(x) = 3x + 4.
