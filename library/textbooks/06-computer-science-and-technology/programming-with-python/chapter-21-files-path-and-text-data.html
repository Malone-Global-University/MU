Chapter 21 — Front Matter

Chapter Introduction and Learning Objectives

Every program you have written so far forgets everything the moment it stops running. Variables disappear. Results are gone. The next time you run the program, you start from zero. Files change that. A file is data that lives on disk. When your program reads a file, it pulls stored information into memory. When it writes a file, it saves information that will still exist after the program ends, after the computer restarts, and after months have passed.

This chapter teaches you how to work with text files in Python. You will learn how the file system is organized, how paths tell Python where files live, how to open and close files safely, how to read text from files, how to write and append text to files, and how to handle common file errors. By the end of this chapter, your programs will be able to store and retrieve real data.

Opening Question

A grade-tracking program needs to remember every student's score even after it closes. A note-taking app needs your notes to still be there tomorrow. A weather station logs a temperature reading every hour, forever. None of these programs can rely on variables alone — variables vanish when the program ends. What do all three have in common? Each one depends on files: data written to disk that outlives the program that created it. This chapter shows you exactly how that works in Python.

Section 21.1

Why Programs Use Files

Think about programs you use every day. A text editor saves your writing. A spreadsheet saves your numbers. A chat app stores your message history. A game saves your progress. Every one of these programs uses files.

Python programs use files for many of the same reasons:

  • Persistence — saving information between runs so it is not lost
  • Input — reading data that was prepared before the program ran
  • Output — writing results that can be shared, printed, or opened later
  • Logging — recording what happened during a run for debugging or auditing
  • Configuration — reading settings that control how the program behaves
  • Data exchange — sharing information with other programs or people

Consider a grade-tracking program. Without files, the teacher must retype all the student grades every time she runs the program. With files, the grades are stored on disk. The program reads them at startup and writes updated grades when it finishes.

The shift from no-file programs to file-using programs is a big step. It means your programs can accumulate information over time, interact with data that already exists, and produce lasting results.

Study Note

Every program in Chapters 1 through 20 lost its data the moment it stopped. Files are the first tool in this book that breaks that limitation. Pay attention to how different this feels — it is a genuine turning point in what your programs can do.

Section 21.2

Files, Folders, and Paths

Before writing any Python, you need a clear picture of how files are organized.

The File System

Your computer stores files in a file system — a hierarchical structure of folders (also called directories) and files. A folder can contain files and other folders. This nesting creates a tree of locations from a starting point called the root.

On Windows, the root of a drive looks like C:\. On macOS and Linux, the root looks like /. Everything else branches from there.

What a Path Is

A path is a string that describes where a file or folder is located. Think of it as the address of a file on disk.

C:\Users\Rosa\Documents\grades.txt       (Windows)
/home/rosa/documents/grades.txt          (macOS / Linux)

Paths use separators between each folder name. Windows uses backslashes (\). macOS and Linux use forward slashes (/). Python on Windows also accepts forward slashes in many situations, and pathlib (covered in Appendix J) handles this portably.

What a File Name Is

The file name is the last part of the path. It typically has a base name and an extension that suggests what type the file is:

grades.txt        base name: grades     extension: .txt
notes.md          base name: notes      extension: .md
report.csv        base name: report     extension: .csv

The extension is a convention. Python does not enforce what you put inside a .txt file, but good naming helps you and others understand the file's purpose.

Section 21.3

Relative and Absolute Paths

There are two ways to describe a file's location.

Absolute Paths

An absolute path gives the full address from the root. It works regardless of where your program is running.

C:\Users\Rosa\Documents\grades.txt
/home/rosa/documents/grades.txt

Absolute paths are unambiguous. But they are fragile — if you move your project to a different computer or folder, the absolute path breaks.

Relative Paths

A relative path describes a location relative to the current working directory — the folder Python considers its base when it runs.

If Python is running in /home/rosa/projects/grades/, then:

grades.txt                     means /home/rosa/projects/grades/grades.txt
data/scores.csv                means /home/rosa/projects/grades/data/scores.csv
../notes.txt                   means /home/rosa/projects/notes.txt

The .. means "go up one folder level."

Which One to Use?

For beginner projects, relative paths are usually the right choice. Put your program file and your data files in the same folder, and use just the file name as the path. This keeps the project portable and simple.

# If grades.txt is in the same folder as your program:
filename = "grades.txt"

If you need subfolders:

filename = "data/grades.txt"
Good Habit

When you start a project, organize it before writing code. Create a folder for the project. Put your .py files and your data files in sensible locations inside that folder. Knowing where your files are before you code will save you frustration.

Section 21.4

Opening and Closing Files

Before Python can read or write a file, you must open it. Opening a file creates a connection between your program and the file on disk. Python calls this connection a file object.

The open() Function

The built-in open() function opens a file and returns a file object.

file = open("notes.txt", "r")

The first argument is the path. The second argument is the mode — a string that tells Python what you want to do with the file.

ModeMeaning
"r"Read. Opens an existing file for reading. Error if the file does not exist.
"w"Write. Creates a new file or overwrites an existing one.
"a"Append. Creates a new file or adds to the end of an existing one.
"x"Exclusive creation. Creates a new file. Error if it already exists.

If you omit the mode, Python defaults to "r".

Closing the File

When you are done with a file, you should close it. Closing releases the connection and ensures that any data waiting to be written actually gets saved to disk.

file = open("notes.txt", "r")
# ... do work with the file ...
file.close()

Forgetting to close a file can cause problems: data may not be saved, other programs may be locked out of the file, and your program may consume resources unnecessarily.

The with Statement — the Right Way to Open Files

Python gives you a cleaner and safer way to handle files: the with statement. It automatically closes the file when the block ends, even if an error occurs inside the block.

with open("notes.txt", "r") as file:
    # work with file here
    pass
# file is automatically closed here

The as file part gives the file object a name inside the block. Once the with block ends, the file is closed. You do not have to call file.close() yourself.

Rule

Always use the with statement when working with files. It is safer, cleaner, and shorter than manually opening and closing.

Section 21.5

Reading Text Files

Once a file is open for reading, Python gives you several ways to get the content.

Reading the Entire File at Once with .read()

The .read() method returns the entire file as one string.

with open("message.txt", "r") as file:
    content = file.read()

print(content)

This is simple and works well for small files. For very large files, loading everything into one string uses a lot of memory.

message.txt
Hello, Rosa.
Your report is ready.
Please review it by Friday.
Output
Hello, Rosa. Your report is ready. Please review it by Friday.

Reading One Line at a Time with .readline()

The .readline() method reads one line and moves a pointer forward. The next call reads the next line.

with open("message.txt", "r") as file:
    line1 = file.readline()
    line2 = file.readline()

print(line1)
print(line2)
Output
Hello, Rosa. Your report is ready.

Notice the blank lines. Each line from the file already ends with a newline character (\n), and print() adds another. You can strip the trailing newline with .rstrip():

line1 = file.readline().rstrip("\n")

Or more commonly, .strip() removes all leading and trailing whitespace:

line1 = file.readline().strip()

Reading All Lines into a List with .readlines()

The .readlines() method returns a list where each element is one line from the file, including the newline character.

with open("message.txt", "r") as file:
    lines = file.readlines()

print(lines)
Output
['Hello, Rosa.\n', 'Your report is ready.\n', 'Please review it by Friday.\n']

This is useful when you want to work with all the lines as a list.

Iterating Over a File Directly — the Cleanest Approach

The most Pythonic way to read a file line by line is to iterate over the file object directly using a for loop. This is memory-efficient and clean.

with open("message.txt", "r") as file:
    for line in file:
        print(line.strip())
Output
Hello, Rosa. Your report is ready. Please review it by Friday.
Best Practice

For reading files line by line, use a for loop directly over the file object. It is readable, memory-efficient, and handles large files gracefully.

Worked Example
Counting lines and words in a file
line_count = 0
word_count = 0

with open("message.txt", "r") as file:
    for line in file:
        line = line.strip()
        line_count += 1
        words = line.split()
        word_count += len(words)

print("Lines:", line_count)
print("Words:", word_count)
Output
Lines: 3 Words: 11
  • Iteration 1line is "Hello, Rosa." → line_count becomes 1. words is ["Hello,", "Rosa."] → word_count becomes 2.
  • Iteration 2line is "Your report is ready." → line_count becomes 2. words is ["Your", "report", "is", "ready."] → word_count becomes 6.
  • Iteration 3line is "Please review it by Friday." → line_count becomes 3. words is ["Please", "review", "it", "by", "Friday."] → word_count becomes 11.
  • LessonIt is easy to misjudge a running total by eye. Tracing word counts line by line — 2, then 6, then 11 — catches mistakes that a quick mental estimate would miss. This is exactly why tracing tables matter.
Section 21.6

Writing Text Files

Opening a file in "w" mode lets you write to it.

Warning

Opening an existing file in "w" mode destroys everything already in the file. The old content is gone immediately when you call open() with mode "w". There is no undo.

Writing with .write()

The .write() method writes a string to the file. It does not automatically add a newline — you must include \n yourself if you want line breaks.

with open("output.txt", "w") as file:
    file.write("Line one\n")
    file.write("Line two\n")
    file.write("Line three\n")
After running this, output.txt contains
Line one
Line two
Line three

Writing a List of Lines with .writelines()

The .writelines() method writes all strings in a list. Like .write(), it does not add newlines automatically.

lines = ["apple\n", "banana\n", "cherry\n"]

with open("fruits.txt", "w") as file:
    file.writelines(lines)
fruits.txt now contains
apple
banana
cherry
Worked Example
Generating a report file
names = ["Aisha", "Ben", "Clara"]
scores = [88, 74, 95]

with open("report.txt", "w") as file:
    file.write("Student Score Report\n")
    file.write("====================\n")
    for i in range(len(names)):
        line = f"{names[i]}: {scores[i]}\n"
        file.write(line)
    file.write("====================\n")
    average = sum(scores) / len(scores)
    file.write(f"Class average: {average:.1f}\n")

print("Report saved to report.txt")
Contents of report.txt
Student Score Report ==================== Aisha: 88 Ben: 74 Clara: 95 ==================== Class average: 85.7
  • What happensThe program does not print the report to the screen, except for the confirmation message. It writes it to a file that can be opened, shared, or printed later.
Section 21.7

Appending Text

When you open a file in "w" mode, you erase what was there. When you open a file in "a" (append) mode, Python adds new content at the end of the existing file. If the file does not exist, Python creates it.

with open("log.txt", "a") as file:
    file.write("Program started at 9:00 AM\n")

Run this several times. Each run adds another line to log.txt. The old content is preserved.

Worked Example
A simple log file
import datetime

def log_event(message):
    timestamp = datetime.datetime.now().strftime("%Y-%m-%d %H:%M:%S")
    entry = f"[{timestamp}] {message}\n"
    with open("events.log", "a") as file:
        file.write(entry)

log_event("Program started")
log_event("User entered a name")
log_event("Program finished")
  • Each runAdds three new lines to events.log. The function opens the file, writes one entry, and closes it again — three separate with blocks in total.
  • After two runsevents.log contains six lines in total, each with its own timestamp, in the order the events occurred.
events.log after two runs
[2025-03-12 10:14:02] Program started [2025-03-12 10:14:02] User entered a name [2025-03-12 10:14:03] Program finished [2025-03-12 10:15:30] Program started [2025-03-12 10:15:31] User entered a name [2025-03-12 10:15:31] Program finished

This pattern — opening in append mode, writing a timestamped entry, closing — is the basis of log files in almost every real program.

When to Use Each Mode

Use "w" when: You want to create or completely replace a file.

Use "a" when: You want to add to an existing file without losing what is already there.

Section 21.8

The with Statement

The with statement is Python's way of managing resources that need to be opened and closed properly. It is called a context manager.

with open("data.txt", "r") as file:
    content = file.read()

What happens step by step:

Execution Order
How a with block runs
  • Step 1Python calls open("data.txt", "r") and gets a file object.
  • Step 2The file object is assigned to file.
  • Step 3The code inside the block runs.
  • Step 4When the block ends — whether normally or because of an error — Python automatically calls file.close().

This guarantees the file is always closed, even if an exception occurs inside the block.

Compare the two approaches:

Without with — risky
file = open("data.txt", "r")
content = file.read()
file.close()       # If an error happens above, this never runs
With with — safe
with open("data.txt", "r") as file:
    content = file.read()
# file is always closed here

The with version is not just shorter. It is genuinely safer. Make it a habit from the start.

Opening Multiple Files at Once

You can open two files in one with statement using a comma:

with open("input.txt", "r") as infile, open("output.txt", "w") as outfile:
    for line in infile:
        outfile.write(line.upper())

This reads every line from input.txt and writes it in uppercase to output.txt.

Section 21.9

Newlines and Text Formatting

Text files are sequences of characters. Lines end with a newline character, written in Python as \n. Understanding newlines prevents a lot of confusing bugs.

What Newlines Look Like in Python

When you read a line from a file, the newline character is included at the end:

with open("items.txt", "r") as file:
    line = file.readline()
    print(repr(line))

If items.txt contains apple, the output will be:

'apple\n'

The repr() function shows you the string exactly as Python stores it, including invisible characters. This is a useful debugging trick.

Stripping Newlines

Most of the time you want to remove the trailing newline before working with the line. Use .strip() or .rstrip("\n"):

with open("items.txt", "r") as file:
    for line in file:
        item = line.strip()
        print(item)

.strip() removes all leading and trailing whitespace — spaces, tabs, and newlines. This is usually what you want.

Adding Newlines When Writing

When you write to a file, Python does not add newlines for you. If you forget \n, all your output runs together on one line:

with open("test.txt", "w") as file:
    file.write("first")
    file.write("second")
    file.write("third")
Contents of test.txt
firstsecondthird

That is one long line, not three separate lines. Add \n to each:

with open("test.txt", "w") as file:
    file.write("first\n")
    file.write("second\n")
    file.write("third\n")
Contents of test.txt
first second third

Cross-Platform Newlines

Windows uses \r\n (carriage return + newline) as the line ending. macOS and Linux use \n. Python's open() handles this automatically in text mode — it converts \r\n to \n when reading and writes the platform-appropriate ending when writing. You do not usually need to think about this, but it is good to know if you ever see mysterious \r characters in your data.

Section 21.10

Common File Errors

File operations can fail in many ways. Understanding the common errors helps you debug quickly and design programs that handle problems gracefully.

FileNotFoundError

This is the most common file error. It happens when you try to open a file for reading and the file does not exist at the path you specified.

with open("missing.txt", "r") as file:
    content = file.read()
Error
FileNotFoundError: [Errno 2] No such file or directory: 'missing.txt'

Common causes:

  • The file name is misspelled
  • The file is in a different folder than you think
  • The file has not been created yet
  • You used an absolute path on one computer but the file is not there on another

Debugging approach: Print the path before opening to confirm it is what you expect. Use os.path.exists() to check.

import os
path = "missing.txt"
print("File exists:", os.path.exists(path))

PermissionError

This happens when your program tries to open a file it is not allowed to access — for example, a system file, a file owned by another user, or a file that another program has locked.

PermissionError: [Errno 13] Permission denied: '/etc/passwd'

This is less common in student projects but can appear if you try to write to a read-only location or a protected system directory.

IsADirectoryError

This happens when you pass a folder path where Python expects a file path.

with open("my_folder", "r") as file:
    pass
Error
IsADirectoryError: [Errno 21] Is a directory: 'my_folder'

Writing to a File Fails Silently

One tricky error is not an exception at all. If you open a file in "r" mode and try to call .write(), you get an error. But sometimes students open in "r" mode and then wonder why their data was never saved — they forgot that "r" is read-only.

with open("output.txt", "r") as file:
    file.write("hello\n")    # io.UnsupportedOperation: not writable

Overwriting Instead of Appending

This is a logic error, not a Python exception. If you mean to add to a file but open it with "w" instead of "a", the old content is silently erased.

# Bug: this erases the file each time instead of adding to it
with open("log.txt", "w") as file:
    file.write("New entry\n")

Fix: use "a" mode when you want to preserve existing content.

Handling File Errors with try/except

You will learn exception handling fully in Chapter 23, but here is a preview of how to handle FileNotFoundError safely:

filename = "notes.txt"

try:
    with open(filename, "r") as file:
        content = file.read()
    print(content)
except FileNotFoundError:
    print(f"Error: the file '{filename}' was not found.")
    print("Please check the file name and location.")

This lets your program continue gracefully instead of crashing when a file is missing.
