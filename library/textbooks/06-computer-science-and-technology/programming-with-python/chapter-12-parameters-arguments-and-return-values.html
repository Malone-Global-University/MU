Chapter 12 — Front Matter

Chapter Introduction and Learning Objectives

Chapter 11 introduced functions as named, reusable blocks of code. You learned how to define a function, how to call it, and how execution jumps into a function and returns when it finishes. But those early functions had a significant limitation: they always did exactly the same thing every time. They could not adapt to different situations, work with different values, or hand a result back to the rest of the program.

This chapter removes that limitation. You will learn how to give information to a function using parameters and arguments. You will learn how functions send results back using return values. You will learn one of the most important distinctions in beginner Python programming: the difference between printing a result and returning a result. And you will learn how to design function interfaces that are clear, reusable, and easy to test.

Why This Chapter Matters

These ideas are foundational. Everything that follows in this textbook — data structures, files, classes, and projects — depends on writing functions that receive information and return results correctly. Confusing print() with return is one of the most common bugs beginning programmers write, and it is also one of the easiest to fix once you understand the difference clearly.

Section 12.1

Giving Information to Functions

Imagine you want a function that calculates the area of a rectangle. A rectangle's area is its width multiplied by its height.

Here is a version of that function that does not receive any information:

def calculate_area(): width = 5 height = 3 area = width * height print(area)

This function works, but it always calculates the area of a rectangle that is exactly 5 units wide and 3 units tall. It cannot calculate the area of any other rectangle. To calculate a different area, you would have to rewrite the function or write a new one.

That defeats the purpose of writing a function.

The solution is to design the function so that the width and height are supplied by whoever calls it. The function should work for any width and any height, not just specific values baked into the function body.

This is what parameters are for.

def calculate_area(width, height): area = width * height print(area)

Now the function receives the width and height from outside. Every time you call it, you can supply different values. The function does the same calculation every time, but it works on whatever values it receives.

Core Idea

Functions are more useful when they receive the information they need rather than assuming fixed values. A function with parameters can solve a whole family of problems, not just one specific case.

Section 12.2

Parameters

A parameter is a variable that appears inside a function's definition. It acts as a placeholder for a value that will be supplied when the function is called.

Parameters are listed inside the parentheses of the def line, separated by commas.

def greet(name): print("Hello,", name)

In this function, name is a parameter. It is a variable that does not exist yet when the function is defined. It will receive a value only when the function is called.

A function can have more than one parameter:

def describe_pet(animal, pet_name): print("I have a", animal, "named", pet_name)

Here animal and pet_name are two parameters. Both receive values when the function is called.

Key Points
What you need to remember about parameters
  • LocationParameters are defined inside the parentheses on the def line.
  • ScopeParameters are local variables. They only exist inside the function.
  • CountA function can have zero, one, or many parameters.
  • SourceParameters receive their values from arguments when the function is called.

Parameters are not yet real values when you write the function definition. They are named slots. The real values arrive from outside, at the moment of the function call.

Section 12.3

Arguments

An argument is the actual value that is passed into a function when you call it. Arguments fill the parameter slots.

greet("Maria")

Here, "Maria" is the argument. When the function call executes, name receives the value "Maria". From that point on, inside the function, name is "Maria".

describe_pet("cat", "Luna")

Here, "cat" is the argument for animal and "Luna" is the argument for pet_name.

Arguments are matched to parameters by position. The first argument goes to the first parameter, the second argument to the second parameter, and so on.

def calculate_area(width, height): area = width * height print(area) calculate_area(10, 4)

When this runs:

  • width receives 10
  • height receives 4
  • area becomes 40
  • The function prints 40

If you reverse the arguments, the parameters receive different values:

calculate_area(4, 10)

For calculate_area, the result is the same because multiplication is commutative. But for other functions — such as one that calculates a price after discount — the order matters significantly. Be careful to supply arguments in the correct order.

Section 12.4

Multiple Parameters

Functions can accept multiple parameters. Each parameter is a separate variable inside the function, and each receives a corresponding argument.

def calculate_total(price, quantity, tax_rate): subtotal = price * quantity tax = subtotal * tax_rate total = subtotal + tax print(total) calculate_total(12.99, 3, 0.08)

Tracing this call:

  • price receives 12.99
  • quantity receives 3
  • tax_rate receives 0.08
  • subtotal becomes 38.97
  • tax becomes 3.1176
  • total becomes 42.0876
  • The function prints 42.0876

You must supply exactly the right number of arguments. If a function has three parameters and you call it with two arguments, Python raises a TypeError:

calculate_total(12.99, 3)
TypeErrorTypeError: calculate_total() missing 1 required positional argument: 'tax_rate'

The error message is informative. It tells you which argument is missing. Read error messages carefully — they are your first debugging tool.

Similarly, supplying too many arguments is also an error:

calculate_total(12.99, 3, 0.08, "extra")
TypeErrorTypeError: calculate_total() takes 3 positional arguments but 4 were given
Worked Example
A function with three parameters
def describe_trip(destination, distance_km, days): print("Destination:", destination) print("Distance:", distance_km, "km") print("Trip length:", days, "days") print("Average per day:", distance_km / days, "km") describe_trip("Oslo", 480, 3)
OutputDestination: Oslo Distance: 480 km Trip length: 3 days Average per day: 160.0 km

The function works for any destination, distance, and number of days. Try calling it with different arguments.

Section 12.5

Return Values

So far, every function in this chapter has displayed its result using print(). That works for showing information to a user, but it has a serious limitation: the result vanishes after it is printed. No other part of the program can use it.

Consider this function:

def calculate_area(width, height): area = width * height print(area)

If you call calculate_area(10, 4), it prints 40. But what if you want to use that area in a larger calculation — say, to figure out the cost of flooring at a certain price per square unit? You cannot, because the value 40 was only printed, not stored or passed anywhere.

The solution is for the function to return its result.

The return statement sends a value back to wherever the function was called. The calling code can then store it, use it in an expression, print it, or pass it to another function.

def calculate_area(width, height): area = width * height return area

Now when you call this function, it produces a value that the rest of the program can use:

room_area = calculate_area(10, 4) print("Area:", room_area)
OutputArea: 40

You can also use the returned value directly in an expression:

price_per_unit = 3.50 total_cost = calculate_area(10, 4) * price_per_unit print("Total cost:", total_cost)
OutputTotal cost: 140.0
What return Does

1. It sends a value back to the calling code.

2. It immediately ends the function's execution.

Any code written after a return statement in the same function will not run:

def calculate_area(width, height): area = width * height return area print("This line never runs.") # unreachable code

The return statement is one of the most important tools in function design. Learn to use it consistently.

Section 12.6

Printing Versus Returning

One of the most common mistakes beginners make is confusing print() and return. This section explains the difference clearly and directly.

The Core Distinction

print() displays a value on the screen. That is all it does. The value is shown to the user, and then it is gone. No other part of the program can work with it further. return sends a value back to the calling code. The calling code can store it, use it, pass it forward, or print it. The value lives on.

Here are two versions of the same function. Read them carefully:

Version A — Prints the Result
def double(number): print(number * 2)
Version B — Returns the Result
def double(number): return number * 2

Now watch what happens when you try to use the result:

result = double(5) print(result)

With Version A:

  • double(5) runs and prints 10 to the screen immediately.
  • The function returns None (nothing).
  • result receives None.
  • The print(result) line prints None.
Output (Version A)10 None

With Version B:

  • double(5) runs silently.
  • The function returns 10.
  • result receives 10.
  • The print(result) line prints 10.
Output (Version B)10

The difference matters enormously when you want to use a function's result in further calculations, store it in a list, write it to a file, or pass it to another function. A function that only prints cannot do any of those things. A function that returns can.

General Rule

If a function computes a value that the rest of the program needs, use return. Use print() only when the function's specific job is to display information. A practical test: ask yourself, "Does the rest of my program need this value?" If yes, use return. If the function's only job is to show something to the user, print() is appropriate — but even then, it is often cleaner to return the value and let the calling code decide whether to print it.

Section 12.7

Using Function Results in Expressions

When a function returns a value, that value can be used anywhere an expression is valid. You can use it in arithmetic, comparisons, assignments, other function calls, or print statements.

def square(n): return n * n print(square(5)) # prints 25 print(square(3) + square(4)) # prints 25 (the Pythagorean triple) total = square(10) + square(2) print(total) # prints 104

You can also pass the result of one function directly as an argument to another:

def double(n): return n * 2 def square(n): return n * n print(square(double(3))) # double(3) returns 6, square(6) returns 36
Execution Trace
square(double(3))
  • Step 1double(3) is called. Returns 6.
  • Step 2square(6) is called with 6. Returns 36.
  • Step 3print(36) prints 36.

This kind of composition — using the result of one function as the argument to another — is a powerful and natural pattern in Python. Keep it simple and readable. If a chain of function calls becomes hard to read, break it into separate assignment steps.

Harder to Read
print(square(double(int(input("Enter a number: ")))))
Easier to Read
raw = input("Enter a number: ") number = int(raw) doubled = double(number) result = square(doubled) print(result)
Style Note

Clarity is more valuable than compactness, especially while you are learning.

Section 12.8

None and Missing Return Values

In Python, every function call produces a value. If a function does not have a return statement, or if execution reaches the end of the function without hitting return, the function automatically returns a special value called None.

None represents the absence of a value. It is not zero, not an empty string, not False. It is its own type, called NoneType.

def greet(name): print("Hello,", name) result = greet("Sam") print(result)
OutputHello, Sam None

greet does not have a return statement, so it returns None. When you assign the result of a call to greet to a variable, that variable holds None.

This is often a source of bugs. A common mistake looks like this:

def calculate_tip(bill_amount, rate): tip = bill_amount * rate print(tip) # prints, but does not return tip_amount = calculate_tip(45.00, 0.18) total = 45.00 + tip_amount # TypeError
TypeErrorTypeError: unsupported operand type(s) for +: 'float' and 'NoneType'

The programmer printed the tip but forgot to return it. When the code tries to add tip_amount to the bill, it gets a TypeError because tip_amount is None, not a number.

The Fix

Replace print(tip) with return tip.

Checking for None: you can check whether a variable holds None using is None:

result = greet("Sam") if result is None: print("The function returned nothing.")
Use is None, Not == None

The is operator checks identity, which is the correct way to test for None in Python.

Summary

Functions without return produce None.

Functions that hit a bare return with no value also produce None.

Storing or using None where a number or string is expected causes errors.

Always check whether your function is supposed to return something, and if so, make sure it does.

Section 12.9

Default Parameters at an Introductory Level

Sometimes a function has a parameter that usually takes the same value, but occasionally needs a different one. Python lets you assign a default value to a parameter. If the caller does not supply that argument, the default is used automatically.

def greet(name, greeting="Hello"): print(greeting + ",", name)

This function has two parameters: name and greeting. The parameter greeting has a default value of "Hello".

You can call it with one argument:

greet("Maria")
OutputHello, Maria

Or with two arguments to override the default:

greet("Maria", "Good morning")
OutputGood morning, Maria

Default parameters make functions more flexible without requiring callers to always supply every detail.

Another Example
A tip calculator with a default rate
def calculate_tip(bill_amount, rate=0.18): tip = bill_amount * rate return tip print(calculate_tip(50.00)) # uses default rate of 18% print(calculate_tip(50.00, 0.20)) # caller specifies 20%
Output9.0 10.0
Rules for Default Parameters

Default parameters must come after non-default parameters in the function definition.

This is valid: def f(a, b=10):

This is a SyntaxError: def f(a=10, b): — you cannot have a required parameter after a default parameter.

Default parameters are useful for optional settings, standard rates, default messages, and any case where one value makes sense for most callers but the function should still accept alternatives.

You do not need to use default parameters in every function. Apply them when they genuinely make a function easier to use without hiding important decisions.

Section 12.10

Designing Useful Function Interfaces

A function's interface is the combination of its name, its parameters, and its return value. The interface is what other parts of the program see and use. The body is the hidden implementation.

A well-designed function interface has these qualities:

Quality 1
A Clear Name
The name describes what the function does. Good names are clear and readable: calculate_area, is_valid_password, get_average, format_name. Avoid vague names like do_stuff, helper, or process.
Quality 2
Necessary Parameters Only
The parameters represent the information the function genuinely needs. Give the function exactly what it needs to do its job, and no more.
Quality 3
A Useful Return Value
The return value is the result the rest of the program needs. If the function computes something, return it. Let the caller decide whether to print it, store it, or use it further.
Quality 4
One Job, Done Well
A function called get_and_print_and_save_the_total is trying to do too much. Separate concerns into separate functions.
Worked Example
Designing a function from scratch
  • TaskWrite a function that takes a student's score and the maximum possible score, and returns the percentage as a float.
  • Step 1Identify the inputs: score and maximum.
  • Step 2Identify the output: the percentage as a float.
  • Step 3Name the function: calculate_percentage is clear and specific.
def calculate_percentage(score, maximum): percentage = (score / maximum) * 100 return percentage

Step 5 — Test it:

pct = calculate_percentage(38, 50) print(pct) # should print 76.0 pct = calculate_percentage(100, 100) print(pct) # should print 100.0 pct = calculate_percentage(0, 50) print(pct) # should print 0.0

This function is short, clear, and testable. It does one thing. It returns its result. It does not print anything. The caller decides what to do with the percentage.

Common Interface Design Mistakes

Mixing computing and printing in the same function makes the function harder to reuse:

Hard to Reuse — Prints, Cannot Be Used Further
def calculate_percentage(score, maximum): percentage = (score / maximum) * 100 print(f"Your score: {percentage:.1f}%")

Hard-coding values that should be parameters locks the function into one specific case:

Only Works for Scores Out of 100
def calculate_percentage(score): percentage = (score / 100) * 100 return percentage
Watch Out

Taking too many parameters makes a function hard to call correctly. If you find yourself writing a function with five or more parameters, consider whether it should be broken into smaller functions.

Good function design is a skill that develops with practice. Start simple. Keep functions small. Return values rather than printing them. Test each function separately before combining them.

Key Terms

parameter
A variable defined in a function's def line that acts as a placeholder for information the function will receive when called.
argument
The actual value passed to a function in a function call. Arguments fill parameter slots by position.
positional argument
An argument whose value is assigned to a parameter based on its position in the call.
return value
The value a function sends back to the calling code using the return statement.
return statement
A Python statement that ends a function immediately and sends a value back to the caller.
None
A special Python value representing the absence of a value. Functions without a return statement return None automatically.
NoneType
The type of None. When None is used where a number or string is expected, Python raises a TypeError.
default parameter
A parameter that has a preset value, used when the caller does not supply that argument.
function interface
The combination of a function's name, its parameters, and its return value — the "contract" between the function and the rest of the program.
composition
Using the return value of one function as an argument to another function.
unreachable code
Code that appears after a return statement in a function body and can never execute.

Preview — Chapter 13
Scope, Local Variables, and Program State

Chapter 12 gave you tools for passing information into functions and getting results back. But there is an important question we have not fully answered yet: what happens to variables that are defined inside a function? Can the rest of the program see them? Can the function see variables defined outside it?

These questions are about scope — where a variable exists and how long it lives. Chapter 13 explores local variables, global variables, variable lifetime, and program state. You will learn why variables defined inside a function are separate from variables with the same name outside it, why functions should usually communicate through parameters and return values rather than sharing variables, and how to avoid the confusing bugs that arise when scope is misunderstood.

Understanding scope is essential before functions can be used to build larger, more organized programs.
