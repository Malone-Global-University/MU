Chapter 24 — Front Matter

Chapter Introduction and Learning Objectives

Every program you have written so far has produced the same output every time it runs — given the same inputs, the same results appear. But many useful programs need to work differently. A quiz program should shuffle its questions. A game should generate unpredictable outcomes. A scheduling tool should know today's date. A scientific model should run thousands of slightly different trials and report what usually happens.

This chapter introduces two standard-library tools — random and datetime — along with the idea of simulation: using a program to model a situation, run it many times, and observe the results. These tools appear throughout computing, from games and testing to scientific research and probability modeling.

By the end of this chapter you will be able to generate random numbers and make random choices, work with dates and times, build simple simulations, run repeated trials, and interpret what simulation results mean.

Connecting to Earlier Chapters

Chapter 15 gave you a first look at import random and import math. Chapter 21 used import csv. This chapter goes deeper into random and introduces datetime. The loops, functions, lists, and dictionaries from earlier parts of the book all reappear here — randomness and dates become most powerful when you combine them with the structures you already know.

Section 24.1

Useful Tools from the Standard Library

You have already used the standard library throughout this course. Chapter 15 introduced math and gave you a first look at random. Chapter 21 used the csv module. Chapter 23 used exception types like ValueError and FileNotFoundError. The standard library is a large collection of modules included with every Python installation — you do not need to install anything extra to use them.

This chapter focuses on two modules that are especially useful for practical programs:

  • random — for generating unpredictable values, making selections, and shuffling sequences
  • datetime — for working with dates, times, and durations

Both modules follow the same import pattern you already know.

Python — importing both modules
import random
import datetime

After importing, you call their functions using dot notation: random.randint(...), datetime.date.today(), and so on. This chapter also introduces simulation, which is not a module but a programming technique. A simulation uses a program to model a real-world situation — often one involving randomness — and runs it many times to observe patterns.

Study Note

The random module generates what are technically called pseudo-random numbers — numbers that pass statistical tests for randomness but are actually produced by a deterministic algorithm. For the purposes of this course, they behave as random numbers. For cryptographic applications, Python provides the secrets module, which uses a different approach.

Section 24.2

Random Numbers

Random numbers are numbers chosen without a predictable pattern. Python's random module provides several functions for generating them.

random.randint(a, b)

Returns a random integer between a and b, inclusive. Both endpoints can be chosen.

Python — simulating a six-sided die
import random

roll = random.randint(1, 6)
print("You rolled:", roll)

Every time this program runs, roll is one of: 1, 2, 3, 4, 5, or 6. All six values are equally likely.

random.random()

Returns a random float between 0.0 and 1.0, not including 1.0. This is useful when you want a probability value or a percentage.

Python — a random probability value
import random

chance = random.random()
print(chance)   # something like 0.4732891...

random.uniform(a, b)

Returns a random float between a and b. Use this when you need a decimal value in a specific range.

Python — simulated temperature reading
import random

temperature = random.uniform(36.1, 37.2)
print(f"Simulated temperature: {temperature:.1f} degrees")

Tracing programs that use random values

Because random values change every run, tracing a program that uses randomness works differently from tracing one with fixed values. When you trace, choose a representative example value and trace with that — then trace again with a different value near a boundary.

Section 24.3

Random Choices

Sometimes you do not want a random number — you want a random item from a list. The random module provides three functions for this, and understanding the differences between them matters.

random.choice(sequence)

Returns one randomly selected item from a sequence. The sequence must not be empty.

Python — picking a random color
import random

colors = ["red", "green", "blue", "yellow"]
picked = random.choice(colors)
print("Chosen color:", picked)

random.shuffle(list)

Randomly rearranges the items of a list in place. The list itself is modified. Nothing is returned.

Python — shuffling a list of questions
import random

questions = ["Q1", "Q2", "Q3", "Q4", "Q5"]
random.shuffle(questions)
print(questions)   # same five items, unpredictable order
Common Mistake — shuffle returns None

random.shuffle() modifies the list directly and returns None. A very common error is assigning the result back to the variable: questions = random.shuffle(questions). This sets questions to None and loses your list. Call random.shuffle(questions) on its own line, without assignment.

random.sample(sequence, k)

Returns a new list of k items chosen from sequence without repetition. The original sequence is unchanged.

Python — selecting 3 random students
import random

students = ["Ana", "Ben", "Cara", "Diego", "Eve", "Finn"]
selected = random.sample(students, 3)
print("Selected:", selected)   # 3 unique names, order varies

Section 24.4

Simulating Repeated Trials

One of the most powerful uses of randomness in programming is running a process many times and observing what usually happens. This is called a simulation. The basic structure of any simulation is:

Python — simulation template
import random

trials = 1000
count_of_interest = 0

for i in range(trials):
    # Simulate one trial
    outcome = ...  # some random result
    if outcome # meets some condition:
        count_of_interest += 1

proportion = count_of_interest / trials
print(f"Proportion: {proportion:.3f}")
Example 24.4.1
Simulating 10,000 coin flips
  • GoalVerify that a fair coin lands heads approximately 50% of the time over many flips.
  • DesignWrite a flip_coin() function that returns "heads" or "tails" with equal probability. Run 10,000 trials. Count heads and compute the proportion.
  • ResultThe proportion should be close to 0.500 but not exactly equal. Randomness produces variation — with more trials the result gets closer to the true probability.
Python — coin flip simulation
import random

def flip_coin():
    return random.choice(["heads", "tails"])

trials = 10000
heads_count = 0

for i in range(trials):
    result = flip_coin()
    if result == "heads":
        heads_count += 1

proportion = heads_count / trials
print(f"Heads in {trials} flips: {heads_count}")
print(f"Proportion: {proportion:.3f}")
Sample output
Heads in 10000 flips: 4987
Proportion: 0.499
Example 24.4.2
Simulating two dice — probability of rolling 7
  • QuestionWhat proportion of rolls with two six-sided dice total exactly 7?
  • Math checkThere are 36 equally likely outcomes. Six of them sum to 7: (1,6), (2,5), (3,4), (4,3), (5,2), (6,1). So the probability is 6/36 ≈ 0.167.
  • SimulationRun 100,000 trials. For each, roll two dice and check if the total is 7.
Python — two-dice simulation
import random

def roll_two_dice():
    die1 = random.randint(1, 6)
    die2 = random.randint(1, 6)
    return die1 + die2

trials = 100000
sevens = 0

for i in range(trials):
    total = roll_two_dice()
    if total == 7:
        sevens += 1

print(f"Sevens: {sevens}")
print(f"Proportion: {sevens / trials:.3f}")
Sample output
Sevens: 16614
Proportion: 0.166

For simple cases like dice, exact probabilities are easy to calculate. But many real situations are complicated enough that simulation is the practical approach: What proportion of customers wait more than 5 minutes? How likely is a student to pass if they guess randomly on a 10-question quiz? How many days does a package typically take to arrive? Simulation answers these questions without advanced mathematics.

Section 24.5

Randomness and Testing

Randomness creates a testing challenge: if a program produces different results every time, how do you test it? There are two strategies.

Strategy 1: Test the structure, not the value

Test that the result is in the right range or has the right type — not that it equals a specific value.

Python — testing range, not exact value
import random

result = random.randint(1, 10)
assert 1 <= result <= 10, "Result out of range"

Strategy 2: Use a seed

A seed is a starting value that controls the sequence of random numbers generated. If you set the same seed before running, you get the same sequence every time. This makes randomness predictable for testing purposes.

Python — seeding for reproducible results
import random

random.seed(42)
print(random.randint(1, 6))   # same value every run when seed is 42
print(random.randint(1, 6))
print(random.randint(1, 6))
When to use seeds

Use random.seed(value) when you want to test a random program and need consistent output, when you want someone else to reproduce your simulation results exactly, or when you are debugging a problem that only appears with certain random values. Call random.seed() with no argument — or remove the seed call — when you want genuine unpredictability in the finished program.

Seed Behavior Summary

random.seed(42) — sets the seed to 42; the same sequence of "random" numbers is produced every run. random.seed() — uses the current system time as the seed; different every run. No seed call — same as calling random.seed(); different every run.

Section 24.6

Dates and Times

Many useful programs need to work with dates and times: What day is today? How many days until a deadline? How long has it been since a file was created? What day of the week does a birthday fall on? Python's datetime module handles all of these.

The module provides several types. The two most useful for beginners are datetime.date (a calendar date: year, month, day) and datetime.datetime (a date and a time together). This chapter focuses on datetime.date.

Getting today's date

Python — today's date
import datetime

today = datetime.date.today()
print(today)
Sample output (date when code runs)
2025-03-14

Dates display in YYYY-MM-DD format by default. Section 24.7 shows how to change this.

Creating a specific date

Python — constructing a specific date
import datetime

birthday = datetime.date(2007, 9, 22)   # arguments: (year, month, day)
print(birthday)   # 2007-09-22

Reading parts of a date

Python — accessing year, month, day
import datetime

today = datetime.date.today()
print("Year:",  today.year)
print("Month:", today.month)
print("Day:",   today.day)
Sample output
Year: 2025
Month: 3
Day: 14
Section 24.7

Formatting Dates

The default date format (YYYY-MM-DD) is machine-readable but not always what you want to show a user. The strftime() method converts a date into a formatted string. Its name stands for "string format time."

Using strftime()

Python — formatting a date as readable text
import datetime

today = datetime.date.today()
formatted = today.strftime("%B %d, %Y")
print(formatted)
Sample output
March 14, 2025

Python — several format examples for the same date
import datetime

d = datetime.date(2025, 9, 22)
print(d.strftime("%A, %B %d, %Y"))   # Monday, September 22, 2025
print(d.strftime("%m/%d/%Y"))          # 09/22/2025
print(d.strftime("%d %b %Y"))           # 22 Sep 2025

Parsing a date string with strptime()

Sometimes you receive a date as a string — from user input, a file, or a CSV — and need to convert it to a date object. datetime.datetime.strptime() does this. The second argument must match the format of the string exactly.

Python — parsing a date string
import datetime

date_string = "2025-09-22"
parsed = datetime.datetime.strptime(date_string, "%Y-%m-%d").date()
print(parsed)         # 2025-09-22
print(parsed.year)    # 2025
strftime vs. strptime — the direction matters

strftime() — takes a date object and converts it to a formatted string (date → string). strptime() — takes a date string and converts it back to a datetime object (string → date). The "f" stands for format (output); the "p" stands for parse (input).

Section 24.8

Simple Timers and Durations

Date arithmetic with timedelta

The difference between two dates is called a duration. Python represents this with datetime.timedelta. Subtracting two date objects produces a timedelta. Its .days attribute gives the number of days in the duration.

Python — days until a future date
import datetime

today     = datetime.date.today()
new_year  = datetime.date(2026, 1, 1)
days_away = new_year - today
print(f"Days until New Year: {days_away.days}")

Adding or subtracting time

You can add a timedelta to a date to find a future or past date.

Python — adding and subtracting timedelta
import datetime

today      = datetime.date.today()
one_week   = datetime.timedelta(days=7)
next_week  = today + one_week
print("Today:",           today)
print("One week from now:", next_week)

# Also works for deadlines
deadline      = datetime.date(2025, 12, 31)
buffer        = datetime.timedelta(days=14)
reminder_date = deadline - buffer
print("Set reminder for:", reminder_date)

Days between two dates

Python — semester length in days
import datetime

start    = datetime.date(2025, 9, 1)
end      = datetime.date(2025, 12, 20)
duration = end - start
print(f"Semester length: {duration.days} days")

Timing program execution

Python — measuring elapsed time
import datetime

start_time = datetime.datetime.now()

# some code that takes time
total = 0
for i in range(10_000_000):
    total += i

end_time = datetime.datetime.now()
elapsed  = end_time - start_time
print(f"Time elapsed: {elapsed}")
Precision timing

For precise measurement of short code segments, the standard library also provides time.perf_counter(), which is more accurate than datetime.datetime.now() for very short durations. For the exercises in this chapter, datetime is clear and readable.

Section 24.9

Building Simple Simulations

Every simulation follows a similar four-step structure:

  • Set up parameters — how many trials, what rules apply
  • Run each trial using randomness
  • Collect results from each trial
  • Summarize or display the results
Example 24.9.1
Quiz guessing simulation — what score does random guessing produce?
  • SetupA 10-question multiple-choice quiz has 4 choices per question. A student guesses randomly on every question.
  • ExpectedThe expected score is 10 × (1/4) = 2.5 correct answers.
  • VerifyRun 10,000 simulated quizzes and check the average.
Python — quiz guessing simulation
import random

def simulate_quiz(num_questions, num_choices):
    """Simulate a student guessing randomly on every question."""
    correct = 0
    for q in range(num_questions):
        guess  = random.randint(1, num_choices)
        answer = 1   # correct answer is always choice 1
        if guess == answer:
            correct += 1
    return correct

trials        = 10000
total_correct = 0

for t in range(trials):
    total_correct += simulate_quiz(10, 4)

average = total_correct / trials
print(f"Average score from random guessing: {average:.2f} out of 10")
print(f"Expected: 2.50 out of 10")
Sample output
Average score from random guessing: 2.51 out of 10
Expected: 2.50 out of 10

Collecting a distribution

Instead of averaging, you can collect how often each score appears — a distribution. Use a dictionary where the keys are scores and the values are counts.

Python — collecting score distribution
import random

def simulate_quiz(num_questions, num_choices):
    correct = 0
    for q in range(num_questions):
        if random.randint(1, num_choices) == 1:
            correct += 1
    return correct

trials       = 10000
score_counts = {}

for t in range(trials):
    score = simulate_quiz(10, 4)
    if score not in score_counts:
        score_counts[score] = 0
    score_counts[score] += 1

print("Score | Count | Proportion")
print("------+-------+-----------")
for score in sorted(score_counts):
    count      = score_counts[score]
    proportion = count / trials
    print(f"  {score:2d}  | {count:5d} | {proportion:.3f}")
Sample output

Most random guesses score 1, 2, or 3. Very few score above 5. This matches what we would expect from probability — the distribution reveals a pattern that a single average cannot.

Section 24.10

Interpreting Simulation Results

Running a simulation produces numbers. Interpreting those numbers correctly requires care. Here are the key principles.

Principle 1
Results are approximate
Simulation results have variation — running the same simulation twice gives slightly different numbers unless you use a seed. For simple simulations, use at least 10,000 trials. For more precise estimates, use 100,000 or more.
Principle 2
Proportion estimates probability
A proportion from a simulation is an estimate of a probability, not the probability itself. If your simulation shows 16.7% of two-dice rolls total 7, the actual probability is 6/36 = 16.67%. The simulation is close — not exact.
Principle 3
Averages are expected values
An average from a simulation is the expected value — the typical result over many trials. It does not describe any single trial. A student guessing randomly might score 0 or 5 on one quiz, but averages about 2.5 over many quizzes.
Principle 4
Garbage in, garbage out
A simulation is only as good as its model. If you model a biased coin as a fair one, your results will be wrong regardless of trial count. Always ask: does this simulation accurately represent the situation being modeled?

When simulation is most useful

Simulation is useful when…Simulation is less useful when…
The exact probability requires complicated mathematics The exact answer is easy to calculate directly
You want to model how a system behaves under different conditions The model does not accurately represent the real situation
You want to test assumptions about randomness or fairness The number of trials is too small to give reliable estimates
You are designing a game and want to check balance The program runs too slowly to complete enough trials
