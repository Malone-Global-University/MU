Chapter 5 — Front Matter

Chapter Introduction and Learning Objectives

Every useful program eventually has to make a choice. Should it approve the login or reject it? Is the temperature above the danger threshold? Did the user enter a valid age? To answer questions like these, programs need a way to evaluate conditions and produce a result that means either yes or no. Python gives you that tool through Boolean values and comparison operators.

This chapter introduces the logical foundation that powers every decision a Python program can make. You will learn what Boolean values are, how comparison operators produce them, how logical operators combine them, and how to design reliable decision conditions. This chapter prepares you for Chapter 6, where you will use these ideas inside if, elif, and else statements to control which parts of your program actually run.

Opening Question

How does a program decide whether a student passed a test, whether a password is strong enough, or whether a transaction looks suspicious? In every case, the program is not "deciding" the way a person does — it is evaluating an expression that reduces a real-world question down to exactly one of two values: True or False. By the end of this chapter, you will understand exactly how that reduction works, and you will be able to design conditions that capture what you actually mean.

Section 5.1

Why Programs Need Decisions

Most programs you write will not just run the same instructions every time. They will need to respond to different situations — different inputs, different data, different states — and behave differently as a result.

Think about how this works in everyday life:

  • If it is raining, bring an umbrella. Otherwise, leave it at home.
  • If a student scored 90 or above, the grade is an A. If they scored 80 or above, the grade is a B.
  • If the user enters the correct password, allow access. Otherwise, show an error.
  • If a number is negative, reject it. If it is zero, warn the user. If it is positive, continue.

Each of those situations involves a condition — a question that has a yes-or-no answer — and a decision about what to do depending on the answer.

Programs make decisions using conditions, and conditions produce Boolean values: either True or False. Before you can write decision-making code, you need to understand how Python evaluates conditions and what makes a condition True or False.

Looking Ahead

This chapter teaches you the foundation. The decisions themselves — the if statements — come in Chapter 6.

Section 5.2

Boolean Values

Python has a data type called bool, short for Boolean. A Boolean value can only be one of two things:

True False

Notice that both are capitalized. Python is case-sensitive. true and false (lowercase) are not Boolean values in Python — they are simply undefined names that will cause a NameError if you use them.

You can assign Boolean values to variables just like any other value:

is_raining = True has_permission = False account_active = True

You can check the type of a Boolean value:

print(type(True)) # <class 'bool'> print(type(False)) # <class 'bool'>

Boolean values may seem simple — just two possible values — but they are the foundation of all decision-making in programs. Every condition a program evaluates eventually produces either True or False, and the program uses that result to decide what to do next.

Booleans as Numbers

Python treats True and False as special cases of integers: True equals 1 and False equals 0. You will occasionally see this come up in arithmetic. For now, just know that this relationship exists — it will make more sense when you encounter it in later chapters.

print(True + True) # 2 print(False + 1) # 1
Study Note

You do not need to use Booleans as numbers in most programs. This is just useful background knowledge.

Section 5.3

Comparison Operators

You rarely type True or False directly into a decision condition. Instead, you write an expression that Python evaluates to produce True or False. The most common way to do this is with a comparison operator.

Python's comparison operators compare two values and produce a Boolean result.

OperatorMeaningExampleResult
==Equal to5 == 5True
!=Not equal to5 != 3True
<Less than3 < 10True
>Greater than10 > 20False
<=Less than or equal to5 <= 5True
>=Greater than or equal to4 >= 7False

Try these in the Python shell to see the results:

>>> 10 == 10 True >>> 10 == 9 False >>> 7 != 3 True >>> 5 > 5 False >>> 5 >= 5 True >>> 3 < 0 False

Each comparison is an expression. Python evaluates the expression and produces a Boolean value. You can store that result in a variable:

score = 87 passed = score >= 60 print(passed) # True

Or you can use a comparison directly in a print() statement to see the result:

temperature = 104 print(temperature > 100) # True

Comparisons Work on Strings Too

Python can compare strings using the same operators. For equality, this is especially useful:

answer = "yes" print(answer == "yes") # True print(answer == "no") # False print(answer != "maybe") # True

String comparisons using <, >, <=, and >= compare alphabetical order (technically, Unicode code point order), which can be useful but also surprising. For now, focus on == and != with strings, since those are most commonly used in beginner programs.

print("apple" < "banana") # True print("zebra" > "ant") # True
Section 5.4

Equality and Assignment Are Different

One of the most common beginner mistakes in Python — and in nearly every programming language — is confusing the equality operator with the assignment operator.

SymbolPurposeExample
=Assignmentx = 10
==Equality checkx == 10

Assignment (=) stores a value in a variable. It does not produce True or False. It is a statement that changes program state.

Equality (==) compares two values and produces True or False. It does not change anything.

Look at the difference:

x = 10 # Assignment: x now stores the value 10 x == 10 # Equality: produces True, but doesn't do anything with it print(x == 10) # True print(x == 5) # False

If you write = when you meant == inside a condition, Python will usually give you a SyntaxError because you cannot assign a value inside most comparison contexts. This error is Python protecting you from a common mistake.

Code Example — Common Mistake
# This is a mistake — do not do this if x = 10: # SyntaxError print("ten")

The correct version uses ==:

if x == 10: # Correct equality check print("ten")
Study Habit

Developing the habit of reading = as "gets" and == as "equals?" will help you avoid this mistake.

Section 5.5

Less Than, Greater Than, and Ranges

Numeric comparisons are especially common in programs that work with ages, scores, temperatures, quantities, prices, and similar values. Let's look at some realistic examples.

Simple Comparisons

age = 17 print(age >= 18) # False — not yet an adult print(age < 18) # True — still a minor speed = 72 print(speed > 65) # True — over the speed limit print(speed <= 65) # False

Testing Whether a Value Is in a Range

Python supports chained comparisons, which let you test a range naturally:

score = 75 print(60 <= score <= 100) # True — score is between 60 and 100 print(0 <= score < 60) # False — score is not below 60

This is a feature unique to Python among common programming languages. It reads almost like natural language: "is score between 60 and 100, inclusive?"

You can also write range checks using and (covered in section 5.6), which is more common when you first learn Boolean logic:

score = 75 print(score >= 60 and score <= 100) # True — equivalent to above

Both forms work. The chained form is more readable when the condition is a simple range.

Boundary Values

Pay careful attention to whether your comparisons are strict (<, >) or inclusive (<=, >=). Boundaries are a common source of logic errors.

For example, if passing is defined as 60 or above:

score = 60 print(score > 60) # False — exactly 60 does NOT pass (strict) print(score >= 60) # True — exactly 60 DOES pass (inclusive)
Watch the Boundary

When you design a condition, ask yourself: should the boundary value be included or excluded? This single question prevents one of the most frequent logic errors in beginner programs.

Section 5.6

Logical Operators: and, or, and not

Single comparisons can only test one thing at a time. Often you need to combine conditions: a user must be at least 18 and must have a valid account. A temperature warning should trigger if it is too hot or too cold. A value is valid only if it is not negative.

Python provides three logical operators for combining Boolean values:

OperatorMeaningExample
andBoth conditions must be Truea and b
orAt least one condition must be Truea or b
notInverts a Boolean valuenot a

The and Operator

and produces True only when both conditions are True. If either condition is False, the result is False.

age = 20 has_id = True print(age >= 18 and has_id) # True — both conditions are met age = 16 print(age >= 18 and has_id) # False — age condition fails

Think of and as a strict gate: both keys are required.

The or Operator

or produces True when at least one condition is True. It only produces False when both conditions are False.

temperature = 38 print(temperature < 0 or temperature > 35) # True — too hot temperature = 20 print(temperature < 0 or temperature > 35) # False — comfortable range

Think of or as a flexible gate: either key will open it.

The not Operator

not flips a Boolean value. not True is False, and not False is True.

is_logged_in = False print(not is_logged_in) # True — the user is NOT logged in account_locked = True print(not account_locked) # False — account IS locked

not is useful when you want to check whether something is absent, disabled, or missing.

Combining Logical Operators

You can combine multiple logical operators in one expression. Use parentheses to make the grouping clear:

age = 25 income = 45000 has_criminal_record = False eligible = (age >= 18) and (income >= 30000) and (not has_criminal_record) print(eligible) # True
Style Tip

Without parentheses, Python uses operator precedence to decide the order of evaluation. Parentheses make your intent clear and prevent errors.

Section 5.7

Compound Conditions

A compound condition is a Boolean expression that combines multiple comparisons using logical operators. You have already seen several examples. This section focuses on reading and writing them carefully.

Reading Compound Conditions

When you read a compound condition, work through it piece by piece:

score = 82 grade = "B" print(score >= 80 and score < 90 and grade == "B")
Code Tracing
Stepping through a compound condition
  • Piece 1score >= 80 → 82 >= 80 → True
  • Piece 2score < 90 → 82 < 90 → True
  • Piece 3grade == "B" → "B" == "B" → True
  • CombineTrue and True and True → True
x = 5 print(x < 0 or x > 10)
Code Tracing
Stepping through an or condition
  • Piece 1x < 0 → 5 < 0 → False
  • Piece 2x > 10 → 5 > 10 → False
  • CombineFalse or False → False

Writing Compound Conditions Clearly

Long conditions can become hard to read. A few habits help.

Use parentheses to group sub-conditions:

# Harder to read result = score >= 80 and score < 90 or grade == "A" # Clearer with parentheses result = (score >= 80 and score < 90) or (grade == "A")

Break very long conditions across lines:

eligible = ( age >= 18 and income >= 30000 and not has_criminal_record )

Give the condition a meaningful name:

in_passing_range = score >= 60 and score <= 100 has_access = is_logged_in and not account_locked

Naming a condition makes later code much easier to read:

if in_passing_range: print("Congratulations, you passed.")

Section 5.9

Designing Decision Rules

Writing correct Boolean conditions requires more than knowing the operators. It requires thinking carefully about what you actually want the program to decide.

Here is a process for designing a condition.

Design Process
Five steps from plain English to a working condition
  • Step 1State the rule in plain English. Write out the decision in natural language before writing any code. Be specific. For example: "A student passes if their score is 60 or above." "A password is strong if it has at least 8 characters and contains a digit." "A transaction is suspicious if it is over 10,000 or it comes from a flagged account."
  • Step 2Identify the variables involved. What information does the program have? What values are you comparing? For example: score (a number), password (a string and its length), transaction amount and account status (numbers and Booleans).
  • Step 3Translate each part into a comparison. For example: score >= 60; len(password) >= 8 (and checking for a digit — more on this later); amount > 10000 or account_flagged.
  • Step 4Combine comparisons using logical operators. For example: score >= 60 (just one comparison here); len(password) >= 8 and has_digit; amount > 10000 or account_flagged.
  • Step 5Test your condition mentally against examples. Pick a few example values and evaluate the condition by hand before trusting the code.

Example: "Pass if score >= 60"

Scorescore >= 60ExpectedMatch?
75TruePass✓
60TruePass✓
59FalseFail✓
0FalseFail✓
Why This Matters

Testing your condition against expected inputs before running the program is an early form of test-case thinking. It will save you from many logic errors.

Section 5.10

Common Boolean Mistakes

Boolean expressions are small but they carry a lot of logic. Several mistakes appear over and over in beginner Python programs.

Mistake 1: Using = Instead of ==

# WRONG if x = 10: # SyntaxError # CORRECT if x == 10:

Mistake 2: Comparing to True or False Unnecessarily

# REDUNDANT — works but is noisy if is_valid == True: print("valid") # BETTER — simpler and more readable if is_valid: print("valid")

Similarly:

# REDUNDANT if is_valid == False: # BETTER if not is_valid:

Python conditions do not need to be compared against True or False. A Boolean variable already is a Boolean.

Mistake 3: Misplacing not

# WRONG — checks if score is "not 60" and then uses and not score == 60 and score < 80 # ambiguous # BETTER — use parentheses to make intent clear (not (score == 60)) and score < 80 # or more naturally: score != 60 and score < 80

Mistake 4: Trying to Write Range Checks the Wrong Way

# WRONG — this does not check a range in most languages, but in Python # it actually works, but beginners often write it wrong like this: 60 <= score <= 100 # This IS valid Python and works correctly # This is wrong: score >= 60 and <= 100 # SyntaxError — missing second operand

Always include the variable on both sides when not using Python's chained comparison:

score >= 60 and score <= 100 # Correct

Mistake 5: Forgetting or Requires Both Sides to Be Full Comparisons

# WRONG — reads naturally in English, but Python doesn't parse it this way if score == 90 or 95: # This doesn't check score == 95 # CORRECT if score == 90 or score == 95:
Why This Is Dangerous

The expression score == 90 or 95 is evaluated as (score == 90) or (95). Since 95 is a non-zero number, Python treats it as True. So this condition is always True, regardless of the score — a hidden logic bug.

Mistake 6: Confusing and with or

Read conditions carefully:

  • "Accept if the user is an admin or a moderator."
  • "Reject if the user is banned and their account is expired."

Getting and and or backwards will produce the opposite of what you intend. Use a truth table or mental test cases to verify.

Key Terms

Boolean
A data type with exactly two possible values: True or False.
bool
Python's built-in Boolean type.
Comparison operator
An operator that compares two values and produces a Boolean result: ==, !=, <, >, <=, >=.
Equality operator (==)
Compares two values to determine whether they are equal. Produces True or False.
Assignment operator (=)
Stores a value in a variable. Does not produce a Boolean.
Logical operator
An operator that combines Boolean values: and, or, not.
and
A logical operator that produces True only when both operands are True.
or
A logical operator that produces True when at least one operand is True.
not
A logical operator that inverts a Boolean value.
Compound condition
A Boolean expression that combines multiple comparisons using logical operators.
Chained comparison
A Python expression that tests a range using two comparison operators together, such as 60 <= score <= 100.
Truth table
A table showing all possible input combinations for a Boolean expression and their corresponding output.
Condition
A Boolean expression used to represent a yes-or-no question that a program evaluates.
