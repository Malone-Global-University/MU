Chapter 9 — Front Matter

Chapter Purpose and Learning Objectives

Chapter 8 introduced the while loop: a loop that keeps running as long as a condition is true. You learned how to initialize variables, update them inside the loop body, and give the loop a clear path toward stopping.

This chapter introduces the for loop. A for loop is Python's tool for repeating work a definite number of times, or for stepping through every item in a sequence one by one. Where a while loop asks keep going as long as this is true, a for loop asks do this once for each thing in this collection.

What This Chapter Covers

You will learn how the for loop works, how to use range() to control it, how to count up and count down, how to compute totals and averages, how to step through the characters of a string, and how to design simple formatted reports. You will also learn how to choose between a for loop and a while loop when both could work. By the end of this chapter you will be able to read, trace, and write for loops confidently — and you will be ready for lists, files, and data processing in later chapters.

Section 9.1

What a for Loop Does

A for loop repeats a block of code a definite number of times, or once for every item in a sequence.

Here is the simplest possible example:

for i in range(5):
    print(i)
Output
0
1
2
3
4

Five lines. Five numbers. The loop ran exactly five times, once for each value that range(5) produced.

Compare this to a while loop that does the same thing:

i = 0
while i < 5:
    print(i)
    i = i + 1

Both programs produce identical output. But notice what the for version does not need: it does not need you to initialize i before the loop, and it does not need you to update i at the bottom of the loop body. The for loop handles both of those things automatically.

This is the central practical advantage of a for loop when you know in advance how many times something should repeat. The loop takes care of the bookkeeping so you can focus on the body.

The General Structure

for variable in sequence:
    body

The word for starts the loop. The variable receives each value from the sequence, one at a time. The colon ends the header. The indented body runs once for each value. When the sequence is exhausted, the loop ends and Python moves on.

Reading the Loop Aloud

It helps to read a for loop as a sentence: "For each value of i in range(5), print i." This reading reminds you that the loop is stepping through a collection — not just counting.

Mental Model
A stack of flashcards
  • Picture thisA stack of five flashcards. You pick up each card, read it, set it down, and pick up the next. When the stack is empty, you stop.
  • The loopA for loop works the same way: it picks up each value from the sequence, uses it in the body, and moves to the next one.
Section 9.2

The range() Function

Most for loops over numbers use range() to generate the sequence of values. range() is a built-in function that produces a series of integers.

range() With One Argument

range(n)

Produces integers from 0 up to but not including n.

for i in range(4):
    print(i)
Output
0
1
2
3

The upper boundary is never included. range(4) gives 0, 1, 2, 3 — four values.

Worth Memorizing

range(n) gives n values, starting at 0.

range() With Two Arguments

range(start, stop)

Produces integers from start up to but not including stop.

for i in range(3, 8):
    print(i)
Output
3
4
5
6
7

range() With Three Arguments

range(start, stop, step)

Produces integers from start up to but not including stop, advancing by step each time.

for i in range(0, 10, 2):
    print(i)
Output
0
2
4
6
8

The step controls how far the value jumps between iterations.

What range() Actually Is

range() does not produce a list. It produces a range object — a description of a sequence. Python generates each value on demand as the loop runs, rather than storing all the values in memory at once. For small loops this makes no practical difference, but it is good to know that range(1000000) does not create a million numbers in memory.

Tracing range() Calls

Before writing a loop, it helps to ask: what values will range() produce here?

CallValues Produced
range(5)0, 1, 2, 3, 4
range(1, 6)1, 2, 3, 4, 5
range(0, 10, 3)0, 3, 6, 9
range(5, 0, -1)5, 4, 3, 2, 1
range(10, 0, -2)10, 8, 6, 4, 2

Counting Up and Counting Down

Counting Up From 1

Many problems count from 1 rather than 0. Use a two-argument range():

for i in range(1, 6):
    print(i)
Output
1
2
3
4
5

Counting Through a Specific Range

Count from 10 through 20:

for i in range(10, 21):
    print(i)

The stop value is 21, not 20, because the boundary is excluded.

Counting by Steps

Count the multiples of 5 from 5 through 50:

for i in range(5, 51, 5):
    print(i)
Output
5

Counting Down

Use a negative step:

for i in range(5, 0, -1):
    print(i)
Output
5
4
3
2
1

This is a countdown. Note that the stop value 0 is excluded, so the sequence ends at 1.

A countdown to zero:

for i in range(3, -1, -1):
    print(i)

The stop value is -1 so that 0 is included in the sequence.

Section 9.4

Loop Variables

The variable that appears after for is called the loop variable. On each iteration, Python assigns it the next value from the sequence.

for temperature in range(20, 26):
    print("Temperature:", temperature, "°C")
Output
Temperature: 20 °C
Temperature: 21 °C
Temperature: 22 °C
Temperature: 23 °C
Temperature: 24 °C
Temperature: 25 °C

The loop variable is named temperature here instead of i to make the code easier to read. The name i is fine for a generic counter, but when the loop variable has a clear meaning, a meaningful name is better.

The Loop Variable After the Loop

After a for loop finishes, the loop variable still exists and holds the last value it was assigned:

for i in range(1, 4):
    print(i)

print("Loop ended. i is now:", i)
Output
1
2
3
Loop ended. i is now: 3

This is rarely useful, but it is good to know so you are not surprised if you see a loop variable used after the loop.

Do Not Modify the Loop Variable Inside the Body

You can read the loop variable inside the loop body, but modifying it does not change how the loop iterates. Python reassigns the loop variable from the sequence at the start of each iteration, regardless of what you do inside the body.

for i in range(3):
    print("Before:", i)
    i = 100      # This does not skip ahead
    print("After:", i)
Output
Before: 0
After: 100
Before: 1
After: 100
Before: 2
After: 100

The loop still ran three times, with i starting as 0, 1, and 2 on each iteration. Assigning i = 100 changed the value for that iteration only.

Section 9.6

Totals and Averages

Two of the most common for loop patterns are computing a total and computing an average.

Pattern: Total (Accumulator)

An accumulator starts at zero and adds to itself on each iteration.

total = 0
for i in range(1, 6):
    total = total + i

print("Total:", total)

Trace:

Iterationitotal beforetotal after
1101
2213
3336
44610
551015
Output
Total: 15

This is the sum of 1 + 2 + 3 + 4 + 5.

Pattern: Count

A counter starts at zero and increases by 1 on each iteration that meets a condition.

count = 0
for i in range(1, 21):
    if i % 3 == 0:
        count = count + 1

print("Multiples of 3 between 1 and 20:", count)
Output
Multiples of 3 between 1 and 20: 6

Pattern: Average

An average requires both a total and a count. For a loop over a fixed range(), the count is simply the number of iterations.

total = 0
count = 0

for score in range(60, 96, 5):
    total = total + score
    count = count + 1

average = total / count
print("Average:", average)
Output
Average: 77.5
Worked Example
Sum of even numbers

Add all even numbers from 2 through 100.

total = 0
for i in range(2, 101, 2):
    total = total + i

print("Sum of even numbers from 2 to 100:", total)
Output
Sum of even numbers from 2 to 100: 2550
Common Mistake: Forgetting to Initialize the Accumulator

If the accumulator is never set before the loop, Python raises a NameError the first time the loop body tries to use it. Always initialize your accumulator to 0 (or the appropriate starting value) before the loop begins.

# This will cause a NameError
for i in range(1, 6):
    total = total + i   # 'total' is not defined yet
Common Mistake: Resetting the Accumulator Inside the Loop

The accumulator must be initialized once, outside the loop. If it is reset inside the loop body, it never accumulates anything across iterations.

# This resets total every iteration — total never grows
for i in range(1, 6):
    total = 0           # Wrong: this runs every iteration
    total = total + i

print(total)  # Always prints 5, not 15

Section 9.7

Iterating Over Strings at an Introductory Level

A for loop can step through the characters of a string one by one. You will work with strings in much more depth in Chapter 16, but it is useful to see strings and for loops together now.

Stepping Through Each Character

word = "Python"
for character in word:
    print(character)
Output
P
y
t
h
o
n

The loop variable character receives each character of the string "Python" in order, from left to right. The loop runs six times because "Python" has six characters.

Counting Characters

Count how many times the letter 'a' appears in a word:

word = "banana"
count = 0
for character in word:
    if character == "a":
        count = count + 1

print("Number of a's:", count)
Output
Number of a's: 3

Examining Each Character

Print each character with its position number. Use range() with the length of the string:

word = "hello"
for i in range(len(word)):
    print("Position", i, ":", word[i])
Output
Position 0 : h
Position 1 : e
Position 2 : l
Position 3 : l
Position 4 : o

len(word) returns the number of characters in the string. range(len(word)) produces 0, 1, 2, 3, 4 — valid positions in the string.

Not Yet, But Soon

Do not worry too much about indexing yet — Chapter 16 covers it thoroughly. The point here is that for loops work with strings, not just numbers.

Iterating Directly Versus Iterating by Index

Direct iteration (stepping through each character) is simpler when you only need each character:

for character in word:
    print(character)

Index-based iteration is necessary when you also need to know the position:

for i in range(len(word)):
    print(i, word[i])

Use the simpler form when it is enough.

Section 9.8

Repeated Output and Simple Reports

One practical use of for loops is generating repeated output in a consistent format — what we might call a simple report.

Horizontal Lines and Separators

Print a line of dashes:

for i in range(20):
    print("-", end="")
print()
Output
--------------------

The end="" argument to print() suppresses the newline that print() normally adds, so all twenty dashes appear on one line. The final print() with no arguments adds the newline at the end.

Nested Loops: Rows and Columns

A loop inside a loop can generate a grid of output. This is called a nested loop. The outer loop controls the rows; the inner loop controls the columns.

for row in range(1, 4):
    for col in range(1, 4):
        print(row * col, end="  ")
    print()
Output
1  2  3  
2  4  6  
3  6  9
Reading This Carefully
How the nested loop executes
  • Outer loopRuns 3 times (rows 1, 2, 3).
  • Inner loopOn each outer iteration, the inner loop runs 3 times (columns 1, 2, 3).
  • Each inner stepPrints one number without a newline (end=" ").
  • After inner loopprint() moves to the next line.
  • Total iterations3 × 3 = 9.
Keep It Simple

Nested loops are powerful, but they can get complicated quickly. Keep them simple while you are learning.

Section 9.9

Choosing Between for and while

Both for and while loops repeat code. Knowing which to use is an important judgment call.

Use For
Definite Repetition
You know in advance how many times the loop should run. You are stepping through every value in a sequence from beginning to end. The loop variable is determined by the sequence, not by conditions inside the body.
Use While
Indefinite Repetition
You do not know in advance how many iterations you need. The loop should continue until some condition becomes false, and the condition depends on what happens inside the loop. You are waiting for user input, a sentinel value, or an event.

Examples for for: iterating over range(10), printing a times table, computing the sum of the numbers 1 through n, stepping through each character in a string.

Examples for while: asking a user to keep entering numbers until they type "done", repeating until a correct password is entered, running until a game is over.

Side by Side

# for: print numbers 1 through 5
for i in range(1, 6):
    print(i)

# while: print numbers 1 through 5
i = 1
while i <= 5:
    print(i)
    i = i + 1

Both work. The for version is shorter and clearer when you know the range in advance.

# while: keep asking until valid input
response = input("Enter a number greater than zero: ")
while not response.isdigit() or int(response) <= 0:
    response = input("Invalid. Try again: ")

A for loop cannot naturally express "keep going until the user gives a valid response," because you do not know how many tries it will take. Use while.

The General Rule

If you can answer "exactly how many times should this loop run?" before the loop starts, use for. If you cannot, use while.

Section 9.10

Common for Loop Errors

Even simple for loops have common mistake patterns. Learning to recognize them saves debugging time.

Error 1
Off-by-one with range()
# Intended: print 1 through 5
for i in range(5):      # Wrong: produces 0, 1, 2, 3, 4
    print(i)
Fix: use range(1, 6). Remember: range(n) starts at 0, not 1. If you want 1 through n, use range(1, n + 1).
Error 2
Missing the stop value adjustment
# Intended: print 10 through 20 inclusive
for i in range(10, 20):   # Wrong: stops at 19
    print(i)
Fix: use range(10, 21). The stop value is always excluded. Add 1 to include it.
Error 3
Accumulator initialized inside the loop
for i in range(1, 6):
    total = 0           # Wrong: resets every iteration
    total = total + i

print(total)
Fix: 
total = 0               # Correct: initialized once before the loop
for i in range(1, 6):
    total = total + i

print(total)
Error 4
Modifying the loop variable expecting to skip iterations
for i in range(10):
    if i == 3:
        i = 7    # This does not jump to iteration 7
    print(i)
Output
0
1
2
7
4
5
6
7
8
9
On the iteration where i starts as 3, the body changes i to 7 and prints 7. But then the next iteration starts, and i becomes 4 as normal. Changing the loop variable inside the body does not skip or repeat iterations.
Error 5
Counting iterations incorrectly
# How many times does this loop run?
for i in range(3, 9, 2):
    print(i)
Output
3
5
7
Three times, not six. The sequence is 3, 5, 7 — three values. When using a step, count the values explicitly rather than assuming.
Error 6
Wrong direction with a negative step
# Intended: count from 5 down to 1
for i in range(5, 1, -1):    # Produces 5, 4, 3, 2
    print(i)
The 1 is excluded. Fix: use range(5, 0, -1) to produce 5, 4, 3, 2, 1.
Error 7
Forgetting that string iteration gives characters, not positions
word = "hello"
for i in word:
    print(i + 1)   # TypeError: can't add str and int
When iterating directly over a string, the loop variable holds a character (a string), not an integer index. You cannot add a number to it. If you need numeric positions, use range(len(word)).
Error 8
Indentation errors in the loop body
for i in range(5):
    total = total + i
print(i)        # Wrong: this line is outside the loop
versus 
for i in range(5):
    total = total + i
    print(i)    # Correct: this line is inside the loop
Python uses indentation to define what is inside the loop body. A line that is not indented is not part of the loop and runs only once, after the loop finishes.
Chapter 9 — Back Matter

Chapter Summary

A for loop repeats a block of code once for each value in a sequence. The sequence is most commonly produced by range(), but it can also be a string or, in later chapters, a list or other collection.

range() accepts one, two, or three arguments: a stop value, a start and stop value, or a start, stop, and step value. The stop value is always excluded. range(n) produces n values starting at 0.

The loop variable receives each value from the sequence at the start of each iteration. It can be used inside the loop body, but modifying it does not change how the loop iterates.

Accumulators — variables initialized before the loop that are updated inside the body — are the standard way to compute totals, counts, products, and averages with a for loop.

for loops can also step through the characters of a string, one character at a time.

Use a for loop when you know in advance how many times the loop should run. Use a while loop when the number of iterations depends on conditions that arise during execution.

Common for loop errors include off-by-one mistakes in range() calls, initializing accumulators inside the loop, and confusing character iteration with index-based iteration.

In the next chapter you will consolidate everything you have learned about control flow — conditionals, while loops, and for loops — through systematic tracing, debugging, and test design.

Key Terms

for loop
A loop that repeats its body once for each value in a sequence.
range()
A built-in function that produces a sequence of integers. It accepts one, two, or three arguments: range(stop), range(start, stop), or range(start, stop, step).
loop variable
The variable that receives each successive value from the sequence at the start of each iteration. Defined in the for statement.
iteration
One complete execution of the loop body.
accumulator
A variable initialized before a loop that is updated on each iteration to collect a running total, count, product, or other cumulative result.
total
An accumulator that sums values.
count
An accumulator that counts how many times a condition is met.
average
Total divided by count.
nested loop
A loop whose body contains another loop. The inner loop runs completely on each iteration of the outer loop.
step
The third argument to range(). Controls how much the value changes between consecutive iterations.
off-by-one error
A mistake in which a loop runs one too many or one too few times, often caused by incorrect range() boundaries.
definite repetition
Repetition where the number of iterations is fixed and known before the loop starts. for loops typically express definite repetition.
indefinite repetition
Repetition where the number of iterations depends on conditions discovered during execution. while loops typically express indefinite repetition.

Mini-ProjectSimple Report Generator

Background. A teacher wants a program that produces a formatted report showing the grade and status for each student score in a set. You will build this report generator using a for loop.

Program Requirements

1. The program should use a fixed list of test scores:

scores = [88, 72, 95, 61, 83, 77, 90, 55, 68, 79]
names = ["Alice", "Bob", "Carmen", "David", "Elena",
         "Frank", "Grace", "Henry", "Isla", "James"]

2. The program should print a formatted report with:

  • A title and a separator line.
  • One row per student showing their name, score, letter grade, and status (Pass or Fail).
  • A summary at the bottom showing: total students, students who passed, students who failed, class average, highest score, lowest score.

3. Use this grading scale:

  • 90 and above: A (Pass)
  • 80–89: B (Pass)
  • 70–79: C (Pass)
  • 60–69: D (Fail)
  • Below 60: F (Fail)
