Chapter 8 — Front Matter

Chapter Introduction and Learning Objectives

In Chapter 7, you learned that loops are how programs repeat work without repeating code. You studied the idea of iteration, counters, accumulators, and the difference between definite and indefinite repetition. You saw that loops have bodies, that loop variables change, and that tracing a loop step by step is the key to understanding it.

Now it is time to meet Python's first loop structure: the while loop.

The while loop is Python's tool for condition-controlled repetition. It keeps running as long as a condition is true, and it stops when that condition becomes false. This makes it the natural choice whenever you do not know in advance how many times a loop should run.

A while loop is powerful precisely because it is flexible. It can run once, ten times, or a thousand times depending on what happens while it runs. But that flexibility comes with a responsibility: you must design your while loop so that it always has a clear, reliable path toward stopping. A loop that never stops is not a useful program. It is a problem.

What This Chapter Covers

This chapter teaches you how while loops work, how to design them carefully, how to trace them accurately, how to avoid infinite loops, and how to use them for realistic programming tasks: counting, validating input, using sentinel values, and building the number guessing game mini-project at the end.

Section 8.1

What a while Loop Does

A while loop runs a block of code repeatedly, checking a condition before each repetition. If the condition is True, the loop body runs. When the body finishes, the condition is checked again. If it is still True, the body runs again. This continues until the condition becomes False, at which point the loop ends and the program moves on to the next statement after the loop.

Here is the general structure:

while condition:
    # loop body
    # (this block runs as long as condition is True)

And here is a simple example:

count = 1

while count <= 3:
    print(count)
    count = count + 1

print("Done.")
Output
1
2
3
Done.

Let's trace this carefully, because tracing is the most important skill for understanding loops.

Before the loop starts: count = 1

Iteration 1: The condition count <= 3 is checked. 1 <= 3 is True. The body runs: print(1) displays 1. Then count = count + 1 updates count to 2.

Iteration 2: The condition is checked again. 2 <= 3 is True. The body runs: print(2) displays 2. Then count becomes 3.

Iteration 3: The condition is checked. 3 <= 3 is True. The body runs: print(3) displays 3. Then count becomes 4.

Iteration 4 check: The condition is checked. 4 <= 3 is False. The loop ends. Execution moves to print("Done.").

The Loop Rhythm

Every while loop follows this same rhythm: check the condition — run the body — update — check again.

The condition is tested before the body runs each time. This means if the condition starts out False, the loop body never runs at all:

count = 10

while count <= 3:
    print(count)
    count = count + 1

print("Loop finished without running.")
Output
Loop finished without running.

The condition 10 <= 3 is immediately False, so the body is skipped entirely.

Reading while Loops

When you read a while loop, ask yourself four questions before you trace it:

  1. What is the condition being checked?
  2. What variable (or variables) does the condition depend on?
  3. Where is that variable initialized before the loop?
  4. Where is that variable updated inside the loop?
Study Note

If you can answer all four questions, you can usually trace the loop correctly. If you cannot find the update step, the loop will probably run forever.

Section 8.2

Loop Conditions

A loop condition is a Boolean expression — the same kind of expression you learned about in Chapter 5. It evaluates to either True or False. The loop continues as long as the condition is True.

Loop conditions often compare a variable to a limit:

temperature > 100
score < 60
attempts <= 3
response != "quit"

They can also use compound conditions with and or or:

while score < 100 and attempts > 0:
    ...

And they can use the value of any expression that produces a Boolean result.

What Makes a Good Loop Condition?

A well-written loop condition is:

  • Clear — you can read it and understand immediately what it is testing.
  • Accurate — it reflects exactly the situation where the loop should continue.
  • Reachable — the condition can actually become False through normal execution.
Common Pitfall

A poorly written condition is often the source of infinite loops and logic errors. If your loop runs more or fewer times than expected, checking the condition is a good first debugging step.

Common Condition Patterns

Pattern 1
Count-Based
Run while a counter is within a range.
while count < 10:
Pattern 2
Flag
Run while a Boolean flag is True.
while running:
Pattern 3
Sentinel
Run while a value is not a special stopping value.
while response != "quit":
Pattern 4
Validation
Run while input is invalid.
while number < 0:

You will use all of these patterns in this chapter.

Section 8.3

Initialization Before a Loop

Every while loop depends on at least one variable in its condition. Before the loop starts, that variable must have a value. This is called initialization.

If you forget to initialize a variable before a loop, Python will raise a NameError because the variable does not exist yet when the condition is first checked:

while count <= 5:       # NameError: count is not defined
    print(count)
    count = count + 1

The fix is simple: initialize the variable before the loop.

count = 1               # initialization

while count <= 5:
    print(count)
    count = count + 1

Initialization Is a Design Decision

Where you initialize a variable determines what value the loop starts with, and that affects how many times it runs and what it produces.

Consider the difference between these two programs:

# Version A
total = 0
count = 1
while count <= 5:
    total = total + count
    count = count + 1
print(total)            # 15  (1 + 2 + 3 + 4 + 5)
# Version B
total = 0
count = 0               # starts at 0, not 1
while count <= 5:
    total = total + count
    count = count + 1
print(total)            # 15  (0 + 1 + 2 + 3 + 4 + 5 = 15)

Wait — both give 15 here. What about this:

# Version C
total = 0
count = 2               # starts at 2
while count <= 5:
    total = total + count
    count = count + 1
print(total)            # 14  (2 + 3 + 4 + 5)
Key Idea

The starting value of count changes the result. Initialization matters.

Initializing for Input Loops

When a loop depends on user input, initialization often requires an initial value that forces the loop to run at least once. This is a common pattern:

response = ""           # initialize to empty string, not "quit"

while response != "quit":
    response = input("Enter a command (or 'quit' to stop): ")
    print(f"You entered: {response}")

If response had been initialized to "quit", the loop would never run. The initialization value is chosen deliberately to let the loop begin.

Section 8.4

Updating Values Inside a Loop

Initialization gets a variable ready before the loop. But the loop will run forever unless something changes inside the body so the condition can eventually become False. That change is called the update step.

The update step is one of the most important parts of any while loop. Without it, the condition never changes, and the loop runs forever.

Counter Updates

The most basic update is incrementing a counter:

count = 1
while count <= 5:
    print(count)
    count = count + 1   # update: count grows each iteration

You can also decrement:

count = 5
while count >= 1:
    print(count)
    count = count - 1   # update: count shrinks each iteration

Accumulator Updates

Accumulators change with each iteration by adding, multiplying, or combining values:

total = 0
number = 1

while number <= 10:
    total = total + number
    number = number + 1

print(f"Sum from 1 to 10: {total}")    # 55

Input-Based Updates

Sometimes the update step is not arithmetic — it is reading new input. The new value from input() is what changes the condition variable:

response = ""

while response != "quit":
    response = input("Enter command: ")
    # response is updated by the user's input each time

Placing the Update Correctly

The update step must be inside the loop body. If you put it outside, the loop variable never changes during the loop:

count = 1
while count <= 3:
    print(count)
# count = count + 1   <-- if this were here, infinite loop!
count = count + 1     # this runs only after the loop ends
Common Pitfall

The update must also be reachable. If the update is inside an if statement that rarely runs, the loop may run much longer than expected.

Section 8.5

Infinite Loops

An infinite loop is a loop whose condition never becomes False. It runs forever until the program is forcibly stopped.

Infinite loops are one of the most common beginner mistakes with while loops, and they happen in predictable ways.

How Infinite Loops Happen

Missing update:

count = 1
while count <= 5:
    print(count)
    # forgot to update count — loops forever printing 1

Update that never reaches the stopping condition:

count = 1
while count <= 5:
    print(count)
    count = count - 1   # count goes 1, 0, -1, -2, ... never reaches > 5

Condition that can never be False:

while True:
    print("Hello")      # this runs forever
Note

while True: is sometimes used intentionally with an explicit break inside — see Section 8.8. But an unconditional while True: with no exit path is an infinite loop.

Wrong variable being updated:

count = 1
total = 0
while count <= 5:
    total = total + count
    total = total + 1   # updating total instead of count — count never changes

How to Stop an Infinite Loop

If your program appears to hang or print forever, you are probably in an infinite loop. Press Ctrl+C in the terminal to interrupt the program. Python will display a KeyboardInterrupt message.

How to Prevent Infinite Loops

Before running a while loop, ask yourself:

  • What variable does the condition check?
  • Is that variable initialized before the loop?
  • Is that variable definitely updated inside the loop?
  • Will the update actually move the variable toward the stopping condition?

If you can answer yes to all four, you are unlikely to create an infinite loop.

Tracing to Verify

When in doubt, trace the first few iterations on paper. If the condition variable is moving in the right direction, the loop will eventually stop. If the variable stays the same or moves in the wrong direction, you have found your bug.

Section 8.6

Sentinel-Controlled Loops

A sentinel value is a special value used to signal the end of input. When the user enters the sentinel, the loop stops.

Sentinel-controlled loops are useful when you do not know in advance how many items the user will enter. You keep asking for input until the user provides the sentinel.

Example 8.6.1
Summing numbers until the user types "done"
total = 0
entry = input("Enter a number (or 'done' to finish): ")

while entry != "done":
    number = float(entry)
    total = total + number
    entry = input("Enter a number (or 'done' to finish): ")

print(f"Total: {total}")
Sample Run
Enter a number (or 'done' to finish): 10
Enter a number (or 'done' to finish): 25
Enter a number (or 'done' to finish): 8
Enter a number (or 'done' to finish): done
Total: 43.0

The sentinel here is the string "done". When the user types "done", the condition entry != "done" becomes False and the loop ends.

Choosing a Good Sentinel

A good sentinel value is something that cannot be confused with a real data value. If you are collecting integers, 0 makes a poor sentinel (zero might be a real answer). The word "done" or -1 in a context where negatives are impossible works better.

SentinelBest Used When
"done", "quit", "stop", "exit"Collecting text input
-1Collecting non-negative integer input
0Collecting input where zero is not a valid value

The "Read Before the Loop" Pattern

Notice the structure in the example above: the first input() call happens before the loop, and then the same input() call happens at the end of the loop body. This is the standard sentinel loop pattern:

read first value
while value is not the sentinel:
    process the value
    read the next value

This ensures the condition is checked before any processing, so the sentinel itself is never processed as a real value.

Section 8.7

Input Validation Loops

One of the most practical uses of while loops is input validation: repeatedly asking the user for input until they provide something acceptable.

Without validation, programs break easily. A user might type a letter when the program expects a number, or type a negative number when only positive numbers make sense.

Basic Structure of a Validation Loop

age = int(input("Enter your age: "))

while age < 0 or age > 120:
    print("That is not a valid age. Please try again.")
    age = int(input("Enter your age: "))

print(f"Your age is {age}.")

This loop keeps running as long as the input is invalid. Once the user provides a valid age (0 through 120), the condition becomes False and the loop ends.

Validating That Input Is a Positive Number

score = int(input("Enter a score (0 to 100): "))

while score < 0 or score > 100:
    print("Score must be between 0 and 100.")
    score = int(input("Enter a score (0 to 100): "))

print(f"Score accepted: {score}")

Validating a Yes/No Answer

answer = input("Continue? (yes/no): ").lower()

while answer != "yes" and answer != "no":
    print("Please type yes or no.")
    answer = input("Continue? (yes/no): ").lower()

if answer == "yes":
    print("Continuing...")
else:
    print("Stopping.")
Study Note

Note the .lower() call — this converts whatever the user types to lowercase so that "Yes", "YES", "yes" all match.

The Design Principle

A validation loop should:

  1. Ask for input once before the loop (or collect input at the start of the body, entering with a deliberately invalid initial value).
  2. Check whether the input meets requirements.
  3. If not, explain what was wrong and ask again.
  4. Continue until the input is valid.

Using an Invalid Initial Value

An alternative pattern initializes the variable to an invalid value that forces the loop to run at least once:

age = -1    # deliberately invalid initial value

while age < 0 or age > 120:
    age = int(input("Enter your age: "))

print(f"Your age is {age}.")

This version always asks at least once and keeps asking until the input is valid. Both patterns work; choose whichever reads more clearly.

Section 8.8

break and continue at an Introductory Level

Python provides two special statements for controlling loops: break and continue. They are useful tools, but beginners should use them carefully because they can make loops harder to trace.

break: Exit the Loop Immediately

The break statement causes the loop to stop immediately, regardless of the condition. Execution jumps to the first statement after the loop.

count = 1

while count <= 10:
    if count == 5:
        break           # stop the loop when count reaches 5
    print(count)
    count = count + 1

print("Loop ended.")
Output
1
2
3
4
Loop ended.

The loop would normally run up to 10, but break stops it at 5.

When break Is Useful

break is most naturally used inside a while True: loop where you want to keep running until some condition inside the body is met:

while True:
    response = input("Enter a command (or 'quit' to stop): ")
    if response == "quit":
        break
    print(f"You entered: {response}")

print("Goodbye.")

The loop runs forever by design — the condition True never becomes False. The only way out is the break statement inside the body.

Why This Pattern Reads Well

This pattern reads well: "keep going until you decide to stop." It is a clean approach for menu-driven programs and command loops.

continue: Skip to the Next Iteration

The continue statement skips the rest of the current loop body and returns to the condition check. The loop does not stop — it just moves on to the next iteration without finishing the current one.

count = 0

while count < 6:
    count = count + 1
    if count == 3:
        continue        # skip the rest when count is 3
    print(count)
Output
1
2
4
5
6

The value 3 is skipped because when count == 3, the continue statement sends the loop back to the condition check before print(count) runs.

Using continue Carefully

continue can be confusing because it jumps over code. Make sure the update step (like count = count + 1) happens before the continue if the loop condition depends on that variable. Otherwise, you could accidentally create an infinite loop:

count = 0

while count < 6:
    if count == 3:
        continue        # PROBLEM: count never gets updated when count == 3
    print(count)
    count = count + 1   # this is skipped when count == 3
Why This Fails

This creates an infinite loop because when count is 3, the loop skips the update, checks the condition (3 < 6 is still True), runs again, sees count == 3, skips the update again — forever.

The fix: move the update before the continue:

count = 0

while count < 6:
    count = count + 1
    if count == 3:
        continue
    print(count)

A Note on Style

break and continue are valid Python, but overusing them can make loops hard to follow. For many situations, you can redesign the loop condition or the if statement to avoid them entirely. When break and continue genuinely make the code clearer, use them. When they are being used to patch a poorly designed loop, consider restructuring instead.

Section 8.9

Designing Safe while Loops

A safe while loop is one that:

  • Terminates for every valid input
  • Produces correct results for typical cases
  • Handles edge cases without breaking
  • Is easy to read and trace

Designing safe while loops is a skill that improves with practice. Here is a structured approach.

Design Process
Six steps to a safe while loop
  • Step 1Identify what controls the loop. Ask: "What condition should keep this loop running?" Write the condition clearly before you write any code. Example: "Keep asking until the user enters a positive number." Condition: number <= 0
  • Step 2Identify the variables the condition depends on. Which variables appear in the condition? Those are the variables you must initialize and update. In this case: number
  • Step 3Initialize those variables before the loop. Choose an initial value that makes the condition True so the loop will run at least once, or that matches the actual starting state of the problem.
  • Step 4Write the loop body. The body should do the work of one iteration and always update the condition variable.
  • Step 5Check the exit. After the loop ends, trace through what happens. Is the variable in the state you expect? Does the rest of the program use it correctly?
  • Step 6Test with boundary values. What happens if the user enters the exact limit? What happens on the first try? What happens after several wrong answers?

Initialization choices for Step 3:

number = -1    # forces the loop to run at least once

or:

number = int(input("Enter a positive number: "))    # read first value before loop

And the loop body for Step 4:

while number <= 0:
    print("Please enter a positive number.")
    number = int(input("Enter a positive number: "))

Complete Safe Loop Example

# Safe loop: collect a positive integer
number = int(input("Enter a positive number: "))

while number <= 0:
    print("That is not positive. Please try again.")
    number = int(input("Enter a positive number: "))

print(f"Thank you. You entered {number}.")

Trace with inputs -3, 0, 5:

InputCondition CheckedResult
-3-3 <= 0True — print error, read again
00 <= 0True — print error, read again
55 <= 0False — loop ends, print result
Section 8.10

Debugging Loop Control Problems

Loop bugs are among the most frustrating in programming because they can cause programs to run forever, skip values, process values incorrectly, or terminate too early. This section shows the most common loop bugs and how to fix them.

Bug 1: Missing Update (Infinite Loop)

# Buggy version
count = 1
while count <= 5:
    print(count)
    # forgot count = count + 1
Symptom
The program prints 1 forever.
Fix: Add the update inside the loop body.
count = 1
while count <= 5:
    print(count)
    count = count + 1

Bug 2: Update in the Wrong Direction (Infinite Loop)

# Buggy version
count = 1
while count <= 5:
    print(count)
    count = count - 1   # moving away from the stopping condition
Symptom
The program prints 1, 0, -1, -2, ... forever.
Fix: Change the update direction to match the condition.
count = 1
while count <= 5:
    print(count)
    count = count + 1

Bug 3: Off-by-One in the Condition

# Intended to print 1 through 5
count = 1
while count < 5:        # should be <= 5
    print(count)
    count = count + 1
Symptom
Prints 1 through 4, stopping one iteration too early.
Fix: Change < to <=. This is one of the most common subtle bugs. Trace the last iteration to check: when count is 5, should the loop run? If yes, the condition must be <= 5.
while count <= 5:

Bug 4: Wrong Initialization

# Intended to sum 1 through 10
total = 0
count = 0               # starts at 0, adding 0 in first iteration

while count <= 10:
    total = total + count
    count = count + 1

print(total)            # prints 55, which happens to be correct here
                         # but if problem said "1 through 10", starting at 0 adds nothing

Whether this is a bug depends on the problem. Trace the first iteration and confirm the starting state is what you intended.

Bug 5: Sentinel Not Matching the Input

response = input("Enter 'quit' to stop: ")

while response != "Quit":   # case mismatch — user types "quit", not "Quit"
    print(f"You entered: {response}")
    response = input("Enter 'quit' to stop: ")
Symptom
Typing "quit" does not stop the loop.
Fix: Normalize the case before comparing.
while response.lower() != "quit":

Bug 6: Processing the Sentinel Value

entry = input("Enter a number or 'done': ")

while entry != "done":
    number = float(entry)
    entry = input("Enter a number or 'done': ")
    total = total + number   # BUG: this line is inside the loop but processes
                              # entry's new value, which might be "done"
Fix
Process the value before reading the next one.
Move the total update so it happens immediately after computing number, before reading the next entry.
while entry != "done":
    number = float(entry)
    total = total + number
    entry = input("Enter a number or 'done': ")

Debugging Strategy for Loops

Loop Debugging Checklist
Work through these in order when a loop misbehaves
  • Print the condition variable at the start of each iteration to see if it is changing correctly.
  • Trace the first three iterations on paper before running the program.
  • Check the last iteration: when the condition becomes False, is the variable in the right state?
  • Test with small limits: instead of looping to 100, test with 3 first.
  • Use Ctrl+C to interrupt an infinite loop, then trace what was happening.

Key Terms

while loop
A loop that repeats its body as long as a condition is True. The condition is checked before each iteration.
loop condition
The Boolean expression tested at the start of each iteration. When it becomes False, the loop ends.
initialization
Setting a variable to a starting value before a loop begins. Required for any variable that appears in the loop condition.
update step
The statement inside a loop body that changes the condition variable. Without an update, the loop may run forever.
infinite loop
A loop whose condition never becomes False. It runs until the program is interrupted.
iteration
One complete execution of the loop body.
sentinel value
A special value used to signal the end of input. When the user enters the sentinel, a sentinel-controlled loop stops.
sentinel-controlled loop
A loop that continues until the user enters a designated stopping value.
input validation loop
A loop that keeps asking for input until the user provides a value that meets specific requirements.
break
A statement that exits a loop immediately, regardless of the condition.
continue
A statement that skips the rest of the current loop body and returns to the condition check.
off-by-one error
A common loop bug where the loop runs one too many or one too few times, usually caused by using < instead of <= or vice versa in the condition.
flag
A Boolean variable used to control a loop. The flag is set to True or False inside the body to eventually stop the loop.

Mini-ProjectNumber Guessing Game

Project Overview

You will write a complete number guessing game program. The program picks a secret number, lets the player guess, gives hints, and tracks the number of attempts.

Requirements

  1. The program selects a random integer between 1 and 100. (Use Python's built-in randint function — the instructions below explain how.)
  2. The program asks the player to guess the number.
  3. After each guess, the program tells the player:
    • "Too low. Try again." — if the guess is below the secret number
    • "Too high. Try again." — if the guess is above the secret number
    • "Correct! You got it in [n] guesses." — if the guess matches
  4. The game keeps accepting guesses until the player guesses correctly.
  5. The program counts and displays the number of guesses.

Using randint

The random module is part of Python's standard library. For now, you only need this one line at the top of your program:

import random

And this line to generate a random number between 1 and 100 (inclusive):

secret = random.randint(1, 100)
Looking Ahead

You will learn much more about the random module in Chapter 15 and Chapter 24. For this project, just use it as a tool.

Starter Structure

Here is one way to organize your program. Fill in the logic:

import random

secret = random.randint(1, 100)
attempts = 0

print("I have chosen a number between 1 and 100.")
print("Try to guess it!")

guess = int(input("Your guess: "))
attempts = attempts + 1

while guess != secret:
    if guess < secret:
        print("Too low. Try again.")
    else:
        print("Too high. Try again.")
    guess = int(input("Your guess: "))
    attempts = attempts + 1

print(f"Correct! You got it in {attempts} guesses.")

Extensions (Optional)

Once the basic version works, try adding:

  1. Guess validation: Only accept guesses between 1 and 100. Reject anything outside that range with a message.
  2. Play again: After the game ends, ask "Play again? (yes/no)". If the player says yes, start a new game.
  3. Limited guesses: Give the player only 7 guesses. If they do not guess correctly in 7 tries, reveal the number and end the game.
  4. Score tracking: Count how many games the player wins and loses across multiple rounds.

Testing Your Game

Before you share your game, test these situations:

  • What happens if the player guesses the number on the first try?
  • What happens if the player guesses very high numbers, then very low numbers?
  • Does the attempt counter increment correctly every time?
  • Does the program end cleanly after a correct guess?

Preview — Chapter 9
For Loops and Ranges

Chapter 9 introduces the for loop — Python's tool for count-controlled and sequence-based repetition.

Where a while loop is driven by a condition, a for loop is driven by a sequence: a range of numbers, a string of characters, or later a list of values. The for loop handles the counting and stopping automatically, without requiring you to manage a counter variable manually.

You will learn how range() generates sequences of numbers, how to count up and count down, how to iterate over the characters of a string, how to compute totals and averages with a for loop, and how to choose between for and while for a given problem.

After Chapter 9, you will have both major loop structures available. The key question — for or while? — will be answered: use for when you know the sequence in advance, and use while when the number of iterations depends on conditions that change during the loop.
