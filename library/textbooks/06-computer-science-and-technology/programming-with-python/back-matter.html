Appendix A

Installing and Running Python

Python Versions

Python has two major version families: Python 2 and Python 3. Python 2 reached its end of life in 2020 and is no longer maintained. This textbook uses Python 3 throughout. Install the most recent stable release of Python 3. Any version of Python 3.8 or later will work for all programs in this book. Check the current release at python.org.

Installing Python

Windows

  1. Go to python.org/downloads and click the button for the latest Python 3 release.
  2. Download the Windows installer (the .exe file) and run it.
  3. On the first screen, check Add Python to PATH. This step is critical — without it, your terminal will not find Python.
  4. Click Install Now and wait for the installation to complete.

macOS

  1. Go to python.org/downloads and download the macOS installer (the .pkg file).
  2. Open the file and follow the installation steps.
  3. After installation, open Terminal and type python3 --version to confirm.

Linux (Ubuntu/Debian)

Terminal
sudo apt update
sudo apt install python3
python3 --version

Checking the Installation

After installation, open a terminal and type:

python --version        # Windows
python3 --version       # macOS and Linux

You should see output like Python 3.12.0. If you see an error, Python was not installed correctly or was not added to your PATH.

Using the Interactive Shell

The Python interactive shell (REPL — Read, Evaluate, Print, Loop) lets you type expressions and see results immediately. Type python or python3 in a terminal:

>>> 2 + 3
5
>>> "hello".upper()
'HELLO'
>>> name = "Ada"
>>> print("Hello,", name)
Hello, Ada

To exit: type exit(), or press Ctrl+D (macOS/Linux) or Ctrl+Z then Enter (Windows). The interactive shell does not save your work — use script files for programs you want to keep.

Running Script Files

A script file is a plain text file containing Python code, ending with .py. Navigate to the file's folder first, then run it:

cd C:\Users\YourName\projects\     # Windows navigation
python hello.py                    # Windows execution

cd /home/yourname/projects/        # macOS/Linux navigation
python3 hello.py                   # macOS/Linux execution

Current Directory

When you open a terminal, you are placed in a current directory. All relative file paths are interpreted from this location.

cd          # Show current directory (Windows)
pwd         # Show current directory (macOS/Linux)

Relative vs. Absolute Paths

TypeDescriptionExample
RelativeStarts from the current directory. Portable — works regardless of where the project lives.data/students.csv
AbsoluteStarts from the root of the file system. Works everywhere but is tied to one machine's layout.C:\Users\Name\projects\data.csv

cd ~                      # Home directory (macOS/Linux)

dir             # Windows
ls              # macOS/Linux
ls -l           # Detailed listing (macOS/Linux)

Running Python Files

python filename.py               # Windows
python3 filename.py              # macOS/Linux
python3 chapter_03/values.py    # File in a subfolder

Path Issues by Platform

Platform Notes
Common path gotchas for each operating system
  • WindowsUses backslashes (\) in paths. In Python strings, write Windows paths as raw strings (r"C:\path\to\file") or use forward slashes ("C:/path/to/file"), which Python accepts. Folder names with spaces must be quoted at the command line: cd "My Documents".
  • macOS/LinuxPaths are case-sensitive — Notes.txt and notes.txt are different files. The home directory is ~, which expands to /Users/yourname (macOS) or /home/yourname (Linux). Folder names with spaces must be quoted: cd "my projects".
Safe File Organization

Always know your current directory before running a script. Keep data files in the same folder as the script that reads them, or in a data/ subfolder. Use pwd or cd to confirm your location before running programs that read or write files.

Appendix D

Common Python Error Messages

When a program encounters a problem it cannot handle, Python raises an exception and prints a traceback. Understanding these messages is one of the most important debugging skills in Python.

How to Read a Traceback

Read the traceback from the bottom up. The last line gives you the error type and message. The lines above show the chain of function calls that led to the error.

Traceback (most recent call last):
  File "calculator.py", line 12, in <module>
    result = divide(10, 0)
  File "calculator.py", line 5, in divide
    return a / b
ZeroDivisionError: division by zero

Reading this: error type is ZeroDivisionError. It happened inside divide() at line 5, which was called at line 12.

Error Reference

SyntaxError
Python cannot parse the code — something is not valid Python syntax
  • Common CausesMissing colon after if, elif, else, for, while, def, or class. Unmatched parentheses, brackets, or quotes. Using = instead of == inside a condition.
  • How to FixRead the line Python points to and the line just before it. Look for missing punctuation.
IndentationError
Unexpected or missing indentation in a code block
  • Common CausesMixing spaces and tabs. Forgetting to indent after a colon. Over-indenting a line that should be at a lower level.
  • How to FixUse 4 spaces per level consistently. Configure your editor to insert spaces instead of tabs.
NameError
A variable or function name is not recognized
  • Common CausesUsing a variable before assigning a value to it. Misspelling a variable or function name. Using a variable outside its scope.
  • How to FixCheck spelling exactly. Confirm the variable was assigned before use.
TypeError
An operation is performed on the wrong type of value
  • Common CausesAdding a string and an integer without converting. Calling something that is not a function. Passing the wrong number of arguments.
  • How to FixCheck types with type(value). Convert where necessary with int(), float(), or str().
ValueError
The type is correct, but the value itself is not acceptable
  • Common CausesConverting a non-numeric string to int or float: int("hello").
  • How to FixValidate input before converting. Use try/except ValueError for user input.
IndexError
An index position does not exist in a list, string, or other sequence
  • Common CausesUsing an index equal to or greater than len(sequence). Off-by-one errors in loops. Valid indexes are 0 through len(sequence) - 1.
  • How to FixCheck the length with len(). Verify loop ranges carefully.
KeyError
A dictionary key does not exist
  • Common CausesMisspelling a key name. Assuming a key is present when it might not be.
  • How to FixUse key in dictionary to check before accessing. Use .get(key, default) for safe lookup.
FileNotFoundError
Python could not find the file at the path specified
  • Common CausesFile does not exist. Misspelled file name. Wrong current directory. Wrong path format.
  • How to FixConfirm file name and extension. Use import os; print(os.getcwd()) to check the working directory.
ZeroDivisionError
A program attempted to divide by zero
  • Common CausesDividing without checking if the divisor could be zero. User input of zero used as a denominator.
  • How to FixAdd a guard: if count != 0: before dividing.
AttributeError
An attribute or method does not exist on the object
  • Common CausesCalling a string method on an integer. Misspelling a method name. Calling a method on None (often because a function forgot to return).
  • How to FixCheck the type of the object. Confirm functions that should return a value use return.
ImportError / ModuleNotFoundError
Python could not find the module you tried to import
  • Common CausesMisspelled module name (e.g. import maths instead of import math). Module not installed.
  • How to FixCheck spelling. For third-party packages, install using pip install package-name in the terminal.
Appendix E

Debugging Checklist

Use this checklist whenever a program is not working as expected. Work through the steps in order.

Ten-Step Debugging Process
Work through these in order — do not skip ahead
Step 1 — Read the error message
  • Do not skip the error message. Read it completely.
  • Find the error type (such as TypeError or NameError) and the message that follows.
  • The error type tells you what went wrong; the message often names the exact value or variable.
Step 2 — Find the line number
  • The traceback includes a file name and line number. Go to that line.
  • Remember: the true cause is sometimes on an earlier line — a variable assigned the wrong value several lines before the error appeared.
Step 3 — Check spelling and names
  • Look at every variable and function name on and around the problem line.
  • Check for typos, wrong capitalization, names used before assignment, and names used outside their scope.
Step 4 — Check indentation
  • Every code block inside if, elif, else, for, while, def, or class must be indented by exactly 4 spaces.
  • Confirm your editor uses spaces, not tabs.
Step 5 — Check data types
  • What type is this variable right now? (str, int, float, list, dict?)
  • Does the operation you are performing work on this type?
  • Did you forget to convert input from str to int or float?
  • Use print(type(variable)) to inspect types.
Step 6 — Trace variable values
  • Add temporary print() statements to display variable values at key points.
  • Compare actual values to what you expected at each step.
  • Remove these debugging print statements once the bug is fixed.
Step 7 — Test one change at a time
  • Make one change and run the program. Repeat.
  • If you change several things at once, you will not know which change fixed the bug — or broke something new.
Step 8 — Use small test cases
  • Test with the simplest possible input that reproduces the problem.
  • A bug that is hard to see in a full program often becomes obvious with a tiny example.
Step 9 — Check edge cases
  • Test with zero, one element, or very large inputs.
  • Test with empty strings, empty lists, and empty files.
  • Test values at the exact boundary of any comparison (>= vs >).
  • Test negative values when the program assumes positive.
Step 10 — Explain the code out loud
  • Describe every line of the program to yourself or another person in plain language.
  • This technique — called rubber duck debugging — often reveals the difference between what the code does and what you intended it to do.
Appendix F

Code Style Guide

Python's community-wide style guide is called PEP 8. This appendix summarizes the most important conventions for students using this textbook. Well-styled code is easier to read, debug, test, and maintain.

Readable Names

Use lowercase with underscores (snake_case) for variables and functions. Prefer clarity over brevity. Variable names should describe what they store; function names should describe what they do.

# Unclear
x = 92
y = x * 0.9

# Clear
exam_score = 92
discounted_price = exam_score * 0.9

# Variables — noun phrases (what it is)
student_count = 30
passing_grades = []

# Functions — verb phrases (what it does)
def calculate_average(scores): ...
def find_maximum(numbers): ...

Consistent Indentation

Always use 4 spaces per indentation level. Never use tabs. Configure your code editor to automatically insert spaces when you press the Tab key.

Comments — Explain Why, Not What

# Good — explains the decision
# Multiply by 0.9 to apply a 10% discount
final_price = base_price * 0.9

# Unnecessary — restates the obvious
# Set x to 5
x = 5

Docstrings

def calculate_average(numbers):
    """Return the average of a list of numbers.

    Returns 0 if the list is empty.
    """
    if len(numbers) == 0:
        return 0
    return sum(numbers) / len(numbers)

Avoiding Repetition

# Repetitive
print(f"Student: {s1_name}, Grade: {s1_grade}")
print(f"Student: {s2_name}, Grade: {s2_grade}")

# Better — extract a function
def print_student(name, grade):
    print(f"Student: {name}, Grade: {grade}")

print_student(s1_name, s1_grade)
print_student(s2_name, s2_grade)

Small Functions and Clear Conditions

Functions should do one thing. If you cannot describe what a function does in one sentence without the word "and," it probably does too much. For conditions, avoid double negatives and unnecessary comparisons with True or False:

# Unnecessary
if is_active == True:
    ...

# Better
if is_active:
    ...

Project Organization

my_project/
    main.py
    student.py
    gradebook.py
    report.py
    data/
        students.csv
    tests/
        test_gradebook.py
Appendix G

Function Design Reference

When to Write a Function

Write a function when you need to do the same task in more than one place; when a block of code has a clear, nameable purpose; when a section has grown long enough to benefit from a name; or when you want to test a piece of logic separately. Do not write functions just to divide code into pieces without reason.

Parameters and Arguments

Parameters are the variable names in a function definition. Arguments are the actual values passed when calling the function.

def greet(name, greeting):   # name, greeting = parameters
    print(f"{greeting}, {name}!")

greet("Alice", "Hello")      # "Alice", "Hello" = arguments

Print Versus Return

print()return
Makes output visibleYesNo
Produces a usable valueNoYes
Value can be storedNoYes
Value can be used in expressionsNoYes
 
# Printing inside — result cannot be reused
def add_and_print(a, b):
    print(a + b)

x = add_and_print(3, 4)   # prints 7, but x is None

# Returning — result can be reused
def add(a, b):
    return a + b

x = add(3, 4)         # x is 7
y = add(x, 10)        # y is 17
print(x + y)          # prints 24
The Core Rule

Use return for functions that compute or produce values. Use print() only when you explicitly want to display something to the user. A function that prints its result cannot be composed with other functions.

Local Variables and Avoiding Globals

# Fragile — depends on global state
total = 0
def add_to_total(n):
    global total
    total += n

# Better — explicit and testable
def add(current_total, n):
    return current_total + n

total = add(total, 5)

Globals are acceptable for program-wide constants that never change: MAX_SCORE = 100. For everything else, pass values as arguments and return results.

Helper Functions

def is_valid_score(score):
    """Return True if score is between 0 and 100."""
    return 0 <= score <= 100

def letter_grade(score):
    """Return the letter grade for a score."""
    if not is_valid_score(score):
        return "Invalid"
    if score >= 90:
        return "A"
    elif score >= 80:
        return "B"
    elif score >= 70:
        return "C"
    elif score >= 60:
        return "D"
    else:
        return "F"

Testing Functions

print(letter_grade(95))    # A
print(letter_grade(83))    # B
print(letter_grade(60))    # D
print(letter_grade(59))    # F
print(letter_grade(-5))    # Invalid
print(letter_grade(105))   # Invalid

Reading, Writing, and Appending

with open("notes.txt", "r") as f:

# Append to end

Always Use with

The with statement automatically closes the file when the block ends — even if an error occurs. This prevents file corruption and resource leaks. Always use with open(...) as f: instead of calling f.close() manually.

Reading and Writing CSV

import csv

# Read with csv.reader (rows as lists)
with open("students.csv", "r", newline="") as f:
    reader = csv.reader(f)
    header = next(reader)
    for row in reader:
        print(row)

# Read with csv.DictReader (rows as dictionaries)
with open("students.csv", "r", newline="") as f:
    reader = csv.DictReader(f)
    for row in reader:
        print(row["name"], row["score"])

# Write with csv.writer
with open("results.csv", "w", newline="") as f:
    writer = csv.writer(f)
    writer.writerow(["name", "score", "grade"])
    writer.writerow(["Ana", 91, "A"])

# Write with csv.DictWriter
students = [{"name": "Ana", "score": 91}]
with open("results.csv", "w", newline="") as f:
    writer = csv.DictWriter(f, fieldnames=["name", "score"])
    writer.writeheader()
    for row in students:
        writer.writerow(row)

Appendix K

Object-Oriented Programming Reference

Classes, Objects, and Instances

A class is a template for creating objects. An object is a specific instance of a class — each with its own data. Instance and object mean the same thing in Python. Class names use CapWords (PascalCase): BankAccount, StudentRecord.

Attributes, Methods, self, and __init__

class Student:
    def __init__(self, name, grade):  # initializer
        self.name = name              # instance attribute
        self.grade = grade

    def is_passing(self):             # method
        return self.grade >= 60

    def apply_bonus(self, points):
        self.grade += points

    def __str__(self):                # string representation
        return f"{self.name} ({self.grade})"

s = Student("Ben", 55)
print(s.is_passing())    # False
s.apply_bonus(10)
print(s.grade)           # 65
print(s.is_passing())    # True
print(s)                 # Ben (65)
Understanding self

self refers to the specific object on which a method is being called. It is always the first parameter of every method definition, but you do not pass it explicitly — Python passes it automatically. Inside the method, self.name means "the name attribute of this particular object."

Object State

class Counter:
    def __init__(self):
        self.count = 0

    def increment(self):
        self.count += 1

    def reset(self):
        self.count = 0

c = Counter()
c.increment()
c.increment()
print(c.count)     # 2
c.reset()
print(c.count)     # 0

Lists of Objects

students = [
    Student("Ana", 91),
    Student("Ben", 78),
    Student("Cal", 55)
]

for s in students:
    status = "passing" if s.is_passing() else "failing"
    print(f"{s.name}: {status}")

When to Use Classes

Classes are useful when you need to create multiple items of the same kind, each with its own state; when data and operations on it naturally belong together; or when you find yourself grouping related data in a dictionary and writing functions to operate on it — a class bundles these together more cleanly. Do not create a class just to have one. A simple function or dictionary is often the right choice for simple tasks.

# Before — duplicated logic

else: print(f"{name1}: F")

else: print(f"{name2}: F")

print_grade(name1, score1)
print_grade(name2, score2)

Glossary

Alphabetical glossary of Python programming terms used throughout this textbook.

  • argumentA value passed to a function when it is called. Arguments are matched to parameters in the function definition.
  • assignmentThe process of binding a value to a variable name using the = operator.
  • attributeA variable stored inside an object. Accessed with dot notation: object.attribute.
  • BooleanA data type with exactly two values: True and False. Used in conditions and logical operations.
  • bugAn error in a program that causes it to behave incorrectly or crash.
  • callTo invoke a function by writing its name followed by parentheses and any required arguments.
  • classA template for creating objects. Defines the attributes and methods that all instances of the class will have.
  • commentText in a program that Python ignores. Written with #. Used to explain code to human readers.
  • concatenationJoining two strings together using the + operator.
  • conditionAn expression that evaluates to True or False. Used in if statements and loop control.
  • conditionalA program structure that executes different code depending on whether a condition is true. In Python: if, elif, else.
  • constantA variable whose value is not intended to change. By convention, written in ALL_CAPS in Python.
  • CSVComma-Separated Values. A plain text file format for storing table data. Each row is one line; columns are separated by commas.
  • data structureA way of organizing and storing data so it can be accessed and modified efficiently. Common Python structures: list, tuple, set, dictionary.
  • debuggingThe process of finding and fixing errors in a program.
  • decompositionBreaking a complex problem or program into smaller, more manageable pieces.
  • default parameterA parameter that has a preset value used when no argument is provided for it.
  • dictionaryA mutable collection of key-value pairs. Keys must be unique. Values can be any type. Accessed by key, not by position.
  • docstringA string literal at the start of a function, class, or module that describes what it does.
  • edge caseAn input at or near the boundary of valid values. A common source of bugs.
  • exceptionAn error detected during program execution that interrupts normal flow. Can be caught and handled with try/except.
  • expressionA combination of values, variables, and operators that evaluates to a single value.
  • f-stringA formatted string literal, prefixed with f. Allows expressions inside {} to be embedded directly in a string.
  • floatA Python numeric type representing real numbers with decimal points. Example: 3.14.
  • for loopA loop that iterates over each item in a sequence or over a range of values.
  • functionA named, reusable block of code that performs a specific task. May accept parameters and return a value.
  • global variableA variable defined at the top level of a module, accessible throughout the file.
  • immutableCannot be changed after creation. Strings and tuples are immutable.
  • importA statement that loads a module or specific names from a module into the current program.
  • indentationThe whitespace at the beginning of a line used to define code blocks in Python. Four spaces per level is standard.
  • indexAn integer position within a sequence. Python sequences are zero-indexed: the first item is at index 0.
  • infinite loopA loop whose condition never becomes False and never encounters a break. It runs forever unless interrupted.
  • instanceAn individual object created from a class. Each instance has its own state.
  • integerA Python numeric type representing whole numbers. Example: 42.
  • iterationOne pass through a loop body. A loop that runs five times performs five iterations.
  • keyThe identifier used to access a value in a dictionary. Keys must be unique and immutable.
  • keywordA word with special meaning in Python that cannot be used as a variable name. Examples: if, for, def, return, True, None.
  • libraryA collection of modules providing reusable code. The Python standard library is included with Python.
  • listA mutable ordered sequence of values. Items can be of any type.
  • local variableA variable defined inside a function. It exists only during the function's execution.
  • loopA program structure that repeats a block of code multiple times.
  • methodA function defined inside a class. Called on an object using dot notation: object.method().
  • moduleA file containing Python code (functions, classes, variables) that can be imported and reused.
  • mutableCan be changed after creation. Lists, sets, and dictionaries are mutable.
  • namespaceA mapping from names to objects. Functions, classes, and modules each create their own namespaces.
  • NoneA special Python value representing the absence of a value. Functions with no return statement return None.
  • objectAn instance of a class. Bundles data (attributes) and behavior (methods) together.
  • operatorA symbol that performs an operation on values. Examples: +, -, *, /, ==, !=, and, or, not.
  • parameterA variable in a function definition that holds the value passed in by the caller.
  • pathA string that specifies the location of a file or folder. Can be absolute or relative.
  • print-style debuggingUsing print() statements to display variable values and track program execution.
  • promptA message passed to input() that tells the user what to type.
  • refactoringRestructuring code to improve clarity or reduce duplication without changing what the program does.
  • return valueThe value a function sends back to the code that called it, using the return statement.
  • scopeThe region of a program where a variable can be seen and used.
  • sequenceAn ordered collection of items accessible by index. Strings, lists, and tuples are sequences.
  • setA mutable unordered collection of unique values. Fast for membership testing and removing duplicates.
  • sliceA portion of a sequence extracted using [start:stop] notation.
  • stateThe current values of an object's attributes at a given moment in the program.
  • stringAn immutable sequence of characters used to represent text.
  • syntaxThe rules that define what valid Python code looks like.
  • syntax errorAn error caused by code that does not follow Python's syntax rules. Detected before the program runs.
  • test caseA specific input and the expected output used to verify that a program works correctly.
  • tracebackThe error report Python prints when an exception occurs. Shows the chain of calls that led to the error.
  • tracingFollowing a program's execution step by step to track variable values and understand behavior.
  • tupleAn immutable ordered sequence. Like a list, but cannot be changed after creation.
  • typeThe category of a value, which determines what operations are valid. Common types: int, float, str, bool, list, dict, tuple, set.
  • type conversionConverting a value from one type to another using functions like int(), float(), or str().
  • variableA named location in memory that stores a value. Assigned with =.
  • while loopA loop that continues as long as its condition is True.
