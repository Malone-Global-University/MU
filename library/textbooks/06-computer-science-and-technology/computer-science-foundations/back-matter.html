Appendix A
Mathematics and Logic for Computer Science
Arithmetic · Variables · Ratios · Tables · Boolean Logic · Truth Values · Sets · Counting · Estimation

Computer science does not require calculus or advanced algebra at the foundational level, but it does require careful reasoning with numbers, symbols, tables, and logic. This appendix reviews the mathematical tools used throughout the textbook.

A.1 Arithmetic Review

  • IntegerA whole number, positive, negative, or zero. Examples: −5, 0, 3, 100. Used for counts, indexes, bit positions.
  • RemainderWhat is left after integer division. 17 ÷ 5 = 3 remainder 2. In code, the modulo operator (%) returns the remainder: 17 % 5 = 2.
  • ExponentRepeated multiplication. 2³ = 2 × 2 × 2 = 8. Powers of 2 appear throughout computing: 2¹⁰ = 1,024 ≈ 1,000 (one kilobyte).
  • Order of operationsParentheses first, then exponents, then multiplication/division (left to right), then addition/subtraction (left to right). Applies to code expressions as well as mathematics.
Example A.1
Powers of 2 — the scale of digital storage
2¹ = 2 (2 states per bit) 2⁸ = 256 (values in one byte) 2¹⁰ = 1,024 ≈ 1 KB 2²⁰ = 1,048,576 ≈ 1 MB 2³⁰ = 1,073,741,824 ≈ 1 GB

A.2 Variables and Expressions

A variable in mathematics is a symbol representing an unknown or changing value. In computing, variables are named locations in memory. Both uses share the idea of a placeholder. An expression combines values, variables, and operations into a single quantity: 2n + 3 is an expression; cost = price * quantity is an assignment statement in code.

A.3 Ratios, Percentages, and Rates

  • RatioA comparison of two quantities: 3:1, or 3/1 = 3. Compression ratios, clock speeds, and bandwidth are stated as ratios.
  • PercentageA ratio expressed as parts per hundred. An algorithm with 75% accuracy correctly classifies 75 out of every 100 inputs.
  • RateA ratio with different units. 1 Gbps = 1 gigabit per second. Bandwidth and data transfer speeds are rates.

A.4 Tables and Graphs

A table organizes data in rows and columns. Rows represent records; columns represent attributes. Reading a table means identifying which column holds the value you need and which row holds the record. A graph shows the relationship between two quantities. A line graph shows how one value changes as another increases. A bar chart compares discrete values across categories.

A.5 Boolean Logic

Boolean logic operates on two values: True (1) and False (0). The three fundamental operations are AND, OR, and NOT. Boolean expressions are the foundation of conditions in programs, search queries, circuit design, and database queries.

A.8 Basic Set Ideas

  • SetA collection of distinct elements. {0, 1, 2, 3} is the set of digits less than 4. Sets model the values a variable can take, the characters in a character set, or the members of a group.
  • IntersectionElements in both sets. {1, 2, 3} ∩ {2, 3, 4} = {2, 3}. Corresponds to AND in Boolean logic and database JOIN overlap.
  • UnionAll elements from either set. {1, 2} ∪ {2, 3} = {1, 2, 3}. Corresponds to OR in Boolean logic.
  • SubsetSet A is a subset of B if every element of A is also in B. The set {0, 1} is a subset of the integers.

A.9 Counting and Combinations

When a choice has n options at each of k steps, the total number of distinct outcomes is nk. A single bit has 2 options (0 or 1). Eight bits (one byte) have 28 = 256 possible values. This counting principle explains why an 8-bit color channel produces 256 shades and why a 4-character PIN with digits 0–9 has 104 = 10,000 possible values.

A.10 Estimation and Approximation

Computer scientists frequently estimate rather than calculate exactly. "Roughly 1,000" is often close enough when comparing algorithms or capacities. Key approximations: 210 ≈ 103 (one thousand); 220 ≈ 106 (one million); 230 ≈ 109 (one billion). These bridge binary and decimal scale.

B.1 Bits and Bytes

  • BitThe smallest unit of digital information. A single binary digit: 0 or 1.
  • Byte8 bits. A byte can store 2⁸ = 256 distinct values (0–255). The basic unit for character encoding and memory addressing.
  • Nibble4 bits. Half a byte. Represents one hexadecimal digit (0–F).

B.2 Binary Counting — Positional Notation

B.3 Decimal-to-Binary Conversion

Division Method

Divide the decimal number by 2. Record the remainder. Divide the quotient by 2 again. Repeat until the quotient is 0. Read the remainders from bottom to top.

Example: 45 ÷ 2 = 22 R 1 / 22 ÷ 2 = 11 R 0 / 11 ÷ 2 = 5 R 1 / 5 ÷ 2 = 2 R 1 / 2 ÷ 2 = 1 R 0 / 1 ÷ 2 = 0 R 1 → Read up: 101101

B.4 Binary-to-Decimal Conversion

Positional Sum Method

Write the binary number. Under each digit, write the power of 2 for that position (right to left starting at 2⁰). Multiply each digit by its power. Sum all the products where the bit is 1.

1011 → (1×8) + (0×4) + (1×2) + (1×1) = 8 + 0 + 2 + 1 = 11

B.6 Representing Text

  • ASCIIAmerican Standard Code for Information Interchange. 7-bit encoding of 128 characters: uppercase A–Z (65–90), lowercase a–z (97–122), digits 0–9 (48–57), and punctuation. 'A' = 65 = 01000001.
  • UnicodeA universal character encoding standard supporting over 140,000 characters covering most of the world's writing systems. UTF-8 is the dominant encoding on the web.
  • EncodingThe mapping from characters to numeric codes. Different encodings may assign different numbers to the same character, causing garbled text when mismatched.

B.7 Representing Images

  • PixelThe smallest addressable unit of a digital image. Each pixel stores a color value.
  • RGBRed, Green, Blue. Each channel is one byte (0–255). White = (255, 255, 255). Black = (0, 0, 0). Together: 3 bytes = 24-bit color per pixel.
  • ResolutionThe number of pixels in an image, stated as width × height. A 1920×1080 image has 2,073,600 pixels. At 3 bytes each ≈ 6 MB uncompressed.

B.8 Representing Sound

  • SamplingConverting a continuous sound wave to discrete measurements. CD audio samples 44,100 times per second (44.1 kHz).
  • Bit depthThe number of bits per sample. 16-bit audio allows 2¹⁶ = 65,536 amplitude levels per sample.
  • BitrateBits per second of audio. CD: 44,100 samples/sec × 16 bits × 2 channels = 1,411,200 bps ≈ 1.4 Mbps uncompressed.

B.9 Compression

  • LosslessCompression that allows perfect reconstruction of the original. Used for text, programs, and ZIP archives. PNG is lossless.
  • LossyCompression that discards some data to achieve smaller size. JPEG (images) and MP3 (audio) are lossy. Acceptable quality loss is the tradeoff.
  • Run-length encodingA simple lossless technique: replace repeated values with a count and the value. "AAABBBCC" → "3A3B2C".

Common Representation Error

Confusing bits and bytes. Network speeds are typically measured in bits per second (Mbps), while file sizes are measured in bytes (MB). A 100 Mbps connection transfers 100 megabits = 12.5 megabytes per second — not 100 MB/s.

C.5 Algorithm Design Checklist

  • Defined inputs?Every input is named, described, and given a reasonable type. Edge cases (empty input, zero, negative) are considered.
  • Defined outputs?The expected result is clearly stated. The algorithm produces an output for every valid input.
  • Finite?The algorithm terminates. Every loop has a condition that eventually becomes false.
  • Correct?Verified by tracing through at least two test cases, including a normal case and an edge case.
  • Clear?Another person could follow the steps without asking for clarification.

D.10 Readable Code Guidelines

  • Descriptive namesUse student_count not sc. Names should communicate purpose without a comment.
  • One idea per lineAvoid packing multiple operations on a single line unless clarity is gained.
  • Comments on whyComment the reason, not the what: # Cap score at 100 to prevent overflow is better than # Set score to 100.
  • Consistent indentationPython requires indentation. Use 4 spaces per level throughout the file.
  • Short functionsA function that fits on one screen is easier to understand and test than one that scrolls for pages.

E.1 Three Types of Errors

Error TypeWhen It OccursExampleHow to Find It
Syntax errorBefore the program runsif x > 0 (missing colon)Read the error message; find the highlighted line
Runtime errorWhile the program is runningDividing by zero; index out of rangeRead the traceback; identify the failing line
Logic errorProgram runs but produces wrong outputUsing + instead of * in a formulaTrace the program by hand; add print statements

E.2 Reading Error Messages

How to Read a Python Traceback

1. Read from the bottom up. The last line names the error type and the specific problem.

2. The line above it shows the file name, the line number, and the code that caused the error.

3. Higher lines show the call chain — which function called which.

4. Fix the error at the bottom-most indicated line first; other errors may disappear.

E.3 Debugging Strategies

  • Print debuggingInsert print(variable) statements at key points to inspect values at runtime. Remove them when the bug is fixed.
  • Hand tracingExecute the code on paper, one statement at a time, updating variable values manually. Effective for logic errors in small functions.
  • Binary searchComment out half the code. If the bug disappears, it was in the commented half. Divide and conquer until you isolate the problem.
  • Rubber duckExplain your code aloud, line by line, to an imaginary listener. Articulating the logic often reveals where it breaks.
  • Minimal caseReproduce the bug with the simplest possible input. A simpler case is easier to reason about.

E.4 Test Cases

CategoryDescriptionExample (function: compute average)
Normal caseTypical, expected input[80, 90, 70] → 80.0
Edge caseBoundary values[0] → 0.0; [100, 100] → 100.0
Empty inputNo data[] → error or None

E.5 Debugging Checklist

  • Read the errorRead the full error message before changing any code. The message tells you the error type and location.
  • Reproduce itConfirm that you can make the bug appear reliably with a specific input.
  • Isolate itNarrow the location to the smallest section of code that still produces the error.
  • Fix one thingChange one thing at a time. If you change three things and the bug disappears, you cannot know which fix worked.
  • Test the fixRun all test cases again after fixing, not just the one that showed the bug.
  • Document itNote what you changed and why, as a comment or in a change log.

F.2 Linear Search

Linear Search

Check each element one by one from the beginning. Return when you find a match; report not found when the list is exhausted. Works on any list, sorted or unsorted.

Best case: first element matches — O(1) Worst case: last element or not present — O(n) Average: O(n)

F.3 Binary Search

Binary Search

Requires a sorted list. Compare the target to the middle element. If equal, done. If target is smaller, search the left half. If larger, search the right half. Repeat.

Each step halves the search space. Best case: O(1) Worst case: O(log n) A list of 1,000,000 elements needs at most 20 comparisons (log₂ 1,000,000 ≈ 20).

Big-O Rule of Thumb

Big-O describes how an algorithm scales as input grows, not how fast it runs on one specific input. An O(n) algorithm that takes 1 second on 1,000 items will take about 10 seconds on 10,000 items. An O(n²) algorithm may be fine for 100 items but unusable for 100,000.

Appendix G
Computer Systems Reference
CPU · Memory · Storage · I/O · Motherboard · OS · Filesystems · Processes · Permissions · CLI · Tools

G.1 Core Hardware Components

  • CPUCentral Processing Unit. Executes instructions one by one using the fetch-decode-execute cycle. Speed is measured in GHz (billions of cycles per second). Modern CPUs have multiple cores that can run instructions in parallel.
  • RAMRandom Access Memory. Fast, temporary storage for programs and data that are currently running. Volatile — contents are lost when power is removed. Measured in GB.
  • Storage (HDD/SSD)Persistent storage that retains data without power. HDDs use spinning magnetic disks; SSDs use flash memory and have no moving parts. Measured in GB or TB.
  • GPUGraphics Processing Unit. Originally for rendering images; now widely used for parallel computation in AI, scientific simulation, and video encoding.
  • MotherboardThe main circuit board connecting CPU, RAM, storage, and expansion slots. The system bus carries data between components.

G.2 The Fetch-Decode-Execute Cycle

How the CPU Runs a Program

Fetch: The CPU reads the next instruction from memory (at the address in the program counter).

Decode: The control unit determines what the instruction means and which circuit should execute it.

Execute: The appropriate arithmetic or logic unit performs the operation. The result may be written to a register or to memory.

This cycle repeats billions of times per second.

G.3 Operating System Functions

  • Process managementThe OS starts, schedules, pauses, and terminates programs. Each running program is a process. The scheduler allocates CPU time among processes.
  • Memory managementThe OS allocates RAM to processes and protects each process's memory from others. Virtual memory allows processes to use more memory than physically available by swapping to disk.
  • Filesystem managementThe OS organizes files in a hierarchical directory structure. It handles permissions, reading, writing, and deletion.
  • Device driversSoftware that allows the OS to communicate with hardware devices. Drivers translate generic OS commands into device-specific signals.
  • User interfaceThe OS provides either a graphical interface (GUI) with windows and icons or a command-line interface (CLI) with text commands — often both.

Appendix H
Networks and Web Reference
Networks · Packets · Routers · IP · DNS · Protocols · Bandwidth · Clients/Servers · URLs · HTTP · HTML · CSS · JS

H.1 Network Fundamentals

  • NodeAny device connected to a network: a computer, phone, printer, router, or server.
  • PacketA small unit of data sent across a network. A file is split into many packets for transmission; packets may travel different routes and are reassembled at the destination.
  • RouterA device that forwards packets between networks based on IP addresses.
  • ProtocolAn agreed-upon set of rules for how data is formatted, addressed, transmitted, received, and acknowledged. TCP/IP is the foundational protocol suite of the internet.
  • BandwidthThe maximum data transfer rate of a connection, measured in bits per second (bps, Kbps, Mbps, Gbps).
  • LatencyThe time for a packet to travel from source to destination. Measured in milliseconds. Distinct from bandwidth: a satellite link may have high bandwidth but high latency.

H.2 IP Addresses and DNS

  • IP addressA unique numeric label for a device on a network. IPv4: four numbers 0–255 separated by dots (e.g., 192.168.1.1). IPv6: 128-bit hex notation, supporting far more addresses.
  • DNSDomain Name System. Translates human-readable domain names (e.g., www.example.com) into IP addresses. Works like a phone book for the internet.
  • Domain nameA human-readable address for a website: example.com. The rightmost part is the top-level domain (TLD): .com, .org, .edu, .gov.

H.4 A Web Request Step by Step

What Happens When You Type a URL

1. Your browser parses the URL and extracts the domain name.

2. Your device queries a DNS resolver to find the IP address for the domain.

3. Your browser opens a TCP connection to the server at that IP address (port 443 for HTTPS).

4. A TLS handshake establishes an encrypted session.

5. Your browser sends an HTTP GET request for the specified resource.

6. The server responds with an HTTP status code and the resource (HTML, image, etc.).

7. The browser parses the HTML, requests linked CSS and JavaScript, and renders the page.

Appendix I
Cybersecurity and Privacy Reference
Threats · Vulnerabilities · Risk · Malware · Phishing · Auth · Encryption · Backups · Passwords · MFA · Privacy

I.1 Core Security Concepts

  • ThreatAny potential cause of harm to a system. Threats include attackers, malware, accidents, and natural disasters.
  • VulnerabilityA weakness in a system that a threat could exploit. Examples: an unpatched software flaw, a weak password, an open network port.
  • RiskThe combination of likelihood and impact. Risk = threat × vulnerability × asset value. Mitigation reduces likelihood or impact.
  • CIA TriadThe three goals of cybersecurity: Confidentiality (only authorized parties see data), Integrity (data is not altered without authorization), Availability (systems are accessible when needed).

I.3 Authentication and Authorization

  • AuthenticationVerifying that a user is who they claim to be. Based on: something you know (password), something you have (phone or token), something you are (biometric).
  • Multifactor authentication (MFA)Requiring two or more authentication factors. A stolen password alone is insufficient if MFA is enabled.
  • AuthorizationDetermining what an authenticated user is allowed to do. Read, write, execute, admin — each is a separate permission.
  • Principle of least privilegeGrant each user or process only the minimum access needed. Reduces damage if credentials are compromised.

I.4 Encryption

  • EncryptionTransforming data into an unreadable form using an algorithm and a key. Only those with the decryption key can read the data.
  • Symmetric encryptionThe same key is used for both encryption and decryption. Fast; key must be shared securely. Example: AES.
  • Asymmetric encryptionUses a public key (shared freely) to encrypt and a private key (secret) to decrypt. Used for secure key exchange and digital signatures. Example: RSA.
  • HTTPS / TLSTransport Layer Security encrypts web traffic. Combines asymmetric encryption for key exchange with symmetric encryption for speed.

I.5 Privacy Reference

  • Personal dataAny information that can be used to identify a person: name, address, email, phone number, IP address, browsing history.
  • MetadataData about data. A photo's metadata includes location, time, device, and settings — even if the photo shows nothing sensitive.
  • Digital footprintThe trail of data created by online activity: searches, purchases, posts, likes, location data, and login records.
  • TrackingCollecting data about a user's behavior across sites or apps, often to build a profile for advertising or analytics.
  • Password hygieneUse long, unique passwords for each account. Never reuse passwords. Use a password manager to store them. Avoid common words or personal information.
Appendix J
Data Science and AI Reference
Datasets · Features · Labels · Cleaning · Visualization · Statistics · Correlation · ML · Models · Neural Nets · Generative AI · Bias

J.1 Data Science Vocabulary

  • DatasetA structured collection of data, usually organized as a table of rows (records) and columns (features). May contain thousands to billions of records.
  • FeatureA measurable attribute or column in a dataset. For a housing dataset: square footage, number of rooms, location, and age are features.
  • LabelThe output variable that a model is trained to predict. For a housing model, the label is the sale price.
  • Data cleaningPreparing raw data for analysis: removing duplicates, handling missing values, correcting errors, standardizing formats.
  • Descriptive statisticsSummary numbers for a dataset: mean (average), median (middle value), mode (most frequent), minimum, maximum, range, standard deviation.
  • CorrelationA statistical measure of how two variables change together. Positive correlation: both go up. Negative: one goes up, the other goes down. Correlation does not imply causation.

J.2 Machine Learning Concepts

  • Machine learningA branch of AI in which systems improve their performance on a task by learning from data rather than through explicit programming.
  • TrainingThe process of exposing a model to labeled examples so it can learn patterns. Training data is used to fit the model's parameters.
  • ModelA mathematical function that maps inputs to outputs. The model is the result of training; it encodes the patterns learned from data.
  • ClassificationPredicting which category an input belongs to. Examples: spam or not-spam; cat, dog, or bird in an image.
  • RegressionPredicting a continuous numerical value. Examples: predicting tomorrow's temperature or a house's sale price.
  • OverfittingWhen a model learns the training data too specifically and performs poorly on new data. The model memorizes noise rather than learning generalizable patterns.

J.3 Neural Networks and Generative AI

  • Neural networkA machine learning model loosely inspired by the structure of biological neurons. Organized in layers: input, hidden, and output. Each connection has a learned weight.
  • Deep learningNeural networks with many hidden layers. The word "deep" refers to the number of layers, not the depth of insight. Powers image recognition, speech, and language models.
  • Generative AIAI systems that produce new content — text, images, audio, code — rather than just classifying or predicting. Large language models (LLMs) generate text by predicting likely next tokens.
  • PromptThe input given to a generative AI model. The model produces output conditioned on the prompt.

J.4 Bias and Evaluation

  • AccuracyThe fraction of predictions a model gets right. Misleading when classes are imbalanced: a model that always predicts "not spam" in a 99% not-spam dataset is 99% accurate but useless.
  • Training/test splitDividing the dataset so the model trains on one portion and is evaluated on a separate, unseen portion. Prevents the evaluation from being too optimistic.
  • Bias in AIWhen a model's outputs systematically favor or disadvantage a group. Often caused by biased training data. A model trained on historical hiring decisions may replicate past discrimination.
  • Responsible AIDesigning, evaluating, and deploying AI systems with attention to fairness, transparency, accountability, privacy, and safety.
Appendix K
Computing Ethics and Careers Reference
Professional Ethics · Accessibility · Privacy · Fairness · Safety · Environment · Automation · Careers · Learning

K.1 Professional Ethics in Computing

  • Do no harmA foundational principle: computing professionals should not create systems that harm users, communities, or society. Harm can be physical, financial, psychological, or social.
  • HonestyBe truthful about what software does, its limitations, and its risks. Do not conceal defects or misrepresent capabilities.
  • FairnessSystems should not discriminate unjustly based on race, gender, age, disability, or other protected attributes. Fairness in AI requires active design and evaluation effort.
  • PrivacyCollect only the data you need, store it securely, and use it only for the stated purpose. Users should understand what data is collected and how it is used.
  • Intellectual propertyRespect software licenses, copyright, and attribution. Open-source licenses have specific requirements; violating them is unethical and potentially illegal.

K.2 Accessibility and Inclusion

  • AccessibilityDesigning systems that can be used by people with visual, auditory, motor, or cognitive disabilities. Screen readers, keyboard navigation, captions, and sufficient color contrast are examples.
  • WCAGWeb Content Accessibility Guidelines. International standards for web accessibility, organized around four principles: Perceivable, Operable, Understandable, Robust.
  • Universal designDesigning products to be usable by as many people as possible without modification. Accessibility benefits everyone: captions help in noisy environments; large text helps on small screens.

K.3 Environmental Impact

  • Energy consumptionData centers consume large amounts of electricity. Efficient algorithms, hardware, and cooling reduce energy use. Training large AI models can use as much energy as a transatlantic flight.
  • E-wasteDiscarded electronics contain hazardous materials and are a growing environmental problem. Extending device lifetimes and responsible recycling reduce impact.
  • Algorithmic efficiencyA more efficient algorithm uses less CPU time and thus less energy. Choosing O(n log n) over O(n²) is an environmental choice as well as a performance one at scale.

Naming Principles

Names should tell a reader what a variable holds or what a function does — without requiring a comment to explain. Prefer student_count to n, is_valid to flag, and calculate_tax(price) to calc(x). Consistency within a project matters as much as following any one convention.

Glossary
Alphabetical glossary of computer science terms used throughout the textbook — Chapters 1–28

Chapter references in parentheses indicate where the term is introduced or used most fully. Terms appear in the glossary when they are central to understanding the textbook. For a comprehensive alphabetical index of all topics, see Appendix O.

A
AbstractionThe process of hiding unnecessary detail while exposing only the information needed for a given purpose. A function signature is an abstraction over its implementation. (Ch. 3, 13)
AlgorithmA finite, precise, step-by-step procedure for solving a problem or accomplishing a task, with defined inputs and outputs. (Ch. 3, 5)
APIApplication Programming Interface. A defined set of rules for how software components communicate, allowing one program to use the services of another without knowing its internal implementation. (Ch. 17)
ArgumentA specific value passed to a function when it is called. Distinct from a parameter, which is the variable in the function definition. (Ch. 9)
ArrayAn ordered sequence of elements stored at contiguous memory locations, accessed by integer index. In Python, the built-in list is the standard array-like structure. (Ch. 11)
AutomationUsing a system to perform tasks with minimal or no human intervention. Programs, scripts, and robots all represent forms of automation. (Ch. 1, 24)
B
BandwidthThe maximum rate at which data can be transferred over a network connection, measured in bits per second. (Ch. 19)
BinaryThe base-2 number system using only the digits 0 and 1. The foundation of all digital data representation. (Ch. 4)
BitThe smallest unit of digital information. A single binary digit: 0 or 1. (Ch. 4)
BooleanA data type with exactly two possible values: True and False. Used in conditions, comparisons, and logic operations. Named after mathematician George Boole. (Ch. 7)
BugAn error in a program that causes incorrect or unexpected behavior. Bugs may be syntax errors, runtime errors, or logic errors. (Ch. 10)
ByteA group of 8 bits. A byte can represent 2⁸ = 256 distinct values (0–255). The standard unit of character encoding and memory measurement. (Ch. 4)
C
CachingStoring a copy of frequently accessed data in a faster or closer location to reduce retrieval time. Web browsers cache pages; CPUs cache memory contents. (Ch. 12, 15)
ClassificationA machine learning task in which a model predicts which category an input belongs to, such as spam or not-spam. (Ch. 26)
ClientA device or program that requests services from a server. A web browser is a client; the computer running the website is the server. (Ch. 19, 20)
CompilerA program that translates source code in a high-level language into machine code or another language all at once, before the program runs. (Ch. 6)
CompressionReducing the size of data by encoding it more efficiently. Lossless compression allows perfect recovery; lossy compression discards some data for greater reduction. (Ch. 4)
Computational ThinkingA problem-solving approach that uses decomposition, pattern recognition, abstraction, and algorithmic thinking to formulate solutions that can be executed by a computer or person. (Ch. 3)
Control FlowThe order in which statements in a program are executed. Control flow is altered by conditionals, loops, and function calls. (Ch. 8)
CPUCentral Processing Unit. The primary processor of a computer; executes program instructions using the fetch-decode-execute cycle. (Ch. 2, 15)
CSVComma-Separated Values. A plain-text format for storing tabular data where each row is a line and values within a row are separated by commas. (Ch. 14)
D
DataRaw facts, measurements, or observations that have been recorded but not yet interpreted. Data becomes information when organized and given context. (Ch. 1, 2)
DatabaseAn organized collection of structured data, typically managed by a Database Management System (DBMS) that supports querying, updating, and protecting the data. (Ch. 18)
DebuggingThe process of finding, understanding, and fixing errors in a program. (Ch. 10)
DecompositionBreaking a complex problem into smaller, more manageable sub-problems. A key strategy in computational thinking and software design. (Ch. 3, 9)
DictionaryA data structure that stores key-value pairs, allowing retrieval of a value by its associated key in constant average time. Called a map or hash map in other languages. (Ch. 11)
DNSDomain Name System. The distributed system that translates human-readable domain names into numeric IP addresses. (Ch. 19)
E
Edge CaseAn input or situation at the boundary of normal operation: an empty list, zero, the maximum allowed value, or an unexpected type. Edge cases often reveal bugs. (Ch. 10)
EncryptionTransforming data into an unreadable form using an algorithm and a key so that only authorized parties can recover the original. (Ch. 21)
F
FeatureIn machine learning, a measurable attribute of an input used by a model to make predictions. In data science, a column in a dataset. (Ch. 25, 26)
FirewallA network security system that monitors and filters incoming and outgoing traffic based on defined rules. (Ch. 21)
FlowchartA diagram that represents an algorithm or process using standardized shapes connected by arrows indicating flow of control. (Ch. 5)
FunctionA named, reusable block of code that performs a specific task. Functions accept parameters, execute a body of statements, and optionally return a value. (Ch. 9)

HardwareThe physical components of a computing system: CPU, memory, storage, input/output devices, and the circuits that connect them. (Ch. 2, 15)
HTTP / HTTPSHyperText Transfer Protocol (and its secure, TLS-encrypted variant). The protocol used to request and transfer web pages. (Ch. 20)
IndexAn integer that identifies the position of an element in a sequence. Most programming languages use zero-based indexing, so the first element is at index 0. (Ch. 11)
InputData or signals provided to a system or algorithm for processing. Inputs may come from a user, a file, a sensor, or another program. (Ch. 2, 5)
InternetA global network of interconnected networks using the TCP/IP protocol suite. The internet provides the infrastructure that the Web, email, and other services run on. (Ch. 19)
InterpreterA program that executes source code directly, line by line, without compiling it to machine code first. Python is typically run with an interpreter. (Ch. 6)
IP AddressA numeric label assigned to each device on a network. IPv4 addresses have the form A.B.C.D where each part is 0–255. (Ch. 19)
L–N
LatencyThe time delay between sending a request and receiving a response. A network measure distinct from bandwidth. (Ch. 19)
ListAn ordered, mutable sequence of elements in Python. Supports indexing, appending, removing, and sorting. (Ch. 11)
Logic ErrorA bug in which the program runs without crashing but produces incorrect results because the algorithm or expression is wrong. (Ch. 10)
LoopA control flow construct that repeats a block of statements. A while loop repeats while a condition is true; a for loop iterates over a sequence. (Ch. 8)
MalwareMalicious software designed to damage systems, steal data, or gain unauthorized access. Includes viruses, worms, ransomware, and spyware. (Ch. 21)
ModelIn machine learning, a mathematical function learned from data that maps inputs to predictions. In simulation, a simplified representation of a system. (Ch. 13, 26)
NetworkTwo or more connected devices that can exchange data. A local area network (LAN) connects devices in one location; a wide area network (WAN) spans larger distances. (Ch. 19)
O–P
Operating SystemSystem software that manages hardware resources and provides services to applications. Examples: Windows, macOS, Linux, Android. (Ch. 16)
OutputThe result produced by a system after processing. Output may be text on a screen, data written to a file, a signal sent to a device, or a network packet. (Ch. 2, 5)
PacketA small unit of data transmitted over a network. Large messages are split into packets, which may take different routes and are reassembled at the destination. (Ch. 19)
ParameterA variable in a function definition that receives a value (argument) when the function is called. (Ch. 9)
PhishingA social engineering attack in which a fraudulent message impersonates a trusted entity to steal credentials or install malware. (Ch. 21, 22)
ProtocolA set of agreed-upon rules governing how systems communicate. TCP/IP, HTTP, SMTP, and SSH are all protocols. (Ch. 19)
PseudocodeAn informal, language-independent description of an algorithm using human-readable text. Used to design algorithms before coding. (Ch. 5)
Q–R
QueryA request for data from a database or search system, often written in SQL. A query specifies what data is wanted and may include filters, sorts, and joins. (Ch. 18)
RAMRandom Access Memory. Fast, volatile computer memory that holds the programs and data currently in use. Contents are lost when power is removed. (Ch. 15)
Return ValueThe value a function sends back to its caller after completing its task. Functions may return one value, multiple values, or nothing (None in Python). (Ch. 9)
RouterA networking device that forwards packets between networks, choosing the best path based on routing tables and the destination IP address. (Ch. 19)
Runtime ErrorAn error that occurs while a syntactically correct program is running, such as division by zero or accessing an index beyond the end of a list. (Ch. 10)
S–T
ScopeThe region of a program where a variable is defined and accessible. Variables defined inside a function are local to that function and not accessible outside it. (Ch. 9)
ServerA computer or program that provides services, data, or resources to clients. A web server responds to HTTP requests by delivering pages. (Ch. 19, 20)
SimulationA program that models the behavior of a real or hypothetical system over time. Used in science, engineering, economics, and training. (Ch. 13)
SoftwarePrograms, data, and instructions that tell hardware what to do. Contrasted with hardware. Includes operating systems, applications, libraries, and firmware. (Ch. 2, 17)
SQLStructured Query Language. The standard language for querying and manipulating relational databases. Key statements: SELECT, INSERT, UPDATE, DELETE. (Ch. 18)
StackA last-in, first-out (LIFO) data structure. Elements are pushed onto the top and popped from the top. Used for function call management, undo operations, and parsing. (Ch. 11)
StringA sequence of characters. In Python, strings are immutable and enclosed in single or double quotes. (Ch. 7, 11)
Syntax ErrorA violation of the rules of a programming language's grammar. Syntax errors prevent the program from running. (Ch. 6, 10)
Training DataLabeled examples used to teach a machine learning model. The quality, diversity, and representativeness of training data significantly affect model performance and fairness. (Ch. 26)
U–Z
URLUniform Resource Locator. The address of a specific resource on the web. Structure: scheme://domain/path?query. Example: https://www.example.com/page?id=5. (Ch. 20)
UsabilityThe ease with which users can accomplish their goals using a system. Factors include learnability, efficiency, memorability, error recovery, and satisfaction. (Ch. 23)
VariableA named location in memory that holds a value. Variables can be reassigned. In Python, variables are created by assignment and can hold any type. (Ch. 7)
Version ControlA system for tracking changes to files over time, enabling collaboration and the ability to revert to earlier states. Git is the most widely used version control system. (Ch. 17, 27)

Part I — What Computer Science Studies (Chapters 1–4)
  • 1.1Computer science studies computation, information, algorithms, programs, systems, and the human use of technology. It is broader than programming.
  • 1.3Input → Processing → Output. Example: A search engine takes a query (input), runs a ranking algorithm (process), and displays results (output).
  • 2.1Analog information is continuous and can take any value in a range. Digital information is discrete, encoded as bits. A vinyl record is analog; an MP3 is digital.
  • 3.3Decomposition: break the problem "plan a class schedule" into sub-problems: list required courses, find available times, check conflicts, assign rooms.
  • 4.113 in binary: 13 ÷ 2 = 6 R 1; 6 ÷ 2 = 3 R 0; 3 ÷ 2 = 1 R 1; 1 ÷ 2 = 0 R 1. Reading remainders up: 1101.
  • 4.3Binary 10110 to decimal: 1×16 + 0×8 + 1×4 + 1×2 + 0×1 = 16 + 4 + 2 = 22.
  • 4.5The letter 'A' in ASCII has the decimal value 65, which in binary is 01000001.
Part II — Programming Foundations (Chapters 5–10)
  • 5.1An algorithm must be finite (it terminates), precise (each step is unambiguous), and produce a defined output for each valid input.
  • 6.1Syntax error: the grammar of the language is violated and the program cannot run. Logic error: the program runs but produces incorrect output.
  • 7.3Predict output of x = 3; y = x * 2 + 1; print(y): y = 3×2+1 = 7.
  • 8.1Tracing total = 0; for i in range(1, 5): total += i: i takes values 1, 2, 3, 4; total becomes 1, 3, 6, 10. Output: 10.
  • 9.1A function called with argument 5 where def double(n): return n * 2 returns 10.
  • 10.1The bug in if score = 90: is a syntax error — should be == (comparison), not = (assignment).
Part III — Data and Algorithms (Chapters 11–14)
  • 11.1In the list ["red", "green", "blue"], the element at index 2 is "blue". (Zero-based indexing: red=0, green=1, blue=2.)
  • 12.1Linear search checks each element one by one, O(n) worst case. Binary search halves the search space each step, O(log n) worst case — but requires a sorted list.
  • 12.3Binary search on a list of 128 elements: log₂(128) = 7. At most 7 comparisons.
  • 13.1Abstraction: a function named calculate_tax(price) hides the implementation. The caller only needs to know the inputs and what it returns, not how it works internally.
Parts IV–VII — Systems, Networks, Security, AI, Ethics (Chapters 15–28)
  • 15.1RAM is volatile (loses data when power is off) and fast. Hard drive storage is non-volatile (retains data) and slower. RAM holds running programs; storage holds saved files.
  • 19.1DNS translates a domain name like example.com to an IP address like 93.184.216.34, so your browser can connect to the server.
  • 21.1Phishing: a fraudulent email impersonating a bank asks you to click a link and enter your password. Defense: verify sender address, go directly to the bank's site.
  • 22.1Metadata in a photo may include: timestamp, GPS coordinates, camera model, and settings — even if the photo itself reveals nothing sensitive.
  • 26.1A spam classifier trains on labeled emails (spam/not-spam). Features might include specific words, sender reputation, and punctuation density. It outputs a prediction for new emails.
  • 28.1One ethical concern with facial recognition: it has historically shown higher error rates for darker-skinned faces, raising fairness and discrimination concerns, especially in law enforcement use.
