Chapter 7 — Front Matter

Purpose and Overview

Every program that does anything useful has to remember information. It stores a name, a score, a temperature, a password, a running total. It works with that information — comparing values, adding numbers, joining words together, checking whether something is true or false. This chapter is about how programs do that: through variables, types, expressions, and the idea of state.

These ideas are not unique to Python or to any one language. Variables, values, types, expressions, and assignment appear in Python, JavaScript, Java, C, Rust, and nearly every other language you will encounter. The syntax changes from language to language, but the underlying logic stays the same. This chapter uses Python-style examples because Python's syntax is clear and readable for learning. The goal, however, is not to memorize Python. The goal is to build the mental model that will carry you through Chapter 8 and every programming course beyond this one.

Opening Question

A program needs to track a player's score in a game. The score starts at zero, increases when the player earns points, and decreases when the player loses a life. How should the program remember this value? How does it update it? How does it display it on screen? By the end of this chapter, you will know exactly how — and why the same pattern applies to prices, temperatures, names, usernames, and every other piece of information a program manages.

Section 7.1

Values and Variables

What a Value Is

A value is a piece of information a program can use. Values come in many forms: the number 42, the word "hello", the measurement 3.14, or the truth-or-false choice True. Every piece of data in a program is a value. Values by themselves are temporary — if a program computes a result and nothing stores it, that result disappears as soon as the line finishes executing.

What a Variable Is

A variable is a named storage location that holds a value. When a program creates a variable, it gives a name to a piece of memory so that value can be referred to later. Think of a variable as a labeled box: the label is the variable's name, and the contents of the box are the variable's value. The box can hold different contents at different times.

The Variable Mental Model

variable name: score

variable value: 95

A program can read from the box (look at the score), write to the box (change the score), or pass the box's contents somewhere else (display the score, or add it to another number).

Variable Naming Rules

In Python, variable names can contain letters, digits, and underscores. Names must start with a letter or underscore — not a digit. Names are case-sensitive: score, Score, and SCORE are three different variables. Names should be descriptive: total_score is clearer than ts.

Habit of Mind

The habit of writing clear variable names is one of the most valuable habits in programming. It makes programs readable — for other programmers, and for your future self who may return to the code months later with no memory of what x was supposed to mean.

Section 7.2

Assignment

The Assignment Statement

An assignment statement stores a value into a variable. In Python, the assignment operator is a single equals sign (=). The left side is always a variable name; the right side is always a value or expression that produces a value. The = sign does not mean "equals" in the mathematical sense — it means "store this value into this variable."

age = 17
score = 100
name = "Jordan"
temperature = 98.6
is_active = True

After these five lines execute, five variables exist, each holding a value of a different type.

Reassignment

A variable can be reassigned as many times as needed. Each assignment replaces the previous value entirely — the earlier value is gone.

score = 100
score = 95
score = 87
print(score)   # Output: 87

The Right Side Evaluates First

An important rule: the right side of an assignment is fully evaluated before anything is stored. This makes the pattern x = x + 1 valid — and very common in programming.

x = 5
x = x + 1    # look up x (5), add 1, get 6, store 6 in x
print(x)      # Output: 6
Mathematical vs. Programming Meaning

In mathematics, x = x + 1 is impossible — no number equals itself plus one. In programming, = is not equality; it is an instruction. It means: "evaluate the right side, then store the result in the variable on the left." The earlier value of x is used in computing the new value, then replaced.

Trace Tables

A trace table records the values of variables through each step of a program's execution. Tracing is the most reliable way to understand what a program does and to find errors.

Section 7.3

Numbers

Integers

An integer (type int) is a whole number with no decimal point. Integers can be positive, negative, or zero. They are used for counting, indexing, and any situation where fractional values do not make sense — you cannot have 2.7 students.

students = 30
temperature = -5
items = 0
year = 2025

Floats

A float (type float) is a number with a decimal point, representing real-world measurements that are not always whole numbers.

price = 4.99
height = 1.75
pi = 3.14159

Arithmetic Operators

OperatorMeaningExampleResult
+Addition7 + 310
-Subtraction7 - 34
*Multiplication7 * 321
**Exponentiation2 ** 416
//Integer division7 // 32
%Remainder (modulo)7 % 31
/Division (float result)7 / 32.333...

Integer division (//) divides and keeps only the whole number portion, discarding the remainder. 7 // 3 gives 2 because 3 goes into 7 twice with something left over. The remainder operator (%) gives what is left after integer division. 7 % 3 gives 1 because 7 = (3 × 2) + 1.

Example 7.3.1
Converting minutes into hours and minutes
  • ProblemA program receives a total of 137 minutes. Break it into full hours and remaining minutes.
  • Solve minutes = 137 hours = minutes // 60 # 137 // 60 = 2 remaining = minutes % 60 # 137 % 60 = 17
  • Check2 × 60 = 120; 120 + 17 = 137. ✓
Output: 2 full hours, 17 minutes remaining.
Two Division Traps

Dividing by zero causes a ZeroDivisionError and stops the program. Always ensure the divisor cannot be zero before dividing.

Mixing division types: 7 / 2 produces 3.5 in Python — not 3. If you want integer division, use 7 // 2.

Section 7.4

Strings

A string is a sequence of characters — letters, digits, punctuation, spaces, or any other symbol — treated as text. Strings are enclosed in quotation marks. Both single quotes and double quotes work in Python; the convention is to be consistent. The value "42" and the value 42 are completely different: one is a string, one is an integer.

String Operations

OperationOperatorExampleResult
Concatenation (join)+"Hello" + " World""Hello World"
Repetition*"-" * 10"----------"
Lengthlen()len("hello")5
Indexing[]"hello"[1]"e"

Zero-Based Indexing

A string is a sequence of individual characters, each at a numbered position called an index. In Python — and most languages — indexes start at zero, not one.

Index Positions in a String

String: a l g o r i t h m

Index: 0 1 2 3 4 5 6 7 8

The string "algorithm" has 9 characters. The first is at index 0; the last is at index 8. This off-by-one pattern appears throughout programming — lists, arrays, and all sequences behave the same way.

word = "algorithm"
print(word[0])    # a
print(word[4])    # r
print(word[8])    # m
print(len(word)) # 9

f-Strings

An f-string (formatted string literal) lets you embed variable values directly inside a string. Place an f before the opening quote and put variable names or expressions inside {}.

name = "Jordan"
score = 94
print(f"Student: {name}, Score: {score}")
# Output: Student: Jordan, Score: 94
Strings Are Not Numbers

"42" is a string. 42 is an integer. You cannot mix them with + directly. Writing "Score: " + 94 will cause a TypeError. Use an f-string or str(94) to convert first.

Section 7.5

Booleans

A Boolean (named after mathematician George Boole) is a value that is either True or False — exactly two values, nothing else. Booleans represent yes/no, on/off, and present/absent conditions. They are most often produced by comparison expressions.

Boolean Operators and Truth Tables

Three operators combine or modify Boolean values: and (True only if both sides are True), or (True if at least one side is True), and not (reverses the value).

Critical Distinction: = vs ==

= is assignment: stores a value into a variable. It is an instruction.
== is comparison: checks whether two values are equal and produces True or False.

Writing = when you mean == — or vice versa — causes bugs that can be difficult to find. This is one of the most common errors beginners make.

Section 7.6

Expressions and Operators

An expression is any combination of values, variables, and operators that a program can evaluate to produce a single value. 3 + 4 evaluates to 7. score * 2 evaluates to twice the current value of score. age >= 18 evaluates to True or False. Every expression produces exactly one value, which can be stored, passed to a function, or used inside another expression.

Operator Precedence

When an expression contains multiple operators, evaluation follows precedence rules — similar to the order of operations in arithmetic. Higher-precedence operators evaluate first.

When in Doubt, Use Parentheses

Parentheses are always evaluated first and make intent clear. (2 + 3) * 4 is unambiguous even if the reader does not know precedence rules. Parentheses also make code easier to read, which is just as important as making it correct.

Section 7.7

Input and Output

Output: print()

The print() function displays output to the screen. It can take one value, multiple comma-separated values, or an f-string. By default, it separates multiple values with a space and ends with a newline.

print("Hello!")
print(42)
print(3.14)
print("Name:", name, "Age:", age)

Input: input()

The input() function pauses the program and waits for the user to type something. Whatever is typed is returned as a string — always a string, regardless of what the user typed. The argument to input() is the prompt shown to the user.

name = input("What is your name? ")
print("Hello,", name)

Converting Input to Numbers

Because input() always returns a string, you must convert the result if you need a number. If the user types 17, you receive the string "17" — you cannot do arithmetic on it directly without converting it first.

age = int(input("Enter your age: "))
next_year = age + 1
print(f"Next year you will be {next_year}")
Wrong Input Type

If the user types "seventeen" when the program expects an integer, int("seventeen") will fail with a ValueError. Handling this safely requires error handling techniques covered in later courses. For now, always ensure you convert input to the correct type before performing operations on it.

Type Conversion

Type conversion (also called casting) changes a value from one type to another. Python provides built-in functions for the most common conversions.

FunctionConverts ToExampleResult
int()Integerint("42")42
float()Floatfloat("3.14")3.14
str()Stringstr(99)"99"
bool()Booleanbool(0)False

Note that int() applied to a float truncates — it removes the decimal portion without rounding. int(3.9) produces 3, not 4. int(-3.9) produces -3, not -4.

Falsy Values in Python

Every value has a Boolean interpretation. Falsy values — those that behave like False — include: 0, 0.0, "" (empty string), None, and empty collections. Everything else is truthy.

bool(0) → False    bool(1) → True    bool("") → False    bool("hi") → True

Section 7.9

State and Changing Values

A program's state is the collection of all variables and their current values at any moment during execution. State changes as the program runs — variables are created, values are updated, and conditions shift. Understanding state is essential for reading programs and predicting output. When you trace a program, you are tracking its state step by step.

Example 7.9.1

  • Trace

Accumulator Variables

An accumulator is a variable that builds up a value over time through repeated addition or other operations. The pattern total = total + value is so common that Python provides shorthand operators.

ShorthandEquivalentEffect
x += 5x = x + 5Add 5 to x
x -= 3x = x - 3Subtract 3 from x
x *= 2x = x * 2Multiply x by 2
x /= 4x = x / 4Divide x by 4
x //= 2x = x // 2Integer divide x by 2
x %= 3x = x % 3Replace x with x mod 3
 
score = 0
score += 10    # score is now 10
score += 25    # score is now 35
score -= 5     # score is now 30
print(score)   # 30

Swapping Two Variables

Swapping the values of two variables is a classic programming task. A naive approach fails because the first assignment overwrites the original value before the second assignment can read it. The correct approach uses a temporary variable to preserve the original value.

Example 7.9.2
Swapping with a temporary variable
  • Naive (wrong) a = 10 b = 20 a = b # a is now 20 — original value of a is LOST b = a # b is now 20, not 10 # Result: a = 20, b = 20 (WRONG)
  • Correct a = 10 b = 20 temp = a # save a's original value a = b # a gets b's value (20) b = temp # b gets original a's value (10) # Result: a = 20, b = 10 (CORRECT)
The temporary variable preserves the original value of a before it is overwritten.
Python One-Line Swap

Python allows a cleaner swap using tuple unpacking: a, b = b, a. This swaps the values in one line. The right side is fully evaluated before any assignment occurs, so neither value is lost. This is elegant Python — but the temporary variable method remains important to understand.

Section 7.10

Reading Simple Programs

A program is a list of instructions that execute from top to bottom, one at a time. Reading a program means following along and tracking what happens at each step. The habits of a good code reader: start at the top; for each line, ask what it does to the program's state; track variable values in a trace table; predict what each print() will produce before running the code. Do not run the code first — predict the output, then verify.

Example 7.10.1
A simple calculator program
print("Simple Calculator")
x = float(input("Enter first number: "))
y = float(input("Enter second number: "))

total      = x + y
difference = x - y
product    = x * y

print(f"Sum:        {total}")
print(f"Difference: {difference}")
print(f"Product:    {product}")
  • ReadingPrint a header. Get two floats from the user. Compute three results. Print each using f-strings. Five variables: x, y, total, difference, product.
  • For x=6, y=4 Sum: 10.0 Difference: 2.0 Product: 24.0
Example 7.10.2
Name greeting with string operations
first = input("First name: ")
last  = input("Last name: ")
full_name = last + ", " + first
print(f"Hello, {full_name}!")
  • For "Alex" Rivera first = "Alex" last = "Rivera" full_name = "Rivera" + ", " + "Alex" = "Rivera, Alex" Output: Hello, Rivera, Alex!
Example 7.10.3
Temperature conversion with formula
celsius = float(input("Enter temperature in Celsius: "))
fahrenheit = (celsius * 9 / 5) + 32

Chapter Summary

  • Sec 7.1A value is a piece of data. A variable is a named storage location that holds a value. Variables should have clear, descriptive names. Python names start with a letter or underscore and are case-sensitive.
  • Sec 7.2The assignment operator = stores a value into a variable. The right side evaluates first. Reassignment replaces the previous value. Trace tables track variable state step by step.
  • Sec 7.3Integers are whole numbers; floats have decimal points. Arithmetic operators include +, -, *, **, /, //, and %. Integer division and remainder are especially useful for time, indexing, and parity checks.
  • Sec 7.4Strings are sequences of characters. Concatenation uses +; repetition uses *. Strings are indexed from zero. f-strings embed variable values into output cleanly. Strings and numbers are different types and cannot be mixed with + directly.
  • Sec 7.5Booleans are True or False. Comparison operators produce Boolean values. Boolean operators and, or, and not combine or modify them. = is assignment; == is comparison — never confuse them.
  • Sec 7.6Expressions combine values, variables, and operators to produce one value. Precedence rules determine evaluation order; parentheses override them. Mixing integers and floats produces floats. Mixing strings with numbers requires explicit conversion.
  • Sec 7.7print() displays output; input() receives a string from the user. Input must always be converted to the correct type before arithmetic. Type mismatches cause TypeErrors.
  • Sec 7.8Type conversion functions: int(), float(), str(), bool(). int() on a float truncates — it does not round. Falsy values include 0, 0.0, "", and None.
  • Sec 7.9A program's state is all variables and their current values at any moment. Accumulator variables build up values using += and similar operators. Swapping two variables requires a temporary variable to preserve the original value.
  • Sec 7.10Reading a program means tracing execution top to bottom, tracking state at each step. Predict output before running code. The code-reading habit is as important as code-writing and improves debugging, comprehension, and design.

Key Terms Review

value
A piece of information a program can store and use: a number, a string, a Boolean, or other data.
variable
A named storage location in a program that holds a value and can be read or updated.
assignment
The process of storing a value into a variable using the = operator. The right side evaluates first.
integer (int)
A whole number with no decimal point, positive or negative. Used for counting and indexing.
float
A number with a decimal point, used for real-world measurements and calculations requiring fractions.
string (str)
A sequence of characters treated as text, enclosed in quotation marks.
Boolean (bool)
A data type with exactly two values: True and False. Named after mathematician George Boole.
data type
A classification that defines what kind of value a variable holds and what operations are valid on it.
expression
A combination of values, variables, and operators that evaluates to a single value.
operator precedence
Rules determining which operators in an expression are evaluated first when there are multiple operators.
concatenation
Joining two strings together using the + operator to produce a longer string.
index
A numeric position in a sequence, starting at zero in Python. Used to access individual characters or elements.
f-string
A formatted string literal prefixed with f, in which expressions inside {} are evaluated and inserted.
truth table
A table showing all possible input combinations for a Boolean operation and the resulting output.
type conversion
Changing a value from one data type to another, using functions like int(), float(), str(), or bool().
state
The complete collection of all variables and their current values at a given moment during program execution.
trace table
A table used to track variable values through each step of a program's execution, used to understand or debug a program.
accumulator
A variable that builds up a value over time, typically by repeatedly adding to it using the += pattern.
integer division (//)
Division that discards the decimal remainder and returns only the whole number portion.
remainder operator (%)
Returns what is left over after integer division. Also called the modulo operator. Useful for even/odd checks and time conversions.

Looking Ahead
Chapter 8: Conditionals, Loops, and Control Flow

Chapter 7 gave programs the ability to store values, compute expressions, and track state. But programs so far can only execute instructions from top to bottom in a fixed order. Every program in this chapter ran straight through without making any decisions or repeating anything.

Chapter 8 introduces control flow — the mechanisms that let a program take different paths depending on conditions, and repeat actions as many times as needed. If statements ask whether a Boolean expression is true and branch accordingly. While loops repeat a block of code as long as a condition holds. For loops repeat over a sequence of values. The Boolean values and comparison expressions from this chapter become the deciding factor in every branch and every loop. By the end of Chapter 8, programs will be able to make decisions, repeat work efficiently, and process information in flexible ways that begin to resemble the software people use every day.
