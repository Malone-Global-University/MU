Chapter 8 — Front Matter

Purpose and Overview

Every program you have seen so far runs in a straight line: step one, step two, step three, done. That is useful for short procedures, but almost no real program works that way. Real programs make decisions. They do different things depending on what they find. They repeat actions many times without you writing the same line over and over again. They skip steps, branch into alternatives, and keep running until a condition is satisfied.

This chapter introduces the tools that give programs this power. Those tools are called control flow structures. The two most important are conditionals — which let programs choose between paths — and loops — which let programs repeat actions automatically. Understanding these structures is one of the most important transitions in learning to program.

The chapter also introduces program tracing: following a program step by step on paper to predict what it will do. Tracing is one of the most important skills a programmer can develop. It teaches you to think like the computer executing the code, not just the programmer who wrote it.

Opening Scenario

A login system needs to check whether a password matches. If it does, let the user in. If it does not, show an error. A game character should keep moving until it hits a wall. A program should add up every number in a list of 10,000 entries without you writing 10,000 addition steps. All three of these require non-linear control flow — the ability for a program to take different paths, or repeat itself, based on what it discovers while running. By the end of this chapter, you will have the tools to write all of them.

Section 8.1

What Control Flow Means

When a computer runs a program, it executes instructions one at a time. The flow of control is the path the computer takes through those instructions. In the simplest programs, control flows from the first line to the last in a straight, uninterrupted path.

But most real programs cannot follow a straight path. A login system needs to check whether a password is correct. An app needs to display different content depending on a user's settings. A program that sums a list of 10,000 numbers cannot have 10,000 addition lines written by hand. All of these require non-linear control flow — the program's path depends on something it discovers while running.

StructureWhat it doesExample use
ConditionalChooses between paths based on a conditionIf the score is above 60, print "Pass"
LoopRepeats a block of instructions multiple timesAdd each number in a list to a total
Function callJumps to a named block and returns (Chapter 9)Call a function to compute a square root
The Tracing Habit

The most important skill in this chapter is learning to trace — to follow a program's execution step by step, on paper, predicting what each line does and what value each variable holds. A program is not a description. It is a precise set of instructions that execute in a specific order. Tracing teaches you to read programs the way the computer runs them, not the way the programmer intended them.

Section 8.2

Boolean Expressions and Comparisons

Before a program can choose between paths, it needs a way to ask a yes-or-no question. Those answers come from Boolean expressions. A Boolean expression is any expression that evaluates to exactly one of two values: True or False. There is no middle ground, no "maybe," and no "partially true."

Comparison Operators

Critical Distinction — = vs. ==

In Python, a single = is the assignment operator — it sets a variable's value. A double == is the comparison operator — it asks whether two values are equal and returns True or False. Writing if x = 5: is a syntax error. Writing if x == 5: is a comparison. Mixing these up is the single most common beginner mistake in Python.

Comparison — Assignment vs. Equality
x = 10        # Assignment: x now holds the value 10
x == 10       # Comparison: asks "is x equal to 10?" → True
x == 5        # Comparison: asks "is x equal to 5?"  → False

Logical Operators

You can think of these operators as mirrors of everyday language. "The score is above 60 and the student attended class" requires both conditions to hold. "The temperature is below 32 or it is raining" is satisfied by either condition. "The account is not locked" reverses the condition entirely. Boolean expressions become most powerful when they involve variables, because the result changes depending on the variable's value at any given moment.

Section 8.3

If Statements

An if statement tells the program: evaluate this condition. If it is True, execute this block of code. If it is False, skip it entirely. It is the simplest form of decision in programming, and it is the foundation for all the more complex conditional structures that follow.

If Statement — Structure

if condition:

instructions to run if condition is True

The colon after the condition is required. The indented lines form the block that runs when the condition is True. Lines at the same indentation level belong to the same block. Python uses indentation — not brackets or keywords — to group lines together.

A Simple If Statement
score = 85

if score >= 60:
    print("You passed.")
Trace 8.3.1
Tracing a simple if statement — two values
  • score = 8585 >= 60 is True. The indented block runs. Output: "You passed."
  • score = 4242 >= 60 is False. The indented block is skipped entirely. Nothing is printed.
Key insight: the same code produces different behavior based on the variable's value. This is control flow.
Indentation Is Not Decoration

In Python, indentation is part of the language's grammar. If the line inside an if block is not indented, Python raises an IndentationError. If it is indented but the if line is missing its colon, Python raises a SyntaxError. Both errors stop the program before it runs. Standard Python indentation is 4 spaces per level. Never mix tabs and spaces.

Section 8.4

Else and Multiple Branches

An if statement handles the case when a condition is True. But what should happen when it is False? The else clause provides a second path. An if-else structure always runs exactly one of the two blocks: if the condition is True, the if block runs; if it is False, the else block runs. There is no situation where both run, and no situation where neither runs.

If-Else
score = 42

if score >= 60:
    print("You passed.")
else:
    print("You did not pass.")

Many decisions have more than two possibilities. For this, Python provides elif (short for "else if"), which adds additional conditions to check when all preceding conditions were False. Python evaluates the chain from top to bottom and runs only the first branch whose condition is True. Once a branch runs, the rest are skipped.

If-Elif-Else — Grade Classification
grade = 73

if grade >= 90:
    letter = "A"
elif grade >= 80:
    letter = "B"
elif grade >= 70:
    letter = "C"
elif grade >= 60:
    letter = "D"
else:
    letter = "F"

print("Your grade is:", letter)
Trace 8.4.1
Tracing the elif chain — grade = 73
  • Step 1Evaluate 73 >= 90: False. Skip this block.
  • Step 2Evaluate 73 >= 80: False. Skip this block.
  • Step 3Evaluate 73 >= 70: True. Run this block: letter = "C". Stop checking.
  • Step 4The elif grade >= 60 and else branches are never evaluated.
  • OutputYour grade is: C
Only one branch runs. Once a True condition is found, the rest of the chain is skipped.

Section 8.5

Nested Decisions

Sometimes a decision depends on more than one condition checked in sequence — not combined with and, but evaluated one inside the other. This is called nesting: placing one if statement inside another. The inner if only runs when the outer condition is True. This is useful when the second question only makes sense if the first question answered a certain way.

Nested Conditionals — Event Entry
age = 20
has_ticket = True

if age >= 18:
    if has_ticket:
        print("Welcome. Enjoy the event.")
    else:
        print("You need a ticket to enter.")
else:
    print("You must be 18 or older.")
Trace 8.5.1
Nested conditionals — three cases
  • age=20, ticket=True20 >= 18 True → enter outer if. has_ticket True → inner if runs. Output: "Welcome. Enjoy the event."
  • age=20, ticket=False20 >= 18 True → enter outer if. has_ticket False → inner else runs. Output: "You need a ticket to enter."
  • age=15, ticket=True15 >= 18 False → outer else runs. Output: "You must be 18 or older." The inner if is never reached.
When outer condition is False, the entire inner structure is skipped. The inner if only becomes reachable once the outer condition passes.
Nested vs. Combined Conditions

When both conditions need to be True and there are no separate alternatives for each, combining with and is often cleaner: if age >= 18 and has_ticket:. Nesting is preferable when the inner condition has its own else branch that only makes sense in context of the outer condition passing. Avoid nesting more than two or three levels deep — deep nesting is difficult to read and often signals that the logic should be restructured.

Section 8.6

Repetition and Loops

Suppose you need to print the numbers from 1 to 10. One approach is to write ten separate print statements. That works for ten numbers, but it is completely impractical for a thousand, let alone a million. More importantly, it requires you to know the count in advance, write each case by hand, and change every line if the task changes.

Loops solve this problem by describing the pattern once and letting the computer repeat it. A loop is a control structure that repeats a block of instructions multiple times — either a fixed number of times, or until a condition changes. Every loop has two essential components: a loop body (the block of instructions to repeat) and a stopping mechanism (something that eventually causes the loop to end). If a loop never stops, it runs forever. Understanding how loops stop is just as important as understanding how they run.

Loop typeBest used whenStopping mechanism
While loopYou repeat until a condition changes; count unknown in advanceCondition becomes False
For loopYou iterate through every item in a sequence; count is knownSequence is exhausted
Section 8.7

While Loops

A while loop checks a condition before each iteration. If the condition is True, the loop body runs. Then the condition is checked again. This continues until the condition becomes False, at which point the loop ends and execution continues with the next instruction after the loop.

While Loop — Structure

while condition:

loop body

The loop body must include something that eventually makes the condition False. If nothing inside the loop changes the condition, the loop will run without stopping — an infinite loop.

Basic While Loop — Counting 1 to 5
count = 1

while count <= 5:
    print(count)
    count = count + 1

print("Done")

Output: 1 2 3 4 5 Done (each number on its own line, then "Done").

Notice that count = count + 1 is inside the loop body. This is essential: if count never changed, the condition count <= 5 would always be True, and the loop would never stop. Every while loop must include something in the body that moves the program toward the stopping condition.

While Loops for Input Validation

While loops are especially useful when you need to keep asking for input until the user provides a valid response. You do not know in advance how many attempts will be needed — which is exactly when a while loop is more appropriate than a for loop.

Input Validation with While Loop
password = input("Enter password: ")

while password != "open_sesame":
    print("Incorrect. Try again.")
    password = input("Enter password: ")

print("Access granted.")

Section 8.8

For Loops

A for loop repeats a block of code once for each item in a sequence. Unlike a while loop, the for loop automatically moves through each item in order. You do not need to manually track a counter variable or write a condition that eventually becomes False. The loop ends naturally when all items have been processed.

For Loop — Structure

for item in sequence:

loop body using item

Each iteration, item takes on the value of the next element in the sequence. When the sequence is exhausted, the loop ends automatically.

Iterating Over a Range

The range() function generates a sequence of integers. range(5) produces 0, 1, 2, 3, 4. range(1, 6) produces 1, 2, 3, 4, 5. The stop value is always excluded — the sequence goes up to, but does not include, the stop number.

For Loop with range()
for i in range(1, 6):
    print(i)

Iterating Over a List

For Loop Over a List
fruits = ["apple", "banana", "cherry"]

for fruit in fruits:
    print(fruit)

Output: apple, banana, cherry (each on its own line). Each iteration, fruit takes on the next value from the list.

Section 8.9

Counters and Accumulators

Two patterns appear so frequently in loops that every programmer should recognize them immediately: counters and accumulators. Both involve a variable that is set to a starting value before the loop begins and updated systematically inside the loop body.

Counters

A counter is a variable that increases by a fixed amount each iteration, keeping track of how many times some condition has been met.

Counter Pattern — Count Even Numbers
count = 0

for i in range(1, 11):
    if i % 2 == 0:
        count = count + 1

print("Even numbers between 1 and 10:", count)
The Modulo Operator %

The % operator returns the remainder after integer division. 7 % 3 is 1 (7 ÷ 3 = 2 remainder 1). 10 % 2 is 0 (10 divides evenly by 2). i % 2 == 0 is True when i is even, because even numbers leave no remainder when divided by 2. Modulo is used constantly in programming: checking divisibility, wrapping values around a range, and detecting periodic events.

Accumulators

An accumulator is a variable that collects a running total as the loop processes each item. The most common form is a sum accumulator, but accumulators can also concatenate strings, build lists, or track any value that grows with each iteration.

Accumulator Pattern — Running Sum
total = 0
numbers = [12, 7, 25, 3, 41, 18]

for n in numbers:
    total = total + n

print("Sum:", total)
Iterationntotal (before)total (after)

———Output: Sum: 106
Always Initialize Before the Loop

A counter or accumulator must be assigned a starting value before the loop begins. If total = 0 were placed inside the loop body instead of before it, total would be reset to 0 at the start of every iteration, destroying the running total. Initialization outside the loop is not optional — it is part of the pattern.

Combined Counter and Accumulator — Computing an Average
total = 0
count = 0
numbers = [15, 8, 22, 6, 30]

for n in numbers:
    total = total + n
    count = count + 1

average = total / count
print("Count:", count)
print("Total:", total)
print("Average:", average)

Output: Count: 5 / Total: 81 / Average: 16.2. This pattern — accumulating a sum and counting items to compute an average — appears constantly in data processing, science, finance, and everyday programs.

Section 8.10

Infinite Loops and Loop Errors

An infinite loop is a loop that never ends because its stopping condition never becomes False. It is one of the most common and frustrating bugs in programming. The program appears to freeze or hang — it is actually running the same instructions over and over without ever moving forward. Understanding what causes infinite loops, and how to prevent them, is an essential part of writing correct code.

Error 8.10.1
Infinite loop — loop variable not updated
  • Broken 
    count = 1
while count <= 5:
    print(count)
    # count is never changed — loop runs forever
  • ProblemThe condition count <= 5 is always True because count never changes. The program prints 1 indefinitely.
  • FixAdd count = count + 1 inside the loop body. Now count increases each iteration and eventually becomes 6, making the condition False.
Rule: every while loop must include something in the body that moves the program toward the stopping condition.

Off-By-One Errors

An off-by-one error occurs when a loop runs one time too many or one time too few because the boundary condition is written incorrectly. It is called "off-by-one" because the error is usually exactly one iteration in the wrong direction.

Off-by-One Error — range()
# Intended to print 1 through 5
for i in range(1, 5):    # Produces 1, 2, 3, 4 — misses 5
    print(i)

# Correct version
for i in range(1, 6):    # Produces 1, 2, 3, 4, 5
    print(i)

range(1, 5) produces numbers from 1 up to but not including 5. The stop value is always excluded. This trips up many beginners: when you want to include the value 5 in your range, you must write range(1, 6). Similarly, in while loops, the difference between < and <= in the boundary condition determines whether the final value is included.

Debugging Loops

The most reliable method for debugging a loop is to trace it by hand. Take a small version of the data, trace the loop iteration by iteration in a table, and check whether the values change as expected. If the trace reveals unexpected behavior, the cause is usually one of: the wrong initial value, the wrong boundary condition (< vs. <=), or a missing or misplaced update to the loop variable.

You can also add print statements inside the loop temporarily to see what is happening at each step. These diagnostic prints reveal the state of your variables at every iteration. Remove them once the loop is working correctly.

Diagnostic Print Statements for Debugging
total = 0
for i in range(1, 6):
    print("i is:", i, "  total before:", total)
    total = total + i
    print("  total after:", total)

Chapter 8 — Back Matter

Chapter Summary

  • Sec 8.1Control flow is the order in which a program's instructions execute. Non-linear control flow — choosing between paths and repeating instructions — is required for all but the simplest programs. Tracing programs step by step is the fundamental skill for understanding control flow.
  • Sec 8.2Boolean expressions evaluate to True or False. Comparison operators (==, !=, <, >, <=, >=) compare values. Logical operators (and, or, not) combine conditions. The == comparison operator must not be confused with the = assignment operator.
  • Sec 8.3An if statement executes a block of code when its condition is True and skips that block when the condition is False. Indentation defines which lines belong to the block. Indentation in Python is grammatically required, not optional.
  • Sec 8.4Else provides a second path when the if condition is False. Elif adds additional conditions to check in a chain. In any if-elif-else structure, exactly one branch runs. Once a True condition is found, remaining branches are skipped.
  • Sec 8.5Nested conditionals place one if statement inside another. The inner block is only reachable when the outer condition is True. Nesting should be kept shallow for readability. Combined conditions with and are often cleaner when both conditions must hold without separate alternatives.
  • Sec 8.6Loops repeat a block of instructions without requiring the programmer to write each repetition manually. Every loop has a loop body (the instructions to repeat) and a stopping mechanism. Infinite loops result from missing or broken stopping mechanisms.
  • Sec 8.7A while loop checks its condition before each iteration and runs the body as long as the condition is True. The body must include something that eventually makes the condition False. While loops are best when the number of iterations is not known in advance.
  • Sec 8.8A for loop iterates once for each item in a sequence, automatically advancing through each element. The range() function generates integer sequences. The stop value is excluded. For loops are best for counted repetition and processing every item in a known sequence.
  • Sec 8.9Counters track how many times a condition is met. Accumulators collect running totals. Both must be initialized before the loop begins. The % modulo operator returns the remainder after division and is commonly used to check for even numbers and periodicity.
  • Sec 8.10Infinite loops are caused by forgetting to update the loop variable. Off-by-one errors are caused by incorrect boundary conditions in range() or while comparisons. Uninitialized variables cause NameErrors. Tracing by hand and using diagnostic print statements are the primary debugging strategies.

Key Terms Review

accumulator
A variable that collects a running total across loop iterations. Must be initialized to a starting value before the loop begins.
Boolean expression
An expression that evaluates to exactly True or False. Used as conditions in if statements and loops.
branch
One of the possible paths through a conditional structure. Only one branch executes per evaluation of the chain.
comparison operator
A symbol that compares two values and returns True or False: ==, !=, <, >, <=, >=.
condition
A Boolean expression used to control which path a program takes, or whether a loop continues running.
conditional
A control structure that chooses between paths based on a Boolean condition. Includes if, if-else, and if-elif-else.
control flow
The order in which a program's instructions execute. Changed by conditionals, loops, and function calls.
counter
A variable that increases by a fixed amount each loop iteration to count how many times a condition is met.
elif
"Else if." A clause that adds another condition to check when all preceding conditions were False. Runs at most once per chain evaluation.
else
A clause that runs when all preceding conditions in an if-elif-else chain are False.
for loop
A loop that iterates once for each item in a sequence, automatically moving to the next item each iteration.
if statement
A conditional structure that executes a block of code when its Boolean condition is True and skips the block when False.
indentation
Whitespace at the start of a line. Python uses indentation to define which lines belong to the same block. Required in if statements and loops.
infinite loop
A loop whose condition never becomes False, causing the program to run without stopping.
initialization
Assigning a starting value to a variable before it is used, especially before a counter or accumulator enters a loop.
iteration
One complete pass through a loop body.
logical operator
An operator that combines Boolean expressions: and, or, and not.
loop
A control structure that repeats a block of instructions multiple times.
loop body
The indented block of instructions that a loop repeats each iteration.
modulo operator
The % operator. Returns the remainder after integer division. 7 % 3 equals 1.
nesting
Placing one control structure inside another — for example, an if inside an if, or a loop inside a conditional.
off-by-one error
A loop error where the loop runs one iteration too many or too few due to an incorrect boundary condition.
range()
A Python function that generates a sequence of integers. The stop value is excluded. range(1, 6) produces 1, 2, 3, 4, 5.
tracing
Following a program's execution step by step in a table to predict its output and verify its behavior. An essential debugging skill.
while loop
A loop that repeats its body as long as a condition is True. Stops when the condition becomes False. The body must include something that eventually changes the condition.

Looking Ahead
Chapter 9: Functions, Decomposition, and Reuse

Chapter 8 gave you the tools to make programs that decide and repeat. As programs grow larger, writing all the instructions in one block becomes unmanageable — the same logic gets repeated in multiple places, and the code becomes difficult to understand, test, and fix. Chapter 9 introduces functions: named, reusable blocks of code that solve a specific part of a problem. Functions let you write an instruction once, give it a name, and call it as many times as needed from anywhere in a program. You will learn to define functions, pass values into them using parameters, and receive results using return values. Functions are the foundation of decomposition in programming — breaking a large, complex problem into small, clearly named, independently tested parts. Every skill you built in Chapter 8 will be used inside functions in Chapter 9 and beyond.
