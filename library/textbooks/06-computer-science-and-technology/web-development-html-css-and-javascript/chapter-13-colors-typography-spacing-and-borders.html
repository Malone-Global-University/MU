Chapter 13 — Front Matter

Chapter Introduction and Learning Objectives

Chapter 12 taught you how CSS selects elements and how the cascade decides which rule wins when several rules apply. Now you have selectors that reliably reach the right elements. Chapter 13 teaches you what to do once you get there: how to control color, typography, spacing, and borders in ways that make pages readable, clear, and visually organized.

These are the foundational visual properties of CSS. Every page you build will use them. Every layout problem you encounter later — in Flexbox, in Grid, in responsive design — will assume you already understand how margins, padding, borders, and typography behave. Getting comfortable with these properties now will make everything that follows easier.

This chapter also introduces a distinction that runs through all of CSS: the difference between properties that affect how an element looks and properties that affect how much space it occupies. Typography mostly affects appearance. Spacing affects layout. Borders affect both. Learning to tell these apart will help you read CSS more clearly and debug it more effectively.

Opening Question

Two pages can contain the exact same HTML and yet feel completely different to read — one cramped and exhausting, the other calm and inviting. Nothing about the content changed. What changed is color, type, spacing, and borders: the four properties this chapter teaches. By the end of the chapter, you will be able to look at any page and explain, property by property, why it feels the way it does — and you will be able to build that feeling yourself.

Color: Formats, Properties, and Accessibility

Section 13.1

Visual Design Begins with Readability

Before learning individual CSS properties, it helps to understand what those properties are for. Visual design in web development is not decoration applied after the content is ready. It is the craft of making content readable, navigable, and usable.

Every property in this chapter is a tool for one of four goals. Keep these in mind as you read — they explain why a property exists, not just what it does.

Goal 1
Readability
Text that is easy to read. Size, line height, font choice, and color all affect how effortless reading feels.
Goal 2
Contrast
Important elements stand out from each other and from the background. Headings should look different from body text; links should look different from plain text.
Goal 3
Spacing
Elements have room to breathe. Crowded text is harder to scan; generous spacing creates clarity and organization.
Goal 4
Hierarchy
Visual weight reflects importance. A page title should carry more weight than a subheading, which should carry more weight than a paragraph.
Keep This in Mind

The goal of styling is never to make a page look impressive for its own sake. The goal is to serve the reader. Every property in this chapter should be judged against these four goals.

Section 13.2

Color Formats in CSS

CSS provides several ways to specify color. Each format describes the same set of colors but expresses them differently. You will see all of these in real CSS, so you need to be able to read and write all of them.

Named Colors

CSS recognizes a set of named colors written as plain words. CSS has 140 named colors, including common ones like red, blue, and gray, and less common ones like rebeccapurple and papayawhip. Named colors are easy to read and useful for quick experiments, but they are too limited and imprecise for serious design work.

CSS
p { color: black; } h1 { color: navy; } body { background-color: white; }

Hexadecimal Color Values

Hexadecimal values, usually called hex values, are the most common color format in web CSS. A hex value begins with # followed by six characters — three pairs for red, green, and blue. Each character is a digit from 0–9 or a letter from A–F. 00 means none of that channel; ff means the maximum.

CSS
h1 { color: #2c3e50; } p { color: #444444; } body { background-color: #ffffff; }

When both characters in each pair match, you can shorten the value to three characters: #ffffff becomes #fff, and #444444 becomes #444. This shorthand is common in real stylesheets.

RGB Values

RGB values express color as three numbers — red, green, and blue — each ranging from 0 to 255. RGB is mathematically identical to hex; rgb(255, 0, 0) is the same color as #ff0000.

CSS
h1 { color: rgb(44, 62, 80); } .overlay { background-color: rgba(0, 0, 0, 0.5); } .notice { background-color: rgba(255, 200, 0, 0.15); }

The rgba() format adds a fourth value, the alpha channel, ranging from 0 (fully transparent) to 1 (fully opaque). This is frequently used for overlays, shadows, and subtle background tints.

HSL Values

HSL stands for Hue, Saturation, and Lightness — a way to describe color that many designers find more intuitive. Hue is an angle on a color wheel from 0 to 360 (0 and 360 are both red; 120 is green; 240 is blue). Saturation runs from 0% (gray) to 100% (vivid). Lightness runs from 0% (black) to 100% (white), with 50% giving the purest version of the color.

CSS
h1 { color: hsl(210, 29%, 24%); } a { color: hsl(207, 90%, 54%); } .card { background-color: hsl(0, 0%, 97%); }

HSL makes it easy to create color variations: increase lightness for a lighter version, decrease saturation for a more muted one. Like RGB, HSL has an alpha variant: hsla(207, 90%, 54%, 0.8).

FormatExampleBest Use
Namednavy, whiteQuick experiments and prototypes
Hex#2c3e50The most common format; compact and easy to copy from tools
RGB / rgba()rgb(44, 62, 80)When you need transparency or readable numeric channels
HSL / hsla()hsl(210, 29%, 24%)Building systematic color scales and variations
Which Format Should You Use?

For this course, you will mostly use hex values and named colors. Hex values are compact, widely used, and easy to copy from design tools, color pickers, and documentation. As you build CSS variable systems in Chapter 18, HSL may become more useful because its logic makes systematic color scales easier to build.

Section 13.3

The color and background-color Properties

The two most fundamental color properties are color and background-color. color sets the foreground color of an element — for text elements, the color of the text itself, including text decorations such as underlines. background-color sets the background color of the element's box. By default, element backgrounds are transparent, so the parent's background shows through.

CSS
body { background-color: #f8f8f8; color: #333333; } h1 { color: #1a1a2e; } .notice { background-color: #fff8e1; color: #5d4037; }

Color Inheritance

The color property is inherited. If you set color on body, all text inside the page inherits that color unless a more specific rule overrides it. This is why setting color and background-color on body is a common, efficient pattern for establishing base page colors.

background-color is not inherited. Child elements default to a transparent background, allowing the parent's background to show through. If you want a child element to have a different background, you must set it explicitly.

Color Contrast for Accessibility

Color contrast is a critical accessibility requirement, not a design preference. When text lacks sufficient contrast against its background, many users cannot read it — including users with low vision, older users, users in bright sunlight, and users on lower-quality screens.

Accessibility Requirement

The Web Content Accessibility Guidelines (WCAG) define minimum contrast ratios: 4.5 to 1 for normal text and 3 to 1 for large text. Free online tools can calculate the contrast ratio between two colors. Any text-and-background combination you use should pass these minimums before you ship it.

CSS
/* Low contrast — difficult to read */ .low-contrast { background-color: #f0f0f0; color: #cccccc; } /* High contrast — easy to read */ .high-contrast { background-color: #ffffff; color: #333333; }

The first rule places light gray text on a light gray background — the contrast ratio is far too low for comfortable reading. The second places dark gray text on white, providing strong contrast. Bright color combinations can fail contrast checks too: bright yellow text on white looks vivid but is extremely difficult to read. Always check your contrast ratio with a tool before finalizing a color scheme.

Sections 13.4 – 13.8
Typography: Fonts, Size, Weight, and Alignment
Font Stacks · px / em / rem · Line Height · Font Weight · Alignment · Decoration
Section 13.4

Font Families

The font-family property controls which typeface is used for text. On the web, fonts work differently than in a word processor: the browser renders text using fonts installed on the user's operating system, or fonts downloaded from a web font service.

Font Stacks

Because you cannot guarantee which fonts a user has installed, font-family accepts a comma-separated list of font names called a font stack. The browser uses the first font in the list that it can find.

CSS
body { font-family: Georgia, "Times New Roman", Times, serif; } h1 { font-family: "Helvetica Neue", Helvetica, Arial, sans-serif; } code { font-family: "Courier New", Courier, monospace; }

The last value in a font stack should always be a generic family name: serif, sans-serif, monospace, cursive, or fantasy. These are keywords, not specific fonts — they tell the browser which category to fall back to if none of the named fonts are available. Serif fonts have small decorative strokes at the ends of letterforms (Georgia, Times New Roman). Sans-serif fonts have none and tend to look cleaner on screens (Helvetica, Arial). Monospace fonts give every character equal width, which matters for code and terminals.

System Font Stacks

Modern CSS often uses a system font stack that tells the browser to use the operating system's default interface font, avoiding slow font loading and providing a native appearance.

CSS
body { font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Oxygen, Ubuntu, Cantarell, "Open Sans", "Helvetica Neue", sans-serif; }

Each entry is a fallback for a specific operating system: macOS uses San Francisco, Windows uses Segoe UI, Android uses Roboto, Linux uses Ubuntu or Cantarell. If none are found, the browser falls back to a generic sans-serif. System font stacks are widely used where fast loading and native appearance matter more than a specific designed typeface.

Web Fonts

Web fonts are font files loaded from a server — your own, or a service such as Google Fonts. Web fonts give you access to any typeface, but add a network request and a small loading cost.

CSS
@import url('https://fonts.googleapis.com/css2?family=Inter:wght@400;600;700&display=swap'); body { font-family: "Inter", sans-serif; }
A Habit Worth Building Now

You may use system fonts or web fonts freely in this course. The important habit to build now is always ending your font stack with a generic family name.

Section 13.5

Font Size

The font-size property controls how large text appears. Font size can be specified in several units — most commonly pixels (px), em, and rem.

Pixels

Pixels are absolute units. font-size: 16px means the text renders at 16 pixels tall. The typical base size for body text is 16px — the browser default for most elements and a comfortable reading size on most screens. Going below 14px makes reading uncomfortable for many users.

CSS
body { font-size: 16px; } h1 { font-size: 36px; } h2 { font-size: 28px; } small { font-size: 13px; }

em Units

The em unit is relative to the current element's font size. If body font size is 16px, 1em is 16px and 1.5em is 24px. The problem with em is that nested elements compound: a list item inside a container that already has a reduced font size inherits that smaller size, and then em values calculate relative to it again, producing unexpected results.

CSS
body { font-size: 16px; } h1 { font-size: 2.25em; /* 36px when body is 16px */ } small { font-size: 0.875em; /* 14px */ }

rem Units

rem stands for root em. It is relative to the font size of the root <html> element, not the current element. Because rem always refers to the same base size, it does not compound through nesting.

CSS
html { font-size: 16px; } h1 { font-size: 2.25rem; /* always 36px, regardless of nesting */ } p { font-size: 1rem; /* always 16px */ }
UnitBased OnCompounds When Nested?
pxA fixed pixel valueNo — always absolute
emThe current element's font sizeYes — can compound unexpectedly
remThe root html element's font sizeNo — predictable everywhere
Why rem Is Recommended

rem is widely recommended for font sizes in modern CSS because it is predictable and easy to reason about. When you learn CSS custom properties in Chapter 18, you will use rem as the unit for a consistent typographic scale. For this chapter, practice with both px and rem — use px to understand exact pixel sizes directly, and use rem to begin building good habits for scalable typography.

Section 13.6

Line Height

Line height controls the vertical space between lines of text within a paragraph, using the line-height property. Adequate line height dramatically improves readability, especially for longer passages.

CSS
/* Too tight — lines feel cramped */ .cramped { font-size: 16px; line-height: 1; } /* Comfortable — standard for body text */ .comfortable { font-size: 16px; line-height: 1.6; } /* Too loose — looks disconnected */ .loose { font-size: 16px; line-height: 2.5; }

Notice that line-height is set without a unit. A unitless value is a multiplier based on the current font size — line-height: 1.6 means the line height is 1.6 times the font size (25.6px on 16px text). Using a unitless multiplier is recommended because it scales proportionally when font size changes; a fixed pixel value will not scale correctly when the font size differs.

Recommended Line Heights

For body text, a line height between 1.4 and 1.7 is generally comfortable — 1.5 and 1.6 are common defaults. For headings, a line height between 1.1 and 1.3 is usually appropriate, since large text already has letters spaced farther apart and needs less extra space between lines.

CSS
body { font-size: 16px; line-height: 1.6; } h1, h2, h3 { line-height: 1.2; }

This is a widely used pattern: generous line height for readability in body text, and tighter line height for headings, where extra space between lines feels excessive at large sizes.

Section 13.7

Font Weight

Font weight controls how thick or thin letterforms appear. The font-weight property accepts keyword values (normal, bold) and numeric values from 100 to 900 in increments of 100. normal is equivalent to 400 and bold is equivalent to 700.

CSS
p { font-weight: normal; } strong { font-weight: bold; } h1 { font-weight: 700; } .medium-label { font-weight: 500; }
A Limitation Worth Knowing

A font must actually include the weight you request. If you specify font-weight: 300 and the font has no 300 variant, the browser substitutes the nearest available weight. Many system fonts only include 400 and 700. Google Fonts and other web font services often offer multiple weights.

WeightCommon Use
400 (normal)Standard body text
500 – 600 (medium)Subheadings, labels, UI text
700 (bold)Headings and strong emphasis
Section 13.8

Text Alignment and Text Decoration

Text Alignment

The text-align property controls the horizontal alignment of text within its container: left, right, center, and justify.

CSS
p { text-align: left; } h1 { text-align: center; } .price { text-align: right; }

Left alignment is the default in left-to-right languages and is the easiest text to read for most users. Center alignment suits short headings and hero text. Right alignment is useful in tables for numbers and prices. Justify stretches text so both edges align, which can help printed documents but often creates uneven spacing on screens, especially in narrow containers — use it sparingly. For body text in a left-to-right language, text-align: left is almost always the right choice for readability.

Text Decoration

The text-decoration property controls underlines, overlines, and strikethroughs.

CSS
a { text-decoration: underline; } .strikethrough { text-decoration: line-through; } a.button-link { text-decoration: none; }
A Caution About Removing Underlines

Browsers underline anchor elements by default. Removing the underline with text-decoration: none is common for navigation links and buttons styled in other ways. Be careful removing underlines from links inside body text — users expect text links to be underlined, and removing that visual cue can confuse them.

Section 13.9

Margins and Padding

Margins and padding are both spacing properties, but they act in different places. Understanding this distinction is fundamental to CSS layout.

The Key Distinction

Padding is the space between an element's content and its border — inside the box.

Margin is the space between an element's border and neighboring elements — outside the box.

Chapter 14 explores both in depth as part of the complete box model. For now, practice with the basic single-value syntax to build familiarity with how each property changes the appearance of elements.

Padding

CSS
p { padding: 16px; } .card { padding: 24px; } .tight-label { padding: 4px 8px; }

padding: 16px applies 16px of internal space to all four sides. padding: 4px 8px applies the first value (4px) to top and bottom and the second value (8px) to left and right.

Margin

CSS
h1 { margin-bottom: 8px; } p { margin-top: 0; margin-bottom: 16px; } .section { margin-top: 48px; margin-bottom: 48px; }

Individual sides can be set with margin-top, margin-right, margin-bottom, and margin-left — the preferred approach when you only want to control spacing on specific sides. A very common pattern is resetting paragraph top margin to zero and controlling spacing entirely through bottom margin, producing predictable, consistent vertical rhythm down the page.

Margin Auto for Centering

One special and important use of margin is centering a block element horizontally. Setting margin-left: auto and margin-right: auto on a block element with a defined width centers it within its container.

CSS
.container { width: 800px; margin-left: auto; margin-right: auto; }
A Shorthand Worth Memorizing

The shorthand margin: 0 auto sets top and bottom margins to zero and left and right margins to auto, centering the element horizontally. This is one of the most commonly used layout patterns in all of CSS.

Section 13.10

Borders and Border Radius

Borders draw a visible line around the outside of an element's padding area. The border shorthand property accepts three values: width, style, and color.

CSS
.box { border: 1px solid #cccccc; } .alert { border: 2px solid #e53935; } .notice { border: 1px dashed #888888; }
PartCommon ValuesNotes
Width1px, 2px, 3pxBorders of 4px or more look heavy unless used intentionally
Stylesolid, dashed, dotted, nonesolid and none are by far the most common
ColorAny CSS color valueOften subtle gray, or meaningful (red for errors, green for success)

Individual Border Sides

You can set borders on individual sides using border-top, border-right, border-bottom, and border-left.

CSS
.section-heading { border-bottom: 2px solid #1a1a2e; } .left-highlight { border-left: 4px solid #3498db; padding-left: 16px; }

A bottom border under a heading is a common way to create visual separation. A left border combined with left padding creates an indented callout style widely used for quotations and notices.

Border Radius

The border-radius property rounds the corners of an element's border box.

CSS
.card { border: 1px solid #e0e0e0; border-radius: 8px; } .pill-button { border-radius: 999px; } .avatar { border-radius: 50%; }

border-radius: 8px rounds all four corners by 8 pixels. border-radius: 50% creates a perfect circle on an element with equal width and height. border-radius: 999px creates a fully rounded pill shape on a wide rectangle — a common technique for badges and buttons.

Using Borders Without Visible Lines

CSS
.input { border: 1px solid #cccccc; } .input:focus { border: 1px solid #3498db; }

Here, an input field has a gray border by default; on focus, the border color changes to blue. Because both states define a 1px solid border, the layout does not shift when focus activates — only the color changes.

Section 13.11

Visual Hierarchy

Visual hierarchy is the arrangement of elements so their visual weight reflects their importance. Users scan pages before reading them. Visual hierarchy guides the eye, signals what matters most, and makes pages easier to navigate. CSS controls hierarchy through the properties you have learned in this chapter: size, weight, color, contrast, spacing, and borders.

CSS
body { font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif; font-size: 16px; line-height: 1.6; color: #333333; background-color: #ffffff; margin: 0; padding: 0; } h1 { font-size: 2rem; font-weight: 700; color: #1a1a2e; line-height: 1.2; margin-top: 0; margin-bottom: 12px; } h2 { font-size: 1.4rem; font-weight: 600; color: #2c3e50; line-height: 1.3; margin-top: 40px; margin-bottom: 8px; } p { margin-top: 0; margin-bottom: 20px; max-width: 680px; } a { color: #2980b9; text-decoration: underline; } a:hover { color: #1a5276; }
Reading the Decisions
Every rule above is a deliberate choice
  • bodySets the visual foundation: a readable font, 16px size, generous line height, dark gray text on white, and no default margins.
  • h1Large, heavy, and dark — the most visually important element on the page.
  • h2Noticeably smaller than h1 but still distinctly larger and heavier than body text. Extra top margin separates it from content above, creating clear section breaks.
  • pNo top margin and a consistent bottom margin, creating even vertical rhythm down the page.
  • a / a:hoverA recognizable link color with an underline. The hover state darkens the color, giving visual feedback when a user points at the link.

Together, these rules create a clear visual hierarchy without any complex layout, shadows, or decorative elements. Hierarchy is a product of disciplined, consistent decisions — not visual flourishes.

Work through the stylesheet rule by rule and answer these questions for each one: What element or class does this rule target? What visual property does it control? Every rule is a decision — understanding why decisions are made is how you develop design judgment.

  • Section 13.3color sets foreground text color and is inherited. background-color sets background color and is not inherited. Text and background combinations must meet WCAG contrast minimums.
  • Section 13.4Font stacks list fallback fonts ending in a generic family name. System font stacks use the operating system's native font; web fonts load custom typefaces from a server.
  • Section 13.5px is absolute; em is relative to the current element and can compound when nested; rem is relative to the root element and does not compound. rem is recommended for scalable typography.
  • Section 13.6Unitless line-height values scale with font size. 1.4–1.7 suits body text; 1.1–1.3 suits large headings.
  • Section 13.7font-weight ranges from 100 to 900; 400 is normal and 700 is bold. A font must include a weight for it to render distinctly.
  • Section 13.8text-align controls horizontal alignment; left alignment is best for body text readability. text-decoration controls underlines and strikethroughs; removing link underlines requires care.
  • Section 13.9Padding is space inside the border; margin is space outside it. margin: 0 auto centers a block element with a defined width.
  • Section 13.10The border shorthand sets width, style, and color. border-radius rounds corners; 50% creates a circle and large values create pills.
  • Section 13.11Visual hierarchy emerges from deliberate, consistent combinations of size, weight, color, contrast, spacing, and borders — not from decoration.
  • Section 13.12A complete worked example shows color, typography, spacing, and borders working together across a real page. Reading CSS rule by rule, and asking why each exists, builds design judgment.

Key Terms

color contrast
The difference in luminance between a foreground color and its background. Sufficient contrast is required for text to be readable by users with low vision or in difficult viewing conditions.
font stack
A comma-separated list of font family names in a font-family declaration. The browser uses the first available font on the user's system.
generic family name
A keyword at the end of a font stack telling the browser which type category to use as a final fallback: serif, sans-serif, monospace, cursive, or fantasy.
hex value
A color expressed as a six-character hexadecimal code preceded by #. The characters represent red, green, and blue channel values from 00 to ff.
HSL
A color model using Hue, Saturation, and Lightness. Useful for creating predictable color variations.
line height
The vertical space occupied by a line of text. Unitless values like 1.6 are multiples of the current font size.
margin
Space outside an element's border, separating it from neighboring elements. Can be set per side or with a shorthand.
padding
Space inside an element between its content and its border. Padding adds to the visual size of the element's background area.
RGB
A color model using Red, Green, and Blue values from 0 to 255. rgba() adds an alpha transparency value from 0 to 1.
rem
Root em. A relative unit equal to the font size of the root html element. Does not compound through nesting.
visual hierarchy
The arrangement of visual elements so that their weight, size, or prominence corresponds to their importance in the content.
font-weight
A CSS property controlling the thickness of text characters. Numeric values range from 100 to 900; 400 is normal and 700 is bold.
