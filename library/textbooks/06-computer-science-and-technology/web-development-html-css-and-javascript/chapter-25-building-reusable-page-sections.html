Chapter 25 — Front Matter

Chapter Introduction and Learning Objectives

Up to this point, you have learned how to build individual HTML pages, apply CSS styling, create responsive layouts with Flexbox and Grid, and write JavaScript that responds to user actions and updates the DOM. You have the fundamental tools. Now you begin learning how to think like a website builder rather than a page builder.

Real websites are not single pages. They are systems of pages that share a consistent visual language, repeating structural patterns, and reusable building blocks. A well-built website uses the same card style in ten different places. It uses the same button style throughout. Its hero section follows a pattern that could appear on multiple pages with different content. Its spacing, colors, and typography come from a shared design vocabulary rather than being invented fresh for each page.

This chapter teaches you how to design and build those reusable building blocks: page sections. You will learn how to identify sections as patterns, how to write HTML that makes sections reusable, how to write CSS that makes section styles consistent and easy to maintain, and how to assemble a complete page from these sections the way a professional web developer would.

Opening Idea

By the end of this chapter, you will be able to look at any well-built website and recognize its section structure. You will also be able to build a page from your own reusable components rather than styling every element from scratch each time — the same mental model used by modern component-based frameworks like React and Vue, built here with plain HTML and CSS.

Section 25.1

Websites Are Built from Sections

Open almost any website and look carefully at its structure. Beneath the visual design, you will notice the same fundamental pattern: the page is divided into horizontal bands of content, each serving a clear purpose.

A typical homepage might have a large banner at the top with a headline and a call-to-action button, a row of feature cards explaining what the site offers, a section with a bold statement and a supporting description, another row of cards with images and short descriptions, a strip with contact information or a newsletter signup, and a footer with links and copyright information.

Each of these bands is a section: a self-contained unit of content and layout that can be designed once and reused with different content wherever it is needed.

Reason 1
Sections Match How Users Read
Research shows users scan pages top to bottom, pausing at visual landmarks. Sections create those landmarks.
Reason 2
Sections Make Design Consistent
When every card follows the same pattern, the page looks intentional. When every hero follows the same structure, pages feel unified.
Reason 3
Sections Make Development Efficient
Writing a .card style once and applying it in fifty places is far better than writing fifty slightly different card styles.
Reason 4
Sections Make Maintenance Manageable
When a design needs to change, you change one set of CSS rules and the change propagates everywhere that style is used.
Reason 5
Sections Prepare You for Frameworks
Modern frameworks like React and Vue are built almost entirely around reusable components — exactly the mental model taught in this chapter.

The Section as a Unit of Design

Think of a section as having three layers: a container that controls the section's maximum width and horizontal centering on the page, a section wrapper that controls the section's background color, padding, and vertical spacing, and section content that holds the actual text, images, cards, or other elements.

This three-layer structure appears so consistently across professional websites that it is worth memorizing before you write a single line of code.

HTML — the three-layer pattern
<!-- The three-layer pattern -->
<section class="hero">               <!-- Layer 2: background, padding, spacing -->
  <div class="container">            <!-- Layer 3: max-width, centering -->
    <h1>Welcome to My Site</h1>
    <p>A place for learning.</p>
    <a href="lessons.html" class="btn btn-primary">Start Learning</a>
  </div>
</section>
CSS — layers 2 and 3
/* Layer 2: controls the section's visual treatment */
.hero {
  background-color: var(--color-primary);
  padding: var(--space-xl) var(--space-md);
}

/* Layer 3: controls content width and centering */
.container {
  max-width: 1100px;
  margin-inline: auto;
}
Study Note

Notice that the .container class appears inside many different sections. It is a reusable utility: it does not know or care what section it is inside. It simply centers content and limits its width. You will write many sections in this chapter, and every one of them will follow this pattern.

Section 25.2

The Container Pattern

Before you can build any section, you need to understand the container — the single most reused CSS pattern in professional web development.

Without a container, your content stretches the full width of the browser window. On a wide monitor, a paragraph of text might span 1600 pixels across the screen. That is nearly unreadable; comfortable reading line length is roughly 55–75 characters, which in practice means content should usually be no wider than about 700–900 pixels for running text, or up to 1200 pixels or so for wider layouts with multiple columns.

The container fixes this by limiting width and centering the content.

CSS — the container
.container {
  max-width: 1100px;
  margin-inline: auto;
  padding-inline: var(--space-md);
}
Breaking Down Each Declaration
What every line of the container does
  • max-widthThe content will grow to fill the available space, but never exceed 1100 pixels wide. On small screens, where the viewport is narrower than 1100px, the content takes the full width naturally. On large screens, it caps at 1100px.
  • margin-inlineSetting this to auto produces equal left and right margins automatically, centering the container horizontally inside its parent. margin-inline is a modern shorthand for margin-left: auto; margin-right: auto.
  • padding-inlineAdds horizontal padding so content does not press against the screen edges on small screens. Without this, on a narrow phone screen, your text would touch the left and right edges.

Variations on the Container

You may sometimes want a narrower container for reading-focused content and a wider one for full-width layouts.

CSS — container variations
.container {
  max-width: 1100px;
  margin-inline: auto;
  padding-inline: var(--space-md);
}

.container-narrow {
  max-width: 720px;
  margin-inline: auto;
  padding-inline: var(--space-md);
}

.container-wide {
  max-width: 1400px;
  margin-inline: auto;
  padding-inline: var(--space-md);
}

For this chapter's projects and exercises, you will primarily use .container with a max-width of around 1100px.

The Container Does Not Control Background

An important principle: the container controls content width, not background color. If you want a section to have a colored background that stretches full-width across the screen, you apply the background color to the section element, not the container. The container sits inside the section and keeps the content centered.

HTML — correct placement of background
<!-- CORRECT: background on section, container inside -->
<section class="highlight-band">
  <div class="container">
    <h2>Our Mission</h2>
    <p>We help students learn to build for the web.</p>
  </div>
</section>
CSS
.highlight-band {
  background-color: var(--color-accent-light);
  padding-block: var(--space-xl);
}
Why This Matters

This gives you a full-width colored band with centered content inside — a pattern you will see on nearly every professional website.

Section 25.3

Hero Sections

The hero section is the first large section at the top of a page, designed to orient the visitor, convey the site's purpose, and invite action. It typically appears before the navigation on marketing pages, or just after the navigation on informational sites.

A basic hero section contains a headline (the primary message), a subheading or short description (supporting context), one or two call-to-action buttons (the primary action you want visitors to take), and optionally an image or illustration.

HTML — a basic hero section
<section class="hero">
  <div class="container">
    <div class="hero-content">
      <h1 class="hero-title">Learn Web Development from the Ground Up</h1>
      <p class="hero-description">
        Structured lessons, hands-on projects, and everything you need
        to build real websites with HTML, CSS, and JavaScript.
      </p>
      <div class="hero-actions">
        <a href="lessons.html" class="btn btn-primary">Start Learning</a>
        <a href="about.html" class="btn btn-secondary">About This Site</a>
      </div>
    </div>
  </div>
</section>
CSS — hero styles
/* Hero section */
.hero {
  background-color: var(--color-primary-dark);
  color: var(--color-text-on-dark);
  padding-block: var(--space-2xl);
  text-align: center;
}

.hero-title {
  font-size: var(--text-4xl);
  font-weight: 700;
  line-height: 1.15;
  margin-bottom: var(--space-md);
}

.hero-description {
  font-size: var(--text-lg);
  line-height: 1.7;
  max-width: 640px;
  margin-inline: auto;
  margin-bottom: var(--space-lg);
  color: var(--color-text-on-dark-muted);
}

.hero-actions {
  display: flex;
  gap: var(--space-sm);
  justify-content: center;
  flex-wrap: wrap;
}

Analyzing the Hero Design

Deliberate Decisions
Why this CSS is written the way it is
  • Description widthEven though .container limits the overall width, long lines of descriptive text are harder to read. Giving .hero-description a max-width: 640px keeps it comfortably readable without affecting the headline.
  • Wrapping buttons.hero-actions uses Flexbox with flex-wrap: wrap. On narrow screens, if there is not enough room for both buttons side by side, they will wrap to a second line gracefully.
  • CSS variablesThe text uses CSS variables throughout, which makes the hero section easy to retheme. Changing one variable value updates the entire hero.
  • padding-blockThe hero uses padding-block rather than padding. This modern shorthand sets padding-top and padding-bottom together, separating vertical spacing decisions from horizontal ones, which pairs consistently with padding-inline on containers.

Two-Column Hero

A common variation places text on the left and an image on the right in a two-column layout.

HTML — two-column hero
<section class="hero">
  <div class="container">
    <div class="hero-layout">
      <div class="hero-content">
        <h1 class="hero-title">Learn Web Development from the Ground Up</h1>
        <p class="hero-description">
          Structured lessons, hands-on projects, and everything you need
          to build real websites with HTML, CSS, and JavaScript.
        </p>
        <div class="hero-actions">
          <a href="lessons.html" class="btn btn-primary">Start Learning</a>
        </div>
      </div>
      <div class="hero-image">
        <img src="images/hero-illustration.png" alt="Student working at a computer" />
      </div>
    </div>
  </div>
</section>
CSS — two-column layout and mobile stacking
.hero-layout {
  display: grid;
  grid-template-columns: 1fr 1fr;
  gap: var(--space-xl);
  align-items: center;
}

/* On small screens, stack the columns */
@media (max-width: 700px) {
  .hero-layout {
    grid-template-columns: 1fr;
  }

  .hero-image {
    order: -1; /* Move image above text on mobile */
  }
}
Key Idea

This two-column hero uses CSS Grid for layout and includes a responsive breakpoint so that on small screens the two columns collapse into a single vertical stack.

Section 25.4

Content Sections

Not every section of a page is a hero. Most sections carry supporting information: explanations, feature lists, descriptions, quotes, or calls to action. These content sections are the workhorses of a page.

A basic content section has a heading, perhaps a subheading, and supporting text.

HTML — a basic content section
<section class="content-section">
  <div class="container">
    <div class="section-header">
      <h2>What You Will Learn</h2>
      <p class="section-subtitle">
        This course covers everything from your first HTML page to
        publishing a complete multi-page website.
      </p>
    </div>
    <div class="section-body">
      <p>
        You will begin with how the web works, then move through HTML structure,
        CSS styling and layout, JavaScript interaction, multi-page projects,
        and finally accessibility, debugging, and deployment.
      </p>
      <p>
        Each part builds on the last. By the end, you will have built a real
        website you can share with others.
      </p>
    </div>
  </div>
</section>
CSS — content section styles
.content-section {
  padding-block: var(--space-xl);
}

.section-header {
  text-align: center;
  margin-bottom: var(--space-lg);
}

.section-header h2 {
  font-size: var(--text-3xl);
  font-weight: 700;
  margin-bottom: var(--space-sm);
}

.section-subtitle {
  font-size: var(--text-lg);
  color: var(--color-text-muted);
  max-width: 600px;
  margin-inline: auto;
  line-height: 1.7;
}

.section-body {
  max-width: 720px;
  margin-inline: auto;
}

.section-body p {
  margin-bottom: var(--space-md);
  line-height: 1.8;
}

Alternating Section Backgrounds

Pages often alternate between white and light-gray backgrounds to visually separate sections without harsh dividers.

CSS — alternating backgrounds
.content-section {
  padding-block: var(--space-xl);
  background-color: var(--color-bg);
}

.content-section-alt {
  padding-block: var(--space-xl);
  background-color: var(--color-bg-alt);
}
Key Idea

By giving some sections the .content-section-alt class, you create visual rhythm on the page without adding borders or extra markup.

Section 25.5

Cards

Cards are one of the most commonly used patterns in web development. A card is a self-contained visual unit that groups related information — typically an image or icon, a title, a short description, and optionally a link or button.

Cards appear in product listings, article previews, team member profiles, course overviews, feature explanations, and resource directories.

The key property of a card is that it is self-contained. Its styles do not depend on what cards surround it. You can put two cards in a row, three cards in a row, four cards in a grid, or one card alone — the card itself stays consistent.

HTML — a basic card
<div class="card">
  <img
    class="card-image"
    src="images/lesson-html.jpg"
    alt="Code showing HTML document structure"
  />
  <div class="card-body">
    <h3 class="card-title">HTML Structure</h3>
    <p class="card-description">
      Learn how HTML gives every web page its structure, meaning, and
      organization using elements, tags, and attributes.
    </p>
    <a href="lessons/html.html" class="card-link">Start Lesson →</a>
  </div>
</div>
CSS — basic card styles
.card {
  background-color: var(--color-bg-card);
  border-radius: var(--radius-md);
  box-shadow: var(--shadow-sm);
  overflow: hidden;
  display: flex;
  flex-direction: column;
  transition: box-shadow 0.2s ease;
}

.card:hover {
  box-shadow: var(--shadow-md);
}

.card-image {
  width: 100%;
  height: 180px;
  object-fit: cover;
  display: block;
}

.card-body {
  padding: var(--space-md);
  display: flex;
  flex-direction: column;
  flex: 1;
}

.card-title {
  font-size: var(--text-xl);
  font-weight: 600;
  margin-bottom: var(--space-xs);
}

.card-description {
  font-size: var(--text-base);
  color: var(--color-text-muted);
  line-height: 1.6;
  margin-bottom: var(--space-md);
  flex: 1;
}

.card-link {
  font-weight: 600;
  color: var(--color-primary);
  text-decoration: none;
  font-size: var(--text-sm);
  align-self: flex-start;
}

.card-link:hover {
  text-decoration: underline;
}
Why flex: 1 and flex-direction: column Matter in Cards

When you display multiple cards in a row, their heights may differ because their descriptions have different lengths. Without careful CSS, shorter cards would not align with taller ones.

The solution is to make the .card itself a flex column and give .card-description flex: 1. This tells the description to grow and fill whatever space is available. The result: all cards in a row stretch to the same height, with the link always sitting at the bottom. This is a small technique, but it produces significantly more polished results.

Placing Cards in a Grid

Cards are almost always placed inside a card grid section.

HTML — cards inside a grid
<section class="content-section">
  <div class="container">
    <div class="section-header">
      <h2>Course Lessons</h2>
      <p class="section-subtitle">
        Six topic areas, from HTML foundations to publishing your first website.
      </p>
    </div>
    <div class="card-grid">
      <div class="card"><!-- card content --></div>
      <div class="card"><!-- card content --></div>
      <div class="card"><!-- card content --></div>
      <div class="card"><!-- card content --></div>
      <div class="card"><!-- card content --></div>
      <div class="card"><!-- card content --></div>
    </div>
  </div>
</section>
CSS — responsive card grid
.card-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: var(--space-lg);
}

@media (max-width: 900px) {
  .card-grid {
    grid-template-columns: repeat(2, 1fr);
  }
}

@media (max-width: 580px) {
  .card-grid {
    grid-template-columns: 1fr;
  }
}
Key Idea

This three-column grid collapses to two columns on medium screens and one column on small screens. The cards themselves do not change; only their container grid adapts.

Section 25.6

Callouts and Notices

A callout is a visually distinct section of content that draws the reader's attention to something important. Callouts appear in documentation, tutorials, educational materials, and anywhere a note, warning, tip, or important message needs to stand out from surrounding content.

Common callout types are: Note (supplementary information), Tip (a helpful suggestion), Warning (something to be careful about), and Important (critical information the reader must not miss).

HTML — a tip callout
<div class="callout callout-tip">
  <div class="callout-icon" aria-hidden="true">💡</div>
  <div class="callout-content">
    <strong class="callout-label">Tip</strong>
    <p>
      You can test your responsive design directly in the browser.
      Open Developer Tools, click the device toolbar icon, and choose
      a screen size to preview.
    </p>
  </div>
</div>
CSS — callout base and type variations
.callout {
  display: flex;
  gap: var(--space-sm);
  padding: var(--space-md);
  border-radius: var(--radius-md);
  border-left: 4px solid transparent;
  margin-block: var(--space-lg);
}

.callout-icon {
  font-size: 1.5rem;
  flex-shrink: 0;
  line-height: 1;
  margin-top: 0.1em;
}

.callout-content {
  flex: 1;
}

.callout-label {
  display: block;
  font-size: var(--text-sm);
  font-weight: 700;
  text-transform: uppercase;
  letter-spacing: 0.05em;
  margin-bottom: var(--space-xs);
}

.callout p {
  margin: 0;
  font-size: var(--text-base);
  line-height: 1.6;
}

/* Callout type variations */
.callout-note {
  background-color: #e8f0fe;
  border-left-color: #1a73e8;
}

.callout-tip {
  background-color: #e6f4ea;
  border-left-color: #34a853;
}

.callout-warning {
  background-color: #fef7e0;
  border-left-color: #f9ab00;
}

.callout-important {
  background-color: #fce8e6;
  border-left-color: #ea4335;
}
Accessibility Note for Callouts

The aria-hidden="true" attribute on the icon element tells screen readers to skip it. The callout label ("Tip", "Warning") already communicates the callout type in text, so the decorative icon does not need to be announced. If you prefer not to use emoji icons, you can use a simple left-border callout without an icon — the left border alone communicates visual distinction effectively.

Section 25.7

Buttons and Links as Actions

Buttons and links that invite action — calls to action — appear throughout every professional website. A consistent button system makes a site feel designed rather than assembled.

A basic button system includes a primary button for the main action, a secondary button for an alternative action, a ghost button for low-emphasis actions, and a danger button for destructive actions (used in interactive applications).

HTML — button usage
<a href="lessons.html" class="btn btn-primary">Start Learning</a>
<a href="about.html" class="btn btn-secondary">Learn More</a>
<button type="button" class="btn btn-ghost">Cancel</button>
CSS — base and variant button styles
/* Base button styles */
.btn {
  display: inline-flex;
  align-items: center;
  justify-content: center;
  gap: var(--space-xs);
  padding: 0.6em 1.4em;
  font-size: var(--text-base);
  font-weight: 600;
  font-family: inherit;
  line-height: 1;
  border-radius: var(--radius-sm);
  border: 2px solid transparent;
  text-decoration: none;
  cursor: pointer;
  transition: background-color 0.15s ease, border-color 0.15s ease, color 0.15s ease;
}

/* Focus styles — critical for keyboard accessibility */
.btn:focus-visible {
  outline: 3px solid var(--color-focus-ring);
  outline-offset: 2px;
}

/* Primary button */
.btn-primary {
  background-color: var(--color-primary);
  color: var(--color-text-on-primary);
  border-color: var(--color-primary);
}

.btn-primary:hover {
  background-color: var(--color-primary-dark);
  border-color: var(--color-primary-dark);
}

/* Secondary button */
.btn-secondary {
  background-color: transparent;
  color: var(--color-primary);
  border-color: var(--color-primary);
}

.btn-secondary:hover {
  background-color: var(--color-primary);
  color: var(--color-text-on-primary);
}

/* Ghost button */
.btn-ghost {
  background-color: transparent;
  color: var(--color-text);
  border-color: var(--color-border);
}

.btn-ghost:hover {
  background-color: var(--color-bg-alt);
}

Button Accessibility Requirements

Rule 1
Anchor vs. Button
Use <a> for navigation, <button> for actions. If clicking opens a new page, use an anchor. If clicking performs an action on the current page, use a button element. Never use a div or span as a button.
Rule 2
Visible Focus Styles
Always include :focus-visible styles. Keyboard users navigate with the Tab key and need to see clearly which element is focused. :focus-visible applies focus styles only for keyboard navigation, not mouse clicks.
Rule 3
Descriptive Link Text
Avoid "Click here" as link text. Screen readers often present a list of links to users out of context. "Start Learning" is meaningful; "Click here" is not.
Rule 4
Sufficient Contrast
Ensure sufficient color contrast. White text on a medium-blue background may or may not have enough contrast. Use a contrast checker to verify at least 4.5:1 for normal text.
Section 25.8

Feature Lists

A feature list section presents a set of related items, each with an icon (or visual), a short title, and a brief description. Feature lists communicate what a service offers, what a course covers, what a product does, or what an organization stands for.

        that remains relevant regardless of which tools and frameworks emerge.
      </p>
    </div>
    <ul class="feature-list" role="list">
      <li class="feature-item">
        <div class="feature-icon" aria-hidden="true">📐</div>
        <div class="feature-content">
          <h3 class="feature-title">Structural Thinking</h3>
          <p class="feature-description">
            HTML teaches you to organize content meaningfully before
            you think about how it looks.
          </p>
        </div>
      </li>
      <li class="feature-item">
        <div class="feature-icon" aria-hidden="true">🎨</div>
        <div class="feature-content">
          <h3 class="feature-title">Visual Systems</h3>

Why Use <ul> for a Feature List

The feature list is a list of related items, and <ul> is the correct semantic element. The CSS resets its visual appearance — removing bullets and indentation — but the semantic meaning remains. This matters because screen readers announce lists with context ("list, 4 items") that helps users understand the structure. A set of <div> elements does not carry that meaning.

Section 25.9

Reusable CSS Classes

Throughout this chapter, you have seen patterns like .card, .btn, .container, .section-header, and .feature-list. These are all examples of reusable CSS classes — styles that are defined once and applied wherever they are needed.

Reusable classes are the foundation of maintainable CSS. They stand in contrast to overly specific selectors like section#home div.content h2.big-title that only work in one exact place, inline styles that cannot be shared or updated centrally, and duplicated rules that copy the same declarations into multiple places and must be updated individually when something changes.

Design Your Classes Around What They Do, Not Where They Are

A class named .homepage-card describes a location. If you later want to use the same card design on a resources page, the name becomes misleading. A class named .card describes what the element is. It works anywhere.

CSS — fragile vs. reusable naming
/* Fragile: tied to one location */
.homepage-card { ... }
.resources-card { ... }
.about-card { ... }

/* Reusable: works anywhere */
.card { ... }

Modifier Classes

Sometimes you need a card that behaves slightly differently in one context. Use a modifier class alongside the base class rather than duplicating the base styles.

CSS — base card with modifiers
/* Base card */
.card { ... }

/* Modifier: a horizontal card layout */
.card-horizontal {
  flex-direction: row;
}

.card-horizontal .card-image {
  width: 200px;
  height: auto;
  flex-shrink: 0;
}

/* Modifier: a featured card with a different background */
.card-featured {
  background-color: var(--color-primary);
  color: var(--color-text-on-primary);
}
HTML — applying modifier classes
<!-- Standard card -->
<div class="card">...</div>

<!-- Horizontal card -->
<div class="card card-horizontal">...</div>

<!-- Featured card -->
<div class="card card-featured">...</div>
Key Idea

The base .card styles apply to all three examples above. The modifier classes add or override only what differs.

Section 25.10

Consistent Spacing and Alignment

One of the most visible differences between amateur and professional web design is spacing consistency. Amateur designs apply margins and padding with arbitrary numbers: margin-top: 37px here, padding: 22px there. Professional designs use a limited set of spacing values drawn from a spacing scale.

Chapter 18 introduced CSS variables as the foundation of reusable design systems. Here you apply that knowledge to spacing specifically.

A Spacing Scale

A spacing scale is a set of named spacing values that grow in a consistent ratio. Every margin, padding, and gap in your stylesheets should use a value from this scale.

CSS — the spacing scale
:root {
  --space-xs:  0.25rem;  /*  4px */
  --space-sm:  0.5rem;   /*  8px */
  --space-md:  1rem;     /* 16px */
  --space-lg:  1.5rem;   /* 24px */
  --space-xl:  2.5rem;   /* 40px */
  --space-2xl: 4rem;     /* 64px */
}

With this scale, every spacing decision in your stylesheet is one of these six values. Elements look visually related because they share spacing values. Sections breathe because they use larger spacing. Text within a card is comfortable because it uses medium and small spacing.

Vertical Rhythm

When sections stack on a page, each section's top and bottom padding contributes to vertical rhythm: the sense that the page has a consistent beat of content and space. Using consistent padding-block values on sections creates that rhythm.

CSS — vertical rhythm
.content-section {
  padding-block: var(--space-xl);
}

.content-section-large {
  padding-block: var(--space-2xl);
}

Every section on the page has predictable top and bottom breathing room, and the visual rhythm feels intentional.

Alignment Within Sections

Key Idea

Elements within sections should align consistently. If your section header text is centered, center it with text-align: center and a centered max-width block — not by adding large left or right margins on one side. This keeps behavior predictable and responsive-friendly.

Chapter 25 — Back Matter

Chapter Summary

  • Section 25.1Professional websites are built from sections — self-contained bands of content that can be designed once and reused. Sections follow a three-layer pattern: a section wrapper for background and spacing, a container for width and centering, and content for the actual elements.
  • Section 25.2The container pattern uses max-width, margin-inline: auto, and padding-inline to limit content width and center it on the page. Background color belongs on the section, not the container, so full-width colored bands can hold centered content.
  • Section 25.3A hero section presents a headline, supporting description, and one or more call-to-action buttons. Two-column heroes use CSS Grid with a media query that stacks the columns on small screens.
  • Section 25.4Content sections carry supporting information with a section header, subtitle, and body text. Alternating background classes create visual rhythm between sections without extra markup.
  • Section 25.5Cards group an image, title, description, and link into a self-contained, reusable unit. Flexbox with flex-direction: column and flex: 1 on the description keeps cards in a row at equal height. Card grids use CSS Grid with breakpoints that reduce columns on smaller screens.
  • Section 25.6Callouts highlight notes, tips, warnings, and important messages with a colored left border and a text label. Decorative icons use aria-hidden="true" so screen readers rely on the text label instead.
  • Section 25.7A consistent button system includes primary, secondary, and ghost variants. Anchors are used for navigation and buttons for in-page actions. Visible :focus-visible styles, descriptive link text, and sufficient color contrast are accessibility requirements, not optional polish.
  • Section 25.8Feature lists present related items with an icon, title, and description, built on a semantic <ul> with role="list" so the list structure survives CSS resets and remains meaningful to screen readers.
  • Section 25.9Reusable classes are named for what an element is, not where it appears, which keeps styles shareable across pages. Modifier classes like .btn-primary and .card-featured extend a base class without duplicating its rules.
  • Section 25.10A spacing scale of CSS variables — from --space-xs to --space-2xl — keeps every margin, padding, and gap consistent, producing vertical rhythm and predictable alignment across a page.
  • Section 25.11A complete homepage can be assembled entirely from the section patterns in this chapter: a hero, a feature list, a card grid, and a callout, each following the same container and spacing conventions.

Key Terms

hero section
The large top section of a page, typically containing a headline, supporting description, and call-to-action buttons. It orients the visitor and communicates the site's primary purpose.
container
A div element with a max-width and centered margins, used inside sections to limit content width and keep it readable on wide screens.
card
A self-contained visual unit grouping related information, typically with an image or icon, a title, a short description, and a link.
card grid
A CSS Grid layout used to arrange multiple cards in columns, with responsive breakpoints that reduce columns on smaller screens.
callout
A visually distinct block that draws attention to a note, tip, warning, or important message, often with a colored left border and a descriptive label.
reusable class
A CSS class designed to be applied in multiple places across a website, making it possible to change a style in one place and have the change propagate everywhere.
modifier class
A CSS class applied alongside a base class to add or override specific styles for a variation, without duplicating the base class rules.
spacing scale
A set of named CSS variable values for spacing (margin, padding, gap) that ensures all spacing decisions use a consistent set of sizes.
vertical rhythm
The visual consistency created by applying uniform top and bottom padding to stacked sections, giving a page a readable sense of space and pace.
feature list
A section pattern that presents multiple related items, each with an icon, a title, and a short description, arranged in a grid for scannable reading.
call to action (CTA)
A button or link that invites the user to take a specific action, such as "Start Learning", "Contact Us", or "View Resources".
object-fit: cover
A CSS property applied to images that scales the image to fill its container while maintaining aspect ratio and cropping edges if necessary.

  • CSS is in an external file, not inline or in a <style> block
  • The page looks intentional and consistent, not randomly styled
  • The callout uses the correct class variation for its message type
Accessibility Check
Review the completed Mini-Project page using this checklist
Headings
  • The page has one <h1> (the hero title)
  • Section headings are <h2> elements
  • Card titles are <h3> elements
  • Headings descend logically (h1 → h2 → h3) without skipping levels
Images
  • Every <img> has an alt attribute
  • Alt text describes the image content in context, not just "image" or the filename
  • Decorative icons have aria-hidden="true"
Links and Buttons
  • All link text is descriptive in isolation
  • Buttons have visible :focus-visible styles
  • No links use generic text like "click here" or "read more" without context
Structure
  • The <main> element wraps the page's primary content
  • The hero section uses a <section> element, not a bare <div>
  • Feature lists use <ul> with role="list" to preserve list semantics after CSS reset
Color
  • Primary action buttons have at least 4.5:1 contrast between text and background
  • Muted text (.color-text-muted) still meets minimum contrast requirements against its background
